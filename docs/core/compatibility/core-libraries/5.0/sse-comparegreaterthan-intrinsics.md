# <a name="sse-and-sse2-comparegreaterthan-methods-properly-handle-nan-inputs"></a><span data-ttu-id="77297-103">SSE ve SSE2 CompareGreaterThan yöntemleri NaN girdilerini doğru bir şekilde işler</span><span class="sxs-lookup"><span data-stu-id="77297-103">SSE and SSE2 CompareGreaterThan methods properly handle NaN inputs</span></span>

<span data-ttu-id="77297-104">Aşağıdaki <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> ve <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> yöntemleri, girdileri doğru bir şekilde işlemek `NaN` ve sınıfındaki eşdeğer yöntemlerin donanım davranışıyla eşleştirmek için düzeltildi <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> :</span><span class="sxs-lookup"><span data-stu-id="77297-104">The following <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> and <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> methods have been fixed to properly handle `NaN` inputs and match the hardware behavior of the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> class:</span></span>

* `CompareGreaterThan`
* `CompareGreaterThanOrEqual`
* `CompareNotGreaterThan`
* `CompareNotGreaterThanOrEqual`
* `CompareScalarGreaterThan`
* `CompareScalarGreaterThanOrEqual`
* `CompareScalarNotGreaterThan`
* `CompareScalarNotGreaterThanOrEqual`

## <a name="change-description"></a><span data-ttu-id="77297-105">Açıklamayı Değiştir</span><span class="sxs-lookup"><span data-stu-id="77297-105">Change description</span></span>

<span data-ttu-id="77297-106">Daha önce, `NaN` listelenmiş <xref:System.Runtime.Intrinsics.X86.Sse> ve yöntemlere yapılan girişler <xref:System.Runtime.Intrinsics.X86.Sse2> yanlış bir sonuç döndürdü.</span><span class="sxs-lookup"><span data-stu-id="77297-106">Previously, `NaN` inputs to the listed <xref:System.Runtime.Intrinsics.X86.Sse> and <xref:System.Runtime.Intrinsics.X86.Sse2> methods returned an incorrect result.</span></span> <span data-ttu-id="77297-107">Sonuç, sınıfında karşılık gelen yöntem tarafından oluşturulan sonuçtan de farklıydı <xref:System.Runtime.Intrinsics.X86.Avx> .</span><span class="sxs-lookup"><span data-stu-id="77297-107">The result also differed from the result generated by the corresponding method in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="77297-108">.NET 5 ' den başlayarak, bu yöntemler girişleri doğru şekilde işler `NaN` ve sınıftaki karşılık gelen yöntemlerle aynı sonuçları döndürür <xref:System.Runtime.Intrinsics.X86.Avx> .</span><span class="sxs-lookup"><span data-stu-id="77297-108">Starting in .NET 5, these methods correctly handle `NaN` inputs and return the same results as the corresponding methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="77297-109">Streaming SIMD Extensions (SSE) ve Streaming SIMD Extensions 2 (SSE2) endüstri standardı mimarileri (IAS), bu karşılaştırma yöntemleri için doğrudan donanım desteği sağlamamakta ve bu nedenle yazılımda uygulanırlar.</span><span class="sxs-lookup"><span data-stu-id="77297-109">The Streaming SIMD Extensions (SSE) and Streaming SIMD Extensions 2 (SSE2) industry standard architectures (ISAs) don't provide direct hardware support for these comparison methods, so they're implemented in software.</span></span> <span data-ttu-id="77297-110">Daha önce, Yöntemler hatalı şekilde uygulanmıştır ve girdileri yanlış bir şekilde işledi `NaN` .</span><span class="sxs-lookup"><span data-stu-id="77297-110">Previously, the methods were improperly implemented, and they incorrectly handled `NaN` inputs.</span></span> <span data-ttu-id="77297-111">Yerel bilgisayardan gelen kod için hatalı davranış hataları ortaya çıkarabilir.</span><span class="sxs-lookup"><span data-stu-id="77297-111">For code ported from native, the incorrect behavior may introduce bugs.</span></span> <span data-ttu-id="77297-112">256 bitlik bir kod yolu için yöntemler, sınıftaki eşdeğer yöntemlere farklı sonuçlar da üretebilir <xref:System.Runtime.Intrinsics.X86.Avx> .</span><span class="sxs-lookup"><span data-stu-id="77297-112">For a 256-bit code path, the methods can also produce different results to the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="77297-113">Yöntemlerin daha önceden nasıl yanlış olduğunu gösteren bir örnek olarak, `CompareNotGreaterThan(x,y)` `CompareLessThanOrEqual(x,y)` normal tamsayılar için olarak öğesini uygulayabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="77297-113">As an example of how the methods were previously incorrect, you can implement `CompareNotGreaterThan(x,y)` as `CompareLessThanOrEqual(x,y)` for regular integers.</span></span> <span data-ttu-id="77297-114">Ancak, `NaN` girişler için bu mantık yanlış sonucu hesaplar.</span><span class="sxs-lookup"><span data-stu-id="77297-114">However, for `NaN` inputs, that logic computes the wrong result.</span></span> <span data-ttu-id="77297-115">Bunun yerine, `CompareNotLessThan(y,x)` sayılar doğru şekilde karşılaştırılmaktadır *ve* `NaN` girişler dikkate alır.</span><span class="sxs-lookup"><span data-stu-id="77297-115">Instead, using `CompareNotLessThan(y,x)` compares the numbers correctly *and* takes `NaN` inputs into consideration.</span></span>
