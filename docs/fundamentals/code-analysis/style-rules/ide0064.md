## <a name="overview"></a><span data-ttu-id="ba62e-116">Genel Bakış</span><span class="sxs-lookup"><span data-stu-id="ba62e-116">Overview</span></span>

<span data-ttu-id="ba62e-117">Bu kural bir veya daha fazla alan içeren yapıları algılar `readonly` ve ayrıca oluşturucunun dışından bir atama içerir `this` .</span><span class="sxs-lookup"><span data-stu-id="ba62e-117">This rule detects structs that contain one ore more `readonly` fields and also contains an assignment to `this` outside of the constructor.</span></span> <span data-ttu-id="ba62e-118">Kural `readonly` , alanların salt okunur olmayan, yani yazılabilir olarak dönüştürülmesini öneriyor.</span><span class="sxs-lookup"><span data-stu-id="ba62e-118">The rule recommends converting `readonly` fields to non-readonly, i.e. writable.</span></span> <span data-ttu-id="ba62e-119">Bu tür yapı alanlarını ReadOnly olarak işaretlemek, alana atanan değer oluşturucunun dışına atandığında, beklenmeyen davranışa yol açabilir `this` .</span><span class="sxs-lookup"><span data-stu-id="ba62e-119">Marking such struct fields as readonly can lead to unexpected behavior, as the value assigned to the field can change when `this` is assigned outside the constructor.</span></span>

## <a name="example"></a><span data-ttu-id="ba62e-120">Örnek</span><span class="sxs-lookup"><span data-stu-id="ba62e-120">Example</span></span>

```csharp
// Code with violations
struct MyStruct
{
    public readonly int Value;

    public MyStruct(int value)
    {
        Value = value;
    }

    public void Test()
    {
        this = new MyStruct(5);
    }
}

// Fixed code
struct MyStruct
{
    public int Value;

    public MyStruct(int value)
    {
        Value = value;
    }

    public void Test()
    {
        this = new MyStruct(5);
    }
}
```
