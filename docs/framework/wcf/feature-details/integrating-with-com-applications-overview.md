# <a name="integrating-with-com-applications-overview"></a><span data-ttu-id="b4859-102">COM Uygulamaları ile Tümleştirme Genel Bakış</span><span class="sxs-lookup"><span data-stu-id="b4859-102">Integrating with COM Applications Overview</span></span>

<span data-ttu-id="b4859-103">Windows Communication Foundation (WCF), bağlı uygulamalar oluşturmak için zengin bir ortam ile yönetilen kod geliştiricisi sağlar.</span><span class="sxs-lookup"><span data-stu-id="b4859-103">Windows Communication Foundation (WCF) provides the managed code developer with a rich environment for creating connected applications.</span></span> <span data-ttu-id="b4859-104">Ancak, yönetilmeyen COM tabanlı kodda önemli bir yatırımınız varsa ve geçiş yapmak istemiyorsanız, WCF hizmeti bilinen adını kullanarak yine de WCF Web hizmetlerini mevcut kodunuzla tümleştirmeye devam edebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="b4859-104">However, if you have a substantial investment in unmanaged COM-based code and do not want to migrate, you can still integrate WCF Web services directly into your existing code by using the WCF service moniker.</span></span> <span data-ttu-id="b4859-105">Hizmet bilinen adı, Office VBA, Visual Basic 6,0 veya Visual C++ 6,0 gibi çok çeşitli COM tabanlı geliştirme ortamlarından kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="b4859-105">The service moniker can be used from a wide range of COM-based development environments, such as Office VBA, Visual Basic 6.0, or Visual C++ 6.0.</span></span>

> [!NOTE]
> <span data-ttu-id="b4859-106">Hizmet bilinen adı, tüm iletişimler için bir WCF iletişim kanalı kullanır.</span><span class="sxs-lookup"><span data-stu-id="b4859-106">The service moniker uses a WCF communication channel for all communication.</span></span> <span data-ttu-id="b4859-107">Bu kanala yönelik güvenlik ve kimlik mekanizmaları standart COM ve DCOM proxy 'lerinde kullanılanlardan farklıdır.</span><span class="sxs-lookup"><span data-stu-id="b4859-107">The security and identity mechanisms for that channel differ from those used in standard COM and DCOM proxies.</span></span> <span data-ttu-id="b4859-108">Ayrıca, hizmet bilinen adı bir WCF iletişim kanalı kullandığından, tüm çağrılar için varsayılan zaman aşımı süresi bir dakikadır.</span><span class="sxs-lookup"><span data-stu-id="b4859-108">In addition, because the service moniker uses a WCF communication channel the default timeout period is one minute for all calls.</span></span>

<span data-ttu-id="b4859-109">Hizmet bilinen `GetObject` geliştiricisi, WCF Web hizmetlerini çağırmak için kesin olarak belirlenmiş, com 'a özgü bir yaklaşımla birlikte yönetilmeyen geliştiriciyi sağlamak için işleviyle birlikte kullanılır.</span><span class="sxs-lookup"><span data-stu-id="b4859-109">The service moniker is used with the `GetObject` function to provide the unmanaged developer with a strongly-typed, COM-specific approach for calling WCF Web services.</span></span> <span data-ttu-id="b4859-110">Bu, WCF Web hizmeti sözleşmesinin yerel, COM görünebilir tanımını ve kullanılacak bağlamayı gerektirir.</span><span class="sxs-lookup"><span data-stu-id="b4859-110">This requires a local, COM-visible definition of the WCF Web service contract and the binding that is to be used.</span></span> <span data-ttu-id="b4859-111">Diğer WCF istemcileri gibi, hizmet bilinen bir kanal, bu kanal oluşturma ilk yöntem çağrısında COM programcısı tarafından saydam olarak gerçekleşse de, hizmete bir türü belirtilmiş bir kanal oluşturulmalıdır.</span><span class="sxs-lookup"><span data-stu-id="b4859-111">Like other WCF clients, the service moniker must construct a typed channel to the service, though this channel construction occurs transparently to the COM programmer on the first method call.</span></span>

<span data-ttu-id="b4859-112">Diğer WCF istemcileriyle ortak olarak, bilinen ad kullanıldığında, uygulamalar bir hizmetle iletişim kurmak için adresi, bağlamayı ve sözleşmeyi belirtir.</span><span class="sxs-lookup"><span data-stu-id="b4859-112">In common with other WCF clients, when using the moniker, applications specify the address, binding and contract to communicate with a service.</span></span> <span data-ttu-id="b4859-113">Sözleşme aşağıdaki yollarla belirtilebilir:</span><span class="sxs-lookup"><span data-stu-id="b4859-113">The contract can be specified in one of the following ways:</span></span>

- <span data-ttu-id="b4859-114">Yazılı sözleşme – sözleşme, istemci makinesinde COM görünebilir bir tür olarak kaydedilir.</span><span class="sxs-lookup"><span data-stu-id="b4859-114">Typed contract – the contract is registered as a COM visible type on the client machine.</span></span>

- <span data-ttu-id="b4859-115">WSDL sözleşmesi – sözleşme, WSDL belgesi biçiminde sağlanır.</span><span class="sxs-lookup"><span data-stu-id="b4859-115">WSDL contract – the contract is supplied in the form of a WSDL document.</span></span>

- <span data-ttu-id="b4859-116">MEX sözleşmesi – sözleşme, meta veri değişimi (MEX) uç noktasından çalışma zamanında alınır.</span><span class="sxs-lookup"><span data-stu-id="b4859-116">MEX contract – the contract is retrieved at runtime from a Metadata Exchange (MEX) endpoint.</span></span>
