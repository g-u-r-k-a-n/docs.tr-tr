## <a name="manners-of-execution"></a><span data-ttu-id="88b4c-108">Yürütme mananlar</span><span class="sxs-lookup"><span data-stu-id="88b4c-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="88b4c-109">Hemen</span><span class="sxs-lookup"><span data-stu-id="88b4c-109">Immediate</span></span>  

 <span data-ttu-id="88b4c-110">Anında yürütme, veri kaynağının okunduğu ve işlemin sorgunun bildirildiği noktada gerçekleştirildiği anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="88b4c-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="88b4c-111">Tek, Numaralandırılmamış bir sonuç döndüren tüm standart sorgu işleçleri hemen yürütülür.</span><span class="sxs-lookup"><span data-stu-id="88b4c-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="88b4c-112">İlmiş</span><span class="sxs-lookup"><span data-stu-id="88b4c-112">Deferred</span></span>  

 <span data-ttu-id="88b4c-113">Ertelenmiş yürütme, işlemin sorgunun bildirildiği noktada gerçekleştirilmediği anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="88b4c-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="88b4c-114">İşlem yalnızca sorgu değişkeni numaralandırıldıktan sonra, örneğin bir ifade kullanılarak yapılır `For Each` .</span><span class="sxs-lookup"><span data-stu-id="88b4c-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="88b4c-115">Bu, sorguyu yürütmenin sonuçlarının sorgu tanımlandığında değil, sorgu yürütüldüğünde veri kaynağının içeriğine bağlı olduğunu gösterir.</span><span class="sxs-lookup"><span data-stu-id="88b4c-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="88b4c-116">Sorgu değişkeni birden çok kez numaralandırıldıktan sonra sonuçlar her seferinde farklılık gösterebilir.</span><span class="sxs-lookup"><span data-stu-id="88b4c-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="88b4c-117">Dönüş türü <xref:System.Collections.Generic.IEnumerable%601> veya <xref:System.Linq.IOrderedEnumerable%601> ertelenmiş bir şekilde yürütülen tüm standart sorgu işleçleri neredeyse hepsi.</span><span class="sxs-lookup"><span data-stu-id="88b4c-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="88b4c-118">Ertelenmiş yürütmeyi kullanan sorgu işleçleri ayrıca akış veya akış olmayan olarak sınıflandırılabilirler.</span><span class="sxs-lookup"><span data-stu-id="88b4c-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="88b4c-119">Akış</span><span class="sxs-lookup"><span data-stu-id="88b4c-119">Streaming</span></span>  

 <span data-ttu-id="88b4c-120">Akış işleçleri, öğeleri almadan önce tüm kaynak verileri okumak zorunda değildir.</span><span class="sxs-lookup"><span data-stu-id="88b4c-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="88b4c-121">Yürütme sırasında, bir akış işleci her kaynak öğe okunışında işlemini gerçekleştirir ve uygunsa öğeyi verir.</span><span class="sxs-lookup"><span data-stu-id="88b4c-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="88b4c-122">Bir akış işleci, bir sonuç öğesi üretilene kadar kaynak öğeleri okumaya devam eder.</span><span class="sxs-lookup"><span data-stu-id="88b4c-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="88b4c-123">Bu, bir sonuç öğesi oluşturmak için birden fazla kaynak öğesinin okunabileceğini gösterir.</span><span class="sxs-lookup"><span data-stu-id="88b4c-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="88b4c-124">Akış olmayan</span><span class="sxs-lookup"><span data-stu-id="88b4c-124">Non-Streaming</span></span>  

 <span data-ttu-id="88b4c-125">Akış olmayan operatörler, bir sonuç öğesi elde etmeden önce tüm kaynak verileri okummalıdır.</span><span class="sxs-lookup"><span data-stu-id="88b4c-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="88b4c-126">Sıralama veya gruplama gibi işlemler bu kategoriye girer.</span><span class="sxs-lookup"><span data-stu-id="88b4c-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="88b4c-127">Yürütme sırasında akış olmayan sorgu işleçleri tüm kaynak verileri okur, veri yapısına koyar, işlemi gerçekleştirir ve sonuçta elde edilen öğeleri verir.</span><span class="sxs-lookup"><span data-stu-id="88b4c-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
