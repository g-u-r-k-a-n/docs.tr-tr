[karakter kaçışları](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span><span class="sxs-lookup"><span data-stu-id="60a8c-107">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="60a8c-108">Aşağıdaki örnek CS1009 oluşturur.</span><span class="sxs-lookup"><span data-stu-id="60a8c-108">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="60a8c-109">Bu hatanın yaygın bir nedeni, aşağıdaki örnekte gösterildiği gibi bir dosya adında ters eğik çizgi karakterini kullanmaktır.</span><span class="sxs-lookup"><span data-stu-id="60a8c-109">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp

```  
  
 <span data-ttu-id="60a8c-110">Bu hatayı çözmek için, \\ \\ @-quoted Aşağıdaki örnekte gösterildiği gibi "" veya dize değişmez değerini kullanın.</span><span class="sxs-lookup"><span data-stu-id="60a8c-110">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
