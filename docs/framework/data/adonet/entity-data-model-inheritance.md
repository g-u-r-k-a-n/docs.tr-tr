# <a name="entity-data-model-inheritance"></a><span data-ttu-id="0bff1-102">Varlık Veri Modeli: devralma</span><span class="sxs-lookup"><span data-stu-id="0bff1-102">Entity Data Model: Inheritance</span></span>
<span data-ttu-id="0bff1-103">Varlık Veri Modeli (EDM), [varlık türleri](entity-type.md)için devralmayı destekler.</span><span class="sxs-lookup"><span data-stu-id="0bff1-103">The Entity Data Model (EDM) supports inheritance for [entity types](entity-type.md).</span></span> <span data-ttu-id="0bff1-104">EDM 'da devralma, nesne odaklı programlama dillerinde sınıfların devralmasına benzerdir.</span><span class="sxs-lookup"><span data-stu-id="0bff1-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="0bff1-105">Nesne odaklı dillerdeki sınıflar gibi, kavramsal bir modelde, başka bir varlık türünden ( *temel tür*) devralan bir varlık türü ( *türetilmiş bir tür*) tanımlayabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="0bff1-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="0bff1-106">Ancak, nesne odaklı programlamada sınıfların aksine, kavramsal bir modelde türetilmiş tür her zaman temel türün tüm [özelliklerini](property.md) ve [gezinti özelliklerini](navigation-property.md) devralır.</span><span class="sxs-lookup"><span data-stu-id="0bff1-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](property.md) and [navigation properties](navigation-property.md) of the base type.</span></span> <span data-ttu-id="0bff1-107">Türetilmiş bir türdeki devralınan özellikleri geçersiz kılamazsınız.</span><span class="sxs-lookup"><span data-stu-id="0bff1-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="0bff1-108">Kavramsal modelde, türetilmiş bir türün başka bir türetilmiş türden devraldığı devralma hiyerarşileri oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="0bff1-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="0bff1-109">Hiyerarşinin en üstündeki tür (türetilmiş tür olmayan hiyerarşide bir tür) *kök türü*olarak adlandırılır.</span><span class="sxs-lookup"><span data-stu-id="0bff1-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="0bff1-110">Devralma hiyerarşisinde, [varlık anahtarının](entity-key.md) kök türünde tanımlanması gerekir.</span><span class="sxs-lookup"><span data-stu-id="0bff1-110">In an inheritance hierarchy, the [entity key](entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="0bff1-111">Türetilmiş bir türün birden fazla türden devraldığı devralma hiyerarşileri derlenemez.</span><span class="sxs-lookup"><span data-stu-id="0bff1-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="0bff1-112">Örneğin, `Book` varlık türü olan bir kavramsal modelde, `FictionBook` türetilmiş türleri ve her birinin `Book`devraldığı `NonFictionBook` tanımlayabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="0bff1-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="0bff1-113">Ancak, `FictionBook` ve `NonFictionBook` türlerinden devralan bir tür tanımlayamazsınız.</span><span class="sxs-lookup"><span data-stu-id="0bff1-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
