# <a name="type-argument-inference-failed-for-type-parameter-typeparametername1-of-genericproceduresignature"></a><span data-ttu-id="12284-103">' ' Öğesinin ' ' tür parametresi için tür bağımsız değişkeni çıkarımı başarısız oldu \<typeparametername1> \<genericproceduresignature></span><span class="sxs-lookup"><span data-stu-id="12284-103">Type argument inference failed for type parameter '\<typeparametername1>' of '\<genericproceduresignature>'</span></span>

<span data-ttu-id="12284-104">' ' Öğesinin ' ' tür parametresi için tür bağımsız değişkeni çıkarımı başarısız oldu \<typeparametername1> \<genericproceduresignature> .</span><span class="sxs-lookup"><span data-stu-id="12284-104">Type argument inference failed for type parameter '\<typeparametername1>' of '\<genericproceduresignature>'.</span></span> <span data-ttu-id="12284-105">' ' Parametresine geçirilen bağımsız değişkenden çıkarılan tür bağımsız değişkeni, \<parametername1> ' ' parametresine geçirilen bağımsız değişkenden çıkarılan tür bağımsız değişkeniyle çakışıyor \<parametername2> .</span><span class="sxs-lookup"><span data-stu-id="12284-105">Type argument inferred from the argument passed to parameter '\<parametername1>' conflicts with the type argument inferred from the argument passed to parameter '\<parametername2>'.</span></span>  
  
 <span data-ttu-id="12284-106">Genel yordam herhangi bir tür bağımsız değişkeni olmadan çağrılır ve denenen tür çıkarımı tür parametreleri arasında bir veri türü çakışması üretti.</span><span class="sxs-lookup"><span data-stu-id="12284-106">A generic procedure is called without any type arguments, and the attempted type inference has produced a data type conflict among the type parameters.</span></span>  
  
 <span data-ttu-id="12284-107">Normal olarak, genel bir yordamı çağırdığınızda, genel yordamın tanımladığı her tür parametresi için bir tür bağımsız değişkeni sağlarsınız.</span><span class="sxs-lookup"><span data-stu-id="12284-107">Normally, when you call a generic procedure, you supply a type argument for each type parameter that the generic procedure defines.</span></span> <span data-ttu-id="12284-108">Herhangi bir tür bağımsız değişkeni belirtmezseniz, derleyici tür parametrelerine geçirilecek türleri çıkarması için girişimde bulunur.</span><span class="sxs-lookup"><span data-stu-id="12284-108">If you do not supply any type arguments, then the compiler attempts to infer the types to be passed to the type parameters.</span></span> <span data-ttu-id="12284-109">Çağrının bağlamı bir tür parametresi için çakışan veri türü bilgileri sağlıyorsa tür çıkarımı başarısız olur.</span><span class="sxs-lookup"><span data-stu-id="12284-109">If the context of the call provides conflicting data type information for a type parameter, then the type inference fails.</span></span>  
  
 <span data-ttu-id="12284-110">Aşağıdaki kod bu hatayı oluşturabilir.</span><span class="sxs-lookup"><span data-stu-id="12284-110">The following code can generate this error.</span></span>  
  
```vb  
Public Sub takeTwoValues(Of t)(ByVal x As t, ByVal y As t)  
End Sub  
Call takeTwoValues(4, 2.5)  
```  
  
 <span data-ttu-id="12284-111">İlk bağımsız değişken derleyicinin tür parametresi için çıkarmasına neden olduğu için `Integer` `t` ikinci bağımsız değişken aynı tür parametresi için ortaya çıkarmasına yol açacağından, `Double` hangi veri türünün geçebileceği hakkında bir çakışma vardır `t` .</span><span class="sxs-lookup"><span data-stu-id="12284-111">Because the first argument causes the compiler to infer `Integer` for type parameter `t`, while the second argument causes it to infer `Double` for the same type parameter, there is a conflict regarding which data type to pass to `t`.</span></span>  
  
 <span data-ttu-id="12284-112">**Hata kimliği:** BC32116</span><span class="sxs-lookup"><span data-stu-id="12284-112">**Error ID:** BC32116</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="12284-113">Bu hatayı düzeltmek için</span><span class="sxs-lookup"><span data-stu-id="12284-113">To correct this error</span></span>  
  
- <span data-ttu-id="12284-114">Derleyicinin bunları çıkarması gereken tür bağımsız değişkenlerini genel türe sağlayın.</span><span class="sxs-lookup"><span data-stu-id="12284-114">Supply type arguments to the generic type, so that the compiler does not have to infer them.</span></span>  
  
    ```vb  
    Call takeTwoValues(Of Double)(4, 2.5)  
    ```  
  
     <span data-ttu-id="12284-115">Bu durumda, iki normal bağımsız değişkenin farklı veri türlerinde olduğu, çağıran kodun her iki bu veri türünü de barındırabilecek bir tür bağımsız değişkeni ilettiğine dikkat edin.</span><span class="sxs-lookup"><span data-stu-id="12284-115">Notice that in this case, where the two normal arguments are of different data types, the calling code must pass a type argument that can accommodate both of those data types.</span></span> <span data-ttu-id="12284-116">Bu durumda, `Integer` widens to `Double` .</span><span class="sxs-lookup"><span data-stu-id="12284-116">In this case, `Integer` widens to `Double`.</span></span>  
  
     <span data-ttu-id="12284-117">-veya-</span><span class="sxs-lookup"><span data-stu-id="12284-117">-or-</span></span>  
  
- <span data-ttu-id="12284-118">Normal parametreler için farklı tür parametreleri belirtmek üzere genel yordamı yeniden tanımlayarak, bu tür bir çakışma olmaması gerekir.</span><span class="sxs-lookup"><span data-stu-id="12284-118">Redefine the generic procedure to specify different type parameters for the normal parameters, so that there is no conflict inferring the types.</span></span>  
  
    ```vb  
    Public Sub takeTwoValues(Of t1, t2)(ByVal x As t1, ByVal y As t2)  
    ```
