# <a name="collection-type-dependency-properties"></a><span data-ttu-id="93c11-102">Koleksiyon Türü Bağımlılık Özellikleri</span><span class="sxs-lookup"><span data-stu-id="93c11-102">Collection-Type Dependency Properties</span></span>
<span data-ttu-id="93c11-103">Bu konu, özellik türü bir koleksiyon türü olan bir bağımlılık özelliğinin nasıl uygulanacağı için kılavuz ve önerilen desenler sağlar.</span><span class="sxs-lookup"><span data-stu-id="93c11-103">This topic provides guidance and suggested patterns for how to implement a dependency property where the type of the property is a collection type.</span></span>

## <a name="implementing-a-collection-type-dependency-property"></a><span data-ttu-id="93c11-104">Toplama Türü Bağımlılık Özelliği Uygulama</span><span class="sxs-lookup"><span data-stu-id="93c11-104">Implementing a Collection-Type Dependency Property</span></span>  
 <span data-ttu-id="93c11-105">Genel olarak bir bağımlılık özelliği için, izlemeniz gereken uygulama deseni, bu özelliğin bir alan <xref:System.Windows.DependencyProperty> veya başka bir yapı yerine tanımlayıcı tarafından desteklenen bir CLR özellik sarıcı tanımlamanızdır.</span><span class="sxs-lookup"><span data-stu-id="93c11-105">For a dependency property in general, the implementation pattern that you follow is that you define a CLR property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="93c11-106">Bir koleksiyon türü özelliği uygularken aynı deseni izlersiniz.</span><span class="sxs-lookup"><span data-stu-id="93c11-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="93c11-107">Ancak, koleksiyon türü özelliği, koleksiyon içinde bulunan tür kendisi bir <xref:System.Windows.DependencyObject> veya <xref:System.Windows.Freezable> türetilmiş sınıf olduğunda desen bazı karmaşıklık tanıtır.</span><span class="sxs-lookup"><span data-stu-id="93c11-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

## <a name="initializing-the-collection-beyond-the-default-value"></a><span data-ttu-id="93c11-108">Koleksiyonu Varsayılan Değerin Ötesinde Başlatma</span><span class="sxs-lookup"><span data-stu-id="93c11-108">Initializing the Collection Beyond the Default Value</span></span>  
 <span data-ttu-id="93c11-109">Bağımlılık özelliği oluşturduğunuzda, özellik varsayılan değerini başlangıç alanı değeri olarak belirtmezsiniz.</span><span class="sxs-lookup"><span data-stu-id="93c11-109">When you create a dependency property, you do not specify the property default value as the initial field value.</span></span> <span data-ttu-id="93c11-110">Bunun yerine, bağımlılık özelliği meta verileri aracılığıyla varsayılan değeri belirtirsiniz.</span><span class="sxs-lookup"><span data-stu-id="93c11-110">Instead, you specify the default value through the dependency property metadata.</span></span> <span data-ttu-id="93c11-111">Mülkünüz bir başvuru türüyse, bağımlılık özelliği meta verilerinde belirtilen varsayılan değer örnek başına varsayılan değer değildir; bunun yerine, türün tüm örnekleri için geçerli varsayılan bir değerdir.</span><span class="sxs-lookup"><span data-stu-id="93c11-111">If your property is a reference type, the default value specified in dependency property metadata is not a default value per instance; ## <a name="reporting-binding-value-changes-from-collection-properties"></a><span data-ttu-id="93c11-130">Toplama Özelliklerinden Bağlayıcı Değer Değişikliklerini Bildirme</span><span class="sxs-lookup"><span data-stu-id="93c11-130">Reporting Binding Value Changes from Collection Properties</span></span>  
 <span data-ttu-id="93c11-131">Kendisi bir bağımlılık özelliği olan bir toplama özelliği, alt özelliklerindeki değişiklikleri otomatik olarak bildirmez.</span><span class="sxs-lookup"><span data-stu-id="93c11-131">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="93c11-132">Bir koleksiyona bağlama lar oluşturuyorsanız, bu durum bağlamanın değişiklikleri bildirmesini engelleyebilir ve bu nedenle bazı veri bağlama senaryolarını geçersiz kılabilir.</span><span class="sxs-lookup"><span data-stu-id="93c11-132">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="93c11-133">Ancak, koleksiyon türünü <xref:System.Windows.FreezableCollection%601> koleksiyon türünüz olarak kullanırsanız, koleksiyonda bulunan öğelere alt özellik değişiklikleri düzgün şekilde bildirilir ve beklendiği gibi bağlama çalışmaları yapılır.</span><span class="sxs-lookup"><span data-stu-id="93c11-133">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>  
  
 <span data-ttu-id="93c11-134">Bağımlılık nesnesi koleksiyonunda alt özellik bağlamayı etkinleştirmek <xref:System.Windows.FreezableCollection%601>için, türdeki herhangi <xref:System.Windows.DependencyObject> bir sınıfa bu koleksiyon için bir tür kısıtlaması olan tür olarak koleksiyon özelliğini oluşturun.</span><span class="sxs-lookup"><span data-stu-id="93c11-134">To enable subproperty binding in a dependency object collection, create the collection property as type <xref:System.Windows.FreezableCollection%601>, with a type constraint for that collection to any <xref:System.Windows.DependencyObject> derived class.</span></span>
