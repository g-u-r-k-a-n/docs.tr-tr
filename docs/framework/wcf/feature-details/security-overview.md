### <a name="transfer-security"></a><span data-ttu-id="27dac-173">Güvenlik aktarma</span><span class="sxs-lookup"><span data-stu-id="27dac-173">Transfer Security</span></span>  

 <span data-ttu-id="27dac-174">Aktarım güvenliği üç önemli güvenlik işlevini kapsar: bütünlük, gizlilik ve kimlik doğrulama.</span><span class="sxs-lookup"><span data-stu-id="27dac-174">Transfer security encompasses three major security functions: integrity, confidentiality, and authentication.</span></span> <span data-ttu-id="27dac-175">*Bütünlük* , bir iletinin değiştirilip değiştirilmediğini tespit etme olanağıdır.</span><span class="sxs-lookup"><span data-stu-id="27dac-175">*Integrity* is the ability to detect whether a message has been tampered with.</span></span> <span data-ttu-id="27dac-176">*Gizlilik* , bir iletiyi amaçlanan alıcı dışında bir kişi tarafından okunamaz halde tutma olanağıdır; Bu, şifreleme üzerinden sağlanır.</span><span class="sxs-lookup"><span data-stu-id="27dac-176">*Confidentiality* is the ability to keep a message unreadable by anyone other than the intended recipient; this is achieved through cryptography.</span></span> <span data-ttu-id="27dac-177">*Kimlik doğrulama* , istenen bir kimliği doğrulama yeteneğidir.</span><span class="sxs-lookup"><span data-stu-id="27dac-177">*Authentication* is the ability to verify a claimed identity.</span></span> <span data-ttu-id="27dac-178">Bu üç işlev birlikte, iletilerin bir noktadan diğerine güvenli bir şekilde ulaşmasını sağlamaya yardımcı olur.</span><span class="sxs-lookup"><span data-stu-id="27dac-178">Together, these three functions help to ensure that messages securely arrive from one point to another.</span></span>  
  
#### <a name="transport-and-message-security-modes"></a><span data-ttu-id="27dac-179">Aktarım ve Ileti güvenliği modları</span><span class="sxs-lookup"><span data-stu-id="27dac-179">Transport and Message Security Modes</span></span>  

 <span data-ttu-id="27dac-180">WCF 'de aktarım güvenliği uygulamak için iki ana mekanizma kullanılır: *Aktarım* güvenliği modu ve *ileti* güvenlik modu.</span><span class="sxs-lookup"><span data-stu-id="27dac-180">Two main mechanisms are used to implement transfer security in WCF: *transport* security mode and *message* security mode.</span></span>  
  
- <span data-ttu-id="27dac-181">Aktarım *güvenliği modu* , aktarım güvenliğini sağlamak için https gibi bir aktarım düzeyi protokolü kullanır.</span><span class="sxs-lookup"><span data-stu-id="27dac-181">*Transport security mode* uses a transport-level protocol, such as HTTPS, to achieve transfer security.</span></span> <span data-ttu-id="27dac-182">Aktarım modu, çok sayıda platformda sunulan ve daha az hesaplama avantajına sahiptir.</span><span class="sxs-lookup"><span data-stu-id="27dac-182">Transport mode has the advantage of being widely adopted, available on many platforms, and less computationally complex.</span></span> <span data-ttu-id="27dac-183">Ancak, iletilerin yalnızca noktadan noktaya güvenli hale getirilmesi olumsuz bir dezavantajı vardır.</span><span class="sxs-lookup"><span data-stu-id="27dac-183">However, it has the disadvantage of securing messages only from point-to-point.</span></span>  
  
- <span data-ttu-id="27dac-184">*İleti güvenliği modu*, diğer yandan aktarım güvenliği uygulamak için WS-Security (ve diğer belirtimleri) kullanır.</span><span class="sxs-lookup"><span data-stu-id="27dac-184">*Message security mode*, on the other hand, uses WS-Security (and other specifications) to implement transfer security.</span></span> <span data-ttu-id="27dac-185">İleti güvenliği doğrudan SOAP iletilerine uygulandığından ve SOAP zarfların içinde, uygulama verileriyle birlikte bulunduğundan, Aktarım Protokolü 'nün bağımsız, daha genişletilebilir ve uçtan uca güvenlik (noktadan noktaya) sağlama avantajına sahiptir. SOAP iletilerinin XML doğası ile uğraşmak gerektiğinden, aktarım güvenliği modundan birkaç kat daha yavaştır.</span><span class="sxs-lookup"><span data-stu-id="27dac-185">Because the message security is applied directly to the SOAP messages and is contained inside the SOAP envelopes, together with the application data, it has the advantage of being transport protocol-independent, more extensible, and ensuring end-to-end security (versus point-to-point); it has the disadvantage of being several times slower than transport security mode because it has to deal with the XML nature of the SOAP messages.</span></span>  
  
 <span data-ttu-id="27dac-186">Bu farklılıklar hakkında daha fazla bilgi için bkz.  <span data-ttu-id="27dac-187">Üçüncü bir güvenlik modu, önceki modlarını kullanır ve her ikisinin de avantajını getirir.</span><span class="sxs-lookup"><span data-stu-id="27dac-187">A third security mode uses both previous modes and brings advantages of both.</span></span> <span data-ttu-id="27dac-188">Bu mod çağrılır `TransportWithMessageCredential` .</span><span class="sxs-lookup"><span data-stu-id="27dac-188">This mode is called `TransportWithMessageCredential`.</span></span> <span data-ttu-id="27dac-189">Bu modda, istemci kimliğini doğrulamak için ileti güvenliği kullanılır ve sunucunun kimliğini doğrulamak ve ileti gizliliği ve bütünlüğü sağlamak için aktarım güvenliği kullanılır.</span><span class="sxs-lookup"><span data-stu-id="27dac-189">In this mode, message security is used to authenticate the client and transport security is used to authenticate the server and provide message confidentiality and integrity.</span></span> <span data-ttu-id="27dac-190">Bu şekilde, `TransportWithMessageCredential` güvenlik modu, aktarım güvenliği modu kadar hızlı bir şekilde ve istemci kimlik doğrulaması genişletilebilirliği ile ileti güvenliği gibi aynı şekilde sağlanır.</span><span class="sxs-lookup"><span data-stu-id="27dac-190">Thanks to this, the `TransportWithMessageCredential` security mode is almost as fast as transport security mode and provides client authentication extensibility in the same way as message security.</span></span> <span data-ttu-id="27dac-191">Ancak, ileti güvenliği modunun aksine, uçtan uca tam güvenlik sağlamaz.</span><span class="sxs-lookup"><span data-stu-id="27dac-191">However, unlike message security mode, it does not provide complete end-to-end security.</span></span>  
  
### <a name="access-control"></a><span data-ttu-id="27dac-192">Erişim Denetimi</span><span class="sxs-lookup"><span data-stu-id="27dac-192">Access Control</span></span>
