# <a name="property-design"></a><span data-ttu-id="d0b4a-102">Özellik Tasarımı</span><span class="sxs-lookup"><span data-stu-id="d0b4a-102">Property Design</span></span>
<span data-ttu-id="d0b4a-103">Özellikler teknik açıdan yöntemlere çok benzer olsa da, bunlar kullanım senaryolarında oldukça farklıdır.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="d0b4a-104">Akıllı alanlar olarak görülenmelidir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-104">They should be seen as smart fields.</span></span> <span data-ttu-id="d0b4a-105">Alanların çağırma söz dizimi ve yöntemlerin esnekliği vardır.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>

 <span data-ttu-id="d0b4a-106">çağıranın özelliğin değerini değiştirememelidir ✔️ salt-al özellikleri oluşturun.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-106">✔️ DO create get-only properties if the caller should not be able to change the value of the property.</span></span>

 <span data-ttu-id="d0b4a-107">Özelliğin türü kesilebilir bir başvuru türü ise, özellik salt al olsa bile Özellik değerinin değiştirilebileceğini aklınızda bulundurun.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>

 <span data-ttu-id="d0b4a-108">❌, ayarlayıcıdan daha geniş erişilebilirliği olan, salt ayarlı özellikler veya özellikler sağlamaz.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-108">❌ DO NOT provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>

 <span data-ttu-id="d0b4a-109">Örneğin, özellikleri ortak bir ayarlayıcı ve korumalı bir alıcı ile birlikte kullanmayın.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-109">For example, do not use properties with a public setter and a protected getter.</span></span>

 <span data-ttu-id="d0b4a-110">Özellik alıcısı sağlanmadıysa, işlevi bir yöntem olarak uygulayın.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="d0b4a-111">Yöntem adını `Set` ile başlatmayı ve özelliği adlandırdıklarınızı takip etmeyi düşünün.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="d0b4a-112">Örneğin <xref:System.AppDomain>, `CachePath`adlı salt ayarlı bir özelliği olması yerine `SetCachePath` adlı bir yönteme sahiptir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>

 <span data-ttu-id="d0b4a-113">✔️ Tüm özellikler için, varsayılan değerlerin bir güvenlik deliğe veya daha fazla verimsiz bir koda neden olmamasını sağlamak için, tüm özellikler için hazır varsayılan değerler sağlar.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-113">✔️ DO provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>

 <span data-ttu-id="d0b4a-114">✔️, nesnenin geçici olarak geçersiz bir durumuna neden olsa da, özelliklerin herhangi bir sırada ayarlanmasına izin verir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-114">✔️ DO allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>

 <span data-ttu-id="d0b4a-115">İki veya daha fazla özelliğin, bir özelliğin bazı değerlerinin aynı nesne üzerindeki diğer özelliklerin değerleri verildiğinde geçersiz olabileceği bir nokta ile ilişkili olması yaygındır.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="d0b4a-116">Bu gibi durumlarda, ilişkili özellikler gerçekten nesne tarafından birlikte kullanıldığından, geçersiz durumdan kaynaklanan özel durumlar ertelenmelidir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>

 <span data-ttu-id="d0b4a-117">bir özellik ayarlayıcısı özel durum oluşturursa, önceki değeri koruma ✔️.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-117">✔️ DO preserve the previous value if a property setter throws an exception.</span></span>

 <span data-ttu-id="d0b4a-118">❌ özelliğinden özel durumlar oluşturmaktan kaçının.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-118">❌ AVOID throwing exceptions from property getters.</span></span>

 <span data-ttu-id="d0b4a-119">Özellik alıcıları basit işlemler olmalıdır ve herhangi bir önkoşullara sahip olmamalıdır.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="d0b4a-120">Alıcı bir özel durum oluştur, büyük olasılıkla bir yöntem olarak yeniden tasarlanması gerekir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="d0b4a-121">Bu kuralın, bağımsız değişkenlerin doğrulanması sonucunda özel durumlar beklediğimiz Dizin oluşturucular için uygulanmadığından emin olun.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>

### <a name="indexed-property-design"></a><span data-ttu-id="d0b4a-122">Dizinli özellik tasarımı</span><span class="sxs-lookup"><span data-stu-id="d0b4a-122">Indexed Property Design</span></span>
 <span data-ttu-id="d0b4a-123">Dizinli bir özellik, parametrelere sahip olabilecek ve dizi dizine benzer özel söz dizimi ile çağrılabilen özel bir özelliktir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>

 <span data-ttu-id="d0b4a-124">Dizinli özellikler genellikle Dizin oluşturucular olarak adlandırılır.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="d0b4a-125">Dizin oluşturucular yalnızca bir mantıksal koleksiyondaki öğelere erişim sağlayan API 'lerde kullanılmalıdır.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="d0b4a-126">Örneğin, bir dize bir karakter koleksiyonudur ve <xref:System.String?displayProperty=nameWithType> üzerindeki Dizin Oluşturucu, karakterlerine erişmek için eklenmiştir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>

 <span data-ttu-id="d0b4a-127">✔️, iç dizide depolanan verilere erişim sağlamak için Dizin oluşturucular kullanmayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-127">✔️ CONSIDER using indexers to provide access to data stored in an internal array.</span></span>

 <span data-ttu-id="d0b4a-128">✔️ öğe koleksiyonlarını temsil eden türlerde Dizin oluşturucular sağlamayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-128">✔️ CONSIDER providing indexers on types representing collections of items.</span></span>

 <span data-ttu-id="d0b4a-129">❌ birden fazla parametreli dizinli Özellikler kullanmaktan KAÇıNıN.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-129">❌ AVOID using indexed properties with more than one parameter.</span></span>

 <span data-ttu-id="d0b4a-130">Tasarım birden çok parametre gerektiriyorsa, özelliğin bir mantıksal koleksiyona bir erişimciyi temsil edip etmediğini yeniden düşünün.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="d0b4a-131">Yoksa, bunun yerine yöntemleri kullanın.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-131">If it does not, use methods instead.</span></span> <span data-ttu-id="d0b4a-132">Yöntem adını `Get` veya `Set`başlatmayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-132">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="d0b4a-133">❌ <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>veya enum dışında parametre türlerine sahip dizin oluşturuculardan KAÇıNıN.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-133">❌ AVOID indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>

 <span data-ttu-id="d0b4a-134">Tasarımda diğer parametre türleri gerekiyorsa, API 'nin mantıksal bir koleksiyona bir erişimciyi temsil edip etmediğini kesin olarak yeniden değerlendirin.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="d0b4a-135">Değilse, bir yöntemi kullanın.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-135">If it does not, use a method.</span></span> <span data-ttu-id="d0b4a-136">Yöntem adını `Get` veya `Set`başlatmayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-136">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="d0b4a-137">✔️, açıkça daha iyi bir ad olmadıkça, dizinli özellikler için `Item` adı kullanın (ör. `System.String`<xref:System.String.Chars%2A> özelliğine bakın).</span><span class="sxs-lookup"><span data-stu-id="d0b4a-137">✔️ DO use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>

 <span data-ttu-id="d0b4a-138">' C#De, Dizin oluşturucular varsayılan olarak adlandırılmış öğedir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="d0b4a-139">Bu adı özelleştirmek için <xref:System.Runtime.CompilerServices.IndexerNameAttribute> kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>

 <span data-ttu-id="d0b4a-140">❌, hem bir Dizin Oluşturucu hem de anlamsal olarak eşdeğer yöntemler sağlamaz.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-140">❌ DO NOT provide both an indexer and methods that are semantically equivalent.</span></span>

 <span data-ttu-id="d0b4a-141">❌, tek bir türde çok sayıda aşırı yüklenmiş dizin grubu sağlamaz.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-141">❌ DO NOT provide more than one family of overloaded indexers in one type.</span></span>

 <span data-ttu-id="d0b4a-142">Bu, C# derleyici tarafından zorlanır.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-142">This is enforced by the C# compiler.</span></span>

 <span data-ttu-id="d0b4a-143">❌ varsayılan olmayan dizinli özellikleri kullanmaz.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-143">❌ DO NOT use nondefault indexed properties.</span></span>

 <span data-ttu-id="d0b4a-144">Bu, C# derleyici tarafından zorlanır.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-144">This is enforced by the C# compiler.</span></span>

### <a name="property-change-notification-events"></a><span data-ttu-id="d0b4a-145">Özellik değişiklik bildirimi olayları</span><span class="sxs-lookup"><span data-stu-id="d0b4a-145">Property Change Notification Events</span></span>
 <span data-ttu-id="d0b4a-146">Bazen bir özellik değerindeki değişiklikleri kullanıcıya bildiren bir olay sağlamak yararlı olur.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="d0b4a-147">Örneğin, `System.Windows.Forms.Control`, `Text` özelliğinin değeri değiştirildikten sonra `TextChanged` olayını harekete geçirir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>

 <span data-ttu-id="d0b4a-148">✔️, üst düzey API 'lerde (genellikle Tasarımcı bileşenleri) özellik değerleri değiştirildiğinde değişiklik bildirimi olaylarını yükseltmeyi göz önünde bulundurun.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-148">✔️ CONSIDER raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>

 <span data-ttu-id="d0b4a-149">Bir kullanıcının bir nesnenin özelliğinin ne zaman değiştiğini bilmesini sağlamak için iyi bir senaryo varsa, nesne özellik için bir değişiklik bildirimi olayı tetiklemelidir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>

 <span data-ttu-id="d0b4a-150">Bununla birlikte, temel türler veya Koleksiyonlar gibi alt düzey API 'Ler için bu tür olayları yükseltme yüküyle ilgili ek yükün olması düşüktür.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="d0b4a-151">Örneğin, <xref:System.Collections.Generic.List%601>, listeye yeni bir öğe eklendiğinde ve `Count` özelliği değiştiğinde bu tür olayları yükseltemez.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>

 <span data-ttu-id="d0b4a-152">✔️, bir özelliğin değeri dış güçlerle değiştirildiğinde değişiklik bildirimi olaylarını yükseltmeyi göz önünde bulundurun.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-152">✔️ CONSIDER raising change notification events when the value of a property changes via external forces.</span></span>

 <span data-ttu-id="d0b4a-153">Bir özellik değeri, bazı dış zorlanarak (nesne üzerinde Yöntemler çağırma dışında bir şekilde) değişirse, olay oluşturma, geliştiricinin değerin değiştiğini ve değiştiğini gösterir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="d0b4a-154">İyi bir örnek, metin kutusu denetiminin `Text` özelliğidir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="d0b4a-155">Kullanıcı bir `TextBox`metin yazdığında, özellik değeri otomatik olarak değişir.</span><span class="sxs-lookup"><span data-stu-id="d0b4a-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
