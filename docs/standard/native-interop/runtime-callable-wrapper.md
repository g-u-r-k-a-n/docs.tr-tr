# <a name="runtime-callable-wrapper"></a><span data-ttu-id="cf7a5-102">Çalışma Zamanı Aranabilir Sarmalayıcısı</span><span class="sxs-lookup"><span data-stu-id="cf7a5-102">Runtime Callable Wrapper</span></span>
<span data-ttu-id="cf7a5-103">Ortak dil çalışma zamanı, COM nesnelerini, çalışma zamanı çağrılabilir sarmalayıcı (RCW) adlı bir ara sunucu üzerinden kullanıma sunar.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-103">The common language runtime exposes COM objects through a proxy called the runtime callable wrapper (RCW).</span></span> <span data-ttu-id="cf7a5-104">RCW, .NET istemcilerine sıradan bir nesne gibi görünse de, birincil işlevi bir .NET istemcisiyle COM nesnesi arasındaki çağrıları sıralayamaz.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-104">Although the RCW appears to be an ordinary object to .NET clients, its primary function is to marshal calls between a .NET client and a COM object.</span></span>  
  
 <span data-ttu-id="cf7a5-105">Çalışma zamanı, bu nesnede bulunan başvuruların sayısından bağımsız olarak her bir COM nesnesi için tam olarak bir RCW oluşturur.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-105">The runtime creates exactly one RCW for each COM object, regardless of the number of references that exist on that object.</span></span> <span data-ttu-id="cf7a5-106">Çalışma zamanı her bir nesne için işlem başına tek bir RCW tutar.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-106">The runtime maintains a single RCW per process for each object.</span></span>  <span data-ttu-id="cf7a5-107">Bir uygulama etki alanında veya grupta bir RCW oluşturup daha sonra başka bir uygulama etki alanına veya gruba bir başvuru geçirirseniz, ilk nesneye yönelik bir ara sunucu kullanılacaktır.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-107">If you create an RCW in one application domain or apartment, and then pass a reference to another application domain or apartment, a proxy to the first object will be used.</span></span>  <span data-ttu-id="cf7a5-108">Aşağıdaki çizimde gösterildiği gibi, herhangi bir sayıda yönetilen istemci, INEW ve INewer arabirimlerini sunan COM nesnelerine bir başvuru tutabilir.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-108">As the following illustration shows, any number of managed clients can hold a reference to the COM objects that expose INew and INewer interfaces.</span></span>  

<span data-ttu-id="cf7a5-109">Aşağıdaki görüntüde, çalışma zamanında çağrılabilir sarmalayıcı aracılığıyla COM nesnelerine erişme işlemi gösterilmektedir:</span><span class="sxs-lookup"><span data-stu-id="cf7a5-109">The following image shows the process for accessing COM objects through the runtime callable wrapper:</span></span>

 ![RCW aracılığıyla COM nesnelerine erişme işlemi.](./media/runtime-callable-wrapper/runtime-callable-wrapper.gif)  

 <span data-ttu-id="cf7a5-111">Bir tür kitaplığından türetilmiş meta verileri kullanarak, çalışma zamanı hem çağrılan COM nesnesini hem de bu nesne için bir sarmalayıcı oluşturur.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-111">Using metadata derived from a type library, the runtime creates both the COM object being called and a wrapper for that object.</span></span> <span data-ttu-id="cf7a5-112">Her RCW, nesne sarmaladığı COM nesnesinde bir arabirim işaretçileri önbelleği tutar ve RCW artık gerekmiyorsa COM nesnesinde başvurusunu yayınlar.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-112">Each RCW maintains a cache of interface pointers on the COM object it wraps and releases its reference on the COM object when the RCW is no longer needed.</span></span> <span data-ttu-id="cf7a5-113">Çalışma zamanı RCW üzerinde çöp toplama işlemi gerçekleştirir.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-113">The runtime performs garbage collection on the RCW.</span></span>  
  
 <span data-ttu-id="cf7a5-114">Diğer etkinliklerin yanı sıra, RCW, Sarmalanan nesne adına yönetilen ve yönetilmeyen kod arasında veri sıraladığında.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-114">Among other activities, the RCW marshals data between managed and unmanaged code, on behalf of the wrapped object.</span></span> <span data-ttu-id="cf7a5-115">Özellikle, RCW, istemci ve sunucu aralarında geçen verilerin farklı temsillerine sahip olduğunda, yöntem bağımsız değişkenleri ve yöntem dönüş değerleri için sıralama sağlar.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-115">Specifically, the RCW provides marshaling for method arguments and method return values whenever the client and server have different representations of the data passed between them.</span></span>  
  
 <span data-ttu-id="cf7a5-116">Standart sarmalayıcı, yerleşik sıralama kurallarını uygular.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-116">The standard wrapper enforces built-in marshaling rules.</span></span> <span data-ttu-id="cf7a5-117">Örneğin, bir .NET istemcisi bir dize türünü yönetilmeyen bir nesneye bir bağımsız değişkenin parçası olarak geçirdiğinde, sarmalayıcı dizeyi bir BSTR türüne dönüştürür.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-117">For example, when a .NET client passes a String type as part of an argument to an unmanaged object, the wrapper converts the string to a BSTR type.</span></span> <span data-ttu-id="cf7a5-118">COM nesnesi, yönetilen çağıranına bir BSTR döndürmelidir, çağıran bir dize alır.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-118">Should the COM object return a BSTR to its managed caller, the caller receives a String.</span></span> <span data-ttu-id="cf7a5-119">Hem istemci hem de sunucu onlara tanıdık gelen verileri gönderir ve alır.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-119">Both the client and the server send and receive data that is familiar to them.</span></span> <span data-ttu-id="cf7a5-120">Diğer türler dönüştürme gerektirmez.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-120">Other types require no conversion.</span></span> <span data-ttu-id="cf7a5-121">Örneğin, standart bir sarmalayıcı, türü dönüştürmeden yönetilen ve yönetilmeyen kod arasında her zaman 4 baytlık bir tamsayı geçilecektir.</span><span class="sxs-lookup"><span data-stu-id="cf7a5-121">For instance, a standard wrapper will always pass a 4-byte integer between managed and unmanaged code without converting the type.</span></span>
