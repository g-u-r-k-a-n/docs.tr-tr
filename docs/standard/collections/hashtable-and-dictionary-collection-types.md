 <span data-ttu-id="a9ef2-109">Genel <xref:System.Collections.Generic.HashSet%601> sınıf, benzersiz öğeleri kapsayan sıralanmamış bir koleksiyondur.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-109">The generic <xref:System.Collections.Generic.HashSet%601> class is an unordered collection for containing unique elements.</span></span>  
  
 <span data-ttu-id="a9ef2-110">Karma işlevi, bir anahtara dayalı sayısal bir karma kodu döndüren algoritmadır.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-110">A hash function is an algorithm that returns a numeric hash code based on a key.</span></span> <span data-ttu-id="a9ef2-111">Anahtar, depolanan nesnenin bazı özelliğinin değeridir.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-111">The key is the value of some property of the object being stored.</span></span> <span data-ttu-id="a9ef2-112">Karma işlev her zaman aynı anahtar için aynı karma kodu döndürmelidir.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-112">A hash function must always return the same hash code for the same key.</span></span> <span data-ttu-id="a9ef2-113">Bir karma işlevin iki farklı anahtar için aynı karma kodu oluşturması mümkündür, ancak her benzersiz anahtar için benzersiz bir karma kod üreten bir karma işlev, karma tablodaki öğeleri alırken daha iyi performans elde edilir.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-113">It is possible for a hash function to generate the same hash code for two different keys, but a hash function that generates a unique hash code for each unique key results in better performance when retrieving elements from the hash table.</span></span>  
  
 <span data-ttu-id="a9ef2-114">Bir öğesi olarak kullanılan her bir nesne <xref:System.Collections.Hashtable> , yönteminin bir uygulamasını kullanarak kendisi için bir karma kod oluşturabilmelidir <xref:System.Object.GetHashCode%2A> .</span><span class="sxs-lookup"><span data-stu-id="a9ef2-114">Each object that is used as an element in a <xref:System.Collections.Hashtable> must be able to generate a hash code for itself by using an implementation of the <xref:System.Object.GetHashCode%2A> method.</span></span> <span data-ttu-id="a9ef2-115">Ancak, <xref:System.Collections.Hashtable> <xref:System.Collections.Hashtable> parametrelerinden biri olarak bir uygulamayı kabul eden bir oluşturucuyu kullanarak içindeki tüm öğeleri için bir karma işlevi de belirtebilirsiniz <xref:System.Collections.IHashCodeProvider> .</span><span class="sxs-lookup"><span data-stu-id="a9ef2-115">However, you can also specify a hash function for all elements in a <xref:System.Collections.Hashtable> by using a <xref:System.Collections.Hashtable> constructor that accepts an <xref:System.Collections.IHashCodeProvider> implementation as one of its parameters.</span></span>  
  
 <span data-ttu-id="a9ef2-116">Bir nesne öğesine eklendiğinde <xref:System.Collections.Hashtable> , nesnenin karma koduyla eşleşen karma kodla ilişkili demet 'de depolanır.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-116">When an object is added to a <xref:System.Collections.Hashtable>, it is stored in the bucket that is associated with the hash code that matches the object's hash code.</span></span> <span data-ttu-id="a9ef2-117">İçinde için bir değer aranmakta olduğunda <xref:System.Collections.Hashtable> , bu değer için karma kodu oluşturulur ve bu karma kodla ilişkili demet aranır.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-117">When a value is being searched for in the <xref:System.Collections.Hashtable>, the hash code is generated for that value, and the bucket associated with that hash code is searched.</span></span>  
  
 <span data-ttu-id="a9ef2-118">Örneğin, bir dize için bir karma işlevi dizedeki her bir karakterin ASCII kodlarını alabilir ve bir karma kod oluşturmak için bunları birlikte ekleyebilir.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-118">For example, a hash function for a string might take the ASCII codes of each character in the string and add them together to generate a hash code.</span></span> <span data-ttu-id="a9ef2-119">"Picnic" dizesinin "Sepet" dizesinin karma kodundan farklı bir karma kodu olmalıdır; Bu nedenle, "Picnic" ve "Sepet" dizeleri farklı demetlere ait olacaktır.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-119">The string "picnic" would have a hash code that is different from the hash code for the string "basket"; therefore, the strings "picnic" and "basket" would be in different buckets.</span></span> <span data-ttu-id="a9ef2-120">Buna karşılık "stthere" ve "destts" aynı karma koda sahip olur ve aynı demet içinde olur.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-120">In contrast, "stressed" and "desserts" would have the same hash code and would be in the same bucket.</span></span>  
  
 <span data-ttu-id="a9ef2-121"><xref:System.Collections.Generic.Dictionary%602>Ve <xref:System.Collections.Concurrent.ConcurrentDictionary%602> sınıfları sınıfıyla aynı işlevselliğe sahiptir <xref:System.Collections.Hashtable> .</span><span class="sxs-lookup"><span data-stu-id="a9ef2-121">The <xref:System.Collections.Generic.Dictionary%602> and <xref:System.Collections.Concurrent.ConcurrentDictionary%602> classes have the same functionality as the <xref:System.Collections.Hashtable> class.</span></span> <span data-ttu-id="a9ef2-122"><xref:System.Collections.Generic.Dictionary%602>Belirli bir türün (dışındaki <xref:System.Object> ) bir değeri, bir değer türü için daha iyi performans sağlar <xref:System.Collections.Hashtable> .</span><span class="sxs-lookup"><span data-stu-id="a9ef2-122">A <xref:System.Collections.Generic.Dictionary%602> of a specific type (other than <xref:System.Object>) provides better performance than a <xref:System.Collections.Hashtable> for value types.</span></span> <span data-ttu-id="a9ef2-123">Bunun nedeni, öğesinin <xref:System.Collections.Hashtable> türünde olması <xref:System.Object> , bu nedenle kutulama ve kutudan çıkarma genellikle bir değer türünü depoladığınızda veya alırken meydana gelir.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-123">This is because the elements of <xref:System.Collections.Hashtable> are of type <xref:System.Object>; therefore, boxing and unboxing typically occur when you store or retrieve a value type.</span></span> <span data-ttu-id="a9ef2-124"><xref:System.Collections.Concurrent.ConcurrentDictionary%602>Birden çok iş parçacığı koleksiyona aynı anda erişildiğinde sınıf kullanılmalıdır.</span><span class="sxs-lookup"><span data-stu-id="a9ef2-124">The <xref:System.Collections.Concurrent.ConcurrentDictionary%602> class should be used when multiple threads might be accessing the collection simultaneously.</span></span>
