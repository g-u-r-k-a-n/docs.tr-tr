 <span data-ttu-id="a4c4e-166">✔️ bellek tüketimini en aza indirmek ve performansı en üst düzeye çıkarmak için alt düzey API 'lerde dizileri kullanmayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-166">✔️ CONSIDER using arrays in low-level APIs to minimize memory consumption and maximize performance.</span></span>

 <span data-ttu-id="a4c4e-167">✔️ bayt koleksiyonları yerine bayt dizilerini kullanın.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-167">✔️ DO use byte arrays instead of collections of bytes.</span></span>

 <span data-ttu-id="a4c4e-168">Özellik alıcısı her çağrıldığında özelliğin yeni bir dizi (örn. bir iç dizinin kopyası) döndürmesi gerekiyorsa ❌ özellikler için diziler kullanmayın.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-168">❌ DO NOT use arrays for properties if the property would have to return a new array (e.g., a copy of an internal array) every time the property getter is called.</span></span>

## <a name="implementing-custom-collections"></a><span data-ttu-id="a4c4e-169">Özel Koleksiyonlar uygulama</span><span class="sxs-lookup"><span data-stu-id="a4c4e-169">Implementing Custom Collections</span></span>
 <span data-ttu-id="a4c4e-170">✔️, yeni koleksiyonlar tasarlarken `Collection<T>`, `ReadOnlyCollection<T>`veya `KeyedCollection<TKey,TItem>` devralmayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-170">✔️ CONSIDER inheriting from `Collection<T>`, `ReadOnlyCollection<T>`, or `KeyedCollection<TKey,TItem>` when designing new collections.</span></span>

 <span data-ttu-id="a4c4e-171">yeni koleksiyonlar tasarlarken `IEnumerable<T>` uygulama ✔️.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-171">✔️ DO implement `IEnumerable<T>` when designing new collections.</span></span> <span data-ttu-id="a4c4e-172">`ICollection<T>`, hatta anlamlı olduğu yerde `IList<T>` uygulamayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-172">Consider implementing `ICollection<T>` or even `IList<T>` where it makes sense.</span></span>

 <span data-ttu-id="a4c4e-173">Böyle bir özel koleksiyon uygularken, `Collection<T>` tarafından belirlenen API düzenine ve mümkün olduğunca yakından `ReadOnlyCollection<T>` uygulayın.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-173">When implementing such custom collection, follow the API pattern established by `Collection<T>` and `ReadOnlyCollection<T>` as closely as possible.</span></span> <span data-ttu-id="a4c4e-174">Diğer bir deyişle, aynı üyeleri açıkça uygular, parametreleri bu iki koleksiyon gibi adlandırın ve daha fazlasını yapın.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-174">That is, implement the same members explicitly, name the parameters like these two collections name them, and so on.</span></span>

 <span data-ttu-id="a4c4e-175">Koleksiyon genellikle bu arabirimleri giriş olarak alan API 'lere geçiriliyorsa, genel olmayan koleksiyon arabirimlerini (`IList` ve `ICollection`) uygulamayı düşünün ✔️.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-175">✔️ CONSIDER implementing nongeneric collection interfaces (`IList` and `ICollection`) if the collection will often be passed to APIs taking these interfaces as input.</span></span>

 <span data-ttu-id="a4c4e-176">❌ koleksiyon kavramıyla ilgisi olmayan karmaşık API 'Ler olan türlerde koleksiyon arabirimleri uygulamaktan KAÇıNıN.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-176">❌ AVOID implementing collection interfaces on types with complex APIs unrelated to the concept of a collection.</span></span>

 <span data-ttu-id="a4c4e-177">❌, `CollectionBase`gibi genel olmayan taban koleksiyonlarından kalıtımla almalardır.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-177">❌ DO NOT inherit from nongeneric base collections such as `CollectionBase`.</span></span> <span data-ttu-id="a4c4e-178">Bunun yerine `Collection<T>`, `ReadOnlyCollection<T>`ve `KeyedCollection<TKey,TItem>` kullanın.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-178">Use `Collection<T>`, `ReadOnlyCollection<T>`, and `KeyedCollection<TKey,TItem>` instead.</span></span>

### <a name="naming-custom-collections"></a><span data-ttu-id="a4c4e-179">Özel koleksiyonları adlandırma</span><span class="sxs-lookup"><span data-stu-id="a4c4e-179">Naming Custom Collections</span></span>
 <span data-ttu-id="a4c4e-180">Koleksiyonlar (`IEnumerable`uygulayan türler) genellikle iki nedenden dolayı oluşturulur: (1) yapıya özgü işlemlere sahip yeni bir veri yapısı ve genellikle varolan veri yapılarından (örn., <xref:System.Collections.Generic.List%601>, <xref:System.Collections.Generic.LinkedList%601>, <xref:System.Collections.Generic.Stack%601>) ve (2) belirli bir öğe kümesini tutmak için özel bir koleksiyon oluşturmak için (örn., <xref:System.Collections.Specialized.StringCollection>).</span><span class="sxs-lookup"><span data-stu-id="a4c4e-180">Collections (types that implement `IEnumerable`) are created mainly for two reasons: (1) to create a new data structure with structure-specific operations and often different performance characteristics than existing data structures (e.g.,  <xref:System.Collections.Generic.List%601>, <xref:System.Collections.Generic.LinkedList%601>, <xref:System.Collections.Generic.Stack%601>), and (2) to create a specialized collection for holding a specific set of items (e.g.,  <xref:System.Collections.Specialized.StringCollection>).</span></span> <span data-ttu-id="a4c4e-181">Veri yapıları genellikle uygulamaların ve kitaplıkların iç uygulamalarında kullanılır.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-181">Data structures are most often used in the internal implementation of applications and libraries.</span></span> <span data-ttu-id="a4c4e-182">Özel Koleksiyonlar genellikle API 'lerde (özellik ve parametre türleri olarak) gösterilmelidir.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-182">Specialized collections are mainly to be exposed in APIs (as property and parameter types).</span></span>

 <span data-ttu-id="a4c4e-183">✔️ `IDictionary` veya `IDictionary<TKey,TValue>`uygulayan soyutlamalar adlarında "Sözlük" sonekini kullanın.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-183">✔️ DO use the "Dictionary" suffix in names of abstractions implementing `IDictionary` or `IDictionary<TKey,TValue>`.</span></span>

 <span data-ttu-id="a4c4e-184">✔️, `IEnumerable` (veya alt öğelerinden herhangi biri) uygulayan ve öğelerin bir listesini temsil eden türlerin adlarında "Collection" sonekini kullanır.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-184">✔️ DO use the "Collection" suffix in names of types implementing `IEnumerable` (or any of its descendants) and representing a list of items.</span></span>

 <span data-ttu-id="a4c4e-185">✔️ özel veri yapıları için uygun veri yapısı adını kullanın.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-185">✔️ DO use the appropriate data structure name for custom data structures.</span></span>

 <span data-ttu-id="a4c4e-186">❌, koleksiyon soyutlamaları adlarında "LinkedList" veya "Hashtable" gibi belirli bir uygulamadan oluşan herhangi bir sonek kullanmaktan kaçının.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-186">❌ AVOID using any suffixes implying particular implementation, such as "LinkedList" or "Hashtable," in names of collection abstractions.</span></span>

 <span data-ttu-id="a4c4e-187">✔️ koleksiyon adlarını öğe türü adı ile önek olarak kullanmayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-187">✔️ CONSIDER prefixing collection names with the name of the item type.</span></span> <span data-ttu-id="a4c4e-188">Örneğin, `Address` türünde öğeleri depolayan bir koleksiyon (`IEnumerable<Address>`uygulayan) `AddressCollection`olarak adlandırılmalıdır.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-188">For example, a collection storing items of type `Address` (implementing `IEnumerable<Address>`) should be named `AddressCollection`.</span></span> <span data-ttu-id="a4c4e-189">Öğe türü bir arabirimse, öğe türünün "I" öneki atlanabilir.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-189">If the item type is an interface, the "I" prefix of the item type can be omitted.</span></span> <span data-ttu-id="a4c4e-190">Bu nedenle, <xref:System.IDisposable> öğelerinin bir koleksiyonu `DisposableCollection`çağrılabilir.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-190">Thus, a collection of <xref:System.IDisposable> items can be called `DisposableCollection`.</span></span>

 <span data-ttu-id="a4c4e-191">✔️ ilgili yazılabilir bir koleksiyon eklenirse veya çerçevede zaten mevcutsa, salt okunur koleksiyonların adlarında "ReadOnly" önekini kullanmayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-191">✔️ CONSIDER using the "ReadOnly" prefix in names of read-only collections if a corresponding writeable collection might be added or already exists in the framework.</span></span>

 <span data-ttu-id="a4c4e-192">Örneğin, salt okunurdur bir dize koleksiyonu `ReadOnlyStringCollection`çağrılmalıdır.</span><span class="sxs-lookup"><span data-stu-id="a4c4e-192">For example, a read-only collection of strings should be called `ReadOnlyStringCollection`.</span></span>
