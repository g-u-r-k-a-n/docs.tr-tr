 <span data-ttu-id="189c2-167">Birçok derleyicilerin bağımsız değişkenlerini çağrı sitesinde geçici bir diziye değiştirdiğinden, dizi geçici bir nesne olabilir ve bu nedenle dizideki tüm değişiklikler kaybedilir.</span><span class="sxs-lookup"><span data-stu-id="189c2-167">Because of the fact that many compilers turn the arguments to the member into a temporary array at the call site, the array might be a temporary object, and therefore any modifications to the array will be lost.</span></span>  
  
 <span data-ttu-id="189c2-168">**✓ CONSIDER** daha karmaşık bir aşırı kullanılamadı olsa bile bir basit aşırı params anahtar sözcüğü kullanarak.</span><span class="sxs-lookup"><span data-stu-id="189c2-168">**✓ CONSIDER** using the params keyword in a simple overload, even if a more complex overload could not use it.</span></span>  
  
 <span data-ttu-id="189c2-169">Tüm aşırı yüklerde olmasa bile kullanıcılar bir aşırı yüklemede params dizisine sahip olacaksa kendinize sorun.</span><span class="sxs-lookup"><span data-stu-id="189c2-169">Ask yourself if users would value having the params array in one overload even if it wasn’t in all overloads.</span></span>  
  
 <span data-ttu-id="189c2-170">**✓ DO** params anahtar sözcüğü kullanmak mümkün kılmak için sipariş parametreleri deneyin.</span><span class="sxs-lookup"><span data-stu-id="189c2-170">**✓ DO** try to order parameters to make it possible to use the params keyword.</span></span>  
  
 <span data-ttu-id="189c2-171">**✓ CONSIDER** küçük sayıda bağımsız değişken son derece performans duyarlı API'lerde aramaları için özel aşırı ve kod yolları sağlama.</span><span class="sxs-lookup"><span data-stu-id="189c2-171">**✓ CONSIDER** providing special overloads and code paths for calls with a small number of arguments in extremely performance-sensitive APIs.</span></span>  
  
 <span data-ttu-id="189c2-172">Bu, API, az sayıda bağımsız değişkenle çağrıldığında dizi nesneleri oluşturmaktan kaçınılmasını mümkün kılar.</span><span class="sxs-lookup"><span data-stu-id="189c2-172">This makes it possible to avoid creating array objects when the API is called with a small number of arguments.</span></span> <span data-ttu-id="189c2-173">Parametre adlarını, dizi parametresinin tekil bir biçimini alarak ve sayısal bir sonek ekleyerek oluşturur.</span><span class="sxs-lookup"><span data-stu-id="189c2-173">Form the names of the parameters by taking a singular form of the array parameter and adding a numeric suffix.</span></span>  
  
 <span data-ttu-id="189c2-174">Bunu yalnızca bir dizi oluşturun ve daha genel yöntemi çağırmak için kod yolunun tamamına özel durum oluşturacaksanız yapmanız gerekir.</span><span class="sxs-lookup"><span data-stu-id="189c2-174">You should only do this if you are going to special-case the entire code path, not just create an array and call the more general method.</span></span>  
  
 <span data-ttu-id="189c2-175">**✓ DO** bu null params dizisi bağımsız değişken olarak geçirilen unutmayın.</span><span class="sxs-lookup"><span data-stu-id="189c2-175">**✓ DO** be aware that null could be passed as a params array argument.</span></span>  
  
 <span data-ttu-id="189c2-176">İşlemeden önce dizinin null olduğunu doğrulamanız gerekir.</span><span class="sxs-lookup"><span data-stu-id="189c2-176">You should validate that the array is not null before processing.</span></span>  
  
 <span data-ttu-id="189c2-177">**X DO NOT** kullanmak `varargs` yöntemleri, aksi takdirde üç nokta bilinir.</span><span class="sxs-lookup"><span data-stu-id="189c2-177">**X DO NOT** use the `varargs` methods, otherwise known as the ellipsis.</span></span>  
  
 <span data-ttu-id="189c2-178">Gibi bazı CLR dilleri C++, `varargs` Yöntemler adlı değişken parametre listelerinin geçirilmesi için alternatif bir kural destekler.</span><span class="sxs-lookup"><span data-stu-id="189c2-178">Some CLR languages, such as C++, support an alternative convention for passing variable parameter lists called `varargs` methods.</span></span> <span data-ttu-id="189c2-179">Bu kural, CLS uyumlu olmadığından çerçeve içinde kullanılmamalıdır.</span><span class="sxs-lookup"><span data-stu-id="189c2-179">The convention should not be used in frameworks, because it is not CLS compliant.</span></span>  
  
### <a name="pointer-parameters"></a><span data-ttu-id="189c2-180">İşaretçi parametreleri</span><span class="sxs-lookup"><span data-stu-id="189c2-180">Pointer Parameters</span></span>  
 <span data-ttu-id="189c2-181">Genel olarak, işaretçiler iyi tasarlanmış bir yönetilen kod çerçevesinin genel yüzey alanında görünmemelidir.</span><span class="sxs-lookup"><span data-stu-id="189c2-181">In general, pointers should not appear in the public surface area of a well-designed managed code framework.</span></span> <span data-ttu-id="189c2-182">Çoğu zaman işaretçiler kapsüllenmelidir.</span><span class="sxs-lookup"><span data-stu-id="189c2-182">Most of the time, pointers should be encapsulated.</span></span> <span data-ttu-id="189c2-183">Ancak bazı durumlarda, birlikte çalışabilirlik nedenleriyle işaretçiler gereklidir ve bu gibi durumlarda işaretçiler kullanılması uygundur.</span><span class="sxs-lookup"><span data-stu-id="189c2-183">However, in some cases pointers are required for interoperability reasons, and using pointers in such cases is appropriate.</span></span>  
  
 <span data-ttu-id="189c2-184">**✓ DO** işaretçileri CLS ile uyumlu olmadığı için bir işaretçi bağımsız değişken herhangi bir üyesi için bir alternatif sunar.</span><span class="sxs-lookup"><span data-stu-id="189c2-184">**✓ DO** provide an alternative for any member that takes a pointer argument, because pointers are not CLS-compliant.</span></span>  
  
 <span data-ttu-id="189c2-185">**X AVOID** pahalı bağımsız değişkeni işaretçi bağımsız denetimi yapılması.</span><span class="sxs-lookup"><span data-stu-id="189c2-185">**X AVOID** doing expensive argument checking of pointer arguments.</span></span>  
  
 <span data-ttu-id="189c2-186">**✓ DO** işaretçisi ile ilgili genel kurallar işaretçileri üyeleriyle tasarlarken izleyin.</span><span class="sxs-lookup"><span data-stu-id="189c2-186">**✓ DO** follow common pointer-related conventions when designing members with pointers.</span></span>  
  
 <span data-ttu-id="189c2-187">Örneğin, başlangıç dizinini geçirmeniz gerekmez, çünkü basit işaretçi aritmetiği aynı sonucu elde etmek için kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="189c2-187">For example, there is no need to pass the start index, because simple pointer arithmetic can be used to accomplish the same result.</span></span>
