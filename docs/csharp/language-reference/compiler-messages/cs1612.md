# <a name="compiler-error-cs1612"></a><span data-ttu-id="b2c00-102">Derleyici Hatası CS1612</span><span class="sxs-lookup"><span data-stu-id="b2c00-102">Compiler Error CS1612</span></span>
<span data-ttu-id="b2c00-103">Bir değişken olmadığından ' expression ' öğesinin dönüş değeri değiştirilemiyor</span><span class="sxs-lookup"><span data-stu-id="b2c00-103">Cannot modify the return value of 'expression' because it is not a variable</span></span>  
  
 <span data-ttu-id="b2c00-104">Bir ara ifadenin sonucu olarak üretilen ancak bir değişkende depolanmayan bir değer türünü değiştirme girişiminde bulunuldu.</span><span class="sxs-lookup"><span data-stu-id="b2c00-104">An attempt was made to modify a value type that is produced as the result of an intermediate expression but is not stored in a variable.</span></span> <span data-ttu-id="b2c00-105">Bu hata, aşağıdaki örnekte gösterildiği gibi genel bir koleksiyondaki bir yapıyı doğrudan değiştirmeye çalıştığınızda ortaya çıkabilir:</span><span class="sxs-lookup"><span data-stu-id="b2c00-105">This error can occur when you attempt to directly modify a struct in a generic collection, as shown in the following example:</span></span>  
  
```csharp  
List<myStruct> list = {…};  
list[0].Name = "MyStruct42"; //CS1612  
```  
  
 <span data-ttu-id="b2c00-106">Yapıyı değiştirmek için, önce onu yerel bir değişkene atayın, değişkeni değiştirin ve sonra değişkeni koleksiyondaki öğeye geri atayın.</span><span class="sxs-lookup"><span data-stu-id="b2c00-106">To modify the struct, first assign it to a local variable, modify the variable, then assign the variable back to the item in the collection.</span></span>  
  
```csharp  
List<myStruct> list = {…};  
MyStruct ms = list[0];  
ms.Name = "MyStruct42";  
list[0] = ms;  
```  
  
 <span data-ttu-id="b2c00-107">Değer türleri atamaya kopyalanırken bu hata oluşur.</span><span class="sxs-lookup"><span data-stu-id="b2c00-107">This error occurs because value types are copied on assignment.</span></span> <span data-ttu-id="b2c00-108">Bir özellikten veya dizin oluşturucudan bir değer türü aldığınızda, nesnenin kendisi için bir başvuru değil, nesnenin bir kopyasını alıyorsunuz.</span><span class="sxs-lookup"><span data-stu-id="b2c00-108">When you retrieve a value type from a property or indexer, you are getting a copy of the object, not a reference to the object itself.</span></span> <span data-ttu-id="b2c00-109">Döndürülen kopya, depolama konumları (değişkenler) değil, gerçekten Yöntemler olduklarından özellik veya Dizin Oluşturucu tarafından depolanmaz.</span><span class="sxs-lookup"><span data-stu-id="b2c00-109">The copy that is returned is not stored by the property or indexer because they are actually methods, not storage locations (variables).</span></span> <span data-ttu-id="b2c00-110">Değişiklik yapabilmeniz için önce bildirdiğiniz bir değişkene kopyalamanız gerekir.</span><span class="sxs-lookup"><span data-stu-id="b2c00-110">You must store the copy into a variable that you declare before you can modify it.</span></span>  
  
 <span data-ttu-id="b2c00-111">Bu örnekte bir özellik veya Dizin Oluşturucu, depolama konumu olan var olan bir nesneye başvuru döndürdüğünden hata başvuru türlerinde oluşmaz.</span><span class="sxs-lookup"><span data-stu-id="b2c00-111">The error does not occur with reference types because a property or indexer in that case returns a reference to an existing object, which is a storage location.</span></span>  
  
 <span data-ttu-id="b2c00-112">Sınıf veya yapı tanımlıyorsanız, yapı üyelerine erişim sağlamak için özellik bildiriminizi değiştirerek bu hatayı çözebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="b2c00-112">If you are defining the class or struct, you can resolve this error by modifying your property declaration to provide access to the members of a struct.</span></span> <span data-ttu-id="b2c00-113">İstemci kodu yazıyorsanız, kendi yapı örneğinizi oluşturarak, alanlarını değiştirerek ve sonra yapının tamamını özelliğe yeniden atayarak hatayı çözebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="b2c00-113">If you are writing client code, you can resolve the error by creating your own instance of the struct, modifying its fields, and then assigning the entire struct back to the property.</span></span> <span data-ttu-id="b2c00-114">Üçüncü bir alternatif olarak, yapısını bir sınıfa dönüştürebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="b2c00-114">As a third alternative, you can change your struct to a class.</span></span>  
  
## <a name="example"></a><span data-ttu-id="b2c00-115">Örnek</span><span class="sxs-lookup"><span data-stu-id="b2c00-115">Example</span></span>  
 <span data-ttu-id="b2c00-116">CS1612 Ayrıca, aşağıdaki örnekte gösterildiği gibi yapının tamamını döndüren kapsayan bir sınıftaki bir özellik aracılığıyla bir yapının üyesine erişmeye çalıştığınızda da gerçekleşir:</span><span class="sxs-lookup"><span data-stu-id="b2c00-116">CS1612 also occurs when you attempt to access the member of a struct through a property on an enclosing class that is returning the entire struct, as shown in the following example:</span></span>  
  
```csharp  
// CS1612.cs  
using System;  
  
public struct MyStruct  
{  
    public int Width;  
}  
  
public class ListView  
{  
    MyStruct ms;  
    public MyStruct Size  
    {  
        get { return ms; }  
        set { ms = value; }  
    }  
}  
  
public class MyClass  
{  
    public MyClass()  
    {  
        ListView lvi;  
        lvi = new ListView();  
        lvi.Size.Width = 5; // CS1612  
  
        // You can use the following lines instead.  
        // MyStruct ms;  
        // ms.Width = 5;  
        // lvi.Size = ms;
    }  
  
    public static void Main()   
    {  
        MyClass mc = new MyClass();  
        // Keep the console open in debug mode.  
        Console.WriteLine("Press any key to exit.");  
        Console.ReadKey();     
    }  
}  
```
