<span data-ttu-id="71aa5-107">Sunucusuz bilgi işlem, uygulamanızı çalıştırmaya katılan bir sunucu olmadığı anlamına gelmez; kod hala sunucuda çalışır.</span><span class="sxs-lookup"><span data-stu-id="71aa5-107">Serverless computing doesn't mean there isn't a server involved in running your application - the code still runs on a server somewhere.</span></span> <span data-ttu-id="71aa5-108">Ayrım, uygulama geliştirme ekibinin sunucu altyapısını yönetme konusunda artık ilgilenmesini gerektirmez.</span><span class="sxs-lookup"><span data-stu-id="71aa5-108">The distinction is that the application development team no longer needs to concern themselves with managing server infrastructure.</span></span> <span data-ttu-id="71aa5-109">Azure Işlevleri gibi sunucusuz bilgi işlem çözümleri ekiplerin üretkenliğini artırmasına yardımcı olur ve kuruluşların kaynaklarını iyileştirmelerine ve çözümler sunmaya odaklanmasına olanak tanır.</span><span class="sxs-lookup"><span data-stu-id="71aa5-109">Serverless computing solutions like Azure Functions help teams increase their productivity and allow organizations to optimize their resources and focus on delivering solutions.</span></span>

<span data-ttu-id="71aa5-110">Sunucusuz bilgi işlem, uygulamanızı veya uygulamanızın bir bölümünü barındırmak için olay tarafından tetiklenen durum bilgisi olmayan kapsayıcılar kullanır.</span><span class="sxs-lookup"><span data-stu-id="71aa5-110">Serverless computing uses event-triggered stateless containers to host your application or part of your application.</span></span> <span data-ttu-id="71aa5-111">Sunucusuz platformlar, talebe göre isteğe bağlı olarak uygun şekilde ölçeklenebilir ve kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-111">Serverless platforms can scale out and in to meet demand as-needed.</span></span> <span data-ttu-id="71aa5-112">Azure Işlevleri gibi platformlar, kuyruklar, olaylar ve depolama gibi diğer Azure hizmetlerine kolayca doğrudan erişebilir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-112">Platforms like Azure Functions have easy direct access to other Azure services like queues, events, and storage.</span></span>

## <a name="what-scenarios-are-appropriate-for-serverless"></a><span data-ttu-id="71aa5-123">Sunucusuz için hangi senaryolar uygun?</span><span class="sxs-lookup"><span data-stu-id="71aa5-123">What scenarios are appropriate for serverless?</span></span>

<span data-ttu-id="71aa5-124">Sunucusuz, bazı tetikleyicisine yanıt olarak çağrılan tek başına kısa süreli işlevleri kullanır.</span><span class="sxs-lookup"><span data-stu-id="71aa5-124">Serverless uses individual short-running functions that are called in response to some trigger.</span></span> <span data-ttu-id="71aa5-125">Bu, arka plan görevlerinin işlenmesi için ideal hale getirir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-125">This makes them ideal for processing background tasks.</span></span>

<span data-ttu-id="71aa5-126">Örneğin, bir uygulamanın bir isteği işlemenin parçası olarak bir e-posta gönderebilmesi gerekebilir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-126">For example, an application might need to send an email as part of processing a request.</span></span> <span data-ttu-id="71aa5-127">Web isteğini işlemenin parçası olarak e-postayı göndermek yerine, e-postanın ayrıntıları bir kuyruğa yerleştirilecektir ve iletiyi almak ve e-postayı göndermek için bir Azure Işlevi kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-127">Instead of sending the email as part of handling the web request, the details of the email could be placed onto a queue and an Azure Function could be used to pick up the message and send the email.</span></span> <span data-ttu-id="71aa5-128">Uygulamanın birçok farklı bölümü veya çok sayıda uygulama, bu aynı Azure Işlevinden yararlanarak uygulamalar için iyileştirilmiş performans ve ölçeklenebilirlik sağlar ve ile ilgili sorunları önlemek için [kuyruk tabanlı yük dengelemeyi](https://docs.microsoft.com/azure/architecture/patterns/queue-based-load-leveling) kullanabilir e-postalar gönderiliyor.</span><span class="sxs-lookup"><span data-stu-id="71aa5-128">Many different parts of the application, or even many applications, could leverage this same Azure Function, providing improved performance and scalability for the applications and using [queue-based load leveling](https://docs.microsoft.com/azure/architecture/patterns/queue-based-load-leveling) to avoid bottlenecks related to sending the emails.</span></span>

<span data-ttu-id="71aa5-129">Uygulamalar ve Azure Işlevleri arasında bir [yayımcı/abonelik düzeni](https://docs.microsoft.com/azure/architecture/patterns/publisher-subscriber) en yaygın düzen olmakla birlikte, diğer desenler de mümkündür.</span><span class="sxs-lookup"><span data-stu-id="71aa5-129">Although a [Publisher/Subscriber pattern](https://docs.microsoft.com/azure/architecture/patterns/publisher-subscriber) between applications and Azure Functions is the most common pattern, other patterns are possible.</span></span> <span data-ttu-id="71aa5-130">Azure Işlevleri, Azure Blob depolamada yapılan değişiklikler gibi diğer olaylar tarafından tetiklenebilir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-130">Azure Functions can be triggered by other events, such as changes to Azure Blob Storage.</span></span> <span data-ttu-id="71aa5-131">Görüntü karşıya yüklemelerini destekleyen bir uygulama, küçük resimler oluşturmaktan veya karşıya yüklenen görüntüleri tutarlı boyutlara yeniden boyutlandırmaktan veya görüntü boyutunu en iyi duruma getirmekten sorumlu bir Azure Işlevine sahip olabilir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-131">An application that supported image uploads could have an Azure Function responsible for creating thumbnail images, or resizing uploaded images to consistent dimensions, or optimizing image size.</span></span> <span data-ttu-id="71aa5-132">Bu işlevlerin tümü, Azure Blob depolama alanına ekleme yaparak doğrudan tetiklenebilir ve karmaşıklık ve iş yükü uygulamanın kendisi tarafından yapılır.</span><span class="sxs-lookup"><span data-stu-id="71aa5-132">All of this functionality could be triggered directly by inserts to Azure Blob Storage, keeping the complexity and the workload out of the application itself.</span></span>

<span data-ttu-id="71aa5-133">Birçok uygulama, iş akışlarının bir parçası olarak uzun süredir çalışan işlemlere sahiptir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-133">Many applications have long-running processes as part of their workflows.</span></span> <span data-ttu-id="71aa5-134">Genellikle bu görevler kullanıcının uygulamayla etkileşimi kapsamında yapılır, böylece Kullanıcı, deneyimlerini bekleyip olumsuz şekilde etkiler.</span><span class="sxs-lookup"><span data-stu-id="71aa5-134">Often these tasks are done as part of the user's interaction with the application, forcing the user to wait and negatively impacting their experience.</span></span> <span data-ttu-id="71aa5-135">Sunucusuz bilgi işlem, Kullanıcı etkileşimi döngüsünün dışında daha yavaş görevler gerçekleştirmek için harika bir yol sağlar ve bu görevler tüm uygulamanın ölçeklendirilmesine gerek kalmadan isteğe bağlı olarak kolayca ölçeklenebilirler.</span><span class="sxs-lookup"><span data-stu-id="71aa5-135">Serverless computing provides a great way to perform slower tasks outside of the user interaction loop, and these tasks can easily scale with demand without requiring the entire application to scale.</span></span>

<span data-ttu-id="71aa5-137">Sunucusuz bilgi işlem, Kullanıcı arabirimini engellememe görevleri için en iyi seçenektir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-137">Serverless computing is best-used for tasks that don't block the user interface.</span></span> <span data-ttu-id="71aa5-138">Bu, Web uygulamalarını veya Web API 'Lerini doğrudan barındırmak için ideal olmadıkları anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-138">This means they're not ideal for hosting web applications or web APIs directly.</span></span> <span data-ttu-id="71aa5-139">Bunun ana nedeni sunucusuz çözümlerin talep üzerine sağlandığını ve ölçeklendirilmesine yöneliktir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-139">The main reason for this is that serverless solutions are provisioned and scaled on demand.</span></span> <span data-ttu-id="71aa5-140">Bir işlevin yeni bir örneği gerektiğinde, *soğuk başlangıç*olarak bahsedildiğinde, sağlanması zaman alır.</span><span class="sxs-lookup"><span data-stu-id="71aa5-140">When a new instance of a function is needed, referred to as a *cold start*, it takes time to provision.</span></span> <span data-ttu-id="71aa5-141">Bu süre genellikle birkaç saniyedir, ancak çeşitli faktörlere bağlı olarak daha uzun sürebilir.</span><span class="sxs-lookup"><span data-stu-id="71aa5-141">This time is typically a few seconds, but can be longer depending on a variety of factors.</span></span> <span data-ttu-id="71aa5-142">Tek bir örnek genellikle sonsuza kadar (örneğin, düzenli olarak bir istek yaparak) etkin bir şekilde tutulabilir, ancak her zaman ölçeği, her birinin ölçeklendirilmesi gerekiyorsa soğuk başlatma sorunu kalır.</span><span class="sxs-lookup"><span data-stu-id="71aa5-142">A single instance can often be kept alive indefinitely (for instance, by periodically making a request to it), but the cold start issue remains if the number of instances ever needs to scale up.</span></span>
