 <span data-ttu-id="98e17-104">Derleyici açık bir dönüştürme gerektirir.</span><span class="sxs-lookup"><span data-stu-id="98e17-104">The compiler requires an explicit conversion.</span></span> <span data-ttu-id="98e17-105">Örneğin, l değeriyle aynı türde olması için bir r değeri dökmeniz gerekebilir.</span><span class="sxs-lookup"><span data-stu-id="98e17-105">For example, you may need to cast an r-value to be the same type as an l-value.</span></span> <span data-ttu-id="98e17-106">Veya, belirli operatör aşırı yüklemelerini desteklemek için dönüşüm yordamları sağlamanız gerekir.</span><span class="sxs-lookup"><span data-stu-id="98e17-106">Or, you must provide conversion routines to support certain operator overloads.</span></span>

 <span data-ttu-id="98e17-110">`i = lng;`bir atama yapar, ancak atama işlecinin sol ve sağ tarafındaki değişkenlerin veri türleri eşleşmiyor.</span><span class="sxs-lookup"><span data-stu-id="98e17-110">`i = lng;` makes an assignment, but the data types of the variables on the left and right-hand side of the assignment operator don't match.</span></span> <span data-ttu-id="98e17-111">Atamayı yapmadan önce derleyici, yazı uzunluğundaki değişkeni `lng`dolaylı olarak bir int'e dönüştürür.  <span data-ttu-id="98e17-113">Bellekte dönüştürdüğünüz veri türünden daha az depolama alanı kaplayan bir veri türüne dönüştürme yaparken daraltma dönüştürme sayılsa.</span><span class="sxs-lookup"><span data-stu-id="98e17-113">A narrowing conversion exists when converting to a data type that occupies less storage space in memory than the data type we are converting from.</span></span> <span data-ttu-id="98e17-114">Örneğin, uzun bir int dönüştürme daralan bir dönüştürme olarak kabul edilir.</span><span class="sxs-lookup"><span data-stu-id="98e17-114">For example, converting a long to an int would be considered a narrowing conversion.</span></span> <span data-ttu-id="98e17-115">Bir int 4 bayt kaplar ise uzun bellek 8 bayt kaplar.</span><span class="sxs-lookup"><span data-stu-id="98e17-115">A long occupies 8 bytes of memory while an int occupies 4 bytes.</span></span> <span data-ttu-id="98e17-116">Veri kaybının nasıl oluşabileceğini görmek için aşağıdaki örneği göz önünde bulundurun:</span><span class="sxs-lookup"><span data-stu-id="98e17-116">To see how data loss can occur, consider the following sample:</span></span>

 <span data-ttu-id="98e17-117">Değişken `lng` şimdi çok büyük olduğu için değişkende `i` depolanamayan bir değer içerir.</span><span class="sxs-lookup"><span data-stu-id="98e17-117">The variable `lng` now contains a value that cannot be stored in the variable `i` because it is too large.</span></span> <span data-ttu-id="98e17-118">Bu değeri bir int türüne dönüştürecek olsaydık, bazı verilerimizi kaybederiz ve dönüştürülen değer dönüştürmeden önceki değerle aynı olmaz.</span><span class="sxs-lookup"><span data-stu-id="98e17-118">If we were to convert this value to an int type we would be losing some of our data and the converted value would not be the same as the value before the conversion.</span></span>

 <span data-ttu-id="98e17-122">Bu kod örneği ile ilki arasındaki farka dikkat edin.</span><span class="sxs-lookup"><span data-stu-id="98e17-122">Notice the difference between this code sample and the first.</span></span> <span data-ttu-id="98e17-123">Bu kez `lng` değişken atama işlecinin sol tarafında, bu nedenle atamamızın hedefidir.</span><span class="sxs-lookup"><span data-stu-id="98e17-123">This time the variable `lng` is on the left-hand side of the assignment operator, so it is the target of our assignment.</span></span> <span data-ttu-id="98e17-124">Atama yapılmadan önce, derleyicinin dolaylı olarak `i`int türünden olan değişkeni uzun yazmaya dönüştürmesi gerekir.</span><span class="sxs-lookup"><span data-stu-id="98e17-124">Before the assignment can be made, the compiler must implicitly convert the variable `i`, which is of type int, to type long.</span></span> <span data-ttu-id="98e17-125">Bu, 4 bayt bellek (int) içeren bir türden 8 bayt bellek (uzun) kaplayan bir türe dönüştürdüğünüz için genişleyen bir dönüşümdür.</span><span class="sxs-lookup"><span data-stu-id="98e17-125">This is a widening conversion since we are converting from a type that occupies 4 bytes of memory (an int) to a type that occupies 8 bytes of memory (a long).</span></span> <span data-ttu-id="98e17-126">Olası veri kaybı olmadığından örtülü genişletme dönüşümlerine izin verilir.</span><span class="sxs-lookup"><span data-stu-id="98e17-126">Implicit widening conversions are allowed because there is no potential loss of data.</span></span> <span data-ttu-id="98e17-127">Bir int'te depolanabilecek herhangi bir değer de uzun süre saklanabilir.</span><span class="sxs-lookup"><span data-stu-id="98e17-127">Any value that can be stored in an int can also be stored in a long.</span></span>
