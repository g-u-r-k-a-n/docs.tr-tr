# <a name="transport-security"></a>Taşıma Güvenliği
Windows Communication Foundation (WCF) içindeki taşıma güvenliği, seçilen bağlamaya bağlıdır. Bağlamanın uyguladığı aktarım, gerçek güvenlik mekanizmasını belirler. Bu bölümdeki konularda, uygulanan mekanizmalar ve bunların seçenekleri açıklanmaktadır.  
  
## <a name="in-this-section"></a>Bu Bölümde  
 [Aktarım Güvenliğine Genel Bakış](transport-security-overview.md)  
 WCF 'de aktarım güvenliği esaslarını açıklar.  
  
 [HTTP Aktarım Güvenliği](http-transport-security.md)  
 WCF 'nin Güvenli Yuva Katmanı (SSL veya HTTPS) nasıl uyguladığı açıklanmaktadır.  
  
 [HTTP Kimlik Doğrulamasını Anlama](understanding-http-authentication.md)  
 Temel, Özet, NT LAN Manager (NTLM) ve diğerleri gibi HTTP kimlik doğrulama düzenlerini açıklar.  
  
 [Taşıma Güvenliği ile Kimliğe Bürünme Kullanma](using-impersonation-with-transport-security.md)  
 Aktarım güvenliği moduyla mümkün olan beş kimliğe bürünme düzeyini açıklar.  
  
 [Nasıl Yapılır: SSL Sertifikası ile Bir Bağlantı Noktasını Yapılandırma](how-to-configure-a-port-with-an-ssl-certificate.md)  
 SSL için X. 509.440 sertifikası (aktarım) güvenliği olan bir makinede bağlantı noktası yapılandırma temellerini adım adım açıklar.
