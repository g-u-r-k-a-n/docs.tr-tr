# <a name="compiler-error-cs0826"></a><span data-ttu-id="ab4b6-102">Derleyici Hatası CS0826</span><span class="sxs-lookup"><span data-stu-id="ab4b6-102">Compiler Error CS0826</span></span>
<span data-ttu-id="ab4b6-103">Örtülü olarak yazılan dizi için en iyi tür bulunamadı.</span><span class="sxs-lookup"><span data-stu-id="ab4b6-103">No best type found for implicitly typed array.</span></span>  
  
 <span data-ttu-id="ab4b6-104">Dizi öğelerinin tümü, derleyici tarafından kullanılan tür çıkarım kurallarına göre aynı türde veya dolaylı olarak aynı türe dönüştürülebilir olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="ab4b6-104">Array elements must all be the same type or implicitly convertible to the same type according to the type inference rules used by the compiler.</span></span> <span data-ttu-id="ab4b6-105">En iyi tür, dizi ifadesinde bulunan türlerden biri olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="ab4b6-105">The best type must be one of the types present in the array expression.</span></span> <span data-ttu-id="ab4b6-106">Öğeler gibi yeni bir türe `object`dönüştürülmeyecektir.</span><span class="sxs-lookup"><span data-stu-id="ab4b6-106">Elements will not be converted to a new type such as `object`.</span></span> <span data-ttu-id="ab4b6-107">Örtülü olarak yazılan bir dizi için derleyicinin, dizi türünü atanan öğelerin türüne göre çıkarması gerekir.</span><span class="sxs-lookup"><span data-stu-id="ab4b6-107">For an implicitly typed array, the compiler must infer the array type based on the type of elements assigned to it.</span></span>

## <a name="example"></a><span data-ttu-id="ab4b6-112">Örnek</span><span class="sxs-lookup"><span data-stu-id="ab4b6-112">Example</span></span>  
 <span data-ttu-id="ab4b6-113">Dizi öğelerinin tümü aynı türde olmadığından ve derleyicinin tür çıkarım mantığı tek bir en iyi türü bulamadığından aşağıdaki kod CS0826 oluşturur:</span><span class="sxs-lookup"><span data-stu-id="ab4b6-113">The following code generates CS0826 because the array elements are not all the same type, and the compiler's type inference logic does not find a single best type:</span></span>  
  
```csharp  
// cs0826.cs  
public class C  
{  
    public static int Main()  
    {  
        var x = new[] { 1, "str" }; // CS0826  
  
        char c = 'c';  
        short s1 = 0;  
        short s2 = -0;  
        short s3 = 1;  
        short s4 = -1;  
  
        var array1 = new[] { s1, s2, s3, s4, c, '1' }; // CS0826  
        return 1;  
    }  
}  
```
