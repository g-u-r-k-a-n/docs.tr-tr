### <a name="product-versioning-changes-in-the-net-framework-46-and-later-versions"></a>.NET Framework 4.6 ve sonraki sürümlerde ürün sürüm değişiklikleri

|   |   |
|---|---|
|Ayrıntılar|Ürün sürümü .NET Framework'ün önceki sürümlerinden ve özellikle .NET Framework 4, 4.5, 4.5.1 ve 4.5.2'den değişmiştir. Ayrıntılı değişiklikler şunlardır:<ul><li>Anahtardaki <code>Version</code> girişin değeri .NET <code>4.6.xxxxx</code> Framework 4.6 ve onun nokta sürümleri <code>4.7.xxxxx</code> için ve .NET Framework 4.7 ve 4.7.1 için değiştirildi. <code>HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full</code> .NET Framework 4.5, 4.5.1 ve 4.5.2 formatında. <code>4.5.xxxxx</code></li><li>.NET Framework dosyalarının dosya ve ürün sürümü, .NET Framework 4.6 ve puan sürümleri için 4.0.30319.x önceki sürüm şemasından 4.6.X.0'a ve .NET Framework 4.7 ve 4.7.1 için 4.7.X.0'a değiştirildi. Bir dosyaya sağ tıkladıktan sonra dosyanın Özelliklerini görüntülediğinizde bu yeni değerleri görebilirsiniz.</li><li>Yönetilen <xref:System.Reflection.AssemblyFileVersionAttribute> <xref:System.Reflection.AssemblyInformationalVersionAttribute> derlemelerin öznitelikleri ,NET Framework 4.6 ve puan sürümleri için 4.6.X.0 ve .NET Framework 4.7 ve 4.7.1 için 4.7.X.0 şeklinde sürüm değerlerine sahiptir.</li><li>.NET Framework 4.6, 4.6.1, 4.6.2, 4.7 ve 4.7.1'de <xref:System.Environment.Version?displayProperty=nameWithType> özellik <code>4.0.30319.42000</code>sabit sürüm dizesini döndürür. .NET Framework 4, 4.5, 4.5.1 ve 4.5.2'de sürüm dizelerini biçimde <code>4.0.30319.xxxxx</code> döndürür (örneğin, &quot;4.0.30319.18010).&quot; Uygulama kodunun Environment.Version özelliğine yeni bir bağımlılık almasını önermediğimizi unutmayın.</li></ul>Daha fazla bilgi için [bkz: Hangi .NET Framework Sürümlerinin Yüklü olduğunu belirleyin.](~/docs/framework/migration-guide/how-to-determine-which-versions-are-installed.md)|
|Öneri|Genel olarak, uygulamalar .NET Framework'ün çalışma zamanı sürümü ve yükleme dizini gibi şeyleri algılamak için önerilen tekniklere bağlı olmalıdır:<ul><li>.NET Framework'ün çalışma zamanı sürümünü algılamak için [bkz: Hangi .NET Framework Sürümlerinin Yüklü olduğunu belirleyin.](~/docs/framework/migration-guide/how-to-determine-which-versions-are-installed.md)</li><li>.NET Framework için yükleme yolunu belirlemek <code>InstallPath</code> <code>HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full</code> için, anahtardaki giriş değerini kullanın.</li></ul> <blockquote> [!IMPORTANT] Alt anahtar adı <code>NET Framework Setup</code>, <code>.NET Framework Setup</code>değil.</blockquote> <ul><li>.NET Framework ortak dil çalışma zamanına giden dizin <xref:System.Runtime.InteropServices.RuntimeEnvironment.GetRuntimeDirectory?displayProperty=nameWithType> yolunu belirlemek için yöntemi arayın.</li><li>CLR sürümünü almak için <xref:System.Runtime.InteropServices.RuntimeEnvironment.GetSystemVersion?displayProperty=nameWithType> yöntemi arayın. .NET Framework 4 ve onun nokta sürümleri (.NET Framework 4.5, 4.5.1, 4.5.2 ve .NET Framework 4.6, 4.6.1, 4.6.2, 4.7 ve 4.7.1) için v4.0.30319 dizesini döndürür.</li></ul>|
