## <a name="rule-description"></a><span data-ttu-id="91ea9-114">Kural açıklaması</span><span class="sxs-lookup"><span data-stu-id="91ea9-114">Rule description</span></span>

<span data-ttu-id="91ea9-115">`static`Genel türün bir üyesi çağrıldığında tür bağımsız değişkeni türü için belirtilmelidir.</span><span class="sxs-lookup"><span data-stu-id="91ea9-115">When a `static` member of a generic type is called, the type argument must be specified for the type.</span></span> <span data-ttu-id="91ea9-116">Destek çıkarımı desteklenmeyen genel örnek üyesi çağrıldığında tür bağımsız değişkeni üye için belirlenmelidir.</span><span class="sxs-lookup"><span data-stu-id="91ea9-116">When a generic instance member that does not support inference is called, the type argument must be specified for the member.</span></span> <span data-ttu-id="91ea9-117">Aşağıdaki çağrılar gösterildiği gibi, bu iki durumda tür bağımsız değişkenini belirtmenin sözdizimi farklı ve kolayca karıştırılır:</span><span class="sxs-lookup"><span data-stu-id="91ea9-117">The syntax for specifying the type argument in these two cases is different and easily confused, as the following calls demonstrate:</span></span>

' Generic instance method that does not support inference.

// Generic instance method that does not support inference.

<span data-ttu-id="91ea9-118">Genellikle, tür bağımsız değişkeninin üye çağrıldığında belirtilmesi gerekmez, ancak önceki bildirimlerin her ikisi de kaçınılmalıdır.</span><span class="sxs-lookup"><span data-stu-id="91ea9-118">Generally, both of the prior declarations should be avoided so that the type argument does not have to be specified when the member is called.</span></span> <span data-ttu-id="91ea9-119">Bu, genel türler için sözdiziminden farklı olmayan genel türlerde üyeleri çağırmaya yönelik bir söz dizimine neden olur.</span><span class="sxs-lookup"><span data-stu-id="91ea9-119">This results in a syntax for calling members in generics that is no different from the syntax for non-generics.</span></span>
