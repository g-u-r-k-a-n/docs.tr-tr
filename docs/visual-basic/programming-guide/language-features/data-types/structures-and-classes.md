 <span data-ttu-id="a3cea-110">Yapılar ve sınıflar aşağıdaki şekilde benzerdir:</span><span class="sxs-lookup"><span data-stu-id="a3cea-110">Structures and classes are similar in the following respects:</span></span>  
  
- <span data-ttu-id="a3cea-111">Her ikisi de *kapsayıcı* türlerdir, yani diğer türleri üye olarak içerirler.</span><span class="sxs-lookup"><span data-stu-id="a3cea-111">Both are *container* types, meaning that they contain other types as members.</span></span>  
  
- <span data-ttu-id="a3cea-112">Her ikisinde de oluşturucular, Yöntemler, özellikler, alanlar, sabitler, numaralandırmalar, olaylar ve olay işleyicilerini içerebilen Üyeler vardır.</span><span class="sxs-lookup"><span data-stu-id="a3cea-112">Both have members, which can include constructors, methods, properties, fields, constants, enumerations, events, and event handlers.</span></span> <span data-ttu-id="a3cea-113">Ancak, bu üyeleri bir yapının tanımlanmış *öğeleriyle* karıştırmayın.</span><span class="sxs-lookup"><span data-stu-id="a3cea-113">However, do not confuse these members with the declared *elements* of a structure.</span></span>  
  
- <span data-ttu-id="a3cea-114">Her ikisinin de üyeleri, kişiselleştirilmemiş erişim düzeylerine sahip olabilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-114">Members of both can have individualized access levels.</span></span> <span data-ttu-id="a3cea-115">Örneğin, bir üye `Public` ve başka bir `Private`olarak bildirilebilecek.</span><span class="sxs-lookup"><span data-stu-id="a3cea-115">For example, one member can be declared `Public` and another `Private`.</span></span>  
  
- <span data-ttu-id="a3cea-116">Her ikisi de arabirim uygulayabilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-116">Both can implement interfaces.</span></span>  
  
- <span data-ttu-id="a3cea-117">Her ikisinde de, parametresiz veya olmayan paylaşılan oluşturucular olabilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-117">Both can have shared constructors, with or without parameters.</span></span>  
  
- <span data-ttu-id="a3cea-118">Her ikisi de *varsayılan bir özellik*sunabilir, bu özellik en az bir parametre alır.</span><span class="sxs-lookup"><span data-stu-id="a3cea-118">Both can expose a *default property*, provided that property takes at least one parameter.</span></span>  
  
- <span data-ttu-id="a3cea-119">Her ikisi de olayları bildirebilir ve oluşturabilir ve her ikisi de temsilciler bildirebilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-119">Both can declare and raise events, and both can declare delegates.</span></span>  
  
## <a name="differences"></a><span data-ttu-id="a3cea-120">Fark</span><span class="sxs-lookup"><span data-stu-id="a3cea-120">Differences</span></span>  
 <span data-ttu-id="a3cea-121">Yapılar ve sınıflar aşağıdaki bununla farklıdır:</span><span class="sxs-lookup"><span data-stu-id="a3cea-121">Structures and classes differ in the following particulars:</span></span>  
  
- <span data-ttu-id="a3cea-122">Yapılar *değer türleridir*; sınıflar *başvuru türleridir*.</span><span class="sxs-lookup"><span data-stu-id="a3cea-122">Structures are *value types*; classes are *reference types*.</span></span> <span data-ttu-id="a3cea-123">Yapı türünün bir değişkeni, bir sınıf türü olarak veriye bir başvuru eklemek yerine yapının verilerini içerir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-123">A variable of a structure type contains the structure's data, rather than containing a reference to the data as a class type does.</span></span>  
  
- <span data-ttu-id="a3cea-124">Yapılar yığın ayırmayı kullanır; sınıflar yığın ayırmayı kullanır.</span><span class="sxs-lookup"><span data-stu-id="a3cea-124">Structures use stack allocation; classes use heap allocation.</span></span>  
  
- <span data-ttu-id="a3cea-125">Tüm yapı öğeleri varsayılan olarak `Public`; sınıf değişkenleri ve sabitler varsayılan olarak `Private`, diğer sınıf üyeleri varsayılan olarak `Public`.</span><span class="sxs-lookup"><span data-stu-id="a3cea-125">All structure elements are `Public` by default; class variables and constants are `Private` by default, while other class members are `Public` by default.</span></span> <span data-ttu-id="a3cea-126">Sınıf üyeleri için bu davranış, varsayılan Visual Basic 6,0 sistemiyle uyumluluk sağlar.</span><span class="sxs-lookup"><span data-stu-id="a3cea-126">This behavior for class members provides compatibility with the Visual Basic 6.0 system of defaults.</span></span>  
  
- <span data-ttu-id="a3cea-127">Bir yapının en az bir paylaşılmayan değişkeni veya paylaşılmayan olmayan, özel olmayan olay öğesi olması gerekir; bir sınıf tamamen boş olabilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-127">A structure must have at least one nonshared variable or nonshared, noncustom event element; a class can be completely empty.</span></span>  
  
- <span data-ttu-id="a3cea-128">Yapı öğeleri `Protected`olarak bildirilemez; sınıf üyeleri olabilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-128">Structure elements cannot be declared as `Protected`; class members can.</span></span>  
  
- <span data-ttu-id="a3cea-129">Bir yapı yordamı yalnızca [paylaşılan](../../../../visual-basic/language-reference/modifiers/shared.md) bir`Sub` yordamı ve yalnızca [AddHandler ifadesinin](../../../../visual-basic/language-reference/statements/addhandler-statement.md)yoluyla, olayları işleyebilir; herhangi bir sınıf yordamı, [Handles](../../../../visual-basic/language-reference/statements/handles-clause.md) anahtar sözcüğünü veya `AddHandler` ifadesini kullanarak olayları işleyebilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-129">A structure procedure can handle events only if it is a [Shared](../../../../visual-basic/language-reference/modifiers/shared.md)`Sub` procedure, and only by means of the [AddHandler Statement](../../../../visual-basic/language-reference/statements/addhandler-statement.md); any class procedure can handle events, using either the [Handles](../../../../visual-basic/language-reference/statements/handles-clause.md) keyword or the `AddHandler` statement.</span></span> <span data-ttu-id="a3cea-130">Daha fazla bilgi için bkz. [Olaylar](../../../../visual-basic/programming-guide/language-features/events/index.md).</span><span class="sxs-lookup"><span data-stu-id="a3cea-130">For more information, see [Events](../../../../visual-basic/programming-guide/language-features/events/index.md).</span></span>  
  
- <span data-ttu-id="a3cea-131">Yapı değişkeni bildirimleri diziler için başlatıcılar veya başlangıç boyutları belirtemez; sınıf değişkeni bildirimleri olabilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-131">Structure variable declarations cannot specify initializers or initial sizes for arrays; class variable declarations can.</span></span>  
  
- <span data-ttu-id="a3cea-132">Yapılar <xref:System.ValueType?displayProperty=nameWithType> sınıfından dolaylı olarak devralınır ve diğer herhangi bir türden devralınabilir; sınıflar, <xref:System.ValueType?displayProperty=nameWithType>dışındaki herhangi bir sınıftan veya sınıftan devralınabilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-132">Structures implicitly inherit from the <xref:System.ValueType?displayProperty=nameWithType> class and cannot inherit from any other type; classes can inherit from any class or classes other than <xref:System.ValueType?displayProperty=nameWithType>.</span></span>  
  
- <span data-ttu-id="a3cea-133">Yapılar devredilemez; sınıflar.</span><span class="sxs-lookup"><span data-stu-id="a3cea-133">Structures are not inheritable; classes are.</span></span>  
  
- <span data-ttu-id="a3cea-134">Yapılar hiçbir şekilde sonlandırılmadığından, ortak dil çalışma zamanı (CLR) herhangi bir yapıda <xref:System.Object.Finalize%2A> yöntemi çağırılmaz; sınıflar, kalan etkin bir başvuru olmadığını algıladığında bir sınıftaki <xref:System.Object.Finalize%2A> çağıran çöp toplayıcı (GC) tarafından sonlandırılır.</span><span class="sxs-lookup"><span data-stu-id="a3cea-134">Structures are never terminated, so the common language runtime (CLR) never calls the <xref:System.Object.Finalize%2A> method on any structure; classes are terminated by the garbage collector (GC), which calls <xref:System.Object.Finalize%2A> on a class when it detects there are no active references remaining.</span></span>  
  
- <span data-ttu-id="a3cea-135">Bir yapı için bir Oluşturucu gerekmez; bir sınıf.</span><span class="sxs-lookup"><span data-stu-id="a3cea-135">A structure does not require a constructor; a class does.</span></span>  
  
- <span data-ttu-id="a3cea-136">Yapılar yalnızca parametre alıyorsa paylaşılmayan oluşturuculara sahip olabilir; sınıflar, parametrelere sahip veya parametresiz olabilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-136">Structures can have nonshared constructors only if they take parameters; classes can have them with or without parameters.</span></span>  
  
 <span data-ttu-id="a3cea-137">Her yapının parametresiz bir örtük ortak Oluşturucusu vardır.</span><span class="sxs-lookup"><span data-stu-id="a3cea-137">Every structure has an implicit public constructor without parameters.</span></span> <span data-ttu-id="a3cea-138">Bu Oluşturucu, tüm yapının veri öğelerini varsayılan değerlerine başlatır.</span><span class="sxs-lookup"><span data-stu-id="a3cea-138">This constructor initializes all the structure's data elements to their default values.</span></span> <span data-ttu-id="a3cea-139">Bu davranışı yeniden tanımlayamazsınız.</span><span class="sxs-lookup"><span data-stu-id="a3cea-139">You cannot redefine this behavior.</span></span>  
  
## <a name="instances-and-variables"></a><span data-ttu-id="a3cea-140">Örnekler ve değişkenler</span><span class="sxs-lookup"><span data-stu-id="a3cea-140">Instances and Variables</span></span>  
 <span data-ttu-id="a3cea-141">Yapılar değer türleri olduğundan, her yapı değişkeni ayrı bir yapı örneğine kalıcı olarak bağlanır.</span><span class="sxs-lookup"><span data-stu-id="a3cea-141">Because structures are value types, each structure variable is permanently bound to an individual structure instance.</span></span> <span data-ttu-id="a3cea-142">Ancak sınıflar başvuru türleridir ve bir nesne değişkeni farklı zamanlarda çeşitli sınıf örneklerine başvurabilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-142">But classes are reference types, and an object variable can refer to various class instances at different times.</span></span> <span data-ttu-id="a3cea-143">Bu ayrım, yapıların ve sınıfların kullanımını aşağıdaki yollarla etkiler:</span><span class="sxs-lookup"><span data-stu-id="a3cea-143">This distinction affects your usage of structures and classes in the following ways:</span></span>  
  
- <span data-ttu-id="a3cea-144">**Başlatılmasında.**</span><span class="sxs-lookup"><span data-stu-id="a3cea-144">**Initialization.**</span></span> <span data-ttu-id="a3cea-145">Yapı değişkeni, yapının parametresiz oluşturucusunu kullanarak örtük olarak öğelerin başlatılmasını içerir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-145">A structure variable implicitly includes an initialization of the elements using the structure's parameterless constructor.</span></span> <span data-ttu-id="a3cea-146">Bu nedenle, `Dim s As struct1` `Dim s As struct1 = New struct1()`eşdeğerdir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-146">Therefore, `Dim s As struct1` is equivalent to `Dim s As struct1 = New struct1()`.</span></span>  
  
- <span data-ttu-id="a3cea-147">**Değişkenler atanıyor.**</span><span class="sxs-lookup"><span data-stu-id="a3cea-147">**Assigning Variables.**</span></span> <span data-ttu-id="a3cea-148">Bir yapı değişkenini diğerine atadığınızda veya bir yapı örneğini bir yordam bağımsız değişkenine geçirdiğinizde, tüm değişken öğelerinin geçerli değerleri yeni yapıya kopyalanır.</span><span class="sxs-lookup"><span data-stu-id="a3cea-148">When you assign one structure variable to another, or pass a structure instance to a procedure argument, the current values of all the variable elements are copied to the new structure.</span></span> <span data-ttu-id="a3cea-149">Bir nesne değişkenini diğerine atadığınızda veya bir yordama bir nesne değişkeni geçirdiğinizde, yalnızca başvuru işaretçisi kopyalanır.</span><span class="sxs-lookup"><span data-stu-id="a3cea-149">When you assign one object variable to another, or pass an object variable to a procedure, only the reference pointer is copied.</span></span>  
  
- <span data-ttu-id="a3cea-150">**Nothing atama.**</span><span class="sxs-lookup"><span data-stu-id="a3cea-150">**Assigning Nothing.**</span></span> <span data-ttu-id="a3cea-151">Bir yapı değişkenine [Nothing](../../../../visual-basic/language-reference/nothing.md) değeri atayabilirsiniz, ancak örnek değişkenle ilişkilendirilmeye devam eder.</span><span class="sxs-lookup"><span data-stu-id="a3cea-151">You can assign the value [Nothing](../../../../visual-basic/language-reference/nothing.md) to a structure variable, but the instance continues to be associated with the variable.</span></span> <span data-ttu-id="a3cea-152">Kendi yöntemlerini çağırabilirsiniz ve veri öğelerine erişebilirsiniz, ancak değişken öğeleri atama tarafından yeniden başlatılır.</span><span class="sxs-lookup"><span data-stu-id="a3cea-152">You can still call its methods and access its data elements, although variable elements are reinitialized by the assignment.</span></span>

- <span data-ttu-id="a3cea-154">**Birden çok örnek.**</span><span class="sxs-lookup"><span data-stu-id="a3cea-154">**Multiple Instances.**</span></span> <span data-ttu-id="a3cea-155">Bir nesne değişkeni farklı zamanlarda kendisine atanmış farklı sınıf örneklerine sahip olabilir ve çeşitli nesne değişkenleri aynı anda aynı sınıf örneğine başvurabilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-155">An object variable can have different class instances assigned to it at different times, and several object variables can refer to the same class instance at the same time.</span></span> <span data-ttu-id="a3cea-156">Sınıf üyeleri değerlerinde yaptığınız değişiklikler, aynı örneğe işaret eden başka bir değişken aracılığıyla erişildiğinde bu üyeleri etkiler.</span><span class="sxs-lookup"><span data-stu-id="a3cea-156">Changes you make to the values of class members affect those members when accessed through another variable pointing to the same instance.</span></span>  
  
     <span data-ttu-id="a3cea-157">Ancak yapı öğeleri kendi örnekleri içinde yalıtılmıştır.</span><span class="sxs-lookup"><span data-stu-id="a3cea-157">Structure elements, however, are isolated within their own instance.</span></span> <span data-ttu-id="a3cea-158">Değerlerinde yapılan değişiklikler aynı `Structure` bildiriminin diğer örneklerinde bile diğer herhangi bir yapı değişkenine yansıtılmaz.</span><span class="sxs-lookup"><span data-stu-id="a3cea-158">Changes to their values are not reflected in any other structure variables, even in other instances of the same `Structure` declaration.</span></span>  
  
- <span data-ttu-id="a3cea-159">**Eþit.**</span><span class="sxs-lookup"><span data-stu-id="a3cea-159">**Equality.**</span></span> <span data-ttu-id="a3cea-160">İki yapının eşitlik testi, öğe öğesi testi ile gerçekleştirilmelidir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-160">Equality testing of two structures must be performed with an element-by-element test.</span></span> <span data-ttu-id="a3cea-161">İki nesne değişkeni <xref:System.Object.Equals%2A> yöntemi kullanılarak karşılaştırılabilir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-161">Two object variables can be compared using the <xref:System.Object.Equals%2A> method.</span></span> <span data-ttu-id="a3cea-162"><xref:System.Object.Equals%2A> iki değişkenin aynı örneğe işaret edip etmediğini gösterir.</span><span class="sxs-lookup"><span data-stu-id="a3cea-162"><xref:System.Object.Equals%2A> indicates whether the two variables point to the same instance.</span></span>
