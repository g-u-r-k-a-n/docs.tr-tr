## <a name="class-enhancements"></a><span data-ttu-id="fcd52-107">Sınıf Geliştirmeleri</span><span class="sxs-lookup"><span data-stu-id="fcd52-107">Class Enhancements</span></span>  
 <span data-ttu-id="fcd52-108">Bu geliştirmelerin temel özelliği, yüksek hacimli asynchronous soket G/Ç sırasında nesnelerin tekrarlanan ayırma ve eşitlemeden kaçınmasIdır.</span><span class="sxs-lookup"><span data-stu-id="fcd52-108">The main feature of these enhancements is the avoidance of the repeated allocation and synchronization of objects during high-volume asynchronous socket I/O.</span></span> <span data-ttu-id="fcd52-109">Şu anda <xref:System.Net.Sockets.Socket> sınıf tarafından asynchronous soket G/Ç için <xref:System.IAsyncResult?displayProperty=nameWithType> uygulanan Begin/End tasarım deseni, her bir eşzamanlı soket işlemi için bir nesne ayrılmasını gerektirir.</span><span class="sxs-lookup"><span data-stu-id="fcd52-109">The Begin/End design pattern currently implemented by the <xref:System.Net.Sockets.Socket> class for asynchronous socket I/O requires a <xref:System.IAsyncResult?displayProperty=nameWithType> object be allocated for each asynchronous socket operation.</span></span>  
  
 <span data-ttu-id="fcd52-110">Yeni <xref:System.Net.Sockets.Socket> sınıf geliştirmelerinde, eşzamanlı soket işlemleri, uygulama tarafından <xref:System.Net.Sockets.SocketAsyncEventArgs?displayProperty=nameWithType> ayrılan ve sürdürülen yeniden kullanılabilir sınıf nesneleri tarafından tanımlanır.</span><span class="sxs-lookup"><span data-stu-id="fcd52-110">In the new <xref:System.Net.Sockets.Socket> class enhancements, asynchronous socket operations are described by reusable <xref:System.Net.Sockets.SocketAsyncEventArgs?displayProperty=nameWithType> class objects allocated and maintained by the application.</span></span> <span data-ttu-id="fcd52-111">Yüksek performanslı soket uygulamaları, sürdürülmesi gereken çakışan soket işlemlerinin miktarını en iyi şekilde bilir.</span><span class="sxs-lookup"><span data-stu-id="fcd52-111">High-performance socket applications know best the amount of overlapped socket operations that must be sustained.</span></span> <span data-ttu-id="fcd52-112">Uygulama, ihtiyaç duyduğu nesnelerin <xref:System.Net.Sockets.SocketAsyncEventArgs> çoğunu oluşturabilir.</span><span class="sxs-lookup"><span data-stu-id="fcd52-112">The application can create as many of the <xref:System.Net.Sockets.SocketAsyncEventArgs> objects that it needs.</span></span> <span data-ttu-id="fcd52-113">Örneğin, bir sunucu uygulamasının gelen istemci bağlantı oranlarını desteklemek için her zaman bekleyen işlemleri kabul etmek için <xref:System.Net.Sockets.SocketAsyncEventArgs> 15 sokete sahip olması gerekiyorsa, bu amaçla 15 yeniden kullanılabilir nesne ayırabilir.</span><span class="sxs-lookup"><span data-stu-id="fcd52-113">For example, if a server application needs to have 15 socket accept operations outstanding at all times to support incoming client connection rates, it can allocate 15 reusable <xref:System.Net.Sockets.SocketAsyncEventArgs> objects in advance for that purpose.</span></span>  
  
 <span data-ttu-id="fcd52-114">Bu sınıfla bir eşzamanlı soket işlemi gerçekleştirmek için desen aşağıdaki adımlardan oluşur:</span><span class="sxs-lookup"><span data-stu-id="fcd52-114">The pattern for performing an asynchronous socket operation with this class consists of the following steps:</span></span>

2. <span data-ttu-id="fcd52-116">Yapılmak üzere olan işlem için bağlam nesnesi üzerindeki özellikleri ayarlama (örneğin geri arama temsilcisi yöntemi ve veri arabelleği).</span><span class="sxs-lookup"><span data-stu-id="fcd52-116">Set properties on the context object to the operation about to be performed (the callback delegate method and data buffer, for example).</span></span>  
  
3. <span data-ttu-id="fcd52-117">Eşzamanlı işlemi başlatmak için uygun soket yöntemini (xxxAsync) çağırın.</span><span class="sxs-lookup"><span data-stu-id="fcd52-117">Call the appropriate socket method (xxxAsync) to initiate the asynchronous operation.</span></span>  
  
4. <span data-ttu-id="fcd52-118">Asynchronous soket yöntemi (xxxAsync) geri aramadoğru döndürür, tamamlama durumu için bağlam özelliklerini sorgun.</span><span class="sxs-lookup"><span data-stu-id="fcd52-118">If the asynchronous socket method (xxxAsync) returns true in the callback, query the context properties for completion status.</span></span>  
  
5. <span data-ttu-id="fcd52-119">Asynchronous soket yöntemi (xxxAsync) geri aramada yanlış dönerse, işlem eşzamanlı olarak tamamlanır.</span><span class="sxs-lookup"><span data-stu-id="fcd52-119">If the asynchronous socket method (xxxAsync) returns false in the callback, the operation completed synchronously.</span></span> <span data-ttu-id="fcd52-120">İçerik özellikleri işlem sonucu için sorgulanabilir.</span><span class="sxs-lookup"><span data-stu-id="fcd52-120">The context properties may be queried for the operation result.</span></span>  
  
6. <span data-ttu-id="fcd52-121">Başka bir işlem için bağlamı yeniden kullanın, havuza geri koyun veya atın.</span><span class="sxs-lookup"><span data-stu-id="fcd52-121">Reuse the context for another operation, put it back in the pool, or discard it.</span></span>  
  
 <span data-ttu-id="fcd52-122">Yeni asynchronous soket çalışma bağlamı nesnesinin ömrü, uygulama kodundaki referanslar ve eşzamanlı G/Ç referansları ile belirlenir.</span><span class="sxs-lookup"><span data-stu-id="fcd52-122">The lifetime of the new asynchronous socket operation context object is determined by references in the application code and asynchronous I/O references.</span></span> <span data-ttu-id="fcd52-123">Uygulamanın, asynchronous soket çalışma yöntemlerinden birine parametre olarak gönderildikten sonra bir asynchronous soket çalışma bağlamı nesnesine başvuruda bulunmaları gerekli değildir.</span><span class="sxs-lookup"><span data-stu-id="fcd52-123">It is not necessary for the application to retain a reference to an asynchronous socket operation context object after it is submitted as a parameter to one of the asynchronous socket operation methods.</span></span> <span data-ttu-id="fcd52-124">Tamam geri arama dönene kadar başvurulan kalır.</span><span class="sxs-lookup"><span data-stu-id="fcd52-124">It will remain referenced until the completion callback returns.</span></span> <span data-ttu-id="fcd52-125">Ancak uygulamanın bağlam nesnesine başvuruyu tutması avantajlıdır, böylece gelecekteki bir eşzamanlı soket çalışması için yeniden kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="fcd52-125">However it is advantageous for the application to retain the reference to the context object so that it can be reused for a future asynchronous socket operation.</span></span>
