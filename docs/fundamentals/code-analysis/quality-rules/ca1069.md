## <a name="rule-description"></a><span data-ttu-id="ed3fe-113">Kural açıklaması</span><span class="sxs-lookup"><span data-stu-id="ed3fe-113">Rule description</span></span>

<span data-ttu-id="ed3fe-114">Her sabit listesi üyesinin benzersiz bir sabit değeri olmalıdır veya kesin bir şekilde paylaşılan değer amacını göstermek için numaralandırıcıdaki bir önceki üye ile açıkça atanmalıdır.</span><span class="sxs-lookup"><span data-stu-id="ed3fe-114">Every enum member should either have a unique constant value or be explicitly assigned with a prior member in the enum to indicate explicit intent of sharing value.</span></span> <span data-ttu-id="ed3fe-115">Örneğin:</span><span class="sxs-lookup"><span data-stu-id="ed3fe-115">For example:</span></span>

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = 2,   // CA1069: This is not fine. Either assign a different constant value or 'Field2' to indicate explicit intent of sharing value.
}
```

<span data-ttu-id="ed3fe-116">Bu kural, aşağıdaki senaryolardan sunulan işlevsel hataları yakalama konusunda yardımcı olur:</span><span class="sxs-lookup"><span data-stu-id="ed3fe-116">This rule helps in catching functional bugs introduced from the following scenarios:</span></span>

- <span data-ttu-id="ed3fe-117">Yanlışlıkla yazma hataları, kullanıcının birden çok üye için yanlışlıkla aynı sabit değeri yazdığın.</span><span class="sxs-lookup"><span data-stu-id="ed3fe-117">Accidental typing mistakes, where the user accidentally typed the same constant value for multiple members.</span></span>
- <span data-ttu-id="ed3fe-118">Kullanıcının var olan bir üye tanımını kopyaladığı, sonra üyeyi yeniden adlandırdığından ve değeri değiştirmeye unuttu, yapıştırma hatalarını Kopyala.</span><span class="sxs-lookup"><span data-stu-id="ed3fe-118">Copy paste mistakes, where the user copied an existing member definition, then renamed the member but forgot to change the value.</span></span>
- <span data-ttu-id="ed3fe-119">Farklı dallarda, farklı bir adla yeni bir üyenin eklendiği, ancak farklı dallardaki aynı değere sahip olan birden çok daldaki çözümü birleştirin.</span><span class="sxs-lookup"><span data-stu-id="ed3fe-119">Merge resolution from multiple branches, where a new member was added with a different name but the same value in different branches.</span></span>

## <a name="how-to-fix-violations"></a><span data-ttu-id="ed3fe-120">İhlalleri çözme</span><span class="sxs-lookup"><span data-stu-id="ed3fe-120">How to fix violations</span></span>

<span data-ttu-id="ed3fe-121">Bir ihlalin düzeltilmesi için, aynı değeri paylaşmanın açık hedefini göstermek üzere yeni bir benzersiz sabit değer atayın veya numaralandırmasında önceki bir üyeyle atayın.</span><span class="sxs-lookup"><span data-stu-id="ed3fe-121">To fix a violation, either assign a new unique constant value or assign with a prior member in the enum to indicate explicit intent of sharing the same value.</span></span> <span data-ttu-id="ed3fe-122">Örneğin, aşağıdaki kod parçacığı, kural ihlalini ve ihlalin düzeltilmesi için birkaç yol gösterir:</span><span class="sxs-lookup"><span data-stu-id="ed3fe-122">For example, the following code snippet shows a violation of the rule and couple of ways to fix the violation:</span></span>

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = 2,   // CA1069: This is not fine. Either assign a different constant value or 'Field2' to indicate explicit intent of sharing value.
}
```

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = 3,   // This is now fine
}
```

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = Field2,   // This is also fine
}
```
