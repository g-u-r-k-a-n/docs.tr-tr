## <a name="discussion"></a><span data-ttu-id="b93a2-108">Tartışma</span><span class="sxs-lookup"><span data-stu-id="b93a2-108">Discussion</span></span>  
 <span data-ttu-id="b93a2-109">Bu örnek iki bileşenden oluşur: bir WCF hizmeti içeren bir Web uygulaması projesi (hizmet) ve SOAP ve HTTP bağlamaları kullanarak hizmet işlemlerini çağıran bir konsol uygulaması (Istemci).</span><span class="sxs-lookup"><span data-stu-id="b93a2-109">This sample consists of two components: a Web Application project (Service) that contains a WCF service and a console application (Client) that invokes service operations using SOAP and HTTP bindings.</span></span>  
  
 <span data-ttu-id="b93a2-110">WCF hizmeti, giriş olarak geçirilmiş dizeyi yankı eden 2 işlem –`GetData` ve `PutData` sunar.</span><span class="sxs-lookup"><span data-stu-id="b93a2-110">The WCF service exposes 2 operations –`GetData` and `PutData` – that echo the string that was passed as input.</span></span> <span data-ttu-id="b93a2-111">Hizmet işlemlerine <xref:System.ServiceModel.Web.WebGetAttribute> ve <xref:System.ServiceModel.Web.WebInvokeAttribute>açıklama eklenir.</span><span class="sxs-lookup"><span data-stu-id="b93a2-111">The service operations are annotated with <xref:System.ServiceModel.Web.WebGetAttribute> and <xref:System.ServiceModel.Web.WebInvokeAttribute>.</span></span> <span data-ttu-id="b93a2-112">Bu öznitelikler, bu işlemlerin HTTP projeksiyonunu denetler.</span><span class="sxs-lookup"><span data-stu-id="b93a2-112">These attributes control the HTTP projection of these operations.</span></span> <span data-ttu-id="b93a2-113">Bunlara ek olarak, <xref:System.ServiceModel.OperationContractAttribute>ile açıklanırlar ve bu, SOAP bağlamaları üzerinden gösterilmesini sağlar.</span><span class="sxs-lookup"><span data-stu-id="b93a2-113">In addition, they are annotated with <xref:System.ServiceModel.OperationContractAttribute>, which enables them to be exposed over SOAP bindings.</span></span> <span data-ttu-id="b93a2-114">Hizmetin `PutData` yöntemi HTTP durum kodu kullanılarak HTTP üzerinden geri gönderilen ve SOAP üzerinden SOAP hatası olarak geri gönderilen bir <xref:System.ServiceModel.Web.WebFaultException>oluşturur.</span><span class="sxs-lookup"><span data-stu-id="b93a2-114">The service’s `PutData` method throws a <xref:System.ServiceModel.Web.WebFaultException>, which gets sent back over HTTP using the HTTP status code and gets sent back over SOAP as a SOAP fault.</span></span>  
  
 <span data-ttu-id="b93a2-115">Web. config dosyası, WCF hizmetini 3 uç nokta ile yapılandırır:</span><span class="sxs-lookup"><span data-stu-id="b93a2-115">The Web.config file configures the WCF service with 3 endpoints:</span></span>

- <span data-ttu-id="b93a2-118">İstemcilerin HTTP bağlama üzerinden SOAP kullanarak hizmete erişmesine izin veren ~/Service.svc/SOAP uç noktası.</span><span class="sxs-lookup"><span data-stu-id="b93a2-118">The ~/service.svc/soap endpoint that allows the clients to access the service using the SOAP over HTTP binding.</span></span>

 <span data-ttu-id="b93a2-121">İstemci projesi, bir SOAP proxy ( **hizmet başvurusu Ekle**aracılığıyla oluşturulur) kullanarak hizmete erişmeyi ve <xref:System.Net.WebClient>kullanarak hizmete erişimi gösterir.</span><span class="sxs-lookup"><span data-stu-id="b93a2-121">The client project demonstrates accessing the service using a SOAP proxy (generated through **Add Service Reference**) and accessing the service using <xref:System.Net.WebClient>.</span></span>
