# <a name="docker-terminology"></a><span data-ttu-id="30da1-103">Docker terimleri</span><span class="sxs-lookup"><span data-stu-id="30da1-103">Docker terminology</span></span>

<span data-ttu-id="30da1-104">Bu bölümde, Docker'a girmeden önce aşina olmanız gereken terimler ve tanımlar listelenebilmektedir.</span><span class="sxs-lookup"><span data-stu-id="30da1-104">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="30da1-105">Diğer tanımlar için Docker tarafından sağlanan kapsamlı [sözlüklere](https://docs.docker.com/glossary/) bakın.</span><span class="sxs-lookup"><span data-stu-id="30da1-105">For further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker.</span></span>

<span data-ttu-id="30da1-106">**Konteyner görüntüsü**: Bir kapsayıcı oluşturmak için gereken tüm bağımlılıkları ve bilgileri içeren bir paket.</span><span class="sxs-lookup"><span data-stu-id="30da1-106">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="30da1-107">Görüntü, kapsayıcı çalışma zamanı tarafından kullanılacak tüm bağımlılıkları (çerçeveler gibi) ve dağıtım ve yürütme yapılandırmasını içerir.</span><span class="sxs-lookup"><span data-stu-id="30da1-107">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="30da1-108">Genellikle, bir görüntü kapsayıcının dosya sistemini oluşturmak için üst üste yığılmış katmanlar olan birden çok temel görüntüden türetilir.</span><span class="sxs-lookup"><span data-stu-id="30da1-108">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container's filesystem.</span></span> <span data-ttu-id="30da1-109">Bir görüntü oluşturulduktan sonra değişmez.</span><span class="sxs-lookup"><span data-stu-id="30da1-109">An image is immutable once it has been created.</span></span>

<span data-ttu-id="30da1-110">**Dockerfile**: Docker görüntüsünün nasıl oluşturacağına ilişkin yönergeler içeren bir metin dosyası.</span><span class="sxs-lookup"><span data-stu-id="30da1-110">**Dockerfile**: A text file that contains instructions on how to build a Docker image.</span></span> <span data-ttu-id="30da1-111">Bu bir toplu komut dosyası gibi, ilk satır ile başlamak ve daha sonra gerekli programları yüklemek için yönergeleri izleyin temel görüntü devletler, dosyaları kopyalamak, ve benzeri, ihtiyacınız olan çalışma ortamı nı elde edene kadar.</span><span class="sxs-lookup"><span data-stu-id="30da1-111">It's like a batch script, the first line states the base image to begin with and then follow the instructions to install required programs, copy files, and so on, until you get the working environment you need.</span></span>

<span data-ttu-id="30da1-112">**Yapı**: Dockerfile tarafından sağlanan bilgi ve içeriğe ve görüntünün oluşturulduğu klasördeki ek dosyalara dayalı bir kapsayıcı görüntüsü oluşturma eylemi.</span><span class="sxs-lookup"><span data-stu-id="30da1-112">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="30da1-113">Docker **`docker build`** komutu ile görüntüler oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="30da1-113">You can build images with the Docker **`docker build`** command.</span></span>

<span data-ttu-id="30da1-114">**Konteyner**: Docker görüntüsünün bir örneği.</span><span class="sxs-lookup"><span data-stu-id="30da1-114">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="30da1-115">Kapsayıcı, tek bir uygulamanın, işlemin veya hizmetin yürütülmesini temsil eder.</span><span class="sxs-lookup"><span data-stu-id="30da1-115">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="30da1-116">Docker görüntüsünün, yürütme ortamının ve standart bir yönerge kümesinin içeriğinden oluşur.</span><span class="sxs-lookup"><span data-stu-id="30da1-116">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="30da1-117">Bir hizmeti ölçeklerken, aynı görüntüden bir kapsayıcının birden çok örneğini oluşturursunuz.</span><span class="sxs-lookup"><span data-stu-id="30da1-117">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="30da1-118">Veya bir toplu iş, her örne farklı parametreler geçirerek aynı görüntüden birden çok kapsayıcı oluşturabilir.</span><span class="sxs-lookup"><span data-stu-id="30da1-118">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="30da1-119">**Birimler**: Kapsayıcının kullanabileceği yazılabilir bir dosya sistemi sunun.</span><span class="sxs-lookup"><span data-stu-id="30da1-119">**Volumes**: Offer a writable filesystem that the container can use.</span></span> <span data-ttu-id="30da1-120">Görüntüler salt okunur ancak çoğu programın dosya sistemine yazması gerektiğinden, birimler kapsayıcı görüntünün üstüne yazılabilir bir katman ekler, böylece programlar yazılabilir bir dosya sistemine erişebilir.</span><span class="sxs-lookup"><span data-stu-id="30da1-120">Since images are read-only but most programs need to write to the filesystem, volumes add a writable layer, on top of the container image, so the programs have access to a writable filesystem.</span></span> <span data-ttu-id="30da1-121">Program katmanlı bir dosya sistemine erişiyor bilmiyor, her zamanki gibi sadece dosya sistemi.</span><span class="sxs-lookup"><span data-stu-id="30da1-121">The program doesn't know it's accessing a layered filesystem, it's just the filesystem as usual.</span></span> <span data-ttu-id="30da1-122">Birimler ana bilgisayar sisteminde yaşar ve Docker tarafından yönetilir.</span><span class="sxs-lookup"><span data-stu-id="30da1-122">Volumes live in the host system and are managed by Docker.</span></span>

<span data-ttu-id="30da1-123">**Etiket**: Görüntülere uygulayabileceğiniz bir işaret veya etiket, böylece aynı görüntünün farklı görüntüleri veya sürümleri (sürüm numarasına veya hedef ortama bağlı olarak) tanımlanabilir.</span><span class="sxs-lookup"><span data-stu-id="30da1-123">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="30da1-124">**Çok aşamalı Yapı**: Docker 17.05 veya daha yüksek olduğundan, son görüntülerin boyutunu azaltmaya yardımcı olan bir özelliktir.</span><span class="sxs-lookup"><span data-stu-id="30da1-124">**Multi-stage Build**: Is a feature, since Docker 17.05 or higher, that helps to reduce the size of the final images.</span></span> <span data-ttu-id="30da1-125">Birkaç cümlede, çok aşamalı yapıyla, örneğin, uygulamayı derlemek ve yayımlamak ve ardından yayımlama klasörünü küçük bir çalışma süresi yalnızca temel görüntüyle kullanmak için çok daha küçük bir son görüntü oluşturmak için SDK içeren büyük bir temel görüntü kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="30da1-125">In a few sentences, with multi-stage build you can use, for example, a large base image, containing the SDK, for compiling and publishing the application and then using the publishing folder with a small runtime-only base image, to produce a much smaller final image</span></span>

<span data-ttu-id="30da1-126">**Depo (repo)**: Resim sürümünü gösteren bir etiketle etiketlenmiş ilgili Docker görüntülerinin koleksiyonu.</span><span class="sxs-lookup"><span data-stu-id="30da1-126">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="30da1-127">Bazı repos'lar, SDK'lar (daha ağır), yalnızca çalışma süreleri (daha hafif) içeren bir görüntü gibi belirli bir görüntünün birden çok varyantını içerir. Bu türevleri etiketleri ile işaretlenmiş olabilir.</span><span class="sxs-lookup"><span data-stu-id="30da1-127">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="30da1-128">Tek bir repo, Linux görüntüsü ve Windows görüntüsü gibi platform türevlerini içerebilir.</span><span class="sxs-lookup"><span data-stu-id="30da1-128">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="30da1-129">**Kayıt Defteri**: Depolara erişim sağlayan bir hizmet.</span><span class="sxs-lookup"><span data-stu-id="30da1-129">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="30da1-130">Çoğu genel görüntü için varsayılan kayıt defteri [Docker Hub'dır](https://hub.docker.com/) (kuruluş olarak Docker'a aittir).</span><span class="sxs-lookup"><span data-stu-id="30da1-130">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="30da1-131">Kayıt defteri genellikle birden çok takımın depolarını içerir.</span><span class="sxs-lookup"><span data-stu-id="30da1-131">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="30da1-132">Şirketlerin genellikle oluşturdukları görüntüleri depolamak ve yönetmek için özel kayıtları vardır.</span><span class="sxs-lookup"><span data-stu-id="30da1-132">Companies often have private registries to store and manage images they've created.</span></span> <span data-ttu-id="30da1-133">Azure Kapsayıcı Kayıt Defteri başka bir örnektir.</span><span class="sxs-lookup"><span data-stu-id="30da1-133">Azure Container Registry is another example.</span></span>

<span data-ttu-id="30da1-134">**Çok yönlü görüntü**: Çok mimari için, docker'ın çalıştığı platforma göre uygun görüntünün seçilmesini kolaylaştıran bir özelliktir, **`FROM mcr.microsoft.com/dotnet/core/sdk:2.2`** örneğin, bir Dockerfile **`2.2-nanoserver-1709`** **`2.2-nanoserver-1803`** kayıt **`2.2-nanoserver-1809`** **`2.2-stretch`** defterinden bir temel görüntü istediğinde, veya Docker'ın çalıştığı işletim sistemine ve sürümüne bağlı olarak.</span><span class="sxs-lookup"><span data-stu-id="30da1-134">**Multi-arch image**: For multi-architecture, is a feature that simplifies the selection of the appropriate image, according to the platform where Docker is running, for example, when a Dockerfile requests a base image **`FROM mcr.microsoft.com/dotnet/core/sdk:2.2`** from the registry it actually gets **`2.2-nanoserver-1709`**, **`2.2-nanoserver-1803`**, **`2.2-nanoserver-1809`** or **`2.2-stretch`**, depending on the operating system and version where Docker is running.</span></span>

<span data-ttu-id="30da1-135">**Docker Hub**: Resim yüklemek ve onlarla çalışmak için genel kayıt defteri.</span><span class="sxs-lookup"><span data-stu-id="30da1-135">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="30da1-136">Docker Hub Docker görüntü barındırma, kamu veya özel kayıt defteri, tetikleyiciler ve web kancaları oluşturmak ve GitHub ve Bitbucket ile entegrasyon sağlar.</span><span class="sxs-lookup"><span data-stu-id="30da1-136">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="30da1-137">**Azure Kapsayıcı Kayıt Defteri**: Azure'da Docker görüntüleri ve bileşenleriyle çalışmak için genel kaynak.</span><span class="sxs-lookup"><span data-stu-id="30da1-137">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="30da1-138">Bu, Azure'daki dağıtımlarınıza yakın olan ve size erişim üzerinde denetim sağlayan bir kayıt defteri sağlayarak Azure Etkin Dizin gruplarınızı ve izinlerinizi kullanmanızı mümkün kılar.</span><span class="sxs-lookup"><span data-stu-id="30da1-138">This provides a registry that's close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="30da1-139">**Docker Trusted Registry (DTR)**: Kuruluşun veri merkezi ve ağında yaşayabilmesi için şirket içinde kurulabilen Docker kayıt hizmeti (Docker'dan).</span><span class="sxs-lookup"><span data-stu-id="30da1-139">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization's datacenter and network.</span></span> <span data-ttu-id="30da1-140">Kuruluş içinde yönetilmesi gereken özel görüntüler için uygundur.</span><span class="sxs-lookup"><span data-stu-id="30da1-140">It's convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="30da1-141">Docker Trusted Registry, Docker Datacenter ürününün bir parçası olarak yer almaktadır.</span><span class="sxs-lookup"><span data-stu-id="30da1-141">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="30da1-142">Daha fazla bilgi için [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/)adresine bakın.</span><span class="sxs-lookup"><span data-stu-id="30da1-142">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="30da1-143">**Docker Community Edition (CE)**: Kapsayıcıları yerel olarak oluşturmak, çalıştırmak ve test etmek için Windows ve macOS için geliştirme araçları.</span><span class="sxs-lookup"><span data-stu-id="30da1-143">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="30da1-144">Windows için Docker CE, hem Linux hem de Windows Kapsayıcıları için geliştirme ortamları sağlar.</span><span class="sxs-lookup"><span data-stu-id="30da1-144">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="30da1-145">Windows'daki Linux Docker ana bilgisayar, [Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) sanal makinesini temel alın.</span><span class="sxs-lookup"><span data-stu-id="30da1-145">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) virtual machine.</span></span> <span data-ttu-id="30da1-146">Windows Kapsayıcıları için ana bilgisayar doğrudan Windows'a dayanır.</span><span class="sxs-lookup"><span data-stu-id="30da1-146">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="30da1-147">Mac için Docker CE Apple Hypervisor çerçeve ve [xhyve hypervisor](https://github.com/mist64/xhyve)dayanmaktadır , Mac OS X. Docker CE Windows için bir Linux Docker ana bilgisayar sanal makine sağlar ve Mac için Docker Toolbox değiştirir, Oracle VirtualBox dayalı.</span><span class="sxs-lookup"><span data-stu-id="30da1-147">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="30da1-148">**Docker Enterprise Edition (EE)**: Linux ve Windows geliştirme için Docker araçlarının kurumsal ölçekli bir sürümü.</span><span class="sxs-lookup"><span data-stu-id="30da1-148">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="30da1-149">**Beste**: Çoklu kapsayıcı uygulamalarıtanımlamak ve çalıştırmak için meta verilere sahip bir komut satırı aracı ve YAML dosya biçimi.</span><span class="sxs-lookup"><span data-stu-id="30da1-149">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="30da1-150">Ortama bağlı olarak değerleri geçersiz kılabilen bir veya daha fazla .yml dosyasına sahip birden çok görüntüye dayalı tek bir uygulama tanımlarsınız.</span><span class="sxs-lookup"><span data-stu-id="30da1-150">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="30da1-151">Tanımları oluşturduktan sonra, Docker ana bilgisayarda görüntü başına bir kapsayıcı oluşturan tek bir komutla (docker-createto up) tüm çok kapsayıcı uygulamasını dağıtabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="30da1-151">After you've created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="30da1-152">**Küme**: Tek bir sanal Docker ana bilgisayar gibi maruz kalan Docker ana bilgisayarları topluluğu, böylece uygulama küme içinde birden çok ana bilgisayara yayılmış hizmetlerin birden çok örneğine ölçeklenebilir.</span><span class="sxs-lookup"><span data-stu-id="30da1-152">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="30da1-153">Docker kümeleri Kubernetes, Azure Service Fabric, Docker Swarm ve Mezosphere DC/OS ile oluşturulabilir.</span><span class="sxs-lookup"><span data-stu-id="30da1-153">Docker clusters can be created with Kubernetes, Azure Service Fabric, Docker Swarm and Mesosphere DC/OS.</span></span>

<span data-ttu-id="30da1-154">**Orchestrator**: Kümelerin ve Docker ana bilgisayarlarının yönetimini kolaylaştıran bir araçtır.</span><span class="sxs-lookup"><span data-stu-id="30da1-154">**Orchestrator**: A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="30da1-155">Orchestrators bir komut satırı arabirimi (CLI) veya grafik uI üzerinden kendi görüntüleri, kapsayıcılar ve ana bilgisayarları yönetmenize olanak sağlar.</span><span class="sxs-lookup"><span data-stu-id="30da1-155">Orchestrators enable you to manage their images, containers, and hosts through a command-line interface (CLI) or a graphical UI.</span></span> <span data-ttu-id="30da1-156">Konteyner ağlarını, yapılandırmaları, yük dengelemeyi, hizmet bulmayı, yüksek kullanılabilirliği, Docker ana bilgisayar yapılandırması ve daha fazlasını yönetebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="30da1-156">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="30da1-157">Bir orkestratör, düğüm koleksiyonunda iş yüklerini çalıştırmaktan, dağıtmaktan, ölçeklemekten ve iyileştirir.</span><span class="sxs-lookup"><span data-stu-id="30da1-157">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="30da1-158">Tipik olarak, orkestratör ürünleri, pazardaki diğer tekliflerin yanı sıra Kubernetes ve Azure Hizmet Kumaşı gibi küme altyapısı sağlayan ürünlerle aynıdır.</span><span class="sxs-lookup"><span data-stu-id="30da1-158">Typically, orchestrator products are the same products that provide cluster infrastructure, like Kubernetes and Azure Service Fabric, among other offerings in the market.</span></span>
