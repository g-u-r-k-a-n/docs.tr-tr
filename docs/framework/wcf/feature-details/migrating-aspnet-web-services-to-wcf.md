# <a name="migrating-aspnet-web-services-to-wcf"></a><span data-ttu-id="ad4f2-102">ASP.NET Web Hizmetlerini WCF'ye Taşıma</span><span class="sxs-lookup"><span data-stu-id="ad4f2-102">Migrating ASP.NET Web Services to WCF</span></span>

<span data-ttu-id="ad4f2-103">ASP.NET, Web hizmetleri oluşturmaya yönelik .NET Framework sınıf kitaplıkları ve araçları ve Internet Information Services (IIS) içinde barındırma hizmetleri olanakları sağlar.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-103">ASP.NET provides .NET Framework class libraries and tools for building Web services, as well as facilities for hosting services within Internet Information Services (IIS).</span></span> <span data-ttu-id="ad4f2-104">Windows Communication Foundation (WCF), Web Hizmetleri tarafından kullanılanlar da dahil olmak üzere, yazılım varlıklarının herhangi bir protokol kullanarak iletişim kurmasını sağlamak için .NET Framework sınıf kitaplıkları, Araçlar ve barındırma olanakları sağlar.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-104">Windows Communication Foundation (WCF) provides .NET Framework class libraries, tools and hosting facilities for enabling software entities to communicate using any protocols, including those used by Web services.</span></span>  <span data-ttu-id="ad4f2-105">ASP.NET Web hizmetlerini WCF 'ye geçirmek, uygulamalarınızın WCF 'ye özgü yeni özelliklerden ve geliştirmelerden yararlanmasını sağlar.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-105">Migrating ASP.NET Web Services to WCF allows your applications to take advantage of new features and improvements that are unique to WCF.</span></span>  
  
 <span data-ttu-id="ad4f2-106">WCF, ASP.NET Web hizmetlerine göre birçok önemli avantaj sunar.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-106">WCF has several important advantages relative to ASP.NET Web services.</span></span> <span data-ttu-id="ad4f2-107">ASP.NET Web Hizmetleri Araçları yalnızca Web hizmetleri oluşturmaya yönelik olmakla birlikte, WCF, yazılım varlıklarının birbirleriyle iletişim kurmak için yapılması gerektiğinde kullanılabilecek araçlar sağlar.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-107">While ASP.NET Web services tools are solely for building Web services, WCF provides tools that can be used when software entities must be made to communicate with one another.</span></span> <span data-ttu-id="ad4f2-108">Bu, geliştiricilerin farklı yazılım iletişim senaryolarına uyum sağlamak için bilmesi gereken teknolojilerin sayısını azaltır ve bu da yazılım geliştirme kaynaklarının maliyetinin yanı sıra yazılım geliştirme projelerinin tamamlanma süresini azaltır.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-108">This will reduce the number of technologies that developers are required to know in order to accommodate different software communication scenarios, which in turn will reduce the cost of software development resources, as well as the time to complete software development projects.</span></span>  
  
 <span data-ttu-id="ad4f2-109">WCF, Web hizmeti geliştirme projeleri için bile ASP.NET Web Hizmetleri desteğinden daha fazla Web hizmeti protokolünü destekler.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-109">Even for Web service development projects, WCF supports more Web service protocols than ASP.NET Web services support.</span></span> <span data-ttu-id="ad4f2-110">Bu ek protokoller, diğer şeyler, güvenilir oturumlar ve işlemler arasında, ilgili daha karmaşık çözümler sunar.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-110">These additional protocols provide for more sophisticated solutions involving, amongst other things, reliable sessions and transactions.</span></span>  
  
 <span data-ttu-id="ad4f2-111">WCF, ASP.NET Web hizmetlerinden daha fazla ileti aktarmak için daha fazla protokol destekler.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-111">WCF supports more protocols for transporting messages than ASP.NET Web services.</span></span> <span data-ttu-id="ad4f2-112">ASP.NET Web Hizmetleri yalnızca Köprü Metni Aktarım Protokolü (HTTP) kullanarak ileti göndermeyi destekler.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-112">ASP.NET Web services only support sending messages by using the Hypertext Transfer Protocol (HTTP).</span></span> <span data-ttu-id="ad4f2-113">WCF, HTTP ve Iletim Denetimi Protokolü (TCP), adlandırılmış kanallar ve Microsoft Message Queuing (MSMQ) ile ileti göndermeyi destekler.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-113">WCF supports sending messages by using HTTP, as well as the Transmission Control Protocol (TCP), named pipes, and Microsoft Message Queuing (MSMQ).</span></span> <span data-ttu-id="ad4f2-114">Daha önemli olan WCF, ek aktarım protokollerini desteklemek için genişletilebilir.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-114">More important, WCF can be extended to support additional transport protocols.</span></span> <span data-ttu-id="ad4f2-115">Bu nedenle, WCF kullanılarak geliştirilen yazılım daha fazla başka yazılımlarla birlikte çalışarak, yatırımdaki olası dönüşü artırır.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-115">Therefore, software developed using WCF can be adapted to work together with a wider variety of other software, thereby increasing the potential return on the investment.</span></span>  
  
 <span data-ttu-id="ad4f2-116">WCF, ASP.NET Web Services tarafından sağlanan uygulamaları dağıtmak ve yönetmek için çok daha zengin olanaklar sunar.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-116">WCF provides much richer facilities for deploying and managing applications than ASP.NET Web services provides.</span></span> <span data-ttu-id="ad4f2-117">ASP.NET de sahip olduğu bir yapılandırma sistemine ek olarak, WCF bir yapılandırma Düzenleyicisi, gönderenlerden gelen etkinlik izleme ve herhangi bir sayıda aracı, bir izleme Görüntüleyicisi, ileti günlüğü, çok sayıda performans sayacı ve Windows Yönetim Araçları desteği sunar.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-117">In addition to a configuration system, which ASP.NET also has, WCF offers a configuration editor, activity tracing from senders to receivers and back through any number of intermediaries, a trace viewer, message logging, a vast number of performance counters, and support for Windows Management Instrumentation.</span></span>  
  
 <span data-ttu-id="ad4f2-118">Bu olası WCF avantajları, ASP.NET Web hizmetlerine göre, kullanıyorsanız veya ASP.NET Web hizmetlerini kullanmayı düşünüyorsanız, birkaç seçeneğiniz vardır:</span><span class="sxs-lookup"><span data-stu-id="ad4f2-118">Given these potential benefits of WCF relative to ASP.NET Web services, if you are using, or are considering using ASP.NET Web services you have several options:</span></span>  
  
- <span data-ttu-id="ad4f2-119">ASP.NET Web Hizmetleri 'ni kullanmaya devam edin ve WCF tarafından temin edilecek avantajları kullanın.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-119">Continue to use ASP.NET Web services, and forego the benefits proffered by WCF.</span></span>  
  
- <span data-ttu-id="ad4f2-120">Daha sonraki bir zamanda WCF 'yi benimseme amacıyla ASP.NET Web Hizmetleri 'ni kullanmaya devam edin.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-120">Keep using ASP.NET Web services with the intention of adopting WCF at some time in the future.</span></span> <span data-ttu-id="ad4f2-121">Bu bölümdeki konularda, yeni ASP.NET Web hizmeti uygulamalarını gelecek WCF uygulamalarıyla birlikte kullanabilmek için adayların nasıl en üst düzeye çıkarması açıklanmaktadır.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-121">The topics in this section explain how to maximize the prospects for being able to use new ASP.NET Web service applications together with future WCF applications.</span></span> <span data-ttu-id="ad4f2-122">Bu bölümdeki konular Ayrıca yeni ASP.NET Web Hizmetleri oluşturmayı, bu sayede bunları WCF 'ye geçirmeyi kolaylaştırmak için açıklar.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-122">The topics in this section also explain how to build new ASP.NET Web services so as to make it easier to migrate them to WCF.</span></span> <span data-ttu-id="ad4f2-123">Ancak, hizmetlerin güvenliğini sağlamak önemliyse veya güvenilirlik ya da işlem kesintileri gerekliyse veya özel yönetim tesislerinin oluşturulması gerekiyorsa, WCF 'yi benimsemeniz daha iyi bir seçenektir.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-123">However, if securing the services is important, or reliability or transaction assurances are required, or if custom management facilities will have to be constructed, then it is a better option to adopt WCF.</span></span> <span data-ttu-id="ad4f2-124">WCF, tam olarak bu senaryolar için tasarlanmıştır.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-124">WCF is designed for precisely such scenarios.</span></span>  
  
- <span data-ttu-id="ad4f2-125">Yeni geliştirme için WCF 'yi benimseyin, mevcut ASP.NET Web hizmeti uygulamalarınızı sürdürmek için devam edin.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-125">Adopt WCF for new development, while continuing to maintain your existing ASP.NET Web service applications.</span></span> <span data-ttu-id="ad4f2-126">Bu seçenek büyük olasılıkla en iyi seçenektir.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-126">This choice is very likely the optimal one.</span></span> <span data-ttu-id="ad4f2-127">Bu, WCF 'nin avantajlarını verir, Sparing, var olan uygulamaları kullanmak için değiştirme maliyetlidir.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-127">It yields the benefits of WCF, while sparing the cost of modifying the existing applications to use it.</span></span> <span data-ttu-id="ad4f2-128">Bu senaryoda, yeni WCF uygulamaları mevcut ASP.NET uygulamalarıyla birlikte bulunabilir.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-128">In this scenario, new WCF applications can co-exist with existing ASP.NET applications.</span></span> <span data-ttu-id="ad4f2-129">Yeni WCF uygulamaları var olan ASP.NET Web hizmetlerini kullanabiliyor ve WCF, WCF ASP.NET uyumluluk modunun sanallaştırılmasına göre mevcut ASP.NET uygulamalarına yeni işletimsel özellikleri programlamak için kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-129">New WCF applications will be able to use existing ASP.NET Web services, and WCF can be used to program new operational capabilities into existing ASP.NET applications by virtue of WCF ASP.NET compatibility mode.</span></span>  
  
- <span data-ttu-id="ad4f2-130">WCF 'yi benimseyin ve var olan ASP.NET Web hizmeti uygulamalarını WCF 'ye geçirin.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-130">Adopt WCF and migrate existing ASP.NET Web service applications to WCF.</span></span> <span data-ttu-id="ad4f2-131">WCF tarafından sunulan özelliklerle mevcut uygulamaları geliştirmek veya mevcut ASP.NET Web Hizmetleri 'nin işlevlerini yeni, daha güçlü WCF uygulamaları içinde yeniden oluşturmak için bu seçeneği belirleyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-131">You may choose this option to enhance the existing applications with features provided by WCF, or to reproduce the functionality of existing ASP.NET Web services within new, more powerful WCF applications.</span></span>

> <span data-ttu-id="ad4f2-132">WCF hizmeti IIS 5. x ve ASP.NET tarafından barındırılıyorsa, dikkatli olunmalıdır.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-132">Care must be taken if a WCF service is hosted by IIS 5.x and ASP.NET is uninstalled.</span></span> <span data-ttu-id="ad4f2-133">Bir WCF hizmeti IIS 5. x tarafından barındırılıyorsa, ASP.NET kaldırılırsa hizmetin kodu istenebilir.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-133">When a WCF service is hosted by IIS 5.x, the code for the service can be requested if ASP.NET is uninstalled.</span></span> <span data-ttu-id="ad4f2-134">IIS 5. x ve WCF çalıştıran bir işletim sisteminde ASP.NET kaldırıldığında,. svc uzantılı bir dosya bir metin dosyası olarak kabul edilir ve kaynak kod dahil olmak üzere içerik istek sahibine döndürülür.</span><span class="sxs-lookup"><span data-stu-id="ad4f2-134">When ASP.NET is uninstalled on an operating system that is running IIS 5.x and WCF is uninstalled, a file with the .svc extension is considered a text file and the contents, including any source code, is returned to the requester.</span></span>
