# <a name="deferred-execution-and-lazy-evaluation-in-linq-to-xml-c"></a><span data-ttu-id="3a904-102">Linq'te Ertelenmiş Yürütme ve Tembel Değerlendirme XML 'ye (C#)</span><span class="sxs-lookup"><span data-stu-id="3a904-102">Deferred Execution and Lazy Evaluation in LINQ to XML (C#)</span></span>
<span data-ttu-id="3a904-103">Sorgu ve eksen işlemleri genellikle ertelenmiş yürütme kullanmak için uygulanır.</span><span class="sxs-lookup"><span data-stu-id="3a904-103">Query and axis operations are often implemented to use deferred execution.</span></span> <span data-ttu-id="3a904-104">Bu konu, ertelenmiş yürütmenin gereklerini ve avantajlarını ve bazı uygulama hususlarını açıklar.</span><span class="sxs-lookup"><span data-stu-id="3a904-104">This topic explains the requirements and advantages of deferred execution, and some implementation considerations.</span></span>  
  
## <a name="deferred-execution"></a><span data-ttu-id="3a904-105">Ertelenmiş Yürütme</span><span class="sxs-lookup"><span data-stu-id="3a904-105">Deferred Execution</span></span>  
 <span data-ttu-id="3a904-106">Ertelenmiş yürütme, bir ifadenin değerlendirilmesinin *gerçekleşen* değeri gerçekten gerekli olana kadar geciktiği anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="3a904-106">Deferred execution means that the evaluation of an expression is delayed until its *realized* value is actually required.</span></span> <span data-ttu-id="3a904-107">Ertelenmiş yürütme, özellikle zincirlenmiş sorgular veya işlemeler içeren programlarda büyük veri toplamaları işlemek zorunda olduğunuzda performansı büyük ölçüde artırabilir.</span><span class="sxs-lookup"><span data-stu-id="3a904-107">Deferred execution can greatly improve performance when you have to manipulate large data collections, especially in programs that contain a series of chained queries or manipulations.</span></span> <span data-ttu-id="3a904-108">En iyi durumda, ertelenmiş yürütme kaynak koleksiyonu üzerinden yalnızca tek bir yineleme sağlar.</span><span class="sxs-lookup"><span data-stu-id="3a904-108">In the best case, deferred execution enables only a single iteration through the source collection.</span></span>  
  
 <span data-ttu-id="3a904-109">LINQ teknolojileri, hem çekirdek <xref:System.Linq?displayProperty=nameWithType> sınıfların üyeleri hem de çeşitli LINQ ad alanlarındaki uzantı yöntemlerinde <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>ertelenmiş yürütmeyi kapsamlı bir şekilde kullanır.</span><span class="sxs-lookup"><span data-stu-id="3a904-109">The LINQ technologies make extensive use of deferred execution in both the members of core <xref:System.Linq?displayProperty=nameWithType> classes and in the extension methods in the various LINQ namespaces, such as <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="3a904-110">Ertelenmiş yürütme, bir yineleyici bloğu içinde kullanıldığında [verim](../../../language-reference/keywords/yield.md) anahtar sözcüğü `yield-return` (deyim biçiminde) tarafından doğrudan C# dilinde desteklenir.</span><span class="sxs-lookup"><span data-stu-id="3a904-110">Deferred execution is supported directly in the C# language by the [yield](../../../language-reference/keywords/yield.md) keyword (in the form of the `yield-return` statement) when used within an iterator block.</span></span> <span data-ttu-id="3a904-111">Böyle bir yineleyici, tür <xref:System.Collections.IEnumerator> veya <xref:System.Collections.Generic.IEnumerator%601> (veya türetilmiş tür) bir koleksiyon döndürmelidir.</span><span class="sxs-lookup"><span data-stu-id="3a904-111">Such an iterator must return a collection of type <xref:System.Collections.IEnumerator> or <xref:System.Collections.Generic.IEnumerator%601> (or a derived type).</span></span>  
  
## <a name="eager-vs-lazy-evaluation"></a><span data-ttu-id="3a904-112">Istekli vs Tembel Değerlendirme</span><span class="sxs-lookup"><span data-stu-id="3a904-112">Eager vs. Lazy Evaluation</span></span>  
 <span data-ttu-id="3a904-113">Ertelenmiş yürütmeyi uygulayan bir yöntem yazdığınızda, yöntemi tembel değerlendirme veya istekli değerlendirme kullanarak uygulayıp uygulamayacağınıza da karar vermeniz gerekir.</span><span class="sxs-lookup"><span data-stu-id="3a904-113">When you write a method that implements deferred execution, you also have to decide whether to implement the method using lazy evaluation or eager evaluation.</span></span>  
  
- <span data-ttu-id="3a904-114">*Tembel değerlendirmede,* kaynak koleksiyonun tek bir öğesi her çağrı sırasında yineleyiciye işlenir.</span><span class="sxs-lookup"><span data-stu-id="3a904-114">In *lazy evaluation*, a single element of the source collection is processed during each call to the iterator.</span></span> <span data-ttu-id="3a904-115">Bu, yineleyicilerin uygulanabın tipik yoludur.</span><span class="sxs-lookup"><span data-stu-id="3a904-115">This is the typical way in which iterators are implemented.</span></span>  
  
- <span data-ttu-id="3a904-116">*Istekli değerlendirmede,* yineleyiciye yapılan ilk çağrı, tüm koleksiyonun işlenmesiyle sonuçlanır.</span><span class="sxs-lookup"><span data-stu-id="3a904-116">In *eager evaluation*, the first call to the iterator will result in the entire collection being processed.</span></span> <span data-ttu-id="3a904-117">Kaynak koleksiyonun geçici bir kopyası da gerekebilir.</span><span class="sxs-lookup"><span data-stu-id="3a904-117">A temporary copy of the source collection might also be required.</span></span> <span data-ttu-id="3a904-118">Örneğin, yöntemin <xref:System.Linq.Enumerable.OrderBy%2A> ilk öğeyi döndürmeden önce tüm koleksiyonu sıralaması gerekiyor.</span><span class="sxs-lookup"><span data-stu-id="3a904-118">For example, the <xref:System.Linq.Enumerable.OrderBy%2A> method has to sort the entire collection before it returns the first element.</span></span>  
  
 <span data-ttu-id="3a904-119">Denetim, yükün değerlendirilmesi boyunca genel bilgileri eşit olarak dağıttığı ve geçici verilerin kullanımını en aza indirdiği için genellikle daha iyi performans sağlar.</span><span class="sxs-lookup"><span data-stu-id="3a904-119">Lazy evaluation usually yields better performance because it distributes overhead processing evenly throughout the evaluation of the collection and minimizes the use of temporary data.</span></span> <span data-ttu-id="3a904-120">Tabii ki, bazı işlemler için, ara sonuçları hayata getirmekten başka bir seçenek yoktur.</span><span class="sxs-lookup"><span data-stu-id="3a904-120">Of course, for some operations, there is no other option than to materialize intermediate results.</span></span>
