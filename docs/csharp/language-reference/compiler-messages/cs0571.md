# <a name="compiler-error-cs0571"></a><span data-ttu-id="99540-102">Derleyici Hatası CS0571</span><span class="sxs-lookup"><span data-stu-id="99540-102">Compiler Error CS0571</span></span>
<span data-ttu-id="99540-103">'function' : operatör veya aksesuarı açıkça arayamaz</span><span class="sxs-lookup"><span data-stu-id="99540-103">'function' : cannot explicitly call operator or accessor</span></span>  
  
 <span data-ttu-id="99540-104">Bazı operatörlerin iç adları vardır.</span><span class="sxs-lookup"><span data-stu-id="99540-104">Certain operators have internal names.</span></span> <span data-ttu-id="99540-105">Örneğin, **op_Increment** ++ işlecinin iç adıdır.</span><span class="sxs-lookup"><span data-stu-id="99540-105">For example, **op_Increment** is the internal name of the ++ operator.</span></span> <span data-ttu-id="99540-106">Bu tür yöntem adlarını kullanmamalı veya açıkça aramamalısınız.</span><span class="sxs-lookup"><span data-stu-id="99540-106">You should not use or explicitly call such method names.</span></span>  
  
 <span data-ttu-id="99540-107">Aşağıdaki örnek CS0571 oluşturur:</span><span class="sxs-lookup"><span data-stu-id="99540-107">The following sample generates CS0571:</span></span>  
  
```csharp  
// CS0571.cs  
public class MyClass  
{  
   public static MyClass operator ++ (MyClass c)  
   {  
      return null;  
   }  
  
   public static int prop  
   {  
      get  
      {  
         return 1;  
      }  
      set  
      {  
      }  
   }  
  
   public static void Main()  
   {  
      op_Increment(null);   // CS0571  
      // use the increment operator as follows  
      // MyClass x = new MyClass();  
      // x++;  
  
      set_prop(1);      // CS0571  
      // try the following line instead  
      // prop = 1;  
   }  
}  
```
