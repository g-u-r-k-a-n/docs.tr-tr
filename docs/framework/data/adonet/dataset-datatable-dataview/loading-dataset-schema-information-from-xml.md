## <a name="inferxmlschema"></a><span data-ttu-id="01a06-126">InferXmlSchema</span><span class="sxs-lookup"><span data-stu-id="01a06-126">InferXmlSchema</span></span>  

 <span data-ttu-id="01a06-127">Veri **kümesinin,** **veri kümesinin** **InferXmlSchema** yöntemini kullanarak bir XML belgesinden şemasını çıkarmasını da söyleyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="01a06-127">You can also instruct the **DataSet** to infer its schema from an XML document using the **InferXmlSchema** method of the **DataSet**.</span></span> <span data-ttu-id="01a06-128">**InferXmlSchema** **, her ikisi** de de bir **XmlReadMode** for **InferSchema** ( **verileri ve** KERS şeması yükler) ve okunan belge satır içi şema içermiyorsa, her ikisi de aynı şekilde çalışır.</span><span class="sxs-lookup"><span data-stu-id="01a06-128">**InferXmlSchema** functions the same as do both **ReadXml** with an **XmlReadMode** of **InferSchema** (loads data as well as infers schema), and **ReadXmlSchema** if the document being read contains no inline schema.</span></span> <span data-ttu-id="01a06-129">Ancak, **InferXmlSchema** , şema çıkarsandığınızda yok SAYıLACAK belirli XML ad alanlarını belirtmenize olanak tanıyan ek bir özellik sunar.</span><span class="sxs-lookup"><span data-stu-id="01a06-129">However, **InferXmlSchema** provides the additional capability of allowing you to specify particular XML namespaces to be ignored when the schema is inferred.</span></span> <span data-ttu-id="01a06-130">**InferXmlSchema** iki gerekli bağımsız değişkeni alır: bir dosya adı, akış veya **XMLREADER**ile belirtilen XML belgesinin konumu. ve işlem tarafından yok sayılacak XML ad alanlarının dize dizisi.</span><span class="sxs-lookup"><span data-stu-id="01a06-130">**InferXmlSchema** takes two required arguments: the location of the XML document, specified by a file name, a stream, or an **XmlReader**; and a string array of XML namespaces to be ignored by the operation.</span></span>  
  
 <span data-ttu-id="01a06-131">Örneğin, aşağıdaki XML 'i göz önünde bulundurun:</span><span class="sxs-lookup"><span data-stu-id="01a06-131">For example, consider the following XML:</span></span>

  <ProductID od:adotype="20">1</ProductID>

  <Discontinued od:adotype="11">0</Discontinued>

</NewDataSet>

 <span data-ttu-id="01a06-132">Önceki XML belgesindeki öğeler için belirtilen öznitelikler nedeniyle, **ınsecollection** 'ın bir **XmlReadMode** değeri olan **ReadXmlSchema** yöntemi ve **ReadXml** yöntemi, belgedeki her öğe için tablolar oluşturacak: **Kategoriler**, **CategoryID**, **CategoryName**, **Description**, **Products**, **ProductID**, **ReorderLevel**ve **Discontinued**.</span><span class="sxs-lookup"><span data-stu-id="01a06-132">Because of the attributes specified for the elements in the preceding XML document, both the **ReadXmlSchema** method and the **ReadXml** method with an **XmlReadMode** of **InferSchema** would create tables for every element in the document: **Categories**, **CategoryID**, **CategoryName**, **Description**, **Products**, **ProductID**, **ReorderLevel**, and **Discontinued**.</span></span> <span data-ttu-id="01a06-133">(Daha fazla bilgi için bkz. [XML 'Den veri kümesi Ilişkisel yapısını](inferring-dataset-relational-structure-from-xml.md)anlamak.) Ancak, daha uygun bir yapı yalnızca **Kategoriler** ve **ürün** tabloları oluşturmak ve ardından **Kategoriler** tablosunda **CategoryID**, **CategoryName**ve **Description** sütunları, **Products** tablosundaki **ProductID**, **ReorderLevel**ve **Discontinued** sütunları oluşturmak için kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="01a06-133">(For more information, see [Inferring DataSet Relational Structure from XML](inferring-dataset-relational-structure-from-xml.md).)
