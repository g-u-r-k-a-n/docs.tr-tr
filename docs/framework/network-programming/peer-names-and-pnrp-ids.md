<span data-ttu-id="2ab05-103">Eş adı, bir bilgisayar, bir Kullanıcı, Grup, hizmet veya bir IPv6 adresine çözümlenebilen bir eş ile ilişkili herhangi bir şey olabilen iletişim için bir uç noktayı temsil eder.</span><span class="sxs-lookup"><span data-stu-id="2ab05-103">A Peer Name represents an endpoint for communication, which can be a computer, a user, a group, a service, or anything associated with a Peer that can be resolved to an IPv6 address.</span></span> <span data-ttu-id="2ab05-104">Eş adı çözümleme Protokolü (PNRP), bulut üyelerini tanımlamak için kullanılan bir PNRP KIMLIĞI oluşturmak için istatistiksel olarak benzersiz eş adı alır.</span><span class="sxs-lookup"><span data-stu-id="2ab05-104">The Peer Name Resolution Protocol (PNRP) takes the statistically unique Peer Name for the creation of a PNRP ID, which is used to identify cloud members.</span></span>

 <span data-ttu-id="2ab05-106">Eş adları, güvenli olmayan veya güvenli şekilde kaydedilebilir.</span><span class="sxs-lookup"><span data-stu-id="2ab05-106">Peer names can be registered as unsecured or secured.</span></span> <span data-ttu-id="2ab05-107">Güvenli olmayan adlar yalnızca, kimlik sahtekarlığına tabi olan metin dizeleridir ve herkes bir yinelenen güvenli olmayan ad kaydedebilir.</span><span class="sxs-lookup"><span data-stu-id="2ab05-107">Unsecured names are just text strings that are subject to spoofing, as anyone can register a duplicate unsecured name.</span></span> <span data-ttu-id="2ab05-108">Güvenli olmayan adlar, özel veya korumalı ağlarda en iyi şekilde kullanılır.</span><span class="sxs-lookup"><span data-stu-id="2ab05-108">Unsecured names are best used in private or otherwise protected networks.</span></span> <span data-ttu-id="2ab05-109">Güvenli adlar bir sertifikayla ve dijital imzayla korunur.</span><span class="sxs-lookup"><span data-stu-id="2ab05-109">Secured names are protected with a certificate and a digital signature.</span></span> <span data-ttu-id="2ab05-110">Yalnızca özgün yayımcı, güvenli bir adın sahipliğini kanıtlayabilecek.</span><span class="sxs-lookup"><span data-stu-id="2ab05-110">Only the original publisher will be able to prove ownership of a secured name.</span></span>  
  
 <span data-ttu-id="2ab05-111">Bulut ve kapsamın birleşimi, PNRP etkinliğine katılan eşleri için makul bir güvenli ortam sağlar.</span><span class="sxs-lookup"><span data-stu-id="2ab05-111">The combination of cloud and scope provides a reasonably secure environment for peers that participate in PNRP activity.</span></span> <span data-ttu-id="2ab05-112">Bununla birlikte, güvenli bir eş adı kullanılması Ağ uygulamasının genel güvenliğinin sağlanması gerekmez.</span><span class="sxs-lookup"><span data-stu-id="2ab05-112">However, using a secured peer name does not ensure the overall security of the networking application.</span></span> <span data-ttu-id="2ab05-113">Uygulamanın güvenliği uygulamaya bağımlıdır.</span><span class="sxs-lookup"><span data-stu-id="2ab05-113">Security of the application is implementation-dependent.</span></span>  
  
 <span data-ttu-id="2ab05-114">Güvenli eş adları yalnızca sahibi tarafından kaydedilir ve ortak anahtar şifrelemesi ile korunur.</span><span class="sxs-lookup"><span data-stu-id="2ab05-114">Secured peer names are only registered by their owner and are protected with public key cryptography.</span></span> <span data-ttu-id="2ab05-115">Güvenli bir eş adı, karşılık gelen özel anahtara sahip olan eşdüzey varlığa ait olarak değerlendirilir.</span><span class="sxs-lookup"><span data-stu-id="2ab05-115">A secured peer name is considered owned by the peer entity having the corresponding private key.</span></span> <span data-ttu-id="2ab05-116">Sahiplik, özel anahtar kullanılarak imzalanmış sertifikalı eş adresi (CPA) aracılığıyla etkinleştirilebilir.</span><span class="sxs-lookup"><span data-stu-id="2ab05-116">Ownership can be proved via the certified peer address (CPA), which is signed using the private key.</span></span> <span data-ttu-id="2ab05-117">Kötü niyetli bir Kullanıcı, karşılık gelen özel anahtar olmadan bir eş adının sahipliğini taklit edemez.</span><span class="sxs-lookup"><span data-stu-id="2ab05-117">A malicious user cannot forge ownership of a peer name without the corresponding private key.</span></span>  
  
## <a name="pnrp-ids"></a><span data-ttu-id="2ab05-118">PNRP kimlikleri</span><span class="sxs-lookup"><span data-stu-id="2ab05-118">PNRP IDs</span></span>  

 <span data-ttu-id="2ab05-119">![PNRP KIMLIĞI](./media/fdc9e8a0-4a1c-488d-a019-bc3a1973220c.gif "fdc9e8a0-4a1c-488d-a019-bc3a1973220c")</span><span class="sxs-lookup"><span data-stu-id="2ab05-119">![PNRP ID](./media/fdc9e8a0-4a1c-488d-a019-bc3a1973220c.gif "fdc9e8a0-4a1c-488d-a019-bc3a1973220c")</span></span>  
  
 <span data-ttu-id="2ab05-120">PNRP kimlikleri aşağıdakilerden oluşur:</span><span class="sxs-lookup"><span data-stu-id="2ab05-120">PNRP IDs are composed of the following:</span></span>  
  
- <span data-ttu-id="2ab05-121">Eşler arası (P2P) KIMLIĞI olarak bilinen yüksek sıralı 128 bitleri, uç noktaya atanan bir eş adı karmasıdır.</span><span class="sxs-lookup"><span data-stu-id="2ab05-121">The high-order 128 bits, known as the peer-to-peer (P2P) ID, are a hash of a peer name assigned to the endpoint.</span></span> <span data-ttu-id="2ab05-122">Eş adı şu biçimdedir: *Authority. sınıflandırıcı*.</span><span class="sxs-lookup"><span data-stu-id="2ab05-122">The peer name has the following format: *Authority.Classifier*.</span></span> <span data-ttu-id="2ab05-123">Güvenli adlar için, *yetkili* , eş adının genel anahtarının onaltılık karakterlerle GÜVENLI karma ALGORITMASı 1 (SHA1) karmasıdır.</span><span class="sxs-lookup"><span data-stu-id="2ab05-123">For secured names, *Authority* is the Secure Hash Algorithm 1 (SHA1) hash of the public key of the peer name in hexadecimal characters.</span></span> <span data-ttu-id="2ab05-124">Güvenli olmayan adlar için, *yetkili* tek karakter "0" dir.</span><span class="sxs-lookup"><span data-stu-id="2ab05-124">For unsecured names, the *Authority* is the single character "0".</span></span> <span data-ttu-id="2ab05-125">*Sınıflandırıcı* , uygulamayı tanımlayan bir dizedir.</span><span class="sxs-lookup"><span data-stu-id="2ab05-125">*Classifier* is a string that identifies the application.</span></span> <span data-ttu-id="2ab05-126">Eş adı Sınıflandırıcısı, Sonlandırıcı dahil 149 karakterden daha uzun olamaz `null` .</span><span class="sxs-lookup"><span data-stu-id="2ab05-126">No peer name classifier can be greater than 149 characters long, including the `null` terminator.</span></span>  
  
- <span data-ttu-id="2ab05-127">Düşük sıralı 128 bitleri, aynı bulutta aynı P2P KIMLIĞININ farklı örneklerini tanımlayan bir üretilen sayı olan hizmet konumu için kullanılır.</span><span class="sxs-lookup"><span data-stu-id="2ab05-127">The low-order 128 bits are used for the Service Location, which is a generated number that identifies different instances of the same P2P ID in the same cloud.</span></span>  
  
 <span data-ttu-id="2ab05-128">P2P KIMLIĞI ve hizmet konumunun bu birleşimi, birden çok PNRP kimliğinin tek bir bilgisayardan kaydedilmesini sağlar.</span><span class="sxs-lookup"><span data-stu-id="2ab05-128">This combination of P2P ID and Service Location allows multiple PNRP IDs to be registered from a single computer.</span></span>
