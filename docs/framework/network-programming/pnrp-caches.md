 <span data-ttu-id="26e3e-105">Eş düğümü başlatıldığında, PNRP önbelleğini veya Eş Adı Kaydı Koleksiyonu'nu başlatacak şekilde, bir düğüm aşağıdaki yöntemleri kullanabilir:</span><span class="sxs-lookup"><span data-stu-id="26e3e-105">To initialize the PNRP cache, or Peer Name Record Collection, when a peer node starts up, a node can use the following methods:</span></span>

## <a name="scaling-peer-name-resolution-with-a-multi-level-cache"></a><span data-ttu-id="26e3e-108">Eş Adı Çözümünü Çok Düzeyli Önbellekle Ölçekleme</span><span class="sxs-lookup"><span data-stu-id="26e3e-108">Scaling Peer Name Resolution with a Multi-Level Cache</span></span>  
 <span data-ttu-id="26e3e-109">PNRP önbelleklerinin boyutlarını küçük tutmak için eş düğümleri, her düzeyin en fazla sayıda giriş içerdiği çok düzeyli bir önbellek kullanır.</span><span class="sxs-lookup"><span data-stu-id="26e3e-109">To keep the sizes of the PNRP caches small, peer nodes use a multi-level cache, in which each level contains a maximum number of entries.</span></span> <span data-ttu-id="26e3e-110">Önbellekteki her düzey PNRP kimlik numarası alanının onda bir küçük kısmını temsil eder (2<sup>256).</sup></span><span class="sxs-lookup"><span data-stu-id="26e3e-110">Each level in the cache represents a one tenth smaller portion of the PNRP ID number space (2<sup>256</sup>).</span></span> <span data-ttu-id="26e3e-111">Önbellekteki en düşük düzey, yerel olarak kaydedilmiş bir PNRP Kimliği ve sayısal olarak yakın olan diğer PNRP kimlikleri içerir.</span><span class="sxs-lookup"><span data-stu-id="26e3e-111">The lowest level in the cache contains a locally registered PNRP ID and other PNRP IDs that are numerically close to it.</span></span> <span data-ttu-id="26e3e-112">Önbelleğin düzeyi en fazla 20 girişle doldurulduğundan, yeni bir alt düzey oluşturulur.</span><span class="sxs-lookup"><span data-stu-id="26e3e-112">As a level of the cache is filled with a maximum of 20 entries, a new lower level is created.</span></span> <span data-ttu-id="26e3e-113">Önbellekteki maksimum düzey sayısı log10 (Buluttaki Toplam PNRP ID sayısı) sırasına göredir.</span><span class="sxs-lookup"><span data-stu-id="26e3e-113">The maximum number of levels in the cache is on the order of log10(Total number of PNRP IDs in the cloud).</span></span> <span data-ttu-id="26e3e-114">Örneğin, 100 milyon PNRP Kimliğine sahip küresel bir bulut için, önbellekte en fazla 8 (=log10(100.000.000)) düzeyi ve ad çözümlemesi sırasında bir PNRP Kimliğini çözmek için benzer sayıda atlama vardır.</span><span class="sxs-lookup"><span data-stu-id="26e3e-114">For example, for a global cloud with 100 million PNRP IDs, there are no more than 8 (=log10(100,000,000)) levels in the cache and a similar number of hops to resolve a PNRP ID during name resolution.</span></span> <span data-ttu-id="26e3e-115">Bu mekanizma, ilgili EBM'ye sahip eş bulunana kadar PNRP İstek iletilerini en yakın eşe ileterek rasgele bir PNRP Kimliğinin  <span data-ttu-id="26e3e-116">Çözünürlüğün tamamlanınabilmesi için, bir düğüm önbelleğinin en alt düzeye her girişinde, önbelleğin son düzeyindeki tüm düğümlere girişin bir kopyasını sel basıyor.</span><span class="sxs-lookup"><span data-stu-id="26e3e-116">To ensure that resolution can complete, each time a node adds an entry to the lowest level of its cache, it floods a copy of the entry to all the nodes within the last level of the cache.</span></span>  
  
 <span data-ttu-id="26e3e-117">Önbellek girişleri zaman içinde yenilenir.</span><span class="sxs-lookup"><span data-stu-id="26e3e-117">The cache entries are refreshed over time.</span></span> <span data-ttu-id="26e3e-118">Eski önbellek girişleri önbellekten kaldırılır.</span><span class="sxs-lookup"><span data-stu-id="26e3e-118">Cache entries that are stale are removed from the cache.</span></span> <span data-ttu-id="26e3e-119">Sonuç olarak, PNRP ID'lerinin dağıtılmış karma tablosu, adres kayıtlarının ve DNS protokolünün adresle ilişkili düğümün ağda etkin olduğuna dair hiçbir garanti vermediği DNS'nin aksine etkin uç noktaları temel almaktadır.</span><span class="sxs-lookup"><span data-stu-id="26e3e-119">The result is that the distributed hash table of PNRP IDs is based on active endpoints, unlike DNS in which address records and the DNS protocol provide no guarantee that the node associated with the address is actively on the network.</span></span>
