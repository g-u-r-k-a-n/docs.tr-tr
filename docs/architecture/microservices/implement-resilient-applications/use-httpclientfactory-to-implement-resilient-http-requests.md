<span data-ttu-id="bd7ff-161">`HttpClient`İçinden bir nesne aldığınızda `IHttpClientFactory` , yeni bir örnek döndürülür.</span><span class="sxs-lookup"><span data-stu-id="bd7ff-161">Each time you get an `HttpClient` object from the `IHttpClientFactory`, a new instance is returned.</span></span> <span data-ttu-id="bd7ff-162">Ancak `HttpClient` `HttpMessageHandler` , `IHttpClientFactory` süresi dolmadığı sürece kaynak tüketimini azaltmak için tarafından havuza alınmış ve yeniden kullanılan bir kullanır `HttpMessageHandler` .</span><span class="sxs-lookup"><span data-stu-id="bd7ff-162">But each `HttpClient` uses an `HttpMessageHandler` that's pooled and reused by the `IHttpClientFactory` to reduce resource consumption, as long as the `HttpMessageHandler`'s lifetime hasn't expired.</span></span>

<span data-ttu-id="bd7ff-163">Her işleyici genellikle kendi temel aldığı HTTP bağlantılarını yönettiğinden, işleyicilerin havuzlaması istenir; gerekenden daha fazla işleyici oluşturulması bağlantı gecikmeleri oluşmasına neden olabilir.</span><span class="sxs-lookup"><span data-stu-id="bd7ff-163">Pooling of handlers is desirable as each handler typically manages its own underlying HTTP connections; creating more handlers than necessary can result in connection delays.</span></span> <span data-ttu-id="bd7ff-164">Ayrıca, bazı işleyiciler bağlantıları süresiz olarak açık tutar, bu da işleyicinin DNS değişikliklerine yeniden davranmasını engelleyebilir.</span><span class="sxs-lookup"><span data-stu-id="bd7ff-164">Some handlers also keep connections open indefinitely, which can prevent the handler from reacting to DNS changes.</span></span>

<span data-ttu-id="bd7ff-165">`HttpMessageHandler`Havuzdaki nesneler, havuzdaki bir örneğin yeniden kullanılabilmesi için gereken süre olan bir yaşam süresine sahiptir `HttpMessageHandler` .</span><span class="sxs-lookup"><span data-stu-id="bd7ff-165">The `HttpMessageHandler` objects in the pool have a lifetime that's the length of time that an `HttpMessageHandler` instance in the pool can be reused.</span></span> <span data-ttu-id="bd7ff-166">Varsayılan değer iki dakikadır, ancak türü belirlenmiş Istemci başına geçersiz kılınabilir.</span><span class="sxs-lookup"><span data-stu-id="bd7ff-166">The default value is two minutes, but it can be overridden per Typed Client.</span></span> <span data-ttu-id="bd7ff-167">Bunu geçersiz kılmak için, `SetHandlerLifetime()` <xref:Microsoft.Extensions.DependencyInjection.IHttpClientBuilder> aşağıdaki kodda gösterildiği gibi istemciyi oluştururken döndürülen ' i çağırın:</span><span class="sxs-lookup"><span data-stu-id="bd7ff-167">To override it, call `SetHandlerLifetime()` on the <xref:Microsoft.Extensions.DependencyInjection.IHttpClientBuilder> that's returned when creating the client, as shown in the following code:</span></span>


<span data-ttu-id="bd7ff-168">Her tür Istemcinin kendi yapılandırılmış işleyici yaşam süresi değeri olabilir.</span><span class="sxs-lookup"><span data-stu-id="bd7ff-168">Each Typed Client can have its own configured handler lifetime value.</span></span> <span data-ttu-id="bd7ff-169">Kullanım süresini, `InfiniteTimeSpan` işleyicinin süre sonunu devre dışı bırakacak şekilde ayarlayın.</span><span class="sxs-lookup"><span data-stu-id="bd7ff-169">Set the lifetime to `InfiniteTimeSpan` to disable handler expiry.</span></span>

### <a name="implement-your-typed-client-classes-that-use-the-injected-and-configured-httpclient"></a><span data-ttu-id="bd7ff-170">Eklenen ve yapılandırılmış HttpClient kullanan, yazılan Istemci sınıflarınızı uygulama</span><span class="sxs-lookup"><span data-stu-id="bd7ff-170">Implement your Typed Client classes that use the injected and configured HttpClient</span></span>
