## <a name="properties-and-events"></a><span data-ttu-id="b56c7-168">Özellikler ve olaylar</span><span class="sxs-lookup"><span data-stu-id="b56c7-168">Properties and Events</span></span>  

 <span data-ttu-id="b56c7-169">İçindeki özellikleriyle yakından birlikte, [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] özellik değiştirme olaylarının kavramıdır.</span><span class="sxs-lookup"><span data-stu-id="b56c7-169">Closely tied in with the properties in [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] is the concept of property-changed events.</span></span> <span data-ttu-id="b56c7-170">Dinamik özellikler için, istemci uygulamanın bir özellik değerinin değiştiğini bilmesi için bir yol gerekir, böylece bilgi önbelleğini güncelleştirebilir veya yeni bilgilere başka bir şekilde tepki verebilir.</span><span class="sxs-lookup"><span data-stu-id="b56c7-170">For dynamic properties, the client application needs a way to know that a property value has changed, so that it can update its cache of information or react to the new information in some other way.</span></span>  
  
 <span data-ttu-id="b56c7-171">Sağlayıcılar, değişikliklere bir şeyler olduğunda olaylar yükseltir [!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)] .</span><span class="sxs-lookup"><span data-stu-id="b56c7-171">Providers raise events when something in the [!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)] changes.</span></span> <span data-ttu-id="b56c7-172">Örneğin, bir onay kutusu seçiliyse veya silinirse, sağlayıcının Iki durumlu düzenin uygulamasıyla bir özellik değiştirildi olayı tetiklenir.</span><span class="sxs-lookup"><span data-stu-id="b56c7-172">For example, if a check box is selected or cleared, a property-changed event is raised by the provider's implementation of the Toggle pattern.</span></span> <span data-ttu-id="b56c7-173">Sağlayıcılar, herhangi bir istemcinin olayları dinleyip dinlemediğini veya belirli olayları dinlemediğine bağlı olarak olayları seçerek oluşturabilir.</span><span class="sxs-lookup"><span data-stu-id="b56c7-173">Providers can raise events selectively, depending on whether any clients are listening for events, or listening for specific events.</span></span>  
  
 <span data-ttu-id="b56c7-174">Tüm özellik değişiklikleri olay oluşturmaz; Bu, öğesi için UI Otomasyon sağlayıcısının uygulamasına tamamıyla çalışır.</span><span class="sxs-lookup"><span data-stu-id="b56c7-174">Not all property changes raise events; that is entirely up to the implementation of the UI Automation provider for the element.</span></span> <span data-ttu-id="b56c7-175">Örneğin, liste kutuları için standart ara sunucu sağlayıcıları, değişiklikler sırasında bir olay oluşturmaz <xref:System.Windows.Automation.SelectionPattern.SelectionProperty> .</span><span class="sxs-lookup"><span data-stu-id="b56c7-175">For example, the standard proxy providers for list boxes do not raise an event when the <xref:System.Windows.Automation.SelectionPattern.SelectionProperty> changes.</span></span> <span data-ttu-id="b56c7-176">Bu durumda, bunun yerine uygulamanın dinlemesi gerekir <xref:System.Windows.Automation.SelectionItemPattern.ElementSelectedEvent> .</span><span class="sxs-lookup"><span data-stu-id="b56c7-176">In this case, the application instead must listen for an <xref:System.Windows.Automation.SelectionItemPattern.ElementSelectedEvent>.</span></span>  
  
 <span data-ttu-id="b56c7-177">İstemciler bu olaylara abone olarak olay dinler.</span><span class="sxs-lookup"><span data-stu-id="b56c7-177">Clients listen for events by subscribing to them.</span></span> <span data-ttu-id="b56c7-178">Olaylara abone olmak, olayları işleyebilen temsilci yöntemleri oluşturma ve ardından yöntemlerini [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] Bu yöntemlerle ele alacak belirli olaylarla birlikte geçirme anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="b56c7-178">Subscribing to events means creating delegate methods that can handle the events, and then passing the methods to [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] along with the specific events that will be dealt with in those methods.</span></span> <span data-ttu-id="b56c7-179">Özellikle, özellik tarafından değiştirilen olaylar için, istemcilerin uygulaması gerekir <xref:System.Windows.Automation.AutomationPropertyChangedEventHandler> .</span><span class="sxs-lookup"><span data-stu-id="b56c7-179">For property-changed events in particular, clients must implement <xref:System.Windows.Automation.AutomationPropertyChangedEventHandler>.</span></span>
