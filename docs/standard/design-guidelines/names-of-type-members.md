 <span data-ttu-id="497af-107">Yöntemler işlem yapma yöntemi olduğundan, tasarım yönergeleri, yöntem adlarının fiiller veya fiil tümceleri olmasını gerektirir.</span><span class="sxs-lookup"><span data-stu-id="497af-107">Because methods are the means of taking action, the design guidelines require that method names be verbs or verb phrases.</span></span> <span data-ttu-id="497af-108">Bu kılavuz aşağıdaki şekilde, yöntem adlarını özellik ve tür adlarından ayırt etmek için kullanılır, bu da isim veya sıfatıcı tümceciklerdir.</span><span class="sxs-lookup"><span data-stu-id="497af-108">Following this guideline also serves to distinguish method names from property and type names, which are noun or adjective phrases.</span></span>


## <a name="names-of-properties"></a><span data-ttu-id="497af-110">Özelliklerin adları</span><span class="sxs-lookup"><span data-stu-id="497af-110">Names of Properties</span></span>

 <span data-ttu-id="497af-114">✔️ adı, isim tümceciğini veya sıfatıcı kullanarak ad özellikleri YAPıN.</span><span class="sxs-lookup"><span data-stu-id="497af-114">✔️ DO name properties using a noun, noun phrase, or adjective.</span></span>

 <span data-ttu-id="497af-115">❌ Aşağıdaki örnekte olduğu gibi "Get" yöntemlerinin adıyla eşleşen özelliklere sahip DEĞILDIR:</span><span class="sxs-lookup"><span data-stu-id="497af-115">❌ DO NOT have properties that match the name of "Get" methods as in the following example:</span></span>

 <span data-ttu-id="497af-118">✔️ koleksiyon özelliklerini, tekil bir tümceciği, ardından "List" veya "Collection" yerine, koleksiyondaki öğeleri açıklayan bir plural ifadesi ile YAPıN.</span><span class="sxs-lookup"><span data-stu-id="497af-118">✔️ DO name collection properties with a plural phrase describing the items in the collection instead of using a singular phrase followed by "List" or "Collection".</span></span>

 <span data-ttu-id="497af-122">Örneğin, aşağıdaki özellik adlı bir Enum değerini doğru bir şekilde alır ve ayarlar `Color` , bu nedenle özellik şu şekilde adlandırılır `Color` :</span><span class="sxs-lookup"><span data-stu-id="497af-122">For example, the following property correctly gets and sets an enum value named `Color`, so the property is named `Color`:</span></span>

```csharp

    public Color Color { get {...} set {...} }
}
```

## <a name="names-of-events"></a><span data-ttu-id="497af-123">Olay adları</span><span class="sxs-lookup"><span data-stu-id="497af-123">Names of Events</span></span>

 <span data-ttu-id="497af-124">Olaylar her zaman, ya da oluşan bir eyleme (ya da oluşan bir işlem) başvurur.</span><span class="sxs-lookup"><span data-stu-id="497af-124">Events always refer to some action, either one that is happening or one that has occurred.</span></span> <span data-ttu-id="497af-125">Bu nedenle, metotlarda olduğu gibi, olaylar fiiller ile adlandırılır ve etkinliğin ne zaman gerçekleştiğini göstermek için fiil zaman hali kullanılır.</span><span class="sxs-lookup"><span data-stu-id="497af-125">Therefore, as with methods, events are named with verbs, and verb tense is used to indicate the time when the event is raised.</span></span>

 <span data-ttu-id="497af-126">bir fiil veya fiil ifadesi ile ad olayları ✔️.</span><span class="sxs-lookup"><span data-stu-id="497af-126">✔️ DO name events with a verb or a verb phrase.</span></span>

 <span data-ttu-id="497af-127">Örnekler,,, vb `Clicked` `Painting` . içerir `DroppedDown` .</span><span class="sxs-lookup"><span data-stu-id="497af-127">Examples include `Clicked`, `Painting`, `DroppedDown`, and so on.</span></span>

 <span data-ttu-id="497af-128">✔️, var olan ve son kullanılan kullanımı kullanarak olay adlarını önceden ve sonra bir kavram olarak verir.</span><span class="sxs-lookup"><span data-stu-id="497af-128">✔️ DO give events names with a concept of before and after, using the present and past tenses.</span></span>

 <span data-ttu-id="497af-129">Örneğin, bir pencere kapatılmadan önce oluşturulan bir Close olayı çağrılır `Closing` ve pencere kapatıldıktan sonra oluşturulan bir kapatma olayı çağırılır `Closed` .</span><span class="sxs-lookup"><span data-stu-id="497af-129">For example, a close event that is raised before a window is closed would be called `Closing`, and one that is raised after the window is closed would be called `Closed`.</span></span>

 <span data-ttu-id="497af-132">Aşağıdaki örnekte gösterildiği gibi, ad olay işleyicilerini (olay türleri olarak kullanılan temsilciler) "EventHandler" sonekiyle birlikte ✔️:</span><span class="sxs-lookup"><span data-stu-id="497af-132">✔️ DO name event handlers (delegates used as types of events) with the "EventHandler" suffix, as shown in the following example:</span></span>

 <span data-ttu-id="497af-133">✔️ `sender` , `e` olay işleyicilerinde ve adında iki parametre kullanır.</span><span class="sxs-lookup"><span data-stu-id="497af-133">✔️ DO use two parameters named `sender` and `e` in event handlers.</span></span>

 <span data-ttu-id="497af-134">Sender parametresi, olayı oluşturan nesneyi temsil eder.</span><span class="sxs-lookup"><span data-stu-id="497af-134">The sender parameter represents the object that raised the event.</span></span> <span data-ttu-id="497af-135">Gönderen parametresi `object` , daha belirli bir tür kullanmak mümkün olsa bile genellikle türüdür.</span><span class="sxs-lookup"><span data-stu-id="497af-135">The sender parameter is typically of type `object`, even if it is possible to employ a more specific type.</span></span>
