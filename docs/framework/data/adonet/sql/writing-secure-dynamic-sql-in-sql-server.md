<span data-ttu-id="bdce4-103">SQL ekleme, kötü amaçlı bir kullanıcının geçerli giriş yerine Transact-SQL deyimlerini girdiği işlemdir.</span><span class="sxs-lookup"><span data-stu-id="bdce4-103">SQL Injection is the process by which a malicious user enters Transact-SQL statements instead of valid input.</span></span> <span data-ttu-id="bdce4-104">Giriş, doğrulamadan doğrudan sunucuya geçiriliyorsa ve uygulama eklenen kodu yanlışlıkla çalıştırırsa, saldırı, verileri hasar veya yok etme potansiyelini ister.</span><span class="sxs-lookup"><span data-stu-id="bdce4-104">If the input is passed directly to the server without being validated and if the application inadvertently executes the injected code, the attack has the potential to damage or destroy data.</span></span>  
  
 <span data-ttu-id="bdce4-105">SQL Server sözdizimsel açıdan geçerli olan aldığı tüm sorguları yürüttüğü için SQL deyimleri oluşturan her türlü yordam, ekleme güvenlik açıklarına karşı gözden geçirilmelidir.</span><span class="sxs-lookup"><span data-stu-id="bdce4-105">Any procedure that constructs SQL statements should be reviewed for injection vulnerabilities because SQL Server will execute all syntactically valid queries that it receives.</span></span> <span data-ttu-id="bdce4-106">Parametreli veriler bile nitelikli ve belirlenmiş bir saldırgan tarafından yönetilebilir.</span><span class="sxs-lookup"><span data-stu-id="bdce4-106">Even parameterized data can be manipulated by a skilled and determined attacker.</span></span> <span data-ttu-id="bdce4-107">Dinamik SQL kullanıyorsanız, komutlarınızı parametreleştirilediğinizden ve parametre değerlerini hiçbir şekilde doğrudan sorgu dizesine eklemeyin.</span><span class="sxs-lookup"><span data-stu-id="bdce4-107">If you use dynamic SQL, be sure to parameterize your commands, and never include parameter values directly into the query string.</span></span>  
  
## <a name="anatomy-of-a-sql-injection-attack"></a><span data-ttu-id="bdce4-108">SQL ekleme saldırısının anatomumu</span><span class="sxs-lookup"><span data-stu-id="bdce4-108">Anatomy of a SQL Injection Attack</span></span>  

 <span data-ttu-id="bdce4-109">Ekleme işlemi, bir metin dizesini erken sonlandırarak ve yeni bir komut ekleyerek işe yarar.</span><span class="sxs-lookup"><span data-stu-id="bdce4-109">The injection process works by prematurely terminating a text string and appending a new command.</span></span> <span data-ttu-id="bdce4-110">Eklenen komuta, yürütülmeden önce başka dizeler eklenmiş olabileceğinden, malefaktör eklenen dizeyi "--" Açıklama işaretiyle sonlandırır.</span><span class="sxs-lookup"><span data-stu-id="bdce4-110">Because the inserted command may have additional strings appended to it before it is executed, the malefactor terminates the injected string with a comment mark "--".</span></span> <span data-ttu-id="bdce4-111">Sonraki metin, yürütme sırasında yok sayılır.</span><span class="sxs-lookup"><span data-stu-id="bdce4-111">Subsequent text is ignored at execution time.</span></span> <span data-ttu-id="bdce4-112">Noktalı virgül kullanılarak birden çok komut eklenebilir (;) ayırıcı.</span><span class="sxs-lookup"><span data-stu-id="bdce4-112">Multiple commands can be inserted using a semicolon (;) delimiter.</span></span>  
  
 <span data-ttu-id="bdce4-113">Eklenen SQL kodu sözdizimsel olarak doğru olduğundan, bu değişiklik programlı bir şekilde algılanamıyor.</span><span class="sxs-lookup"><span data-stu-id="bdce4-113">As long as injected SQL code is syntactically correct, tampering cannot be detected programmatically.</span></span> <span data-ttu-id="bdce4-114">Bu nedenle, tüm kullanıcı girdilerini doğrulamanız ve kullanmakta olduğunuz sunucuda oluşturulmuş SQL komutlarını yürüten kodu dikkatle incelemeniz gerekir.</span><span class="sxs-lookup"><span data-stu-id="bdce4-114">Therefore, you must validate all user input and carefully review code that executes constructed SQL commands in the server that you are using.</span></span> <span data-ttu-id="bdce4-115">Doğrulanmamış olmayan kullanıcı girişini hiçbir şekilde birleştirme.</span><span class="sxs-lookup"><span data-stu-id="bdce4-115">Never concatenate user input that is not validated.</span></span> <span data-ttu-id="bdce4-116">Dize birleştirme, betik ekleme için birincil giriş noktasıdır.</span><span class="sxs-lookup"><span data-stu-id="bdce4-116">String concatenation is the primary point of entry for script injection.</span></span>  
  
 <span data-ttu-id="bdce4-117">Faydalı yönergeler aşağıda verilmiştir:</span><span class="sxs-lookup"><span data-stu-id="bdce4-117">Here are some helpful guidelines:</span></span>

- <span data-ttu-id="bdce4-124">Dize değişkenlerinin içeriğini test edin ve yalnızca beklenen değerleri kabul edin.</span><span class="sxs-lookup"><span data-stu-id="bdce4-124">Test the content of string variables and accept only expected values.</span></span> <span data-ttu-id="bdce4-125">İkili veri, kaçış dizileri ve Açıklama karakterleri içeren girişleri reddedin.</span><span class="sxs-lookup"><span data-stu-id="bdce4-125">Reject entries that contain binary data, escape sequences, and comment characters.</span></span>  
  
- <span data-ttu-id="bdce4-126">XML belgeleriyle çalışırken, girilen tüm verileri şemaya göre doğrulayın.</span><span class="sxs-lookup"><span data-stu-id="bdce4-126">When you are working with XML documents, validate all data against its schema as it is entered.</span></span>  
  
- <span data-ttu-id="bdce4-127">Çok katmanlı ortamlarda, tüm verilerin güvenilen bölgeye bağlanmadan önce doğrulanması gerekir.</span><span class="sxs-lookup"><span data-stu-id="bdce4-127">In multi-tiered environments, all data should be validated before admission to the trusted zone.</span></span>  
  
- <span data-ttu-id="bdce4-128">Dosya adlarının oluşturulabileceği alanlarda aşağıdaki dizeleri kabul etme: AUX, saat $, COM1 aracılığıyla COM8, CON, CONFIG $, LPT1 ile LPT8, NUL ve PRN.</span><span class="sxs-lookup"><span data-stu-id="bdce4-128">Do not accept the following strings in fields from which file names can be constructed: AUX, CLOCK$, COM1 through COM8, CON, CONFIG$, LPT1 through LPT8, NUL, and PRN.</span></span>  
  
- <span data-ttu-id="bdce4-129"><xref:System.Data.SqlClient.SqlParameter>Tür denetimi ve uzunluk doğrulaması sağlamak için saklı yordam ve komutlarla nesneleri kullanın.</span><span class="sxs-lookup"><span data-stu-id="bdce4-129">Use <xref:System.Data.SqlClient.SqlParameter> objects with stored procedures and commands to provide type checking and length validation.</span></span>
