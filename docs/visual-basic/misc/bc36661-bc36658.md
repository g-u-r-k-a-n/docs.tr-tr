<span data-ttu-id="ea836-104">' TypeName ' içinde tanımlanan ' ' genişletme yöntemindeki tür parametrelerinin veri türleri \<methodname> , aynı türe dönüştürülmediğinden bu bağımsız değişkenlerden çıkarılamıyor.</span><span class="sxs-lookup"><span data-stu-id="ea836-104">Data type(s) of the type parameter(s) in extension method '\<methodname>' defined in 'typename' cannot be inferred from these arguments because they do not convert to the same type.</span></span> <span data-ttu-id="ea836-105">Veri türlerinin açıkça belirtilmesi bu hatayı düzeltebilir.</span><span class="sxs-lookup"><span data-stu-id="ea836-105">Specifying the data type(s) explicitly might correct this error.</span></span>

<span data-ttu-id="ea836-106">Genel genişletme yöntemine yapılan bir çağrı değerlendirilirken tür parametresinin (veya parametrelerinin) veri türünü (veya türlerini) belirlemekte kullanılacak bir girişimde bulunuldu.</span><span class="sxs-lookup"><span data-stu-id="ea836-106">An attempt has been made to use type inference to determine the data type (or types) of the type parameter (or parameters) when evaluating a call to a generic extension method.</span></span> <span data-ttu-id="ea836-107">Derleyici, tüm bağımsız değişkenlerin kısıtlamalarını karşılayan bir veri türü bulamadı.</span><span class="sxs-lookup"><span data-stu-id="ea836-107">The compiler could not find a data type that meets the constraints of all the arguments.</span></span> <span data-ttu-id="ea836-108">Bu nedenle, bu hatayı bildirdi.</span><span class="sxs-lookup"><span data-stu-id="ea836-108">Therefore, it reported this error.</span></span>

> [!NOTE]
> <span data-ttu-id="ea836-109">Bağımsız değişkenlerin belirtilmesi bir seçenek değil (örneğin, sorgu ifadelerinde sorgu işleçleri için), ikinci tümce olmadan hata iletisi görüntülenir.</span><span class="sxs-lookup"><span data-stu-id="ea836-109">When specifying arguments is not an option (for example, for query operators in query expressions), the error message appears without the second sentence.</span></span>

<span data-ttu-id="ea836-110">Aşağıdaki kod hatayı gösterir.</span><span class="sxs-lookup"><span data-stu-id="ea836-110">The following code demonstrates the error.</span></span>

```vb

    Sub Main()

        Dim c1 As New Class1

        '' Not valid. Integer and Date do not convert to the same type.
        'c1.targetMethod(19, #3/4/2007#)

    End Sub

    <System.Runtime.CompilerServices.Extension()> _
    Sub targetMethod(Of T)(ByVal p0 As Class1, ByVal p1 As T, ByVal p2 As T)
    End Sub

    Class Class1
    End Class

```

## <a name="to-correct-this-error"></a><span data-ttu-id="ea836-112">Bu hatayı düzeltmek için</span><span class="sxs-lookup"><span data-stu-id="ea836-112">To correct this error</span></span>

- <span data-ttu-id="ea836-113">Aşağıdaki kodda gösterildiği gibi, bir veya daha fazla bağımsız değişkeni açıkça uyumlu bir türe dönüştürebilirsiniz:</span><span class="sxs-lookup"><span data-stu-id="ea836-113">You might be able to convert one or more arguments explicitly to a compatible type, as shown in the following code:</span></span>

  ```

- <span data-ttu-id="ea836-114">Aşağıdaki kodda gösterildiği gibi, tür parametresi veya bağımsız değişkenlerin dönüştürüleceği parametreler için bir veri türü belirtebilirsiniz:</span><span class="sxs-lookup"><span data-stu-id="ea836-114">You might be able to specify a data type for the type parameter or parameters to which the arguments convert, as shown in the following code:</span></span>

  ```
