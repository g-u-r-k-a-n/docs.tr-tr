<span data-ttu-id="70438-114">Bir tür açık statik yapıcı bildirdiğinde, JIT derleyici her bir statik yöntemi kontrol ekler ve türün yapıcı örneği statik yapıcının daha önceden çağrıldığından emin olur.</span><span class="sxs-lookup"><span data-stu-id="70438-114">When a type declares an explicit static constructor, the just-in-time (JIT) compiler adds a check to each static method and instance constructor of the type to make sure that the static constructor was previously called.</span></span> <span data-ttu-id="70438-115">Statik başlatma, herhangi bir statik üyeye erişildiğinde veya türün bir örneği oluşturulduğunda tetiklenir.</span><span class="sxs-lookup"><span data-stu-id="70438-115">Static initialization is triggered when any static member is accessed or when an instance of the type is created.</span></span> <span data-ttu-id="70438-116">Ancak, türün bir değişkenini bildirirseniz ancak kullanmıyorsanız, başlatma genel durumu değiştirdiğinde önemli olabilecek bir statik başlatma tetiklenmez.</span><span class="sxs-lookup"><span data-stu-id="70438-116">However, static initialization is not triggered if you declare a variable of the type but do not use it, which can be important if the initialization changes global state.</span></span>

- <span data-ttu-id="70438-125">Genel durum üzerindeki etkisi pahalıdır ve tür kullanılmazsa gerekli değildir.</span><span class="sxs-lookup"><span data-stu-id="70438-125">The effect on global state is expensive and is not required if the type is not used.</span></span>
