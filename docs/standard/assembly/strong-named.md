## <a name="why-strong-name-your-assemblies"></a><span data-ttu-id="64fdf-114">Derlemelerinize neden güçlü ad adı veriyor?</span><span class="sxs-lookup"><span data-stu-id="64fdf-114">Why strong-name your assemblies?</span></span>

<span data-ttu-id="64fdf-115">.NET Framework için, tanımlayıcı adlı derlemeler aşağıdaki senaryolarda faydalıdır:</span><span class="sxs-lookup"><span data-stu-id="64fdf-115">For .NET Framework, strong-named assemblies are useful in the following scenarios:</span></span>

- <span data-ttu-id="64fdf-116">Derlemelerinize tanımlayıcı adlı derlemeler tarafından başvurulmak üzere veya `friend` tanımlayıcı adlı diğer derlemelerden derlemelerinize erişim vermek isteyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="64fdf-116">You want to enable your assemblies to be referenced by strong-named assemblies, or you want to give `friend` access to your assemblies from other strong-named assemblies.</span></span>

- <span data-ttu-id="64fdf-117">Uygulamanın aynı derlemenin farklı sürümlerine erişmesi gerekir.</span><span class="sxs-lookup"><span data-stu-id="64fdf-117">An app needs access to different versions of the same assembly.</span></span> <span data-ttu-id="64fdf-118">Bu, çakışma olmadan aynı uygulama etki alanında yan yana yüklemek üzere bir derlemenin farklı sürümlerinin olması gerektiği anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="64fdf-118">This means  you need different versions of an assembly to load side by side in the same app domain without conflict.</span></span> <span data-ttu-id="64fdf-119">Örneğin, aynı basit ada sahip derlemelerde bir API 'nin farklı uzantıları varsa, güçlü adlandırma derlemenin her sürümü için benzersiz bir kimlik sağlar.</span><span class="sxs-lookup"><span data-stu-id="64fdf-119">For example, if different extensions of an API exist in assemblies that have the same simple name, strong-naming provides a unique identity for each version of the assembly.</span></span>

- <span data-ttu-id="64fdf-120">Derlemenizi kullanarak uygulamaların performansını olumsuz şekilde etkilememesini istemezsiniz, bu nedenle derlemenin etki alanı nötr olmasını isteyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="64fdf-120">You do not want to negatively affect performance of apps using your assembly, so you want the assembly to be domain neutral.</span></span> <span data-ttu-id="64fdf-121">Bu, genel derleme önbelleğine bir etki alanı nötr derleme yüklenmesi gerektiğinden, güçlü adlandırma gerektirir.</span><span class="sxs-lookup"><span data-stu-id="64fdf-121">This requires strong-naming because a domain-neutral assembly must be installed in the global assembly cache.</span></span>

- <span data-ttu-id="64fdf-122">Yayımcı ilkesi uygulayarak uygulamanızın bakımını merkezileştirmek istiyorsunuz, bu, derlemenin genel derleme önbelleğinde yüklü olması gerektiği anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="64fdf-122">You want to centralize servicing for your app by applying publisher policy, which means the assembly must be installed in the global assembly cache.</span></span>

<span data-ttu-id="64fdf-123">.NET Core için, tanımlayıcı adlı derlemeler malzeme avantajları sağlamaz.</span><span class="sxs-lookup"><span data-stu-id="64fdf-123">For .NET Core, strong-named assemblies do not provide material benefits.</span></span>

<span data-ttu-id="64fdf-124">Açık kaynaklı bir geliştiricisiyseniz ve tanımlayıcı adlı bir derlemenin kimlik avantajlarından .NET Framework daha iyi uyumluluk sağlamak istiyorsanız, kaynak denetim sisteminize bir derlemeyle ilişkili özel anahtarı iade etmeyi göz önünde bulundurun.</span><span class="sxs-lookup"><span data-stu-id="64fdf-124">If you are an open-source developer and you want the identity benefits of a strong-named assembly for better compatibility with .NET Framework, consider checking in the private key associated with an assembly to your source control system.</span></span>
