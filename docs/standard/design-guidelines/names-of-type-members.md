# <a name="names-of-type-members"></a><span data-ttu-id="fba62-102">Tür üyelerinin adları</span><span class="sxs-lookup"><span data-stu-id="fba62-102">Names of Type Members</span></span>
<span data-ttu-id="fba62-103">Üyeleri türlerine yapılan: yöntemler, özellikler, olaylar, Oluşturucular ve alanları.</span><span class="sxs-lookup"><span data-stu-id="fba62-103">Types are made of members: methods, properties, events, constructors, and fields.</span></span> <span data-ttu-id="fba62-104">Aşağıdaki bölümlerde, tür üyeleri adlandırmak için yönergeler açıklanmaktadır.</span><span class="sxs-lookup"><span data-stu-id="fba62-104">The following sections describe guidelines for naming type members.</span></span>  
  
## <a name="names-of-methods"></a><span data-ttu-id="fba62-105">Yöntemlerin adları</span><span class="sxs-lookup"><span data-stu-id="fba62-105">Names of Methods</span></span>  
 <span data-ttu-id="fba62-106">Yöntemleri eylemde toplanabilmesini olduğundan, tasarım yönergeleri yöntem adları fiilleri ya da tümcelere fiili olması gerekir.</span><span class="sxs-lookup"><span data-stu-id="fba62-106">Because methods are the means of taking action, the design guidelines require that method names be verbs or verb phrases.</span></span> <span data-ttu-id="fba62-107">Bu kılavuz ayrıca aşağıdaki isim veya sıfat tümcecikleri olan özellik ve tür adları, yöntem adlarını ayırt etmek için hizmet verir.</span><span class="sxs-lookup"><span data-stu-id="fba62-107">Following this guideline also serves to distinguish method names from property and type names, which are noun or adjective phrases.</span></span>  
  
 <span data-ttu-id="fba62-108">**✓ DO** fiilleri ya da fiili tümcecikleri yöntemleri adlar verin.</span><span class="sxs-lookup"><span data-stu-id="fba62-108">**✓ DO** give methods names that are verbs or verb phrases.</span></span>  
  
```csharp  
public class String {  
    public int CompareTo(...);  
    public string[] Split(...);  
    public string Trim();  
}  
```  
  
## <a name="names-of-properties"></a><span data-ttu-id="fba62-109">Özelliklerin adları</span><span class="sxs-lookup"><span data-stu-id="fba62-109">Names of Properties</span></span>  
 <span data-ttu-id="fba62-110">Diğer üyeleri özellikleri isim ifade veya sıfat adları verilmelidir.</span><span class="sxs-lookup"><span data-stu-id="fba62-110">Unlike other members, properties should be given noun phrase or adjective names.</span></span> <span data-ttu-id="fba62-111">Bir özellik verileri ifade eder ve özellik adını yansıtılan çünkü olmasıdır.</span><span class="sxs-lookup"><span data-stu-id="fba62-111">That is because a property refers to data, and the name of the property reflects that.</span></span> <span data-ttu-id="fba62-112">PascalCasing her zaman özellik adları için kullanılır.</span><span class="sxs-lookup"><span data-stu-id="fba62-112">PascalCasing is always used for property names.</span></span>  
  
 <span data-ttu-id="fba62-113">**✓ DO** adı bir isim, isim sözcük veya sıfat kullanarak özellikleri.</span><span class="sxs-lookup"><span data-stu-id="fba62-113">**✓ DO** name properties using a noun, noun phrase, or adjective.</span></span>  
  
 <span data-ttu-id="fba62-114">**X DO NOT** aşağıdaki örnekte olduğu gibi "Get" yöntemlerinin adıyla aynı özelliklere sahiptir:</span><span class="sxs-lookup"><span data-stu-id="fba62-114">**X DO NOT** have properties that match the name of "Get" methods as in the following example:</span></span>  
  
 `public string TextWriter { get {...} set {...} }`  
 `public string GetTextWriter(int value) { ... }`  
  
 <span data-ttu-id="fba62-115">Bu düzen, genellikle özelliği bir yöntem gerçekten gerektiğini gösterir.</span><span class="sxs-lookup"><span data-stu-id="fba62-115">This pattern typically indicates that the property should really be a method.</span></span>  
  
 <span data-ttu-id="fba62-116">**✓ DO** koleksiyon özellikleri ardından "List" veya "Koleksiyonu." tekil bir ifade kullanmak yerine koleksiyondaki öğeleri açıklayan çoğul tümcecik adı</span><span class="sxs-lookup"><span data-stu-id="fba62-116">**✓ DO** name collection properties with a plural phrase describing the items in the collection instead of using a singular phrase followed by "List" or "Collection."</span></span>  
  
 <span data-ttu-id="fba62-117">**✓ DO** olumlu bir ifade ile Boole özellikleri adlandırın (`CanSeek` yerine `CantSeek`).</span><span class="sxs-lookup"><span data-stu-id="fba62-117">**✓ DO** name Boolean properties with an affirmative phrase (`CanSeek` instead of `CantSeek`).</span></span> <span data-ttu-id="fba62-118">İsteğe bağlı olarak ayrıca Boole özellikleri öneki "İçin"Olan,"" veya ", ancak yalnızca değer kazandıran burada sahip".</span><span class="sxs-lookup"><span data-stu-id="fba62-118">Optionally, you can also prefix Boolean properties with "Is," "Can," or "Has," but only where it adds value.</span></span>  
  
 <span data-ttu-id="fba62-119">**✓ CONSIDER** özellik türü olarak aynı adı vererek.</span><span class="sxs-lookup"><span data-stu-id="fba62-119">**✓ CONSIDER** giving a property the same name as its type.</span></span>  
  
 <span data-ttu-id="fba62-120">Örneğin, aşağıdaki özelliği doğru alır ve bir enum değeri ayarlar `Color`, özellik adlı `Color`:</span><span class="sxs-lookup"><span data-stu-id="fba62-120">For example, the following property correctly gets and sets an enum value named `Color`, so the property is named `Color`:</span></span>  
  
```csharp  
public enum Color {...}  
public class Control {  
    public Color Color { get {...} set {...} }  
}  
```  
  
## <a name="names-of-events"></a><span data-ttu-id="fba62-121">Olayların adları</span><span class="sxs-lookup"><span data-stu-id="fba62-121">Names of Events</span></span>  
 <span data-ttu-id="fba62-122">Olayları için gerçekleşmesini olan bir ya da gerçekleşen bir bazı eylemler her zaman bakın.</span><span class="sxs-lookup"><span data-stu-id="fba62-122">Events always refer to some action, either one that is happening or one that has occurred.</span></span> <span data-ttu-id="fba62-123">Bu nedenle, yöntemleriyle yönteminde olduğu gibi olayları fiiller ile adlandırılır ve fiili şimdiki zaman olayı zaman belirtmek için kullanılır.</span><span class="sxs-lookup"><span data-stu-id="fba62-123">Therefore, as with methods, events are named with verbs, and verb tense is used to indicate the time when the event is raised.</span></span>  
  
 <span data-ttu-id="fba62-124">**✓ DO** fiil veya fiili tümcecik olaylarlaA adı.</span><span class="sxs-lookup"><span data-stu-id="fba62-124">**✓ DO** name events with a verb or a verb phrase.</span></span>  
  
 <span data-ttu-id="fba62-125">Örnekler `Clicked`, `Painting`, `DroppedDown`ve benzeri.</span><span class="sxs-lookup"><span data-stu-id="fba62-125">Examples include `Clicked`, `Painting`, `DroppedDown`, and so on.</span></span>  
  
 <span data-ttu-id="fba62-126">**✓ DO** mevcut ve geçmiş zamanlarını kullanarak önce ve sonra bir kavramı ile olayları adlar verin.</span><span class="sxs-lookup"><span data-stu-id="fba62-126">**✓ DO** give events names with a concept of before and after, using the present and past tenses.</span></span>  
  
 <span data-ttu-id="fba62-127">Örneğin, bir pencere kapatılmadan hemen önce bir kapatma olayı adlı `Closing`, ve penceresi kapatıldıktan sonra başlatan bir çağrılabilir `Closed`.</span><span class="sxs-lookup"><span data-stu-id="fba62-127">For example, a close event that is raised before a window is closed would be called `Closing`, and one that is raised after the window is closed would be called `Closed`.</span></span>  
  
 <span data-ttu-id="fba62-128">**X DO NOT** "Önce" veya "Sonra" ön ekleri veya öncesi belirtmek için postfixes ve sonrası olayları kullanın.</span><span class="sxs-lookup"><span data-stu-id="fba62-128">**X DO NOT** use "Before" or "After" prefixes or postfixes to indicate pre- and post-events.</span></span> <span data-ttu-id="fba62-129">Kullanımı mevcut ve geçmiş zamanlarını yalnızca tanımlandığı gibi.</span><span class="sxs-lookup"><span data-stu-id="fba62-129">Use present and past tenses as just described.</span></span>  
  
 <span data-ttu-id="fba62-130">**✓ DO** aşağıdaki örnekte gösterildiği gibi "EventHandler" soneki ile ad olay işleyicileri (temsilcileri olay türleri kullanılan):</span><span class="sxs-lookup"><span data-stu-id="fba62-130">**✓ DO** name event handlers (delegates used as types of events) with the "EventHandler" suffix, as shown in the following example:</span></span>  
  
 `public delegate void ClickedEventHandler(object sender, ClickedEventArgs e);`  
  
 <span data-ttu-id="fba62-131">**✓ DO** adlı iki parametre kullanmak `sender` ve `e` olay işleyicileri.</span><span class="sxs-lookup"><span data-stu-id="fba62-131">**✓ DO** use two parameters named `sender` and `e` in event handlers.</span></span>  
  
 <span data-ttu-id="fba62-132">Gönderen parametresi olayı tetikleyen nesne temsil eder.</span><span class="sxs-lookup"><span data-stu-id="fba62-132">The sender parameter represents the object that raised the event.</span></span> <span data-ttu-id="fba62-133">Gönderen parametresi genellikle türüdür `object`bile daha belirli bir tür kullanmak istemiyorsunuz mümkündür.</span><span class="sxs-lookup"><span data-stu-id="fba62-133">The sender parameter is typically of type `object`, even if it is possible to employ a more specific type.</span></span>  
  
 <span data-ttu-id="fba62-134">**✓ DO** "EventArgs" sonek bağımsız değişkeni sınıflarıyla olay adı.</span><span class="sxs-lookup"><span data-stu-id="fba62-134">**✓ DO** name event argument classes with the "EventArgs" suffix.</span></span>

 <span data-ttu-id="fba62-139">**✓ DO** ad alanları bir isim, isim tümceciği veya sıfat kullanarak.</span><span class="sxs-lookup"><span data-stu-id="fba62-139">**✓ DO** name fields using a noun, noun phrase, or adjective.</span></span>
