 <span data-ttu-id="dcf84-114">İşlem kümeleri ve dağıtılmış işleme genellikle finansal/Hava durumu modelleme ve insan DNA kodu çözme gibi büyük ölçekli hesaplamalar için kullanılır.</span><span class="sxs-lookup"><span data-stu-id="dcf84-114">Compute clusters and distributed processing are typically used for large-scale computations, such as financial/weather modeling and decoding human DNA.</span></span> <span data-ttu-id="dcf84-115">Genellikle bu işlem, sunucuların hesaplama kümesine katılan tüm istemcilere ayrı ayrı görev atamasını sağlar.</span><span class="sxs-lookup"><span data-stu-id="dcf84-115">Typically, this is done by having servers individually assign tasks to all clients participating in the computation cluster.</span></span> <span data-ttu-id="dcf84-116">Bu sunucularda ek talepler de olabilir; for example, all tasks may need to be completed within a certain duration, requiring more than one machine for each task.</span></span> <span data-ttu-id="dcf84-117">Ayrıca, bir görevi çalıştıran herhangi bir istemci kapalıysa, başka bir istemci bu görevi ele alabilmeli ve üzerinde iş gerçekleştirebilir.</span><span class="sxs-lookup"><span data-stu-id="dcf84-117">Additionally, if any client running a task goes down, another client must be able to take over that task and perform work on it.</span></span> <span data-ttu-id="dcf84-118">Benzer şekilde, birden fazla istemcinin tutarlı sonuçlar sağlamak için aynı görevi çalıştırması gerekebilir.</span><span class="sxs-lookup"><span data-stu-id="dcf84-118">Likewise, more than one client may have to run the same task to ensure consistent results.</span></span> <span data-ttu-id="dcf84-119">Sunucular bu tür istemci koordinasyonunu çalıştırabilse de, bir görevi alan istemcilerin, görevin bir yanındaki sunucu gereksinimlerini ve bu görevi nasıl tamamlayacağınızı anlamak için bir işlem ağı kullanmasını sağlayan bir eşler arası çözüm oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="dcf84-119">Although servers can run this type of client coordination, you can create a peer-to-peer solution where the clients receiving a task independently determine the server requirements around the task and use a compute mesh to determine how to complete that task.</span></span>
