## <a name="what-was-the-app-doing"></a><span data-ttu-id="45dd4-109">Uygulama ne yapıyordu?</span><span class="sxs-lookup"><span data-stu-id="45dd4-109">What was the app doing?</span></span>

`AppViewModel.Current.LayoutVM.PageMap`  
  
 <span data-ttu-id="45dd4-114">Bu örnekte, `LayoutVM` özelliği `AppViewModel.Current` **null** idi.</span><span class="sxs-lookup"><span data-stu-id="45dd4-114">In this instance, the `LayoutVM` property on `AppViewModel.Current` was **null**.</span></span>  <span data-ttu-id="45dd4-115">Bazı meta veri yokluğu, hafif davranış farkına neden oldu ve uygulamanın beklendiği gibi küme yerine başlatılmamış bir özellik ile sonuçlandı.</span><span class="sxs-lookup"><span data-stu-id="45dd4-115">Some absence of metadata caused a subtle behavior difference and resulted in a property being uninitialized instead of set, as the app expected.</span></span>  <span data-ttu-id="45dd4-116">Kodda başlatılmış olması gereken bir kesme noktası ayarlamak `LayoutVM` durumunda ışık oluşturabilir.</span><span class="sxs-lookup"><span data-stu-id="45dd4-116">Setting a breakpoint in the code where `LayoutVM` should have been initialized might throw light on the situation.</span></span>  <span data-ttu-id="45dd4-117">Ancak, `LayoutVM` türünün olduğunu unutmayın `App.Core.ViewModels.Layout.LayoutApplicationVM` .</span><span class="sxs-lookup"><span data-stu-id="45dd4-117">However, note that `LayoutVM`’s type is `App.Core.ViewModels.Layout.LayoutApplicationVM`.</span></span>  <span data-ttu-id="45dd4-118">rd.xml dosyasında şu ana kadar tek metaveri yönergesi bulunur:</span><span class="sxs-lookup"><span data-stu-id="45dd4-118">The only metadata directive present so far in the rd.xml file is:</span></span>

  
 <span data-ttu-id="45dd4-119">Hata için olası bir aday, `App.Core.ViewModels.Layout.LayoutApplicationVM` farklı bir ad alanında olduğu için meta veriler eksik.</span><span class="sxs-lookup"><span data-stu-id="45dd4-119">A likely candidate for the failure is that `App.Core.ViewModels.Layout.LayoutApplicationVM` is missing metadata because it's in a different namespace.</span></span>  
  
 <span data-ttu-id="45dd4-120">Bu durumda, sorunu çözen bir çalışma zamanı yönergesi ekleme `App.Core.ViewModels` .</span><span class="sxs-lookup"><span data-stu-id="45dd4-120">In this case, adding a runtime directive for `App.Core.ViewModels` resolved the issue.</span></span> <span data-ttu-id="45dd4-121">Kök neden, null döndüren yöntemine yönelik bir API çağrısıdır <xref:System.Type.GetType%28System.String%29?displayProperty=nameWithType> ve **null** uygulama kilitlenme gerçekleşene kadar sorunu sessizce yok sayılır.</span><span class="sxs-lookup"><span data-stu-id="45dd4-121">The root cause was an API call to the <xref:System.Type.GetType%28System.String%29?displayProperty=nameWithType> method that returned **null**, and the app silently ignored the problem until a crash occurred.</span></span>  
  
 <span data-ttu-id="45dd4-122">Dinamik programlamada, .NET Native altında yansıma API 'Leri kullanırken iyi bir uygulama <xref:System.Type.GetType%2A?displayProperty=nameWithType> hata durumunda özel durum oluşturan aşırı yüklemeleri kullanmaktır.</span><span class="sxs-lookup"><span data-stu-id="45dd4-122">In dynamic programming, a good practice when using reflection APIs under .NET Native is to use the <xref:System.Type.GetType%2A?displayProperty=nameWithType> overloads that throw an exception on failure.</span></span>
