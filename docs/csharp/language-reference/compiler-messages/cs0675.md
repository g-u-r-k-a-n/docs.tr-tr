# <a name="compiler-warning-level-3-cs0675"></a><span data-ttu-id="e696d-103">Derleyici Uyarısı (düzey 3) CS0675</span><span class="sxs-lookup"><span data-stu-id="e696d-103">Compiler Warning (level 3) CS0675</span></span>
<span data-ttu-id="e696d-104">Bir işaret genişletilmiş işleneninde bit düzeyinde OR işleci kullanılıyor; önce daha küçük bir işaretsiz türe dönüştürmeyi düşünün</span><span class="sxs-lookup"><span data-stu-id="e696d-104">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="e696d-105">Derleyici örtük olarak iletildiklerinde ve bir değişken genişletti ve sonra elde edilen değeri bit düzeyinde veya işlemde kullanır.</span><span class="sxs-lookup"><span data-stu-id="e696d-105">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="e696d-106">Bu beklenmeyen davranışlara neden olabilir.</span><span class="sxs-lookup"><span data-stu-id="e696d-106">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="e696d-107">Aşağıdaki örnek CS0675 oluşturur:</span><span class="sxs-lookup"><span data-stu-id="e696d-107">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
