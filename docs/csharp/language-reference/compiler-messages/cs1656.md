# <a name="compiler-error-cs1656"></a><span data-ttu-id="41fbf-103">Derleyici Hatası CS1656</span><span class="sxs-lookup"><span data-stu-id="41fbf-103">Compiler Error CS1656</span></span>

<span data-ttu-id="41fbf-104">' Variable ' öğesine atanamıyor çünkü bu bir ' salt okunurdur ' değişken türü.</span><span class="sxs-lookup"><span data-stu-id="41fbf-104">Cannot assign to 'variable' because it is a 'read-only variable type'</span></span>  
  
 <span data-ttu-id="41fbf-105">Bu hata, bir değişkene atama salt okunurdur bir bağlamda oluştuğunda oluşur.</span><span class="sxs-lookup"><span data-stu-id="41fbf-105">This error occurs when an assignment to variable occurs in a read-only context.</span></span> <span data-ttu-id="41fbf-106">Salt okuma bağlamları [foreach](../keywords/foreach-in.md) yineleme değişkenlerini, değişkenleri [kullanarak](../keywords/using-statement.md) ve [sabit](../keywords/fixed-statement.md) değişkenleri içerir.</span><span class="sxs-lookup"><span data-stu-id="41fbf-106">Read-only contexts include [foreach](../keywords/foreach-in.md) iteration variables, [using](../keywords/using-statement.md) variables, and [fixed](../keywords/fixed-statement.md) variables.</span></span> <span data-ttu-id="41fbf-107">Bu hatayı çözmek için `using` bloklar, `foreach` deyimler ve deyimler içindeki deyim değişkenine atamalardan kaçının `fixed` .</span><span class="sxs-lookup"><span data-stu-id="41fbf-107">To resolve this error, avoid assignments to a statement variable in `using` blocks, `foreach` statements, and `fixed` statements.</span></span>  
  
## <a name="example"></a><span data-ttu-id="41fbf-108">Örnek</span><span class="sxs-lookup"><span data-stu-id="41fbf-108">Example</span></span>  

 <span data-ttu-id="41fbf-109">Aşağıdaki örnek, bir içindeki bir koleksiyonun bütün öğelerini bir döngü içinde değiştirmeye çalıştığı için hata CS1656 oluşturur `foreach` .</span><span class="sxs-lookup"><span data-stu-id="41fbf-109">The following example generates error CS1656 because it tries to replace complete elements of a collection inside a `foreach` loop.</span></span> <span data-ttu-id="41fbf-110">Hatayı geçici olarak çözmek için bir yol, `foreach` döngüyü [for](../keywords/for.md) döngüsü olarak değiştirkullanmaktır.</span><span class="sxs-lookup"><span data-stu-id="41fbf-110">One way to work around the error is to change the `foreach` loop to a [for](../keywords/for.md) loop.</span></span> <span data-ttu-id="41fbf-111">Burada görüntülenmemiş başka bir yol, mevcut öğenin üyelerini değiştirmektir; Bu, sınıflarla kullanılamaz, ancak yapılar ile mümkün değildir.</span><span class="sxs-lookup"><span data-stu-id="41fbf-111">Another way, not shown here, is to modify the members of the existing element; this is possible with classes, but not with structs.</span></span>  
  
```csharp  
using System;  
using System.Collections;  
using System.Collections.Generic;  
using System.Text;  
  
namespace CS1656_2  
{  
  
    class Book  
    {  
        public string Title;  
        public string Author;  
        public double Price;  
        public Book(string t, string a, double p)  
        {  
            Title=t;  
            Author=a;  
            Price=p;  
  
        }  
    }  
  
    class Program  
    {  
        private List<Book> list;  
        static void Main(string[] args)  
        {  
            Program prog = new Program();  
            prog.list = new List<Book>();  
            prog.list.Add(new Book ("The C# Programming Language",  
                                    "Hejlsberg, Wiltamuth, Golde",  
                                     29.95));  
            prog.list.Add(new Book ("The C++ Programming Language",  
                                    "Stroustrup",  
                                     29.95));  
            prog.list.Add(new Book ("The C Programming Language",  
                                    "Kernighan, Ritchie",  
                                    29.95));  
            foreach(Book b in prog.list)  
            {  
                // Cannot modify an entire element in a foreach loop
                // even with reference types.  
                // Use a for or while loop instead  
                if (b.Title == "The C Programming Language")  
                    // Cannot assign to 'b' because it is a 'foreach
                    // iteration variable'  
                    b = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95); //CS1656  
            }  
  
            //With a for loop you can modify elements  
            //for(int x = 0; x < prog.list.Count; x++)  
            //{  
            //    if(prog.list[x].Title== "The C Programming Language")  
            //        prog.list[x] = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95);  
            //}  
            //foreach(Book b in prog.list)  
            //    Console.WriteLine(b.Title);  
  
        }  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="41fbf-112">Örnek</span><span class="sxs-lookup"><span data-stu-id="41fbf-112">Example</span></span>  

 <span data-ttu-id="41fbf-113">Aşağıdaki örnek, CS1656 'in bir döngünün yanı sıra diğer bağlamlarda nasıl üretilebileceğini göstermektedir `foreach` :</span><span class="sxs-lookup"><span data-stu-id="41fbf-113">The following sample demonstrates how CS1656 can be generated in other contexts besides a `foreach` loop:</span></span>  
  
```csharp  
// CS1656.cs  
// compile with: /unsafe  
using System;  
  
class C : IDisposable  
{  
    public void Dispose() { }  
}  
  
class CMain  
{  
    unsafe public static void Main()  
    {  
        using (C c = new C())  
        {  
            // Cannot assign to 'c' because it is a 'using variable'  
            c = new C(); // CS1656  
        }  
  
        int[] ary = new int[] { 1, 2, 3, 4 };  
        fixed (int* p = ary)  
        {  
            // Cannot assign to 'p' because it is a 'fixed variable'  
            p = null; // CS1656  
        }  
    }  
}  
```
