<span data-ttu-id="a76b4-103">Linq deseni içinde herhangi bir veri kaynağının sorgulanmasını sağlamak için LINQ'yi genişletmenin çeşitli yolları vardır.</span><span class="sxs-lookup"><span data-stu-id="a76b4-103">There are various ways to extend LINQ to enable any data source to be queried in the LINQ pattern.</span></span> <span data-ttu-id="a76b4-104">Veri kaynağı örneğin bir veri yapısı, Web hizmeti, dosya sistemi veya veritabanı olabilir.</span><span class="sxs-lookup"><span data-stu-id="a76b4-104">The data source might be a data structure, a Web service, a file system, or a database, to name some.</span></span> <span data-ttu-id="a76b4-105">LINQ deseni, sorgunun sözdizimi ve deseni değişmediği için, istemcilerin LINQ sorgusunun etkin olduğu bir veri kaynağını sorgulamasını kolaylaştırır.</span><span class="sxs-lookup"><span data-stu-id="a76b4-105">The LINQ pattern makes it easy for clients to query a data source for which LINQ querying is enabled, because the syntax and pattern of the query does not change.</span></span> <span data-ttu-id="a76b4-106">LINQ'nin bu veri kaynaklarına nasıl genişletilebildiği aşağıdakileri içerir:</span><span class="sxs-lookup"><span data-stu-id="a76b4-106">The ways in which LINQ can be extended to these data sources include the following:</span></span>

## <a name="iqueryable-linq-providers"></a><span data-ttu-id="a76b4-124">IQueryable LINQ Sağlayıcıları</span><span class="sxs-lookup"><span data-stu-id="a76b4-124">IQueryable LINQ Providers</span></span>  
 <span data-ttu-id="a76b4-125">Uygulayan <xref:System.Linq.IQueryable%601> LINQ sağlayıcıları karmaşıklıklarında büyük farklılıklar gösterebilir.</span><span class="sxs-lookup"><span data-stu-id="a76b4-125">LINQ providers that implement <xref:System.Linq.IQueryable%601> can vary widely in their complexity.</span></span> <span data-ttu-id="a76b4-126">Bu bölümde, farklı karmaşıklık düzeyleri açıklanmaktadır.</span><span class="sxs-lookup"><span data-stu-id="a76b4-126">This section discusses the different levels of complexity.</span></span>  
  
 <span data-ttu-id="a76b4-127">Daha az `IQueryable` karmaşık bir sağlayıcı, tek bir Web hizmeti yöntemiyle arabirim yapabilir.</span><span class="sxs-lookup"><span data-stu-id="a76b4-127">A less complex `IQueryable` provider might interface with a single method of a Web service.</span></span> <span data-ttu-id="a76b4-128">Bu türden bir sağlayıcı, işlediği sorgularda belirli bilgiler beklediği için çok özeldir.</span><span class="sxs-lookup"><span data-stu-id="a76b4-128">This type of provider is very specific because it expects specific information in the queries that it handles.</span></span> <span data-ttu-id="a76b4-129">Muhtemelen tek bir sonuç türü ortaya çıkaran kapalı bir tür sistemi vardır.</span><span class="sxs-lookup"><span data-stu-id="a76b4-129">It has a closed type system, perhaps exposing a single result type.</span></span> <span data-ttu-id="a76b4-130">Sorgunun yürütülmesinin çoğu, örneğin standart sorgu işleçlerinin <xref:System.Linq.Enumerable> uygulamalarını kullanarak yerel olarak gerçekleşir.</span><span class="sxs-lookup"><span data-stu-id="a76b4-130">Most of the execution of the query occurs locally, for example by using the <xref:System.Linq.Enumerable> implementations of the standard query operators.</span></span> <span data-ttu-id="a76b4-131">Daha az karmaşık olan bir sağlayıcı, sorguyu temsil eden ifade ağacında yalnızca bir yöntem çağrısı ifadesi inceleyebilir ve kalan sorgu mantığının başka bir yerde işlenmesine izin verebilir.</span><span class="sxs-lookup"><span data-stu-id="a76b4-131">A less complex provider might examine only one method call expression in the expression tree that represents the query, and let the remaining logic of the query be handled elsewhere.</span></span>

 <span data-ttu-id="a76b4-136">Sağlayıcı `IQueryable` gibi karmaşık bir sağlayıcı, tüm LINQ sorgularını SQL gibi etkileyici bir sorgu diline çevirebilir. [!INCLUDE[vbtecdlinq](~/includes/vbtecdlinq-md.md)]</span><span class="sxs-lookup"><span data-stu-id="a76b4-136">A complex `IQueryable` provider, such as the [!INCLUDE[vbtecdlinq](~/includes/vbtecdlinq-md.md)] provider, might translate complete LINQ queries to an expressive query language, such as SQL.</span></span> <span data-ttu-id="a76b4-137">Karmaşık bir sağlayıcı, sorguda çok çeşitli soruları işleyebileceği için daha az karmaşık olan bir sağlayıcıya göre daha geneldir.</span><span class="sxs-lookup"><span data-stu-id="a76b4-137">A complex provider is more general than a less complex provider, because it can handle a wider variety of questions in the query.</span></span> <span data-ttu-id="a76b4-138">Ayrıca, açık bir tür sistemine de sahiptir ve bu nedenle kullanıcı tanımlı türleri eşleştirmek için kapsamlı bir altyapı içermelidir.</span><span class="sxs-lookup"><span data-stu-id="a76b4-138">It also has an open type system and therefore must contain extensive infrastructure to map user-defined types.</span></span> <span data-ttu-id="a76b4-139">Karmaşık bir sağlayıcının geliştirilmesi için önemli ölçüde çaba gerekir.</span><span class="sxs-lookup"><span data-stu-id="a76b4-139">Developing a complex provider requires a significant amount of effort.</span></span>
