## <a name="example"></a><span data-ttu-id="ddb9d-128">Örnek</span><span class="sxs-lookup"><span data-stu-id="ddb9d-128">Example</span></span>

<span data-ttu-id="ddb9d-129">Aşağıdaki kod parçacığı, bu kuralı ihlal eden soyut bir tür içeriyor.</span><span class="sxs-lookup"><span data-stu-id="ddb9d-129">The following code snippet contains an abstract type that violates this rule.</span></span>

:::code language="vb" source="snippets/vb/all-rules/ca1012-abstract-types-should-not-have-constructors_1.vb" id="snippet1":::

```csharp
// Violates this rule
public abstract class Book
{
    public Book()
    {
    }
}
```

<span data-ttu-id="ddb9d-130">Aşağıdaki kod parçacığı, oluşturucusunun erişilebilirliğini ile değiştirerek önceki ihlalin konumunu düzeltir `public` `protected` .</span><span class="sxs-lookup"><span data-stu-id="ddb9d-130">The following code snippet fixes the previous violation by changing the accessibility of the constructor from `public` to `protected`.</span></span>

```csharp
// Does not violate this rule
public abstract class Book
{
    protected Book()
    {
    }
}
```
