<span data-ttu-id="23f25-103">Microsoft .NET çerçevesindeki şema nesne modeli (SOM), şemaları programlı bir şekilde oluşturmanıza, düzenlemenize ve doğrulamanıza olanak sağlayan zengin bir API 'dir.</span><span class="sxs-lookup"><span data-stu-id="23f25-103">The Schema Object Model (SOM) in the Microsoft .NET Framework is a rich API that allows you to create, edit, and validate schemas programmatically.</span></span> <span data-ttu-id="23f25-104">SOM, XML şeması belgelerinde Belge Nesne Modeli (DOM) XML belgelerinde çalışma biçimine benzer şekilde çalışır.</span><span class="sxs-lookup"><span data-stu-id="23f25-104">The SOM operates on XML schema documents similarly to the way the Document Object Model (DOM) operates on XML documents.</span></span> <span data-ttu-id="23f25-105">XML şeması belgeleri, SOM 'a bir kez yüklendikten sonra şemaya uygun diğer XML belgelerinin yapısı ve geçerliliği hakkında anlam veren geçerli XML dosyalarıdır.</span><span class="sxs-lookup"><span data-stu-id="23f25-105">XML schema documents are valid XML files that, once loaded into the SOM, convey meaning about the structure and validity of other XML documents which conform to the schema.</span></span>  
  
 <span data-ttu-id="23f25-106">Şema, belirli bir şema için XML belgelerinin yapısını veya modelini belirterek XML belgelerinin bir sınıfını tanımlayan bir XML belgesidir.</span><span class="sxs-lookup"><span data-stu-id="23f25-106">A schema is an XML document that defines a class of XML documents by specifying the structure or model of XML documents for a particular schema.</span></span> <span data-ttu-id="23f25-107">Bir şema, XML belgelerinin içeriğiyle ilgili kısıtlamaları tanımlar ve uyumlu XML belgelerinin, şema için geçerli olan şemayla birlikte kabul edilebilmesi için izlenmesi gereken sözlük (kurallar veya dilbilgisi) açıklanmaktadır.</span><span class="sxs-lookup"><span data-stu-id="23f25-107">A schema identifies the constraints on the content of the XML documents, and describes the vocabulary (rules or grammar) that compliant XML documents must follow in order to be considered schema-valid with that particular schema.</span></span> <span data-ttu-id="23f25-108">Bir XML belgesinin doğrulanması, belgenin şema tarafından belirtilen dilbilgisine uyduğundan emin olmanızı sağlayan işlemdir.</span><span class="sxs-lookup"><span data-stu-id="23f25-108">Validation of an XML document is the process that ensures that the document conforms to the grammar specified by the schema.</span></span>  
  
 <span data-ttu-id="23f25-109">Aşağıda, .NET Framework 'teki SOM API 'sinin şemaları oluşturmanıza, düzenlemenize ve doğrulamanıza olanak tanıyan yollar verilmiştir.</span><span class="sxs-lookup"><span data-stu-id="23f25-109">The following are ways the SOM API in the .NET Framework enables you to create, edit, and validate schemas.</span></span>

- <span data-ttu-id="23f25-111">Kesin olarak belirlenmiş sınıfları kullanarak bellek içi şemalar oluşturun.</span><span class="sxs-lookup"><span data-stu-id="23f25-111">Create in-memory schemas using strongly typed classes.</span></span>  
  
- <span data-ttu-id="23f25-112"><xref:System.Xml.Schema.XmlSchemaSet>Şemaları önbelleğe alma, derleme ve alma amacıyla sınıfla etkileşime geçin.</span><span class="sxs-lookup"><span data-stu-id="23f25-112">Interact with the <xref:System.Xml.Schema.XmlSchemaSet> class to cache, compile, and retrieve schemas.</span></span>

- <span data-ttu-id="23f25-115">Karmaşık ve XML örnek belgelerinin doğrulamasında kullanılmak üzere kaydedilebilecek bir şemayı dinamik olarak düzenleyin.</span><span class="sxs-lookup"><span data-stu-id="23f25-115">Dynamically edit a schema that can be complied and saved for use in the validation of XML instance documents.</span></span>
