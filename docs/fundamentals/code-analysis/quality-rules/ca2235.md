## <a name="rule-description"></a><span data-ttu-id="571f9-113">Kural açıklaması</span><span class="sxs-lookup"><span data-stu-id="571f9-113">Rule description</span></span>

<span data-ttu-id="571f9-114">Seri hale getirilebilir bir tür, özniteliğiyle işaretlenmiş bir türüdür <xref:System.SerializableAttribute?displayProperty=fullName> .</span><span class="sxs-lookup"><span data-stu-id="571f9-114">A serializable type is one that is marked with the <xref:System.SerializableAttribute?displayProperty=fullName> attribute.</span></span> <span data-ttu-id="571f9-115">Tür serileştirildiğinde, <xref:System.Runtime.Serialization.SerializationException?displayProperty=fullName> tür seri hale getirilebilir olmayan *ve* arabirimi uygulamayan bir türün örnek alanını içeriyorsa, bir özel durum oluşturulur <xref:System.Runtime.Serialization.ISerializable?displayProperty=fullName> .</span><span class="sxs-lookup"><span data-stu-id="571f9-115">When the type is serialized, a <xref:System.Runtime.Serialization.SerializationException?displayProperty=fullName> exception is thrown if the type contains an instance field of a type that's not serializable *and* doesn't implement the <xref:System.Runtime.Serialization.ISerializable?displayProperty=fullName> interface.</span></span>

> [!TIP]
> <span data-ttu-id="571f9-116">CA2235, <xref:System.Runtime.Serialization.ISerializable> kendi serileştirme mantığını sağladığından, uygulayan türlerin örnek alanları için başlatılmıyor.</span><span class="sxs-lookup"><span data-stu-id="571f9-116">CA2235 does not fire for instance fields of types that implement <xref:System.Runtime.Serialization.ISerializable> because they provide their own serialization logic.</span></span>
