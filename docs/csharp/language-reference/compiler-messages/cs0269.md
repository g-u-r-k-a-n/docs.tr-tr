# <a name="compiler-error-cs0269"></a><span data-ttu-id="3ba59-102">Derleyici Hatası CS0269</span><span class="sxs-lookup"><span data-stu-id="3ba59-102">Compiler Error CS0269</span></span>
<span data-ttu-id="3ba59-103">Atanmamış parametre 'parametre' kullanımı</span><span class="sxs-lookup"><span data-stu-id="3ba59-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="3ba59-104">Derleyici, dış parametrenin kullanılmadan önce bir değer atandığını doğrulayamadı; atandığında değeri tanımsız olabilir.</span><span class="sxs-lookup"><span data-stu-id="3ba59-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="3ba59-105">Değere erişmeden önce `out` çağrılan yöntemdeki parametrelere değer atadığından emin olun.</span><span class="sxs-lookup"><span data-stu-id="3ba59-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="3ba59-106">Geçirilen değişkenin değerini kullanmanız gerekiyorsa, bunun yerine `ref` bir parametre kullanın.</span><span class="sxs-lookup"><span data-stu-id="3ba59-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="3ba59-107">Daha fazla bilgi için [Geçişu Parametreleri'ne](../../programming-guide/classes-and-structs/passing-parameters.md)bakın.</span><span class="sxs-lookup"><span data-stu-id="3ba59-107">For more information, see [Passing Parameters](../../programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="3ba59-108">Örnek</span><span class="sxs-lookup"><span data-stu-id="3ba59-108">Example</span></span>  
 <span data-ttu-id="3ba59-109">Aşağıdaki örnek CS0269 üretir:</span><span class="sxs-lookup"><span data-stu-id="3ba59-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="3ba59-110">Örnek</span><span class="sxs-lookup"><span data-stu-id="3ba59-110">Example</span></span>  
 <span data-ttu-id="3ba59-111">Derleyicinin doğrulayamadığı bir try bloğunda bir değişkenin başlatılması gerçekleşirse, bu durum da oluşabilir:</span><span class="sxs-lookup"><span data-stu-id="3ba59-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
