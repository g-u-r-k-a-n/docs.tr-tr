 <span data-ttu-id="a575b-110">Yapılar ve sınıflar aşağıdaki şekilde benzerdir:</span><span class="sxs-lookup"><span data-stu-id="a575b-110">Structures and classes are similar in the following respects:</span></span>

- <span data-ttu-id="a575b-112">Her ikisinde de oluşturucular, Yöntemler, özellikler, alanlar, sabitler, numaralandırmalar, olaylar ve olay işleyicilerini içerebilen Üyeler vardır.</span><span class="sxs-lookup"><span data-stu-id="a575b-112">Both have members, which can include constructors, methods, properties, fields, constants, enumerations, events, and event handlers.</span></span> <span data-ttu-id="a575b-113">Ancak, bu üyeleri bir yapının tanımlanmış *öğeleriyle* karıştırmayın.</span><span class="sxs-lookup"><span data-stu-id="a575b-113">However, do not confuse these members with the declared *elements* of a structure.</span></span>

- <span data-ttu-id="a575b-116">Her ikisi de arabirim uygulayabilir.</span><span class="sxs-lookup"><span data-stu-id="a575b-116">Both can implement interfaces.</span></span>

- <span data-ttu-id="a575b-118">Her ikisi de *varsayılan bir özellik*sunabilir, bu özellik en az bir parametre alır.</span><span class="sxs-lookup"><span data-stu-id="a575b-118">Both can expose a *default property*, provided that property takes at least one parameter.</span></span>  
  
- <span data-ttu-id="a575b-119">Her ikisi de olayları bildirebilir ve oluşturabilir ve her ikisi de temsilciler bildirebilir.</span><span class="sxs-lookup"><span data-stu-id="a575b-119">Both can declare and raise events, and both can declare delegates.</span></span>  
  
## <a name="differences"></a><span data-ttu-id="a575b-120">Farklılıklar</span><span class="sxs-lookup"><span data-stu-id="a575b-120">Differences</span></span>  
 <span data-ttu-id="a575b-121">Yapılar ve sınıflar aşağıdaki bununla farklıdır:</span><span class="sxs-lookup"><span data-stu-id="a575b-121">Structures and classes differ in the following particulars:</span></span>  
  
- <span data-ttu-id="a575b-122">Yapılar *değer türleridir*; sınıflar *başvuru türleridir*.</span><span class="sxs-lookup"><span data-stu-id="a575b-122">Structures are *value types*; classes are *reference types*.</span></span> <span data-ttu-id="a575b-123">Yapı türünün bir değişkeni, bir sınıf türü olarak veriye bir başvuru eklemek yerine yapının verilerini içerir.</span><span class="sxs-lookup"><span data-stu-id="a575b-123">A variable of a structure type contains the structure's data, rather than containing a reference to the data as a class type does.</span></span>  
  
- <span data-ttu-id="a575b-124">Yapılar yığın ayırmayı kullanır; sınıflar yığın ayırmayı kullanır.</span><span class="sxs-lookup"><span data-stu-id="a575b-124">Structures use stack allocation; classes use heap allocation.</span></span>  
  
- <span data-ttu-id="a575b-125">Tüm yapı öğeleri `Public` Varsayılan olarak, sınıf değişkenleri ve sabitler varsayılan olarak `Private` , diğer sınıf üyeleri varsayılan olarak kullanılır `Public` .</span><span class="sxs-lookup"><span data-stu-id="a575b-125">All structure elements are `Public` by default; class variables and constants are `Private` by default, while other class members are `Public` by default.</span></span> <span data-ttu-id="a575b-126">Sınıf üyeleri için bu davranış, varsayılan Visual Basic 6,0 sistemiyle uyumluluk sağlar.</span><span class="sxs-lookup"><span data-stu-id="a575b-126">This behavior for class members provides compatibility with the Visual Basic 6.0 system of defaults.</span></span>  
  
- <span data-ttu-id="a575b-127">Bir yapının en az bir paylaşılmayan değişkeni veya paylaşılmayan olmayan, özel olmayan olay öğesi olması gerekir; bir sınıf tamamen boş olabilir.</span><span class="sxs-lookup"><span data-stu-id="a575b-127">A structure must have at least one nonshared variable or nonshared, noncustom event element; a class can be completely empty.</span></span>  
  
- <span data-ttu-id="a575b-128">Yapı öğeleri olarak bildirilemez `Protected` ; sınıf üyeleri olabilir.</span><span class="sxs-lookup"><span data-stu-id="a575b-128">Structure elements cannot be declared as `Protected`; class members can.</span></span>  
  
- <span data-ttu-id="a575b-129">Bir yapı yordamı yalnızca [paylaşılan](../../../language-reference/modifiers/shared.md) bir `Sub` yordam ve yalnızca [AddHandler ifadesinin](../../../language-reference/statements/addhandler-statement.md)bir yöntemi olduğunda olayları işleyebilir; herhangi bir sınıf yordamı [Handles](../../../language-reference/statements/handles-clause.md) anahtar sözcüğünü veya ifadesini kullanarak olayları işleyebilir `AddHandler` .</span><span class="sxs-lookup"><span data-stu-id="a575b-129">A structure procedure can handle events only if it is a [Shared](../../../language-reference/modifiers/shared.md)`Sub` procedure, and only by means of the [AddHandler Statement](../../../language-reference/statements/addhandler-statement.md); any class procedure can handle events, using either the [Handles](../../../language-reference/statements/handles-clause.md) keyword or the `AddHandler` statement.</span></span> <span data-ttu-id="a575b-130">Daha fazla bilgi için bkz. [Olaylar](../events/index.md).</span><span class="sxs-lookup"><span data-stu-id="a575b-130">For more information, see [Events](../events/index.md).</span></span>  
  
- <span data-ttu-id="a575b-131">Yapı değişkeni bildirimleri diziler için başlatıcılar veya başlangıç boyutları belirtemez; sınıf değişkeni bildirimleri olabilir.</span><span class="sxs-lookup"><span data-stu-id="a575b-131">Structure variable declarations cannot specify initializers or initial sizes for arrays; class variable declarations can.</span></span>

- <span data-ttu-id="a575b-133">Yapılar devredilemez; sınıflar.</span><span class="sxs-lookup"><span data-stu-id="a575b-133">Structures are not inheritable; classes are.</span></span>  
  
- <span data-ttu-id="a575b-134">Yapılar hiçbir zaman sonlandırılmadığından, ortak dil çalışma zamanı (CLR) <xref:System.Object.Finalize%2A> herhangi bir yapıda yöntemi çağırılmaz; sınıflar çöp toplayıcı (GC) tarafından sonlandırılır ve <xref:System.Object.Finalize%2A> kalan etkin bir başvuru olmadığını algıladığında bir sınıfa çağrı yapılır.</span><span class="sxs-lookup"><span data-stu-id="a575b-134">Structures are never terminated, so the common language runtime (CLR) never calls the <xref:System.Object.Finalize%2A> method on any structure; classes are terminated by the garbage collector (GC), which calls <xref:System.Object.Finalize%2A> on a class when it detects there are no active references remaining.</span></span>  
  
- <span data-ttu-id="a575b-135">Bir yapı için bir Oluşturucu gerekmez; bir sınıf.</span><span class="sxs-lookup"><span data-stu-id="a575b-135">A structure does not require a constructor; a class does.</span></span>  
  
- <span data-ttu-id="a575b-136">Yapılar yalnızca parametre alıyorsa paylaşılmayan oluşturuculara sahip olabilir; sınıflar, parametrelere sahip veya parametresiz olabilir.</span><span class="sxs-lookup"><span data-stu-id="a575b-136">Structures can have nonshared constructors only if they take parameters; classes can have them with or without parameters.</span></span>  
  
 <span data-ttu-id="a575b-137">Her yapının parametresiz bir örtük ortak Oluşturucusu vardır.</span><span class="sxs-lookup"><span data-stu-id="a575b-137">Every structure has an implicit public constructor without parameters.</span></span> <span data-ttu-id="a575b-138">Bu Oluşturucu, tüm yapının veri öğelerini varsayılan değerlerine başlatır.</span><span class="sxs-lookup"><span data-stu-id="a575b-138">This constructor initializes all the structure's data elements to their default values.</span></span> <span data-ttu-id="a575b-139">Bu davranışı yeniden tanımlayamazsınız.</span><span class="sxs-lookup"><span data-stu-id="a575b-139">You cannot redefine this behavior.</span></span>  
  
## <a name="instances-and-variables"></a><span data-ttu-id="a575b-140">Örnekler ve değişkenler</span><span class="sxs-lookup"><span data-stu-id="a575b-140">Instances and Variables</span></span>  
 <span data-ttu-id="a575b-141">Yapılar değer türleri olduğundan, her yapı değişkeni ayrı bir yapı örneğine kalıcı olarak bağlanır.</span><span class="sxs-lookup"><span data-stu-id="a575b-141">Because structures are value types, each structure variable is permanently bound to an individual structure instance.</span></span> <span data-ttu-id="a575b-142">Ancak sınıflar başvuru türleridir ve bir nesne değişkeni farklı zamanlarda çeşitli sınıf örneklerine başvurabilir.</span><span class="sxs-lookup"><span data-stu-id="a575b-142">But classes are reference types, and an object variable can refer to various class instances at different times.</span></span> <span data-ttu-id="a575b-143">Bu ayrım, yapıların ve sınıfların kullanımını aşağıdaki yollarla etkiler:</span><span class="sxs-lookup"><span data-stu-id="a575b-143">This distinction affects your usage of structures and classes in the following ways:</span></span>  
  
- <span data-ttu-id="a575b-144">**Başlatılmasında.**</span><span class="sxs-lookup"><span data-stu-id="a575b-144">**Initialization.**</span></span> <span data-ttu-id="a575b-145">Yapı değişkeni, yapının parametresiz oluşturucusunu kullanarak örtük olarak öğelerin başlatılmasını içerir.</span><span class="sxs-lookup"><span data-stu-id="a575b-145">A structure variable implicitly includes an initialization of the elements using the structure's parameterless constructor.</span></span> <span data-ttu-id="a575b-146">Bu nedenle, `Dim s As struct1` ile eşdeğerdir `Dim s As struct1 = New struct1()` .</span><span class="sxs-lookup"><span data-stu-id="a575b-146">Therefore, `Dim s As struct1` is equivalent to `Dim s As struct1 = New struct1()`.</span></span>

- <span data-ttu-id="a575b-150">**Nothing atama.**</span><span class="sxs-lookup"><span data-stu-id="a575b-150">**Assigning Nothing.**</span></span> <span data-ttu-id="a575b-151">Bir yapı değişkenine [Nothing](../../../language-reference/nothing.md) değeri atayabilirsiniz, ancak örnek değişkenle ilişkilendirilmeye devam eder.</span><span class="sxs-lookup"><span data-stu-id="a575b-151">You can assign the value [Nothing](../../../language-reference/nothing.md) to a structure variable, but the instance continues to be associated with the variable.</span></span> <span data-ttu-id="a575b-152">Kendi yöntemlerini çağırabilirsiniz ve veri öğelerine erişebilirsiniz, ancak değişken öğeleri atama tarafından yeniden başlatılır.</span><span class="sxs-lookup"><span data-stu-id="a575b-152">You can still call its methods and access its data elements, although variable elements are reinitialized by the assignment.</span></span>

     <span data-ttu-id="a575b-157">Ancak yapı öğeleri kendi örnekleri içinde yalıtılmıştır.</span><span class="sxs-lookup"><span data-stu-id="a575b-157">Structure elements, however, are isolated within their own instance.</span></span> <span data-ttu-id="a575b-158">Değerlerinde yapılan değişiklikler, aynı bildirimin diğer örneklerinde bile diğer herhangi bir yapı değişkenine yansıtılmaz `Structure` .</span><span class="sxs-lookup"><span data-stu-id="a575b-158">Changes to their values are not reflected in any other structure variables, even in other instances of the same `Structure` declaration.</span></span>
