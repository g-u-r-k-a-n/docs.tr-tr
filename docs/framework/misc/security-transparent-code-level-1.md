## <a name="the-level-1-transparency-model"></a><span data-ttu-id="40fcf-115">Düzey 1 saydamlık modeli</span><span class="sxs-lookup"><span data-stu-id="40fcf-115">The Level 1 Transparency Model</span></span>  

 <span data-ttu-id="40fcf-116">Düzey 1 saydamlık kullandığınızda, kodu güvenlik açısından saydam, güvenlik açısından güvenli-kritik ve güvenlik açısından kritik yöntemlere ayıran bir güvenlik modeli kullanıyorsunuz.</span><span class="sxs-lookup"><span data-stu-id="40fcf-116">When you use Level 1 transparency, you are using a security model that separates code into security-transparent, security-safe-critical, and security-critical methods.</span></span>  
  
 <span data-ttu-id="40fcf-117">Tüm bir derlemeyi, bir derlemedeki bazı sınıfları veya bir sınıftaki bazı yöntemleri güvenlik açısından saydam olarak işaretleyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="40fcf-117">You can mark a whole assembly, some classes in an assembly, or some methods in a class as security-transparent.</span></span> <span data-ttu-id="40fcf-118">Güvenliği saydam kod ayrıcalıkları yükseltemez.</span><span class="sxs-lookup"><span data-stu-id="40fcf-118">Security-transparent code cannot elevate privileges.</span></span> <span data-ttu-id="40fcf-119">Bu kısıtlama üç sonuçlara sahiptir:</span><span class="sxs-lookup"><span data-stu-id="40fcf-119">This restriction has three consequences:</span></span>

- <span data-ttu-id="40fcf-122">Güvenlik açısından saydam kodda yürütülmesi gereken güvenli olmayan (doğrulanamayan) kod, güvenlik izni için tam talebe neden olur <xref:System.Security.Permissions.SecurityPermissionFlag.UnmanagedCode> .</span><span class="sxs-lookup"><span data-stu-id="40fcf-122">Any unsafe (unverifiable) code that must execute in security-transparent code causes a full demand for the <xref:System.Security.Permissions.SecurityPermissionFlag.UnmanagedCode> security permission.</span></span>

 <span data-ttu-id="40fcf-129">Güvenlik açısından kritik, güvenlik açısından şeffaf bir tersidir.</span><span class="sxs-lookup"><span data-stu-id="40fcf-129">Security-critical is the opposite of security-transparent.</span></span> <span data-ttu-id="40fcf-130">Güvenlik açısından kritik kod tam güvenle yürütülür ve tüm ayrıcalıklı işlemleri gerçekleştirebilir.</span><span class="sxs-lookup"><span data-stu-id="40fcf-130">Security-critical code executes with full trust and can perform all privileged operations.</span></span> <span data-ttu-id="40fcf-131">Güvenli kritik kod, kısmen güvenilen çağıranların erişim izni olmayan kaynakları kullanmasına izin vermediğini onaylamak için kapsamlı bir güvenlik denetimi aracılığıyla oluşturulmuş ayrıcalıklı koddur.</span><span class="sxs-lookup"><span data-stu-id="40fcf-131">Security-safe-critical code is privileged code that has been through an extensive security audit to confirm that it does not allow partially trusted callers to use resources they do not have permission to access.</span></span>

> [!IMPORTANT]

## <a name="security-transparency-examples"></a><span data-ttu-id="40fcf-182">Güvenlik saydamlığı örnekleri</span><span class="sxs-lookup"><span data-stu-id="40fcf-182">Security Transparency Examples</span></span>
