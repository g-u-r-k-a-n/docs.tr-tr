# <a name="property-design"></a><span data-ttu-id="64746-102">Özellik Tasarımı</span><span class="sxs-lookup"><span data-stu-id="64746-102">Property Design</span></span>
<span data-ttu-id="64746-103">Özellikler teknik açıdan yöntemlere çok benzer olsa da, bunlar kullanım senaryolarında oldukça farklıdır.</span><span class="sxs-lookup"><span data-stu-id="64746-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="64746-104">Akıllı alanlar olarak görülenmelidir.</span><span class="sxs-lookup"><span data-stu-id="64746-104">They should be seen as smart fields.</span></span> <span data-ttu-id="64746-105">Alanların çağırma söz dizimi ve yöntemlerin esnekliği vardır.</span><span class="sxs-lookup"><span data-stu-id="64746-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>  
  
 <span data-ttu-id="64746-106">**✓ DO** çağıran özelliğin değerini değiştirmek açmaması gereken, yalnızca get özellikleri oluşturun.</span><span class="sxs-lookup"><span data-stu-id="64746-106">**✓ DO** create get-only properties if the caller should not be able to change the value of the property.</span></span>  
  
 <span data-ttu-id="64746-107">Özelliğin türü kesilebilir bir başvuru türü ise, özellik salt al olsa bile Özellik değerinin değiştirilebileceğini aklınızda bulundurun.</span><span class="sxs-lookup"><span data-stu-id="64746-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>  
  
 <span data-ttu-id="64746-108">**X DO NOT** alıcı daha geniş erişilebilirlik sahip ayarlayıcı yalnızca küme özellikleri veya özellikleri sağlar.</span><span class="sxs-lookup"><span data-stu-id="64746-108">**X DO NOT** provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>  
  
 <span data-ttu-id="64746-109">Örneğin, özellikleri ortak bir ayarlayıcı ve korumalı bir alıcı ile birlikte kullanmayın.</span><span class="sxs-lookup"><span data-stu-id="64746-109">For example, do not use properties with a public setter and a protected getter.</span></span>  
  
 <span data-ttu-id="64746-110">Özellik alıcısı sağlanmadıysa, işlevi bir yöntem olarak uygulayın.</span><span class="sxs-lookup"><span data-stu-id="64746-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="64746-111">Yöntem adını `Set` ile başlatmayı ve özelliği adlandırdıklarınızı takip etmeyi düşünün.</span><span class="sxs-lookup"><span data-stu-id="64746-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="64746-112">Örneğin <xref:System.AppDomain>, `CachePath`adlı salt ayarlı bir özelliği olması yerine `SetCachePath` adlı bir yönteme sahiptir.</span><span class="sxs-lookup"><span data-stu-id="64746-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>  
  
 <span data-ttu-id="64746-113">**✓ DO** tüm özellikleri, varsayılan bir güvenlik açığı veya çok verimsiz kodu yol açmamasını sağlamak için duyarlı varsayılan değerler sağlayın.</span><span class="sxs-lookup"><span data-stu-id="64746-113">**✓ DO** provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>  
  
 <span data-ttu-id="64746-114">**✓ DO** bu nesne bir geçici geçersiz durumda sonuçları olsa bile herhangi bir sırada ayarlanacak özellikler izin verir.</span><span class="sxs-lookup"><span data-stu-id="64746-114">**✓ DO** allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>  
  
 <span data-ttu-id="64746-115">İki veya daha fazla özelliğin, bir özelliğin bazı değerlerinin aynı nesne üzerindeki diğer özelliklerin değerleri verildiğinde geçersiz olabileceği bir nokta ile ilişkili olması yaygındır.</span><span class="sxs-lookup"><span data-stu-id="64746-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="64746-116">Bu gibi durumlarda, ilişkili özellikler gerçekten nesne tarafından birlikte kullanıldığından, geçersiz durumdan kaynaklanan özel durumlar ertelenmelidir.</span><span class="sxs-lookup"><span data-stu-id="64746-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>  
  
 <span data-ttu-id="64746-117">**✓ DO** özellik set yordamı bir özel durum oluşturursa önceki değeri korumak.</span><span class="sxs-lookup"><span data-stu-id="64746-117">**✓ DO** preserve the previous value if a property setter throws an exception.</span></span>  
  
 <span data-ttu-id="64746-118">**X AVOID** özellik alıcıları özel durumları atma.</span><span class="sxs-lookup"><span data-stu-id="64746-118">**X AVOID** throwing exceptions from property getters.</span></span>  
  
 <span data-ttu-id="64746-119">Özellik alıcıları basit işlemler olmalıdır ve herhangi bir önkoşullara sahip olmamalıdır.</span><span class="sxs-lookup"><span data-stu-id="64746-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="64746-120">Alıcı bir özel durum oluştur, büyük olasılıkla bir yöntem olarak yeniden tasarlanması gerekir.</span><span class="sxs-lookup"><span data-stu-id="64746-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="64746-121">Bu kuralın, bağımsız değişkenlerin doğrulanması sonucunda özel durumlar beklediğimiz Dizin oluşturucular için uygulanmadığından emin olun.</span><span class="sxs-lookup"><span data-stu-id="64746-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>  
  
### <a name="indexed-property-design"></a><span data-ttu-id="64746-122">Dizinli özellik tasarımı</span><span class="sxs-lookup"><span data-stu-id="64746-122">Indexed Property Design</span></span>  
 <span data-ttu-id="64746-123">Dizinli bir özellik, parametrelere sahip olabilecek ve dizi dizine benzer özel söz dizimi ile çağrılabilen özel bir özelliktir.</span><span class="sxs-lookup"><span data-stu-id="64746-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>  
  
 <span data-ttu-id="64746-124">Dizinli özellikler genellikle Dizin oluşturucular olarak adlandırılır.</span><span class="sxs-lookup"><span data-stu-id="64746-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="64746-125">Dizin oluşturucular yalnızca bir mantıksal koleksiyondaki öğelere erişim sağlayan API 'lerde kullanılmalıdır.</span><span class="sxs-lookup"><span data-stu-id="64746-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="64746-126">Örneğin, bir dize bir karakter koleksiyonudur ve <xref:System.String?displayProperty=nameWithType> üzerindeki Dizin Oluşturucu, karakterlerine erişmek için eklenmiştir.</span><span class="sxs-lookup"><span data-stu-id="64746-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>  
  
 <span data-ttu-id="64746-127">**✓ CONSIDER** iç dizisinde depolanan verilere erişim sağlamak için dizin oluşturucular kullanma.</span><span class="sxs-lookup"><span data-stu-id="64746-127">**✓ CONSIDER** using indexers to provide access to data stored in an internal array.</span></span>  
  
 <span data-ttu-id="64746-128">**✓ CONSIDER** öğe koleksiyonunu temsil eden türlerinde dizin oluşturucular sağlama.</span><span class="sxs-lookup"><span data-stu-id="64746-128">**✓ CONSIDER** providing indexers on types representing collections of items.</span></span>  
  
 <span data-ttu-id="64746-129">**X AVOID** dizin oluşturulmuş özellikleri birden fazla parametresiyle kullanarak.</span><span class="sxs-lookup"><span data-stu-id="64746-129">**X AVOID** using indexed properties with more than one parameter.</span></span>  
  
 <span data-ttu-id="64746-130">Tasarım birden çok parametre gerektiriyorsa, özelliğin bir mantıksal koleksiyona bir erişimciyi temsil edip etmediğini yeniden düşünün.</span><span class="sxs-lookup"><span data-stu-id="64746-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="64746-131">Yoksa, bunun yerine yöntemleri kullanın.</span><span class="sxs-lookup"><span data-stu-id="64746-131">If it does not, use methods instead.</span></span> <span data-ttu-id="64746-132">Yöntem adını `Get` veya `Set`başlatmayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="64746-132">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="64746-133">**X AVOID** dışında parametre türleri oluşturucularla <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, veya enum.</span><span class="sxs-lookup"><span data-stu-id="64746-133">**X AVOID** indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>  
  
 <span data-ttu-id="64746-134">Tasarımda diğer parametre türleri gerekiyorsa, API 'nin mantıksal bir koleksiyona bir erişimciyi temsil edip etmediğini kesin olarak yeniden değerlendirin.</span><span class="sxs-lookup"><span data-stu-id="64746-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="64746-135">Değilse, bir yöntemi kullanın.</span><span class="sxs-lookup"><span data-stu-id="64746-135">If it does not, use a method.</span></span> <span data-ttu-id="64746-136">Yöntem adını `Get` veya `Set`başlatmayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="64746-136">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="64746-137">**✓ DO** adını kullanın `Item` için tabii daha iyi bir adı olmadıkça özellikleri dizine (örn., bkz: <xref:System.String.Chars%2A> özellikte `System.String`).</span><span class="sxs-lookup"><span data-stu-id="64746-137">**✓ DO** use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>  
  
 <span data-ttu-id="64746-138">' C#De, Dizin oluşturucular varsayılan olarak adlandırılmış öğedir.</span><span class="sxs-lookup"><span data-stu-id="64746-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="64746-139">Bu adı özelleştirmek için <xref:System.Runtime.CompilerServices.IndexerNameAttribute> kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="64746-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>  
  
 <span data-ttu-id="64746-140">**X DO NOT** hem bir dizin oluşturucu hem de anlam olarak eşdeğerdir yöntemleri sağlar.</span><span class="sxs-lookup"><span data-stu-id="64746-140">**X DO NOT** provide both an indexer and methods that are semantically equivalent.</span></span>  
  
 <span data-ttu-id="64746-141">**X DO NOT** aşırı yüklenmiş dizin oluşturucular bir türde birden fazla ailesi sağlayın.</span><span class="sxs-lookup"><span data-stu-id="64746-141">**X DO NOT** provide more than one family of overloaded indexers in one type.</span></span>  
  
 <span data-ttu-id="64746-142">Bu, C# derleyici tarafından zorlanır.</span><span class="sxs-lookup"><span data-stu-id="64746-142">This is enforced by the C# compiler.</span></span>  
  
 <span data-ttu-id="64746-143">**X DO NOT** kullanım varsayılan olmayan dizin oluşturulmuş özellikleri.</span><span class="sxs-lookup"><span data-stu-id="64746-143">**X DO NOT** use nondefault indexed properties.</span></span>  
  
 <span data-ttu-id="64746-144">Bu, C# derleyici tarafından zorlanır.</span><span class="sxs-lookup"><span data-stu-id="64746-144">This is enforced by the C# compiler.</span></span>  
  
### <a name="property-change-notification-events"></a><span data-ttu-id="64746-145">Özellik değişiklik bildirimi olayları</span><span class="sxs-lookup"><span data-stu-id="64746-145">Property Change Notification Events</span></span>  
 <span data-ttu-id="64746-146">Bazen bir özellik değerindeki değişiklikleri kullanıcıya bildiren bir olay sağlamak yararlı olur.</span><span class="sxs-lookup"><span data-stu-id="64746-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="64746-147">Örneğin, `System.Windows.Forms.Control`, `Text` özelliğinin değeri değiştirildikten sonra `TextChanged` olayını harekete geçirir.</span><span class="sxs-lookup"><span data-stu-id="64746-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>  
  
 <span data-ttu-id="64746-148">**✓ CONSIDER** değişiklik üst düzey API'leri (genellikle Tasarımcı bileşenleri) özellik değerlerinde değişiklik yapıldığında bildirim olayları oluşturma.</span><span class="sxs-lookup"><span data-stu-id="64746-148">**✓ CONSIDER** raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>  
  
 <span data-ttu-id="64746-149">Bir kullanıcının bir nesnenin özelliğinin ne zaman değiştiğini bilmesini sağlamak için iyi bir senaryo varsa, nesne özellik için bir değişiklik bildirimi olayı tetiklemelidir.</span><span class="sxs-lookup"><span data-stu-id="64746-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>  
  
 <span data-ttu-id="64746-150">Bununla birlikte, temel türler veya Koleksiyonlar gibi alt düzey API 'Ler için bu tür olayları yükseltme yüküyle ilgili ek yükün olması düşüktür.</span><span class="sxs-lookup"><span data-stu-id="64746-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="64746-151">Örneğin, <xref:System.Collections.Generic.List%601>, listeye yeni bir öğe eklendiğinde ve `Count` özelliği değiştiğinde bu tür olayları yükseltemez.</span><span class="sxs-lookup"><span data-stu-id="64746-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>  
  
 <span data-ttu-id="64746-152">**✓ CONSIDER** değişiklik bir özelliğin değerini dış zorlar değiştiğinde bildirim olayları oluşturma.</span><span class="sxs-lookup"><span data-stu-id="64746-152">**✓ CONSIDER** raising change notification events when the value of a property changes via external forces.</span></span>  
  
 <span data-ttu-id="64746-153">Bir özellik değeri, bazı dış zorlanarak (nesne üzerinde Yöntemler çağırma dışında bir şekilde) değişirse, olay oluşturma, geliştiricinin değerin değiştiğini ve değiştiğini gösterir.</span><span class="sxs-lookup"><span data-stu-id="64746-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="64746-154">İyi bir örnek, metin kutusu denetiminin `Text` özelliğidir.</span><span class="sxs-lookup"><span data-stu-id="64746-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="64746-155">Kullanıcı bir `TextBox`metin yazdığında, özellik değeri otomatik olarak değişir.</span><span class="sxs-lookup"><span data-stu-id="64746-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
