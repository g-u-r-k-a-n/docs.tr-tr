# <a name="streaming-message-transfer"></a><span data-ttu-id="45c24-102">İleti Aktarma Akışı</span><span class="sxs-lookup"><span data-stu-id="45c24-102">Streaming Message Transfer</span></span>
<span data-ttu-id="45c24-103">Windows Communication Foundation (WCF) aktarımları, iletileri aktarmaya yönelik iki modu destekler:</span><span class="sxs-lookup"><span data-stu-id="45c24-103">Windows Communication Foundation (WCF) transports support two modes for transferring messages:</span></span>

 <span data-ttu-id="45c24-111">Varsayılan olarak, HTTP, TCP/IP ve adlandırılmış kanal aktarımları, arabelleğe alınmış aktarımları kullanır.</span><span class="sxs-lookup"><span data-stu-id="45c24-111">By default, the HTTP, TCP/IP, and named pipe transports use buffered transfers.</span></span> <span data-ttu-id="45c24-112">Bu belgede, Bu aktarımların arabelleğe alınan aktarım modundan akışlı aktarım moduna nasıl değiştirileceği ve bunu yapmanın sonuçları açıklanmaktadır.</span><span class="sxs-lookup"><span data-stu-id="45c24-112">This document describes how to switch these transports from a buffered to streamed transfer mode and the consequences of doing so.</span></span>
