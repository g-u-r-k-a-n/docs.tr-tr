## <a name="parallelmergeoptions"></a><span data-ttu-id="401c0-117">ParallelMergeOptions</span><span class="sxs-lookup"><span data-stu-id="401c0-117">ParallelMergeOptions</span></span>  
 <span data-ttu-id="401c0-118">Numaralandırma, <xref:System.Linq.ParallelMergeOptions> desteklenen sorgu şekilleri için, sonuçlar tek bir iş parçacığı üzerinde tüketildiğinde sorgunun son çıktısının nasıl veriş olduğunu belirten aşağıdaki seçenekleri içerir:</span><span class="sxs-lookup"><span data-stu-id="401c0-118">The <xref:System.Linq.ParallelMergeOptions> enumeration includes the following options that specify, for supported query shapes, how the final output of the query is yielded when the results are consumed on one thread:</span></span>  
  
- `Not Buffered`  
  
     <span data-ttu-id="401c0-119">Bu <xref:System.Linq.ParallelMergeOptions.NotBuffered> seçenek, işlenen her öğenin üretilir üretilmez her iş parçacığından döndürülmeye neden olur.</span><span class="sxs-lookup"><span data-stu-id="401c0-119">The <xref:System.Linq.ParallelMergeOptions.NotBuffered> option causes each processed element to be returned from each thread as soon as it is produced.</span></span> <span data-ttu-id="401c0-120">Bu davranış, çıktıyı "akış" ile benzer.</span><span class="sxs-lookup"><span data-stu-id="401c0-120">This behavior is analogous to "streaming" the output.</span></span> <span data-ttu-id="401c0-121"><xref:System.Linq.ParallelEnumerable.AsOrdered%2A> İşleç sorguda varsa, `NotBuffered` kaynak öğelerin sırasını korur.</span><span class="sxs-lookup"><span data-stu-id="401c0-121">If the <xref:System.Linq.ParallelEnumerable.AsOrdered%2A> operator is present in the query, `NotBuffered` preserves the order of the source elements.</span></span> <span data-ttu-id="401c0-122">Kullanılabilir `NotBuffered` oldukları anda sonuç vermeye başlasa da, tüm sonuçları üretmek için gereken toplam süre diğer birleştirme seçeneklerinden birini kullanmaktan daha uzun olabilir.</span><span class="sxs-lookup"><span data-stu-id="401c0-122">Although `NotBuffered` starts yielding results as soon as they're available, the total time to produce all the results might still be longer than using one of the other merge options.</span></span>  
  
- `Auto Buffered`  
  
     <span data-ttu-id="401c0-123">Seçenek, <xref:System.Linq.ParallelMergeOptions.AutoBuffered> sorgunun öğeleri arabelleğe toplamasına ve arabellek içeriğini düzenli aralıklarla tüketen iş parçacığına aynı anda teslim almasına neden olur.</span><span class="sxs-lookup"><span data-stu-id="401c0-123">The <xref:System.Linq.ParallelMergeOptions.AutoBuffered> option causes the query to collect elements into a buffer and then periodically yield the buffer contents all at once to the consuming thread.</span></span> <span data-ttu-id="401c0-124">Bu, kaynak verilerin "akış" davranışını `NotBuffered`kullanmak yerine "yığınlar" halinde veri vermeye benzer.</span><span class="sxs-lookup"><span data-stu-id="401c0-124">This is analogous to yielding the source data in "chunks" instead of using the "streaming" behavior of `NotBuffered`.</span></span> <span data-ttu-id="401c0-125">`AutoBuffered`tüketen iş `NotBuffered` parçacığı üzerinde ilk öğeyi kullanılabilir hale getirmek için daha uzun sürebilir.</span><span class="sxs-lookup"><span data-stu-id="401c0-125">`AutoBuffered` may take longer than `NotBuffered` to make the first element available on the consuming thread.</span></span> <span data-ttu-id="401c0-126">Arabelleğin boyutu ve tam verimli davranış yapılandırılamaz ve sorguyla ilgili çeşitli etkenlere bağlı olarak değişebilir.</span><span class="sxs-lookup"><span data-stu-id="401c0-126">The size of the buffer and the exact yielding behavior are not configurable and may vary, depending on various factors that relate to the query.</span></span>  
  
- `FullyBuffered`  
  
     <span data-ttu-id="401c0-127">Seçenek, <xref:System.Linq.ParallelMergeOptions.FullyBuffered> öğelerden herhangi biri verilmeden önce tüm sorgunun çıktısının arabelleğe alınmasına neden olur.</span><span class="sxs-lookup"><span data-stu-id="401c0-127">The <xref:System.Linq.ParallelMergeOptions.FullyBuffered> option causes the output of the whole query to be buffered before any of the elements are yielded.</span></span> <span data-ttu-id="401c0-128">Bu seçeneği kullandığınızda, ilk öğenin tüketen iş parçacığında kullanılabilir olması daha uzun sürebilir, ancak tam sonuçlar diğer seçenekleri kullanmaktan daha hızlı üretilebilir.</span><span class="sxs-lookup"><span data-stu-id="401c0-128">When you use this option, it can take longer before the first element is available on the consuming thread, but the complete results might still be produced faster than by using the other options.</span></span>
