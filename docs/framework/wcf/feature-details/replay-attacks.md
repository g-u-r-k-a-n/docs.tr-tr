# <a name="replay-attacks"></a><span data-ttu-id="38ee9-103">Yeniden Yürütme Saldırıları</span><span class="sxs-lookup"><span data-stu-id="38ee9-103">Replay Attacks</span></span>

<span data-ttu-id="38ee9-104">Bir saldırgan iki taraf arasında bir ileti akışını kopyaladığında veya bir veya daha fazla tarafın akışını yeniden oynadığında bir yeniden *yürütme saldırısı* meydana gelir.</span><span class="sxs-lookup"><span data-stu-id="38ee9-104">A *replay attack* occurs when an attacker copies a stream of messages between two parties and replays the stream to one or more of the parties.</span></span> <span data-ttu-id="38ee9-105">Hafiflemediği sürece, saldırıya tabi olan bilgisayarlar akışı meşru iletiler olarak işler ve bu da bir öğenin gereksiz sıraları gibi hatalı sonuçlar oluşmasına neden olur.</span><span class="sxs-lookup"><span data-stu-id="38ee9-105">Unless mitigated, the computers subject to the attack process the stream as legitimate messages, resulting in a range of bad consequences, such as redundant orders of an item.</span></span>  
  
## <a name="bindings-may-be-subject-to-reflection-attacks"></a><span data-ttu-id="38ee9-106">Bağlamalar, yansıma saldırılarına maruz kalabilir</span><span class="sxs-lookup"><span data-stu-id="38ee9-106">Bindings May Be Subject to Reflection Attacks</span></span>  

 <span data-ttu-id="38ee9-107">*Yansıma saldırıları* , yanıt olarak alıcıdan gelmiş gibi iletiler bir gönderene geri yeniden oynatılır.</span><span class="sxs-lookup"><span data-stu-id="38ee9-107">*Reflection attacks* are replays of messages back to a sender as if they came from the receiver as the reply.</span></span> <span data-ttu-id="38ee9-108">Windows Communication Foundation (WCF) mekanizmasında standart yeniden *yürütme algılaması* bunu otomatik olarak işlemez.</span><span class="sxs-lookup"><span data-stu-id="38ee9-108">The standard *replay detection* in the Windows Communication Foundation (WCF) mechanism does not automatically handle this.</span></span>  
  
 <span data-ttu-id="38ee9-109">WCF hizmet modeli istek iletileri için imzalı bir ileti KIMLIĞI eklediğinden ve yanıt iletilerinde imzalı bir üst bilgi beklediği için yansıma saldırıları varsayılan olarak azaltıldığında `relates-to` .</span><span class="sxs-lookup"><span data-stu-id="38ee9-109">Reflection attacks are mitigated by default because the WCF service model adds a signed message ID to request messages and expects a signed `relates-to` header on response messages.</span></span> <span data-ttu-id="38ee9-110">Sonuç olarak, istek iletisi yanıt olarak yeniden yürütülemez.</span><span class="sxs-lookup"><span data-stu-id="38ee9-110">Consequently, the request message cannot be replayed as a response.</span></span> <span data-ttu-id="38ee9-111">Güvenli güvenilir ileti (RM) senaryolarında, şu nedenle yansıma saldırıları azaltıldığında:</span><span class="sxs-lookup"><span data-stu-id="38ee9-111">In secure reliable message (RM) scenarios, reflection attacks are mitigated because:</span></span>  
  
- <span data-ttu-id="38ee9-112">Oluşturma sırası ve sıra oluşturma yanıtı ileti şemaları farklıdır.</span><span class="sxs-lookup"><span data-stu-id="38ee9-112">The create sequence and create sequence response message schemas are different.</span></span>  
  
- <span data-ttu-id="38ee9-113">Simpleks dizileri için, istemci gönderdiği sıralı iletiler, istemci bu iletileri anlamadığından bu iletiyi yeniden oynamayabilir.</span><span class="sxs-lookup"><span data-stu-id="38ee9-113">For simplex sequences, sequence messages the client sends cannot be replayed back to it because the client cannot understand such messages.</span></span>  
  
- <span data-ttu-id="38ee9-114">Çift yönlü diziler için, iki sıra kimliği benzersiz olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="38ee9-114">For duplex sequences, the two sequence IDs must be unique.</span></span> <span data-ttu-id="38ee9-115">Bu nedenle, bir giden sırası iletisi gelen sıralı ileti olarak yeniden oynatılamaz (tüm sıralı üstbilgiler ve gövdeler, çok fazla).</span><span class="sxs-lookup"><span data-stu-id="38ee9-115">Thus, an outgoing sequence message cannot be replayed back as an incoming sequence message (all sequence headers and bodies are signed, too).</span></span>  
  
 <span data-ttu-id="38ee9-116">Yansıma saldırılarına açık olan tek bağlamalar, WS-Addressing olmayan ve WS-Addressing devre dışı olan özel bağlamalardır ve simetrik anahtar tabanlı güvenliği kullanır.</span><span class="sxs-lookup"><span data-stu-id="38ee9-116">The only bindings that are susceptible to reflection attacks are those without WS-Addressing: custom bindings that have WS-Addressing disabled and use the symmetric key-based security.</span></span> <span data-ttu-id="38ee9-117"><xref:System.ServiceModel.BasicHttpBinding>Varsayılan olarak WS-Addressing kullanmaz, ancak simetrik anahtar tabanlı güvenliği bu saldırıya karşı savunmasız çalışmasına izin verecek şekilde kullanmaz.</span><span class="sxs-lookup"><span data-stu-id="38ee9-117">The <xref:System.ServiceModel.BasicHttpBinding> does not use WS-Addressing by default, but it does not use symmetric key-based security in a way that allows it to be vulnerable to this attack.</span></span>  
  
 <span data-ttu-id="38ee9-118">Özel bağlamalar için azaltma, güvenlik bağlamı kurmamalıdır veya WS-Addressing üst bilgileri gerektirmemelidir.</span><span class="sxs-lookup"><span data-stu-id="38ee9-118">The mitigation for custom bindings is to not establish security context or to require WS-Addressing headers.</span></span>  
  
## <a name="web-farm-attacker-replays-request-to-multiple-nodes"></a><span data-ttu-id="38ee9-119">Web grubu: saldırgan birden çok düğüme Isteği yeniden yürütür</span><span class="sxs-lookup"><span data-stu-id="38ee9-119">Web Farm: Attacker Replays Request to Multiple Nodes</span></span>  

 <span data-ttu-id="38ee9-120">İstemci bir Web çiftliğinde uygulanan bir hizmeti kullanır.</span><span class="sxs-lookup"><span data-stu-id="38ee9-120">A client uses a service that is implemented on a Web farm.</span></span> <span data-ttu-id="38ee9-121">Bir saldırgan gruptaki bir düğüme gönderilen isteği gruptaki başka bir düğüme yeniden yürütür.</span><span class="sxs-lookup"><span data-stu-id="38ee9-121">An attacker replays a request that was sent to one node in the farm to another node in the farm.</span></span> <span data-ttu-id="38ee9-122">Ayrıca, bir hizmet yeniden başlatılırsa, yeniden yürütme önbelleği temizlenir ve bu da bir saldırganın isteği yeniden oynamalarına olanak tanır.</span><span class="sxs-lookup"><span data-stu-id="38ee9-122">In addition, if a service is restarted, the replay cache is flushed, allowing an attacker to replay the request.</span></span> <span data-ttu-id="38ee9-123">(Önbellek kullanılan, daha önce görülen ileti imzası değerlerini içerir ve bu imzaların yalnızca bir kez kullanılabilmesi için yeniden oynatılır.</span><span class="sxs-lookup"><span data-stu-id="38ee9-123">(The cache contains used, previously seen message signature values and prevents replays so those signatures can be used only once.</span></span> <span data-ttu-id="38ee9-124">Yeniden yürütme önbellekleri bir Web grubu genelinde paylaşılmaz.)</span><span class="sxs-lookup"><span data-stu-id="38ee9-124">Replay caches are not shared across a Web farm.)</span></span>  
  
 <span data-ttu-id="38ee9-125">Azaltmaları şunları içerir:</span><span class="sxs-lookup"><span data-stu-id="38ee9-125">Mitigations include:</span></span>

[nasıl yapılır: güvenli bir oturum Için güvenlik bağlamı belirteci oluşturma](how-to-create-a-security-context-token-for-a-secure-session.md).</span><span class="sxs-lookup"><span data-stu-id="38ee9-127">For more information, see [How to: Create a Security Context Token for a Secure Session](how-to-create-a-security-context-token-for-a-secure-session.md).</span></span>  
  
- <span data-ttu-id="38ee9-128">Hizmeti, aktarım düzeyi güvenliği kullanacak şekilde yapılandırın.</span><span class="sxs-lookup"><span data-stu-id="38ee9-128">Configure the service to use transport-level security.</span></span>
