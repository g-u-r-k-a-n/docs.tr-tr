# <a name="compiler-error-cs1656"></a><span data-ttu-id="1ba84-102">Derleyici Hatası CS1656</span><span class="sxs-lookup"><span data-stu-id="1ba84-102">Compiler Error CS1656</span></span>
<span data-ttu-id="1ba84-103">'Salt okunur değişken türü' olduğu için 'değişkene' atayamaz</span><span class="sxs-lookup"><span data-stu-id="1ba84-103">Cannot assign to 'variable' because it is a 'read-only variable type'</span></span>  
  
 <span data-ttu-id="1ba84-104">Bu hata, bir atama için bir atama salt okunur bağlamında oluştuğunda oluşur.</span><span class="sxs-lookup"><span data-stu-id="1ba84-104">This error occurs when an assignment to variable occurs in a read-only context.</span></span> <span data-ttu-id="1ba84-105">Salt okunur bağlamlar, değişkenleri [kullanarak](../keywords/using-statement.md) [her yineleme](../keywords/foreach-in.md) değişkenini ve [sabit](../keywords/fixed-statement.md) değişkenleri içerir.</span><span class="sxs-lookup"><span data-stu-id="1ba84-105">Read-only contexts include [foreach](../keywords/foreach-in.md) iteration variables, [using](../keywords/using-statement.md) variables, and [fixed](../keywords/fixed-statement.md) variables.</span></span> <span data-ttu-id="1ba84-106">Bu hatayı gidermek için, bloklar, `using` `foreach` deyimler ve `fixed` deyimler içinde bir deyim değişkenine atamalar kaçının.</span><span class="sxs-lookup"><span data-stu-id="1ba84-106">To resolve this error, avoid assignments to a statement variable in `using` blocks, `foreach` statements, and `fixed` statements.</span></span>  
  
## <a name="example"></a><span data-ttu-id="1ba84-107">Örnek</span><span class="sxs-lookup"><span data-stu-id="1ba84-107">Example</span></span>  
 <span data-ttu-id="1ba84-108">Aşağıdaki örnek, bir `foreach` döngü içinde bir koleksiyonun tüm öğelerini değiştirmeye çalıştığından CS1656 hatası oluşturur.</span><span class="sxs-lookup"><span data-stu-id="1ba84-108">The following example generates error CS1656 because it tries to replace complete elements of a collection inside a `foreach` loop.</span></span> <span data-ttu-id="1ba84-109">Hatayı aşmanın bir yolu, `foreach` döngüyü [for](../keywords/for.md) döngüsüne değiştirmektir.</span><span class="sxs-lookup"><span data-stu-id="1ba84-109">One way to work around the error is to change the `foreach` loop to a [for](../keywords/for.md) loop.</span></span> <span data-ttu-id="1ba84-110">Burada gösterilmeyen başka bir yol da varolan öğenin üyelerini değiştirmektir; bu sınıflar ile mümkündür, ancak yapılarla değil.</span><span class="sxs-lookup"><span data-stu-id="1ba84-110">Another way, not shown here, is to modify the members of the existing element; this is possible with classes, but not with structs.</span></span>  
  
```csharp  
using System;  
using System.Collections;  
using System.Collections.Generic;  
using System.Text;  
  
namespace CS1656_2  
{  
  
    class Book  
    {  
        public string Title;  
        public string Author;  
        public double Price;  
        public Book(string t, string a, double p)  
        {  
            Title=t;  
            Author=a;  
            Price=p;  
  
        }  
    }  
  
    class Program  
    {  
        private List<Book> list;  
        static void Main(string[] args)  
        {  
            Program prog = new Program();  
            prog.list = new List<Book>();  
            prog.list.Add(new Book ("The C# Programming Language",  
                                    "Hejlsberg, Wiltamuth, Golde",  
                                     29.95));  
            prog.list.Add(new Book ("The C++ Programming Language",  
                                    "Stroustrup",  
                                     29.95));  
            prog.list.Add(new Book ("The C Programming Language",  
                                    "Kernighan, Ritchie",  
                                    29.95));  
            foreach(Book b in prog.list)  
            {  
                // Cannot modify an entire element in a foreach loop
                // even with reference types.  
                // Use a for or while loop instead  
                if (b.Title == "The C Programming Language")  
                    // Cannot assign to 'b' because it is a 'foreach
                    // iteration variable'  
                    b = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95); //CS1656  
            }  
  
            //With a for loop you can modify elements  
            //for(int x = 0; x < prog.list.Count; x++)  
            //{  
            //    if(prog.list[x].Title== "The C Programming Language")  
            //        prog.list[x] = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95);  
            //}  
            //foreach(Book b in prog.list)  
            //    Console.WriteLine(b.Title);  
  
        }  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="1ba84-111">Örnek</span><span class="sxs-lookup"><span data-stu-id="1ba84-111">Example</span></span>  
 <span data-ttu-id="1ba84-112">Aşağıdaki örnek, CS1656'nın döngü `foreach` dışında diğer bağlamlarda nasıl oluşturulabileceğini göstermektedir:</span><span class="sxs-lookup"><span data-stu-id="1ba84-112">The following sample demonstrates how CS1656 can be generated in other contexts besides a `foreach` loop:</span></span>  
  
```csharp  
// CS1656.cs  
// compile with: /unsafe  
using System;  
  
class C : IDisposable  
{  
    public void Dispose() { }  
}  
  
class CMain  
{  
    unsafe public static void Main()  
    {  
        using (C c = new C())  
        {  
            // Cannot assign to 'c' because it is a 'using variable'  
            c = new C(); // CS1656  
        }  
  
        int[] ary = new int[] { 1, 2, 3, 4 };  
        fixed (int* p = ary)  
        {  
            // Cannot assign to 'p' because it is a 'fixed variable'  
            p = null; // CS1656  
        }  
    }  
}  
```
