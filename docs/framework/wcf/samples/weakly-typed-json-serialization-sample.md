<span data-ttu-id="82733-103">Kullanıcı tanımlı bir türü belirli bir tel biçimine seri hale getirdiğinizde veya bir tel biçimini kullanıcı tanımlı bir türe dönüştürdüğünde, verilen kullanıcı tanımlı tür hem hizmette hem de istemcide kullanılabilir olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="82733-103">When serializing a user-defined type to a given wire format, or deserializing a wire format back into a user-defined type, the given user-defined type must be available on both the service and the client.</span></span> <span data-ttu-id="82733-104">Genellikle bunu gerçekleştirmek <xref:System.Runtime.Serialization.DataContractAttribute> için, öznitelik bu kullanıcı tanımlı <xref:System.Runtime.Serialization.DataMemberAttribute> türlere uygulanır ve öznitelik üyelerine uygulanır.</span><span class="sxs-lookup"><span data-stu-id="82733-104">Usually to accomplish this, the <xref:System.Runtime.Serialization.DataContractAttribute> attribute is applied to these user-defined types and the <xref:System.Runtime.Serialization.DataMemberAttribute> attribute is applied to their members.</span></span> <span data-ttu-id="82733-105">Bu mekanizma, JavaScript Object Notation (JSON) nesneleri ile çalışırken de geçerlidir, konu nasıl açıklandığı [gibi: Serialize ve Deserialize JSON Verileri](../../../../docs/framework/wcf/feature-details/how-to-serialize-and-deserialize-json-data.md).</span><span class="sxs-lookup"><span data-stu-id="82733-105">This mechanism also applies when working with JavaScript Object Notation (JSON) objects, as described in the topic [How to: Serialize and Deserialize JSON Data](../../../../docs/framework/wcf/feature-details/how-to-serialize-and-deserialize-json-data.md).</span></span>  
  
 <span data-ttu-id="82733-106">Bazı senaryolarda, bir Windows Communication Foundation (WCF) hizmeti veya istemci, geliştiricinin denetimi dışında bir hizmet veya istemci tarafından oluşturulan JSON nesnelerine erişmelidir.</span><span class="sxs-lookup"><span data-stu-id="82733-106">In some scenarios, a Windows Communication Foundation (WCF) service or client must access JSON objects generated by a service or client that is outside of the control of the developer.</span></span> <span data-ttu-id="82733-107">Daha fazla Web hizmeti JSON API'lerini genel olarak ortaya çıkarırken, WCF geliştiricisinin rasgele JSON nesnelerini deserialize etmek için yerel kullanıcı tanımlı türleri oluşturması pratik hale gelebilir.</span><span class="sxs-lookup"><span data-stu-id="82733-107">As more Web services publicly expose JSON APIs, it can become impractical for the WCF developer to construct local user-defined types into which to deserialize arbitrary JSON objects.</span></span> <span data-ttu-id="82733-108">Bu örnek, WCF geliştiricilerinin kullanıcı tanımlı türler oluşturmadan deserialized, rasgele JSON nesneleri ile çalışmasını sağlayan bir mekanizma sağlar.</span><span class="sxs-lookup"><span data-stu-id="82733-108">This sample provides a mechanism that enables WCF developers to work with deserialized, arbitrary JSON objects, without creating user-defined types.</span></span> <span data-ttu-id="82733-109">JSON nesnesinin seri sini olarak deserialize ettiği tür derleme zamanında bilinmedığından, bu JSON nesnelerinin *zayıf yazılı serileştirmesi* olarak bilinir.</span><span class="sxs-lookup"><span data-stu-id="82733-109">This is known as *weakly-typed serialization* of JSON objects, because the type into which a JSON object deserializes is not known at compile time.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="82733-110">Bu örnek için kurulum yordamı ve yapı yönergeleri bu konunun sonunda yer alır.</span><span class="sxs-lookup"><span data-stu-id="82733-110">The setup procedure and build instructions for this sample are located at the end of this topic.</span></span>  
  
 <span data-ttu-id="82733-111">Örneğin, genel web hizmeti API'si, hizmetin kullanıcısı hakkında bazı bilgileri açıklayan aşağıdaki JSON nesnesini döndürür.</span><span class="sxs-lookup"><span data-stu-id="82733-111">For example, a public Web service API returns the following JSON object, which describes some information about a user of the service.</span></span>  
  
```json  
{"personal": {"name": "Paul", "age": 23, "height": 1.7, "isSingle": true, "luckyNumbers": [5,17,21]}, "favoriteBands": ["Band ABC", "Band XYZ"]}  
```  
  
 <span data-ttu-id="82733-112">Bu nesneyi deserialize etmek için, bir WCF istemcisi aşağıdaki kullanıcı tanımlı türleri uygulamak gerekir.</span><span class="sxs-lookup"><span data-stu-id="82733-112">To deserialize this object, a WCF client must implement the following user-defined types.</span></span>  
  
```csharp  
[DataContract]  
 public class MemberProfile  
 {  
     [DataMember]  
     public PersonalInfo personal;  
  
     [DataMember]  
     public string[] favoriteBands;  
 }  
  
 [DataContract]  
 public class PersonalInfo  
 {  
     [DataMember]  
     public string name;  
  
     [DataMember]  
     public int age;  
  
     [DataMember]  
     public double height;  
  
     [DataMember]  
     public bool isSingle;  
  
     [DataMember]  
     public int[] luckyNumbers;  
 }  
```  
  
 <span data-ttu-id="82733-113">Bu hantal olabilir, özellikle istemci JSON nesnesinin birden fazla tür işlemek zorunda.</span><span class="sxs-lookup"><span data-stu-id="82733-113">This can be cumbersome, especially if the client has to handle more than one type of JSON object.</span></span>  
  
 <span data-ttu-id="82733-114">Bu `JsonObject` örnek tarafından sağlanan tür, deserialized JSON nesnesinin zayıf bir şekilde yazılmış bir temsilini tanıtır.</span><span class="sxs-lookup"><span data-stu-id="82733-114">The `JsonObject` type provided by this sample introduces a weakly-typed representation of the deserialized JSON object.</span></span> <span data-ttu-id="82733-115">`JsonObject`JSON nesneleri ve .NET Framework sözlükleri arasındaki doğal eşleme ile JSON dizileri ile .NET Framework dizileri arasındaki eşleme temeline dayanır.</span><span class="sxs-lookup"><span data-stu-id="82733-115">`JsonObject` relies on the natural mapping between JSON objects and .NET Framework dictionaries, and the mapping between JSON arrays and .NET Framework arrays.</span></span> <span data-ttu-id="82733-116">Aşağıdaki kod `JsonObject` türünü gösterir.</span><span class="sxs-lookup"><span data-stu-id="82733-116">The following code shows the `JsonObject` type.</span></span>  
  
```csharp  
// Instantiation of JsonObject json omitted  
  
string name = json["root"]["personal"]["name"];  
int age = json["root"]["personal"]["age"];  
double height = json["root"]["personal"]["height"];  
bool isSingle = json["root"]["personal"]["isSingle"];  
int[] luckyNumbers = {  
                                     json["root"]["personal"]["luckyNumbers"][0],  
                                     json["root"]["personal"]["luckyNumbers"][1],  
                                     json["root"]["personal"]["luckyNumbers"][2]
                                 };  
string[] favoriteBands = {  
                                        json["root"]["favoriteBands"][0],  
                                        json["root"]["favoriteBands"][1]  
                                    };  
```  
  
 <span data-ttu-id="82733-117">JSON nesnelerine ve dizilerine derleme zamanında kendi türünü bildirmeye gerek kalmadan "göz atabileceğinizi" unutmayın.</span><span class="sxs-lookup"><span data-stu-id="82733-117">Note that you can "browse" JSON objects and arrays without the need to declare their type at compile time.</span></span> <span data-ttu-id="82733-118">Üst düzey `["root"]` nesne gereksiniminin açıklaması için [JSON ve XML arasındaki](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md)konu Eşleme bölümüne bakın.</span><span class="sxs-lookup"><span data-stu-id="82733-118">For an explanation of the requirement for the top-level `["root"]` object, see the topic [Mapping Between JSON and XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span></span>  
  
> [!NOTE]
> <span data-ttu-id="82733-119">Sınıf `JsonObject` yalnızca örnek olarak sağlanır.</span><span class="sxs-lookup"><span data-stu-id="82733-119">The `JsonObject` class is provided as an example only.</span></span> <span data-ttu-id="82733-120">Ayrıntılı olarak test edilmemiştir ve üretim ortamlarında kullanılmamalıdır.</span><span class="sxs-lookup"><span data-stu-id="82733-120">It has not been thoroughly tested, and should not be used in production environments.</span></span> <span data-ttu-id="82733-121">Zayıf yazılı JSON serileştirme bariz bir sonucu ile `JsonObject`çalışırken tip güvenliği eksikliğidir.</span><span class="sxs-lookup"><span data-stu-id="82733-121">An obvious implication of weakly-typed JSON serialization is the lack of type-safety when working with `JsonObject`.</span></span>  
  
 <span data-ttu-id="82733-122">`JsonObject` Türü kullanmak için istemci işlem sözleşmesinin iade türü olarak kullanılması <xref:System.ServiceModel.Channels.Message> gerekir.</span><span class="sxs-lookup"><span data-stu-id="82733-122">To use the `JsonObject` type, the client operation contract must use <xref:System.ServiceModel.Channels.Message> as its return type.</span></span>  
  
```csharp  
[ServiceContract]  
    interface IClientSideProfileService  
    {  
        // There is no need to write a DataContract for the complex type returned by the service.  
        // The client will use a JsonObject to browse the JSON in the received message.  
  
        [OperationContract]  
        [WebGet(ResponseFormat = WebMessageFormat.Json)]  
        Message GetMemberProfile();  
    }  
```  
  
 <span data-ttu-id="82733-123">Daha `JsonObject` sonra aşağıdaki kodda gösterildiği gibi anında.</span><span class="sxs-lookup"><span data-stu-id="82733-123">The `JsonObject` is then instantiated as shown in the following code.</span></span>  
  
```csharp  
// Code to instantiate IClientSideProfileService channel omitted…  
  
// Make a request to the service and obtain the Json response  
XmlDictionaryReader reader = channel.GetMemberProfile().GetReaderAtBodyContents();  
  
// Go through the Json as though it is a dictionary. There is no need to map it to a .NET CLR type.  
JsonObject json = new JsonObject(reader);  
```  
  
 <span data-ttu-id="82733-124">Yapıcı, `JsonObject` <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A> yöntemle <xref:System.Xml.XmlDictionaryReader>elde edilen bir , alır.</span><span class="sxs-lookup"><span data-stu-id="82733-124">The `JsonObject` constructor takes a <xref:System.Xml.XmlDictionaryReader>, which is obtained through the <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A> method.</span></span> <span data-ttu-id="82733-125">Okuyucu, istemci tarafından alınan JSON iletisinin XML temsilini içerir.</span><span class="sxs-lookup"><span data-stu-id="82733-125">The reader contains an XML representation of the JSON message received by the client.</span></span> <span data-ttu-id="82733-126">Daha fazla bilgi için [JSON ve XML arasındaki](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md)konu Eşleme'ye bakın.</span><span class="sxs-lookup"><span data-stu-id="82733-126">For more information, see the topic [Mapping Between JSON and XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span></span>
