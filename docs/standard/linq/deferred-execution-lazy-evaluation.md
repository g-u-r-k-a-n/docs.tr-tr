## <a name="deferred-execution"></a><span data-ttu-id="d20b3-106">Ertelenmiş yürütme</span><span class="sxs-lookup"><span data-stu-id="d20b3-106">Deferred execution</span></span>

<span data-ttu-id="d20b3-107">Ertelenmiş yürütme, *İstenen değer gerçekten* gerekli olana kadar bir ifadenin değerlendirmesinin gecikildiği anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="d20b3-107">Deferred execution means that the evaluation of an expression is delayed until its *realized* value is actually required.</span></span> <span data-ttu-id="d20b3-108">Ertelenmiş yürütme, özellikle de bir dizi zincirleme sorgu veya işleme içeren programlarda büyük veri koleksiyonlarını işlemek gerektiğinde performansı önemli ölçüde iyileştirebilir.</span><span class="sxs-lookup"><span data-stu-id="d20b3-108">Deferred execution can greatly improve performance when you have to manipulate large data collections, especially in programs that contain a series of chained queries or manipulations.</span></span> <span data-ttu-id="d20b3-109">En iyi durumda ertelenmiş yürütme, kaynak koleksiyon aracılığıyla yalnızca tek bir yinelemeyi mümkün bir şekilde sunar.</span><span class="sxs-lookup"><span data-stu-id="d20b3-109">In the best case, deferred execution enables only a single iteration through the source collection.</span></span>

<span data-ttu-id="d20b3-110">LINQ teknolojileri, hem çekirdek sınıfların üyelerinde hem de gibi <xref:System.Linq?displayProperty=nameWithType> ÇEŞITLI LINQ ad alanlarında uzantı yöntemlerinde ertelenmiş yürütmenin yoğun bir şekilde kullanılmasını kolaylaştırır <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType> .</span><span class="sxs-lookup"><span data-stu-id="d20b3-110">The LINQ technologies make extensive use of deferred execution in both the members of core <xref:System.Linq?displayProperty=nameWithType> classes and in the extension methods in the various LINQ namespaces, such as <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span></span>

<span data-ttu-id="d20b3-111">Ertelenmiş yürütme, bir yineleyici bloğunda kullanılan [yield (C# Reference)](../../csharp/language-reference/keywords/yield.md) anahtar sözcüğü (deyimin biçiminde) tarafından doğrudan C# dilinde desteklenir `yield-return` .</span><span class="sxs-lookup"><span data-stu-id="d20b3-111">Deferred execution is supported directly in the C# language by the [yield (C# Reference)](../../csharp/language-reference/keywords/yield.md) keyword (in the form of the `yield-return` statement) when used within an iterator block.</span></span> <span data-ttu-id="d20b3-112">Bu tür bir yineleyici, <xref:System.Collections.IEnumerator> ya da <xref:System.Collections.Generic.IEnumerator%601> (ya da türetilmiş bir tür) bir koleksiyon döndürmelidir.</span><span class="sxs-lookup"><span data-stu-id="d20b3-112">Such an iterator must return a collection of type <xref:System.Collections.IEnumerator> or <xref:System.Collections.Generic.IEnumerator%601> (or a derived type).</span></span>

## <a name="eager-vs-lazy-evaluation"></a><span data-ttu-id="d20b3-113">Eager ve geç değerlendirme karşılaştırması</span><span class="sxs-lookup"><span data-stu-id="d20b3-113">Eager vs. lazy evaluation</span></span>

<span data-ttu-id="d20b3-114">Ertelenmiş yürütmeyi uygulayan bir yöntem yazdığınızda, yavaş değerlendirme veya Eager değerlendirmesi kullanarak yöntemi uygulayıp uygulamamaya de karar vermeniz gerekir.</span><span class="sxs-lookup"><span data-stu-id="d20b3-114">When you write a method that implements deferred execution, you also have to decide whether to implement the method using lazy evaluation or eager evaluation.</span></span>

- <span data-ttu-id="d20b3-115">*Yavaş değerlendirmede*, her Yineleyici çağrısı sırasında kaynak koleksiyonun tek bir öğesi işlenir.</span><span class="sxs-lookup"><span data-stu-id="d20b3-115">In *lazy evaluation*, a single element of the source collection is processed during each call to the iterator.</span></span> <span data-ttu-id="d20b3-116">Yineleyicilerin uygulanma tipik yoludur.</span><span class="sxs-lookup"><span data-stu-id="d20b3-116">This is the typical way in which iterators are implemented.</span></span>
- <span data-ttu-id="d20b3-117">*Eager değerlendirmesi*sırasında, Yineleyici için ilk çağrı, tüm koleksiyonun işlenmesine neden olur.</span><span class="sxs-lookup"><span data-stu-id="d20b3-117">In *eager evaluation*, the first call to the iterator will result in the entire collection being processed.</span></span> <span data-ttu-id="d20b3-118">Kaynak koleksiyonun geçici bir kopyası da gerekebilir.</span><span class="sxs-lookup"><span data-stu-id="d20b3-118">A temporary copy of the source collection might also be required.</span></span> <span data-ttu-id="d20b3-119">Örneğin, <xref:System.Linq.Enumerable.OrderBy%2A> yöntemi ilk öğeyi döndürmadan önce tüm koleksiyonu sıralamak zorunda olur.</span><span class="sxs-lookup"><span data-stu-id="d20b3-119">For example, the <xref:System.Linq.Enumerable.OrderBy%2A> method has to sort the entire collection before it returns the first element.</span></span>

<span data-ttu-id="d20b3-120">Yavaş değerlendirme genellikle koleksiyonun değerlendirmesi boyunca ek yük işlemeyi eşit bir şekilde dağıttığı ve geçici verilerin kullanımını en aza indirecek için daha iyi performans verir.</span><span class="sxs-lookup"><span data-stu-id="d20b3-120">Lazy evaluation usually yields better performance because it distributes overhead processing evenly throughout the evaluation of the collection and minimizes the use of temporary data.</span></span> <span data-ttu-id="d20b3-121">Tabii ki, bazı işlemler için, ara sonuçları gerçekleştirme dışında başka bir seçenek yoktur.</span><span class="sxs-lookup"><span data-stu-id="d20b3-121">Of course, for some operations, there is no other option than to materialize intermediate results.</span></span>
