# <a name="version-compatibility"></a><span data-ttu-id="f2a87-102">Sürüm uyumluluğu</span><span class="sxs-lookup"><span data-stu-id="f2a87-102">Version compatibility</span></span>

<span data-ttu-id="f2a87-103">Geriye dönük uyumluluk, bir platformun belirli bir sürümü için geliştirilen bir uygulamanın bu platformun sonraki sürümlerinde çalışacağı anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="f2a87-103">Backward compatibility means that an app that was developed for a particular version of a platform will run on later versions of that platform.</span></span> <span data-ttu-id="f2a87-104">.NET Framework geriye dönük uyumluluğu en üst düzeye çıkarmaya çalışır: .NET Framework'ün bir sürümü için yazılan kaynak kodu .NET Framework'ün sonraki sürümlerinde derlenmelidir ve .NET Framework'ün bir sürümünde çalışan ikililer .NET Framework'ün sonraki sürümlerinde aynı şekilde kullanılmalıdır.</span><span class="sxs-lookup"><span data-stu-id="f2a87-104">.NET Framework tries to maximize backward compatibility: Source code written for one version of .NET Framework should compile on later versions of .NET Framework, and binaries that run on one version of .NET Framework should behave identically on later versions of .NET Framework.</span></span>

## <a name="version-compatibility-for-apps"></a><a name="Apps"></a><span data-ttu-id="f2a87-105">Uygulamalar için sürüm uyumluluğu</span><span class="sxs-lookup"><span data-stu-id="f2a87-105">Version compatibility for apps</span></span>

<span data-ttu-id="f2a87-106">Varsayılan olarak, bir uygulama .NET Framework sürümünde çalışır.</span><span class="sxs-lookup"><span data-stu-id="f2a87-106">By default, an app runs on the version of .NET Framework that it was built for.</span></span> <span data-ttu-id="f2a87-107">Bu sürüm yoksa ve uygulama yapılandırma dosyası desteklenen sürümleri tanımlamıyorsa, .NET Framework başlatma hatası oluşabilir.</span><span class="sxs-lookup"><span data-stu-id="f2a87-107">If that version isn't present and the app configuration file doesn't define supported versions, a .NET Framework initialization error may occur.</span></span> <span data-ttu-id="f2a87-108">Bu durumda, uygulamayı çalıştırma girişimi başarısız olur.</span><span class="sxs-lookup"><span data-stu-id="f2a87-108">In this case, the attempt to run the app will fail.</span></span>

<span data-ttu-id="f2a87-109">Uygulamanızın çalıştığı belirli sürümleri tanımlamak için, uygulamanızın yapılandırma dosyasına bir veya daha fazla [ \<desteklenen Runtime>](../configure-apps/file-schema/startup/supportedruntime-element.md) öğesi ekleyin.</span><span class="sxs-lookup"><span data-stu-id="f2a87-109">To define the specific versions on which your app runs, add one or more [\<supportedRuntime>](../configure-apps/file-schema/startup/supportedruntime-element.md) elements to your app's configuration file.</span></span> <span data-ttu-id="f2a87-110">Her `<supportedRuntime>` öğe, ilk en çok tercih edilen sürümü ve en son en az tercih edilen sürümü belirten ile çalışma zamanıdesteklenen bir sürümünü listeler.</span><span class="sxs-lookup"><span data-stu-id="f2a87-110">Each `<supportedRuntime>` element lists a supported version of the runtime, with the first specifying the most preferred version and the last specifying the least preferred version.</span></span>

```xml

   <startup>
      <supportedRuntime version="v2.0.50727" />
      <supportedRuntime version="v4.0" />
   </startup>
</configuration>
```

<span data-ttu-id="f2a87-111">Daha fazla bilgi için [bkz: Bir Uygulamayı Destekleyecek şekilde yapılandırın .NET Framework 4 veya 4.x](../migration-guide/how-to-configure-an-app-to-support-net-framework-4-or-4-5.md).</span><span class="sxs-lookup"><span data-stu-id="f2a87-111">For more information, see [How to: Configure an App to Support .NET Framework 4 or 4.x](../migration-guide/how-to-configure-an-app-to-support-net-framework-4-or-4-5.md).</span></span>

## <a name="version-compatibility-for-components"></a><span data-ttu-id="f2a87-112">Bileşenler için sürüm uyumluluğu</span><span class="sxs-lookup"><span data-stu-id="f2a87-112">Version compatibility for components</span></span>

<span data-ttu-id="f2a87-113">Bir uygulama çalıştığı .NET Framework sürümünü denetleyebilir, ancak bileşen denetemez.</span><span class="sxs-lookup"><span data-stu-id="f2a87-113">An app can control the version of the .NET Framework on which it runs, but a component can't.</span></span> <span data-ttu-id="f2a87-114">Bileşenler ve sınıf kitaplıkları belirli bir uygulama bağlamında yüklenir ve bu nedenle uygulamanın çalıştığı .NET Framework sürümünde otomatik olarak çalıştırılırlar.</span><span class="sxs-lookup"><span data-stu-id="f2a87-114">Components and class libraries are loaded in the context of a particular app, and that's why they automatically run on the version of the .NET Framework that the app runs on.</span></span>

## <a name="backward-compatibility"></a><span data-ttu-id="f2a87-118">Geriye dönük uyumluluk</span><span class="sxs-lookup"><span data-stu-id="f2a87-118">Backward compatibility</span></span>

<span data-ttu-id="f2a87-119">.NET Framework 4.5 ve sonraki sürümler,.NET Framework'ün önceki sürümleriyle oluşturulmuş uygulamalarla geriye dönük uyumludur.</span><span class="sxs-lookup"><span data-stu-id="f2a87-119">The .NET Framework 4.5 and later versions are backward-compatible with apps that were built with earlier versions of the .NET Framework.</span></span> <span data-ttu-id="f2a87-120">Başka bir deyişle, önceki sürümlerle oluşturulmuş uygulamalar ve bileşenler .NET Framework 4.5 ve sonraki sürümlerde değişiklik yapılmadan çalışır.</span><span class="sxs-lookup"><span data-stu-id="f2a87-120">In other words, apps and components built with previous versions will work without modification on the .NET Framework 4.5 and later versions.</span></span> <span data-ttu-id="f2a87-121">Ancak, varsayılan olarak, uygulamalar geliştirildiği ortak dil çalışma zamanı sürümünde çalışır, bu nedenle uygulamanızın .NET Framework 4.5 veya sonraki sürümlerinde çalışmasını sağlamak için bir yapılandırma dosyası sağlamanız gerekebilir.</span><span class="sxs-lookup"><span data-stu-id="f2a87-121">However, by default, apps run on the version of the common language runtime for which they were developed, so you may have to provide a configuration file to enable your app to run on the .NET Framework 4.5 or later versions.</span></span> <span data-ttu-id="f2a87-122">Daha fazla bilgi için, bu makalenin başlarında uygulamalar için [Sürüm uyumluluğu](#Apps) bölümüne bakın.</span><span class="sxs-lookup"><span data-stu-id="f2a87-122">For more information, see the [Version compatibility for apps](#Apps) section earlier in this article.</span></span>

<span data-ttu-id="f2a87-123">Uygulamada, bu uyumluluk .NET Framework'deki önemsiz gibi görünen değişiklikler ve programlama tekniklerinde yapılan değişikliklerle bozulabilir.</span><span class="sxs-lookup"><span data-stu-id="f2a87-123">In practice, this compatibility can be broken by seemingly inconsequential changes in the .NET Framework and changes in programming techniques.</span></span> <span data-ttu-id="f2a87-124">Örneğin, .NET Framework 4.5'teki performans iyileştirmeleri, önceki sürümlerde oluşmayan bir yarış koşulunu ortaya çıkarabilir.</span><span class="sxs-lookup"><span data-stu-id="f2a87-124">For example, performance improvements in the .NET Framework 4.5 can expose a race condition that did not occur on earlier versions.</span></span> <span data-ttu-id="f2a87-125">Benzer şekilde, .NET Framework derlemelerine sabit kodlanmış bir yol kullanmak, .NET Framework'ün belirli bir sürümüyle eşitlik karşılaştırması yapmak ve yansımayı kullanarak özel bir alanın değerini almak geriye doğru uyumlu uygulamalar değildir.</span><span class="sxs-lookup"><span data-stu-id="f2a87-125">Similarly, using a hard-coded path to .NET Framework assemblies, performing an equality comparison with a particular version of the .NET Framework, and getting the value of a private field by using reflection are not backward-compatible practices.</span></span> <span data-ttu-id="f2a87-126">Buna ek olarak, .NET Framework'ün her sürümü, hata düzeltmeleri ve bazı uygulamaların ve bileşenlerin uyumluluğunu etkileyebilecek güvenlikle ilgili değişiklikleri içerir.</span><span class="sxs-lookup"><span data-stu-id="f2a87-126">In addition, each version of the .NET Framework includes bug fixes and security-related changes that can affect the compatibility of some apps and components.</span></span>
