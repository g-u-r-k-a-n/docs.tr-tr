# <a name="host-lock-renewal-period"></a><span data-ttu-id="6368f-103">Konak Kilidi Yenileme Dönemi</span><span class="sxs-lookup"><span data-stu-id="6368f-103">Host Lock Renewal Period</span></span>

<span data-ttu-id="6368f-104">SQL Iş akışı örneği deposunun **konak kilit yenileme süresi** özelliği, konağın bir iş akışı örneği üzerinde kilidini yenilediğini belirten süreyi belirtmenize olanak tanır.</span><span class="sxs-lookup"><span data-stu-id="6368f-104">The **Host Lock Renewal Period** property of the SQL Workflow Instance Store lets you specify the time period within which the host renews its lock on a workflow instance.</span></span> <span data-ttu-id="6368f-105">Kilit, ana bilgisayar kilidi yenileme süresi + 30 saniye için geçerli kalır.</span><span class="sxs-lookup"><span data-stu-id="6368f-105">The lock remains valid for Host Lock Renewal Period + 30 seconds.</span></span> <span data-ttu-id="6368f-106">Ana bilgisayar kilidi yenilemezse (diğer bir deyişle, kirayı bu zaman aralığında uzatırsa), kilidin süresi dolar ve kalıcılık sağlayıcısı örneğin kilidini açar.</span><span class="sxs-lookup"><span data-stu-id="6368f-106">If the host fails to renew the lock (in other words, extend the lease) within this time period, the lock expires and the persistence provider unlocks the instance.</span></span> <span data-ttu-id="6368f-107">Bu özelliğin değeri "hh: mm: ss" biçiminde TimeSpan türündedir.</span><span class="sxs-lookup"><span data-stu-id="6368f-107">The value for this property is of type TimeSpan of the form "hh:mm:ss".</span></span> <span data-ttu-id="6368f-108">İzin verilen en düşük değer "00:00:01" değeridir (1 saniye).</span><span class="sxs-lookup"><span data-stu-id="6368f-108">The minimum permitted value is "00:00:01" (1 second).</span></span> <span data-ttu-id="6368f-109">Bu özelliğin varsayılan değeri "00:00:30" (30 saniye) değeridir.</span><span class="sxs-lookup"><span data-stu-id="6368f-109">The default value of this property is "00:00:30" (30 seconds).</span></span>  
  
 <span data-ttu-id="6368f-110">Bu özellik, bir iş akışı hizmeti konağının, sahip olduğu bir iş akışı hizmeti örneğinin kilidini açmasından önce başarısız olduğu senaryolarda önemli bir özelliktir.</span><span class="sxs-lookup"><span data-stu-id="6368f-110">This property is significant in scenarios where a workflow service host fails before it can unlock a workflow service instance that it owns.</span></span> <span data-ttu-id="6368f-111">Bu senaryoda, aynı bilgisayar üzerinde çalışan başka bir iş akışı hizmeti konağının veya bir sunucu grubundaki başka bir bilgisayar kilidi alabilir ve iş akışı hizmet örneğini en son kalıcı durumundan devam ettirmeye devam etmek için belleğe yükleyebilir, bu durumda, kalıcılık veritabanındaki iş akışı hizmeti örneğindeki kilit, kilitlenme sağlayıcısı tarafından kaldırılır.</span><span class="sxs-lookup"><span data-stu-id="6368f-111">In this scenario, the lock on the workflow service instance in the persistence database is removed by the persistence provider after the lock expires so that another workflow service host running on the same computer or another computer in a server farm can acquire the lock and load the workflow service instance into memory to resume its execution from its last persisted state.</span></span>  
  
 <span data-ttu-id="6368f-112">Bu özellik için daha yüksek bir değer ayarlandığında, iş akışı hizmeti örneklerinin Kalıcılık veritabanında daha uzun bir süre kilitlenmesine neden olur ve bu nedenle örneği son Kalıcılık noktasından kurtarmaya gecikir.</span><span class="sxs-lookup"><span data-stu-id="6368f-112">Setting a higher value for this property causes the workflow service instances to be locked in the persistence database for a longer time and therefore delays the recovery of the instance from the last persistence point.</span></span> <span data-ttu-id="6368f-113">Bu özellik için kısa bir Aralık ayarlamak, iş akışı hizmet ana bilgisayarının yeni örneğinin başarısız iş akışı hizmeti örneğini hızla seçmesini sağlar, ancak iş akışı hizmeti ana bilgisayarı ve SQL Server veritabanı için iş yükünde artışa neden olur.</span><span class="sxs-lookup"><span data-stu-id="6368f-113">Setting a short interval for this property causes the new instance of the workflow service host to pick up the failed workflow service instance quickly, but causes an increase in workload for the workflow service host and the SQL Server database.</span></span>  
  
 <span data-ttu-id="6368f-114">SQL Iş akışı örneği deposu, düzenli aralıklarla uyandığı iç bir görev çalıştırır ve bunlar üzerinde zaman aşımına uğradı.</span><span class="sxs-lookup"><span data-stu-id="6368f-114">The SQL Workflow Instance Store runs an internal task that periodically wakes up and detects instances with expired locks on them.</span></span> <span data-ttu-id="6368f-115">Süre dolma kilitleri olan örnekleri bulduğunda, örnekleri Runnableınstances tablosuna koyar, böylece iş akışı ana bilgisayarı bu örnekleri alabilir ve çalıştırabilir.</span><span class="sxs-lookup"><span data-stu-id="6368f-115">When it finds instances with expired locks, it places the instances in the RunnableInstances table so that a workflow host can pick up and run these instances.</span></span>
