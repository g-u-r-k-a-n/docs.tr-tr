## <a name="workspace"></a><span data-ttu-id="51bce-108">Çalışma alanı</span><span class="sxs-lookup"><span data-stu-id="51bce-108">Workspace</span></span>

<span data-ttu-id="51bce-109">Çalışma alanı, çözümünüzün her biri belge koleksiyonuna sahip projeler topluluğu olarak etkin bir temsilidir.</span><span class="sxs-lookup"><span data-stu-id="51bce-109">A workspace is an active representation of your solution as a collection of projects, each with a collection of documents.</span></span> <span data-ttu-id="51bce-110">Çalışma alanı genellikle, kullanıcı türleri olarak sürekli değişen veya özellikleri manipüle eden bir ana bilgisayar ortamına bağlıdır.</span><span class="sxs-lookup"><span data-stu-id="51bce-110">A workspace is typically tied to a host environment that is constantly changing as a user types or manipulates properties.</span></span>

<span data-ttu-id="51bce-111">Çözümün <xref:Microsoft.CodeAnalysis.Workspace> geçerli modeline erişim sağlar.</span><span class="sxs-lookup"><span data-stu-id="51bce-111">The <xref:Microsoft.CodeAnalysis.Workspace> provides access to the current model of the solution.</span></span> <span data-ttu-id="51bce-112">Ana bilgisayar ortamında bir değişiklik olduğunda, çalışma alanı ilgili <xref:Microsoft.CodeAnalysis.Workspace.CurrentSolution?displayProperty=nameWithType> olayları çalıştırıyor ve özellik güncelleştirilir.</span><span class="sxs-lookup"><span data-stu-id="51bce-112">When a change in the host environment occurs, the workspace fires corresponding events, and the <xref:Microsoft.CodeAnalysis.Workspace.CurrentSolution?displayProperty=nameWithType> property is updated.</span></span> <span data-ttu-id="51bce-113">Örneğin, kullanıcı kaynak belgelerden birine karşılık gelen bir metin düzenleyicisinde yazdığında, çalışma alanı çözümün genel modelinin değiştiğini ve hangi belgenin değiştirildiğini bildirmek için bir olay kullanır.</span><span class="sxs-lookup"><span data-stu-id="51bce-113">For example, when the user types in a text editor corresponding to one of the source documents, the workspace uses an event to signal that the overall model of the solution has changed and which document was modified.</span></span> <span data-ttu-id="51bce-114">Daha sonra, yeni modeli doğruluk için çözümleyerek, önem alanlarını vurgulayarak veya kod değişikliği için bir öneride bulunarak bu değişikliklere tepki verebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="51bce-114">You can then react to those changes by analyzing the new model for correctness, highlighting areas of significance, or making a suggestion for a code change.</span></span>

<span data-ttu-id="51bce-115">Ayrıca, ana bilgisayar ortamından bağlantısı kesilen veya ana bilgisayar ortamı olmayan bir uygulamada kullanılan bağımsız çalışma alanları da oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="51bce-115">You can also create stand-alone workspaces that are disconnected from the host environment or used in an application that has no host environment.</span></span>

## <a name="solutions-projects-documents"></a><span data-ttu-id="51bce-116">Çözümler, projeler, belgeler</span><span class="sxs-lookup"><span data-stu-id="51bce-116">Solutions, projects, documents</span></span>

<span data-ttu-id="51bce-117">Bir anahtara her basıldığında çalışma alanı değişebilir, ancak çözüm modeliyle birlikte yalıtımda çalışabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="51bce-117">Although a workspace may change every time a key is pressed, you can work with the model of the solution in isolation.</span></span>

<span data-ttu-id="51bce-123">Proje, genel değişmez çözüm modelinin bir parçasıdır.</span><span class="sxs-lookup"><span data-stu-id="51bce-123">A project is a part of the overall immutable solution model.</span></span> <span data-ttu-id="51bce-124">Tüm kaynak kodu belgelerini, ayrıştırma ve derleme seçeneklerini ve hem derlemeyi hem de projeden projeye başvurularını temsil eder.</span><span class="sxs-lookup"><span data-stu-id="51bce-124">It represents all the source code documents, parse and compilation options, and both assembly and project-to-project references.</span></span> <span data-ttu-id="51bce-125">Projeden, proje bağımlılıklarını belirlemenize veya kaynak dosyalarını ayrıştırmaya gerek kalmadan ilgili derlemeye erişebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="51bce-125">From a project, you can access the corresponding compilation without needing to determine project dependencies or parse any source files.</span></span>

<span data-ttu-id="51bce-126">Belge aynı zamanda genel değişmez çözüm modelinin bir parçasıdır.</span><span class="sxs-lookup"><span data-stu-id="51bce-126">A document is also a part of the overall immutable solution model.</span></span> <span data-ttu-id="51bce-127">Belge, dosyanın metnine, sözdizimi ağacına ve anlamsal modele erişebileceğiniz tek bir kaynak dosyayı temsil eder.</span><span class="sxs-lookup"><span data-stu-id="51bce-127">A document represents a single source file from which you can access the text of the file, the syntax tree, and the semantic model.</span></span>
