## <a name="general-guidelines"></a><span data-ttu-id="93d30-169">Genel Yönergeler</span><span class="sxs-lookup"><span data-stu-id="93d30-169">General Guidelines</span></span>

<span data-ttu-id="93d30-170">Bir nesnenin kodunu değiştirerek veya bunu yapmak makul ve mümkün olduğunda yeni bir tür üreterek işlevsellik eklemek hala tercih edilebilir olarak kabul edilirken, uzantı yöntemleri .NET boyunca yeniden kullanılabilir işlevsellik oluşturmak için önemli bir seçenek haline gelmiştir Ekosistem.</span><span class="sxs-lookup"><span data-stu-id="93d30-170">While it's still considered preferable to add functionality by modifying an object's code or deriving a new type whenever it's reasonable and possible to do so, extension methods have become a crucial option for creating reusable functionality throughout the .NET ecosystem.</span></span> <span data-ttu-id="93d30-171">Özgün kaynağın denetimialtında olmadığı, türetilen bir nesnenin uygunsuz veya imkansız olduğu veya işlevselliğin geçerli kapsamının dışında açığa çıkmaması gereken durumlar için Uzantı yöntemleri mükemmel bir seçimdir.</span><span class="sxs-lookup"><span data-stu-id="93d30-171">For those occasions when the original source isn't under your control, when a derived object is inappropriate or impossible, or when the functionality shouldn't be exposed beyond its applicable scope, Extension methods are an excellent choice.</span></span>

<span data-ttu-id="93d30-172">Türetilen türler hakkında daha fazla bilgi için [Kalıtım'a](./inheritance.md)bakın.</span><span class="sxs-lookup"><span data-stu-id="93d30-172">For more information on derived types, see [Inheritance](./inheritance.md).</span></span>

<span data-ttu-id="93d30-173">Kaynak kodu denetiminde olmayan bir türü genişletmek için bir uzantı yöntemi kullanırken, türün uygulanmasındayapılacak bir değişikliğin uzantı yönteminizin kırılmasına neden olacağı riskini taşırsınız.</span><span class="sxs-lookup"><span data-stu-id="93d30-173">When using an extension method to extend a type whose source code you aren't in control of, you run the risk that a change in the implementation of the type will cause your extension method to break.</span></span>

<span data-ttu-id="93d30-174">Belirli bir tür için uzantı yöntemleri uygularsanız, aşağıdaki noktaları unutmayın:</span><span class="sxs-lookup"><span data-stu-id="93d30-174">If you do implement extension methods for a given type, remember the following points:</span></span>

- <span data-ttu-id="93d30-175">Türden tanımlı yöntemle aynı imzaya sahip değilse genişletme yöntemi asla çağrılmaz.</span><span class="sxs-lookup"><span data-stu-id="93d30-175">An extension method will never be called if it has the same signature as a method defined in the type.</span></span>
- <span data-ttu-id="93d30-176">Uzantı yöntemleri ad alanı seviyesinde kapsama alınır.</span><span class="sxs-lookup"><span data-stu-id="93d30-176">Extension methods are brought into scope at the namespace level.</span></span> <span data-ttu-id="93d30-177">Örneğin, adlandırılmış `Extensions`tek bir ad alanında uzantı yöntemleri içeren birden çok statik sınıfınvarsa, `using Extensions;` bunların tümü yönerge tarafından kapsamına getirilir.</span><span class="sxs-lookup"><span data-stu-id="93d30-177">For example, if you have multiple static classes that contain extension methods in a single namespace named `Extensions`, they'll all be brought into scope by the `using Extensions;` directive.</span></span>

<span data-ttu-id="93d30-178">Uygulanan bir sınıf kitaplığı için derleme sürüm numarasının artıyor olmasını önlemek için uzantı yöntemleri kullanmamanız gerekir.</span><span class="sxs-lookup"><span data-stu-id="93d30-178">For a class library that you implemented, you shouldn't use extension methods to avoid incrementing the version number of an assembly.</span></span> <span data-ttu-id="93d30-179">Kaynak koduna sahip olduğunuz kitaplığa önemli işlevsellik eklemek isterseniz, derleme sürüm oluşturma için standart .NET Framework yönergelerini izlemeniz gerekir.</span><span class="sxs-lookup"><span data-stu-id="93d30-179">If you want to add significant functionality to a library for which you own the source code, you should follow the standard .NET Framework guidelines for assembly versioning.</span></span> <span data-ttu-id="93d30-180">Daha fazla bilgi için [Montaj Sürümü'ne](../../../standard/assembly/versioning.md)bakın.</span><span class="sxs-lookup"><span data-stu-id="93d30-180">For more information, see [Assembly Versioning](../../../standard/assembly/versioning.md).</span></span>
