# <a name="entity-references-are-expanded-and-not-preserved"></a><span data-ttu-id="460f6-102">Varlık Başvuruları Genişletilir ve Korunmaz</span><span class="sxs-lookup"><span data-stu-id="460f6-102">Entity References are Expanded and Not Preserved</span></span>

<span data-ttu-id="460f6-103">Varlık başvurusu genişletildiğinde ve gösterdiği metin tarafından değiştirildiğinde, **XmlEntityReference** düğümü oluşturulmaz.</span><span class="sxs-lookup"><span data-stu-id="460f6-103">When the entity reference is expanded and replaced by the text it represents, the **XmlEntityReference** node is not created.</span></span> <span data-ttu-id="460f6-104">Bunun yerine, varlık bildirimi ayrıştırılır ve bildirimdeki içerikten oluşturulan düğümler **XmlEntityReference** yerine kopyalanır.</span><span class="sxs-lookup"><span data-stu-id="460f6-104">Instead, the entity declaration is parsed, and nodes created from the content in the declaration are copied in the place of the **XmlEntityReference**.</span></span> <span data-ttu-id="460f6-105">Bu nedenle, `&publisher;` örnekte `&publisher;` kaydedilmez, ancak bunun yerine **XmlText** düğümü oluşturulur.</span><span class="sxs-lookup"><span data-stu-id="460f6-105">Therefore, in the `&publisher;` example, the `&publisher;` is not saved, but instead, an **XmlText** node is created.</span></span>  
  
 <span data-ttu-id="460f6-106">![Genişletilmiş ağaç yapısı](media/xmlentityref-expanded-nodes.gif "xmlentityref_expanded_nodes")</span><span class="sxs-lookup"><span data-stu-id="460f6-106">![expanded tree structure](media/xmlentityref-expanded-nodes.gif "xmlentityref_expanded_nodes")</span></span>  
<span data-ttu-id="460f6-107">Genişletilmiş varlık başvuruları için ağaç yapısı</span><span class="sxs-lookup"><span data-stu-id="460f6-107">Tree structure for entity references that are expanded</span></span>  
  
 <span data-ttu-id="460f6-108">Veya gibi karakter varlıkları `B` `<` korunmaz.</span><span class="sxs-lookup"><span data-stu-id="460f6-108">Character entities such as `B` or `<` are not preserved.</span></span> <span data-ttu-id="460f6-109">Bunun yerine, her zaman genişletilir ve metin düğümleri olarak gösterilir.</span><span class="sxs-lookup"><span data-stu-id="460f6-109">Instead, they are always expanded and represented as text nodes.</span></span>  
  
 <span data-ttu-id="460f6-110">**XmlEntityReference** düğümlerini ve ona iliştirilmiş Varlık başvurusunun alt düğümlerini korumak Için, **EntityHandling** bayrağını **ExpandCharEntities** olarak ayarlayın.</span><span class="sxs-lookup"><span data-stu-id="460f6-110">To preserve **XmlEntityReference** nodes and child nodes of the entity reference attached to it, set the **EntityHandling** flag to **ExpandCharEntities**.</span></span> <span data-ttu-id="460f6-111">Aksi halde, **EntityHandling** bayrağını varsayılan konumda bırakın, bu, **ExpandEntities**.</span><span class="sxs-lookup"><span data-stu-id="460f6-111">Otherwise, leave the **EntityHandling** flag at the default, which is to **ExpandEntities**.</span></span> <span data-ttu-id="460f6-112">Bu durumda, DOM 'da varlık başvurusu düğümlerini görmezsiniz.</span><span class="sxs-lookup"><span data-stu-id="460f6-112">In this case, you will not see entity reference nodes in the DOM.</span></span> <span data-ttu-id="460f6-113">Düğümler, varlık bildiriminin alt düğümlerinin kopyaları olan düğümlerle değiştirilmiştir.</span><span class="sxs-lookup"><span data-stu-id="460f6-113">The nodes are replaced by the nodes that are copies of the child nodes of the entity declaration.</span></span>  
  
 <span data-ttu-id="460f6-114">Varlık başvurularını korumadan bir yan etkisi, belge kaydedilip başka bir uygulamaya geçirildiğinde, alıcı uygulamanın düğümlerin bir varlık başvurusu tarafından oluşturulduğunu bilmez.</span><span class="sxs-lookup"><span data-stu-id="460f6-114">One side effect of not preserving entity references is that when the document is saved and passed on to another application, the receiving application does not know that the nodes were generated by an entity reference.</span></span> <span data-ttu-id="460f6-115">Ancak, varlık başvuruları korunduğunda, alıcı bir uygulama bir varlık başvurusu görür ve alt düğümleri okur.</span><span class="sxs-lookup"><span data-stu-id="460f6-115">However, when entity references are preserved, a receiving application sees an entity reference and reads the child nodes.</span></span> <span data-ttu-id="460f6-116">Alt düğümlerin varlık bildiriminde bulunan bilgileri temsil ettiği görünür.</span><span class="sxs-lookup"><span data-stu-id="460f6-116">It is apparent that the child nodes represent the information that was in the entity declaration.</span></span> <span data-ttu-id="460f6-117">Örneğin, varlık başvuruları korunsa, DOM teorik olarak aşağıdaki yapıya sahiptir.</span><span class="sxs-lookup"><span data-stu-id="460f6-117">For example, the DOM theoretically has the following structure if entity references are preserved.</span></span>  
  
 <span data-ttu-id="460f6-118">XmlElement: Yayımcı</span><span class="sxs-lookup"><span data-stu-id="460f6-118">XmlElement: publisher</span></span>  
  
 <span data-ttu-id="460f6-119">XmlEntityReference: `&publisher;`</span><span class="sxs-lookup"><span data-stu-id="460f6-119">XmlEntityReference: `&publisher;`</span></span>  
  
 <span data-ttu-id="460f6-120">XmlText: Microsoft Press</span><span class="sxs-lookup"><span data-stu-id="460f6-120">XmlText: Microsoft Press</span></span>  
  
 <span data-ttu-id="460f6-121">Varlık başvuruları DOM 'da genişletilmişse (varsayılan yöntem), yapı bu ağaç türüne sahiptir:</span><span class="sxs-lookup"><span data-stu-id="460f6-121">If entity references are expanded in the DOM, which is the default method, the structure has this type of tree:</span></span>  
  
 <span data-ttu-id="460f6-122">XmlElement: Yayımcı</span><span class="sxs-lookup"><span data-stu-id="460f6-122">XmlElement: publisher</span></span>  
  
 <span data-ttu-id="460f6-123">XmlText: Microsoft Press</span><span class="sxs-lookup"><span data-stu-id="460f6-123">XmlText: Microsoft Press</span></span>  
  
 <span data-ttu-id="460f6-124">Varlık başvurusu düğümünün kaybolduğuna ve alıcı uygulamanın "Microsoft Press" içeren **XmlText** düğümünün bir varlık bildiriminden oluşturulduğunu söylüdiğine dikkat edin.</span><span class="sxs-lookup"><span data-stu-id="460f6-124">Notice that the entity reference node is gone, and the receiving application cannot tell that the **XmlText** node containing "Microsoft Press" was created from an entity declaration.</span></span>
