# <a name="compiler-error-cs0413"></a><span data-ttu-id="17083-103">Derleyici Hatası CS0413</span><span class="sxs-lookup"><span data-stu-id="17083-103">Compiler Error CS0413</span></span>
<span data-ttu-id="17083-104">' Type Parameter ' tür parametresi, bir sınıf türü kısıtlaması veya bir ' class ' kısıtlaması olmadığından ' As ' işleciyle kullanılamaz</span><span class="sxs-lookup"><span data-stu-id="17083-104">The type parameter 'type parameter' cannot be used with the 'as' operator because it does not have a class type constraint nor a 'class' constraint</span></span>  
  
<span data-ttu-id="17083-105">Genel bir tür [as](../operators/type-testing-and-cast.md#as-operator) işlecini kullanıyorsa, ancak bu genel türün bir sınıf türü kısıtlaması yoksa, bu hata oluşur.</span><span class="sxs-lookup"><span data-stu-id="17083-105">This error occurs if a generic type uses the [as](../operators/type-testing-and-cast.md#as-operator) operator, but that generic type does not have a class type constraint.</span></span> <span data-ttu-id="17083-106">`as`İşlecine yalnızca başvuru ve null atanabilir değer türleriyle izin verilir, bu nedenle tür parametresinin değer türü olmadığından emin olmak için kısıtlamalı olması gerekir.</span><span class="sxs-lookup"><span data-stu-id="17083-106">The `as` operator is only allowed with reference and nullable value types, so the type parameter must be constrained to guarantee that it is not a value type.</span></span> <span data-ttu-id="17083-107">Bu hatayı önlemek için bir sınıf türü kısıtlaması veya bir başvuru türü kısıtlaması kullanın.</span><span class="sxs-lookup"><span data-stu-id="17083-107">To avoid this error, use a class type constraint or a reference type constraint.</span></span>
  
<span data-ttu-id="17083-108">Bunun nedeni, `as` işlecin `null` bir değer türü için olası bir değer olmaması ve tür parametresinin bir sınıf türü kısıtlaması veya bir başvuru türü kısıtlaması olmadığı sürece değer türü olarak değerlendirilmesi gerekir.</span><span class="sxs-lookup"><span data-stu-id="17083-108">This is because the `as` operator could return `null`, which is not a possible value for a value type, and the type parameter must be treated as a value type unless it is a class type constraint or a reference type constraint.</span></span>
  
## <a name="example"></a><span data-ttu-id="17083-109">Örnek</span><span class="sxs-lookup"><span data-stu-id="17083-109">Example</span></span>

<span data-ttu-id="17083-110">Aşağıdaki örnek CS0413 oluşturur.</span><span class="sxs-lookup"><span data-stu-id="17083-110">The following sample generates CS0413.</span></span>  
  
```csharp  
// CS0413.cs  
// compile with: /target:library  
class A {}  
class B : A {}  
  
class CMain  
{  
   A a = null;  
   public void G<T>()  
   {  
      a = new A();  
      System.Console.WriteLine (a as T);  // CS0413  
   }  
  
   // OK  
   public void H<T>() where T : A  
   {  
      a = new A();  
      System.Console.WriteLine (a as T);  
   }  
}  
```
