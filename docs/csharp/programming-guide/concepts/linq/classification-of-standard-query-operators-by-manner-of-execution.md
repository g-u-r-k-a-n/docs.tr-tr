<span data-ttu-id="0db14-103">Standart sorgu işleci yöntemlerinin Nesnelere LINQ uygulamaları iki ana yoldan biriyle yürütülür: hemen veya ertelenmiş.</span><span class="sxs-lookup"><span data-stu-id="0db14-103">The LINQ to Objects implementations of the standard query operator methods execute in one of two main ways: immediate or deferred.</span></span> <span data-ttu-id="0db14-104">Ertelenmiş yürütme kullanan sorgu işleçleri ayrıca iki kategoriye ayrılabilir: akış ve akış dışı.</span><span class="sxs-lookup"><span data-stu-id="0db14-104">The query operators that use deferred execution can be additionally divided into two categories: streaming and non-streaming.</span></span> <span data-ttu-id="0db14-105">Farklı sorgu işleçleri nasıl yürütülür biliyorsanız, belirli bir sorgudan elde ettiğiniz sonuçları anlamanıza yardımcı olabilir.</span><span class="sxs-lookup"><span data-stu-id="0db14-105">If you know how the different query operators execute, it may help you understand the results that you get from a given query.</span></span> <span data-ttu-id="0db14-106">Bu, özellikle veri kaynağı değişiyorsa veya başka bir sorgunun üstüne bir sorgu oluşturuyorsanız geçerlidir.</span><span class="sxs-lookup"><span data-stu-id="0db14-106">This is especially true if the data source is changing or if you are building a query on top of another query.</span></span> <span data-ttu-id="0db14-107">Bu konu, standart sorgu işleçlerini yürütme biçimlerine göre sınıfa alır.</span><span class="sxs-lookup"><span data-stu-id="0db14-107">This topic classifies the standard query operators according to their manner of execution.</span></span>  
  
## <a name="manners-of-execution"></a><span data-ttu-id="0db14-108">İdam Görgüleri</span><span class="sxs-lookup"><span data-stu-id="0db14-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="0db14-109">Hemen</span><span class="sxs-lookup"><span data-stu-id="0db14-109">Immediate</span></span>  
 <span data-ttu-id="0db14-110">Hemen yürütme, veri kaynağının okunduğu ve işlemin sorgunun bildirildiği koddaki noktada gerçekleştirildiği anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="0db14-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="0db14-111">Tek bir, sayısal olmayan sonuç döndüren tüm standart sorgu işleçleri hemen yürütülür.</span><span class="sxs-lookup"><span data-stu-id="0db14-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="0db14-112">Ertelen -miş</span><span class="sxs-lookup"><span data-stu-id="0db14-112">Deferred</span></span>  
 <span data-ttu-id="0db14-113">Ertelenmiş yürütme, işlemin sorgunun bildirildiği koddaki noktada gerçekleştirilmediği anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="0db14-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="0db14-114">İşlem yalnızca sorgu değişkeni numaralandırıldığında, örneğin bir `foreach` deyim kullanılarak gerçekleştirilir.</span><span class="sxs-lookup"><span data-stu-id="0db14-114">The operation is performed only when the query variable is enumerated, for example by using a `foreach` statement.</span></span> <span data-ttu-id="0db14-115">Bu, sorgutanımlandığında değil, sorgu yürütüldüğünde sorgunun yürütülmesinin sonuçlarının veri kaynağının içeriğine bağlı olduğu anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="0db14-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="0db14-116">Sorgu değişkeni birden çok kez numaralandırılırsa, sonuçlar her seferinde farklı olabilir.</span><span class="sxs-lookup"><span data-stu-id="0db14-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="0db14-117">İade türü ertelenmiş bir şekilde <xref:System.Collections.Generic.IEnumerable%601> olan <xref:System.Linq.IOrderedEnumerable%601> veya çalıştırılan hemen hemen tüm standart sorgu işleçleri.</span><span class="sxs-lookup"><span data-stu-id="0db14-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="0db14-118">Ertelenmiş yürütme kullanan sorgu işleçleri ayrıca akış veya akış dışı olarak sınıflandırılabilir.</span><span class="sxs-lookup"><span data-stu-id="0db14-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="0db14-119">Akış</span><span class="sxs-lookup"><span data-stu-id="0db14-119">Streaming</span></span>  
 <span data-ttu-id="0db14-120">Akış işleçleri, öğeleri verim önce tüm kaynak verileri okumak zorunda değildir.</span><span class="sxs-lookup"><span data-stu-id="0db14-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="0db14-121">Yürütme sırasında, bir akış işleci okundukça her kaynak öğesi üzerinde işlemini gerçekleştirir ve uygunsa öğeyi verir.</span><span class="sxs-lookup"><span data-stu-id="0db14-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="0db14-122">Bir akış işleci, sonuç öğesi üretilene kadar kaynak öğeleriokumaya devam ediyor.</span><span class="sxs-lookup"><span data-stu-id="0db14-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="0db14-123">Bu, bir sonuç öğesi oluşturmak için birden fazla kaynak öğesinin okunabileceği anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="0db14-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="0db14-124">Akış Dışı</span><span class="sxs-lookup"><span data-stu-id="0db14-124">Non-Streaming</span></span>  
 <span data-ttu-id="0db14-125">Akış dışı işleçler, bir sonuç öğesi verebilmeleri için tüm kaynak verileri okumalıdır.</span><span class="sxs-lookup"><span data-stu-id="0db14-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="0db14-126">Sıralama veya gruplandırma gibi işlemler bu kategoriye girer.</span><span class="sxs-lookup"><span data-stu-id="0db14-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="0db14-127">Yürütme sırasında, akış sızak sorgu işleçleri tüm kaynak verileri okur, bir veri yapısına koyar, işlemi gerçekleştirir ve elde edilen öğeleri verir.</span><span class="sxs-lookup"><span data-stu-id="0db14-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
