# <a name="differences-between-passing-an-argument-by-value-and-by-reference-visual-basic"></a><span data-ttu-id="d5bf0-102">Değere ve Başvuruya Göre Bağımsız Değişken Geçirme Arasındaki Farklar (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="d5bf0-102">Differences Between Passing an Argument By Value and By Reference (Visual Basic)</span></span>
<span data-ttu-id="d5bf0-103">Bir yordama bir veya daha fazla bağımsız değişken geçirdiğinizde, her bağımsız değişken çağıran koddaki temel bir programlama öğesine karşılık gelir.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-103">When you pass one or more arguments to a procedure, each argument corresponds to an underlying programming element in the calling code.</span></span> <span data-ttu-id="d5bf0-104">Bu temel öğenin veya buna bir başvurunun değerini geçirebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-104">You can pass either the value of this underlying element, or a reference to it.</span></span> <span data-ttu-id="d5bf0-105">Bu, *geçirme mekanizması*olarak bilinir.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-105">This is known as the *passing mechanism*.</span></span>  
  
## <a name="passing-by-value"></a><span data-ttu-id="d5bf0-106">Değere göre geçirme</span><span class="sxs-lookup"><span data-stu-id="d5bf0-106">Passing by Value</span></span>  
 <span data-ttu-id="d5bf0-107">Yordam tanımındaki karşılık gelen parametrenin [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) anahtar sözcüğünü belirterek *değere göre* bir bağımsız değişken geçirirsiniz.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-107">You pass an argument *by value* by specifying the [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="d5bf0-108">Bu geçen mekanizmayı kullandığınızda Visual Basic, temeldeki programlama öğesinin değerini yordamdaki bir yerel değişkene kopyalar.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-108">When you use this passing mechanism, Visual Basic copies the value of the underlying programming element into a local variable in the procedure.</span></span> <span data-ttu-id="d5bf0-109">Yordam kodu, çağıran koddaki temel alınan öğeye hiçbir erişime sahip değil.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-109">The procedure code does not have any access to the underlying element in the calling code.</span></span>  
  
## <a name="passing-by-reference"></a><span data-ttu-id="d5bf0-110">Başvuruya göre geçirme</span><span class="sxs-lookup"><span data-stu-id="d5bf0-110">Passing by Reference</span></span>  
 <span data-ttu-id="d5bf0-111">Yordam tanımında karşılık gelen parametre için [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) anahtar sözcüğünü belirterek bir bağımsız değişkeni *başvuruya göre* geçirirsiniz.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-111">You pass an argument *by reference* by specifying the [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="d5bf0-112">Bu geçiş mekanizmasını kullandığınızda Visual Basic, yordama çağıran koddaki temeldeki programlama öğesine doğrudan başvuru verir.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-112">When you use this passing mechanism, Visual Basic gives the procedure a direct reference to the underlying programming element in the calling code.</span></span>  
  
## <a name="passing-mechanism-and-element-type"></a><span data-ttu-id="d5bf0-113">Geçirme mekanizması ve öğe türü</span><span class="sxs-lookup"><span data-stu-id="d5bf0-113">Passing Mechanism and Element Type</span></span>  
 <span data-ttu-id="d5bf0-114">Geçirme mekanizması seçimi, temel öğe türünün sınıflandırmasıyla aynı değildir.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-114">The choice of passing mechanism is not the same as the classification of the underlying element type.</span></span> <span data-ttu-id="d5bf0-115">Değere veya başvuruya göre geçirme, yordam koduna ne Visual Basic temin eder.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-115">Passing by value or by reference refers to what Visual Basic supplies to the procedure code.</span></span> <span data-ttu-id="d5bf0-116">Bir değer türü veya başvuru türü, bir programlama öğesinin bellekte nasıl depolandığını belirtir.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-116">A value type or reference type refers to how a programming element is stored in memory.</span></span>  
  
 <span data-ttu-id="d5bf0-117">Ancak, geçen mekanizma ve öğe türü birbirleriyle ilişkilidir.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-117">However, the passing mechanism and element type are interrelated.</span></span> <span data-ttu-id="d5bf0-118">Başvuru türünün değeri, belleğin başka bir yerindeki verilerin bir işaretçisidir.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-118">The value of a reference type is a pointer to the data elsewhere in memory.</span></span> <span data-ttu-id="d5bf0-119">Yani, bir başvuru türü değere göre geçirdiğinizde, yordam kodunun temeldeki öğenin kendisine erişemese de, temel alınan öğenin verilerine yönelik bir işaretçisi vardır.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-119">This means that when you pass a reference type by value, the procedure code has a pointer to the underlying element's data, even though it cannot access the underlying element itself.</span></span> <span data-ttu-id="d5bf0-120">Örneğin, öğe bir dizi değişkenidir, yordam kodu değişkene erişemez, ancak dizi üyelerine erişebilir.</span><span class="sxs-lookup"><span data-stu-id="d5bf0-120">For example, if the element is an array variable, the procedure code does not have access to the variable itself, but it can access the array members.</span></span>
