1. <span data-ttu-id="9f1c6-107">Aralık değişkeni açıkça yazılmışsa, türün, yineleme yaptığı koleksiyondaki öğelerin türü ile aynı ya da örtülü olarak dönüştürülebilir olduğundan emin olun.</span><span class="sxs-lookup"><span data-stu-id="9f1c6-107">If the range variable is explicitly typed, make sure that the type is either the same as, or implicitly convertible from, the type of the elements in the collection it iterates.</span></span> <span data-ttu-id="9f1c6-108">Aralık değişkenine önünde `var` anahtar sözcüğü varsa `var`kaldırın.</span><span class="sxs-lookup"><span data-stu-id="9f1c6-108">If the range variable is preceded with the `var` keyword, remove `var`.</span></span>  
  
## <a name="example"></a><span data-ttu-id="9f1c6-109">Örnek</span><span class="sxs-lookup"><span data-stu-id="9f1c6-109">Example</span></span>

<span data-ttu-id="9f1c6-110">Aşağıdaki kod CS1942 oluşturur:</span><span class="sxs-lookup"><span data-stu-id="9f1c6-110">The following code generates CS1942:</span></span>  

```csharp
// cs1942.cs  
class Program  
    {  
        static void Main(string[] args)  
        {  
            var x = from var i in Enumerable.Range(1, 100) // CS1949  
                    select i; //CS1942  
        }  
    }  
```

<span data-ttu-id="9f1c6-111">CS1942, CS1949 ile ilgilidir çünkü `var` bir tür olmadığından, bir Aralık değişkeniyle birlikte `var` kullanımı temeldeki `Cast<T>` işleminin başarısız olmasına neden olur.</span><span class="sxs-lookup"><span data-stu-id="9f1c6-111">CS1942 is related to CS1949 because the use of `var` with a range variable causes the underlying `Cast<T>` operation to fail because `var` is not a type.</span></span>
