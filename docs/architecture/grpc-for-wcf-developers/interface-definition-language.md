# <a name="interface-definition-language"></a><span data-ttu-id="e34e3-103">Arabirim Tanımlama Dili</span><span class="sxs-lookup"><span data-stu-id="e34e3-103">Interface Definition Language</span></span>

<span data-ttu-id="e34e3-104">Windows Communication Foundation (WCF) ile, Hizmetler Web hizmeti tanım dili (WSDL) kullanarak açıklama meta verilerini kullanıma sunabilir.</span><span class="sxs-lookup"><span data-stu-id="e34e3-104">With Windows Communication Foundation (WCF), services can expose description metadata by using the Web Service Definition Language (WSDL).</span></span> <span data-ttu-id="e34e3-105">WSDL, çalışma zamanında .NET Reflection kullanılarak dinamik olarak oluşturulur.</span><span class="sxs-lookup"><span data-stu-id="e34e3-105">WSDL is generated dynamically by using .NET reflection at runtime.</span></span> <span data-ttu-id="e34e3-106">Geliştiriciler bu meta verileri, büyük olasılıkla HTTP üzerinden SOAP gibi platformdan bağımsız bir bağlama kullanıyorsa diğer dillerde Bu hizmetler için istemciler oluşturmak üzere kullanabilir.</span><span class="sxs-lookup"><span data-stu-id="e34e3-106">Developers can use this metadata to generate clients for those services, potentially in other languages if they're using a platform-neutral binding such as SOAP over HTTP.</span></span>

<span data-ttu-id="e34e3-107">gRPC, protokol arabelleklerinden arabirim tanım dilini (IDL) kullanır.</span><span class="sxs-lookup"><span data-stu-id="e34e3-107">gRPC uses the Interface Definition Language (IDL) from Protocol Buffers.</span></span> <span data-ttu-id="e34e3-108">Protokol arabellekleri IDL, açık bir belirtimle özel, platformdan bağımsız bir dildir.</span><span class="sxs-lookup"><span data-stu-id="e34e3-108">The Protocol Buffers IDL is a custom, platform-neutral language with an open specification.</span></span> <span data-ttu-id="e34e3-109">Geliştiriciler `.proto` , kendi giriş ve çıkışlarıyla birlikte Hizmetleri tanımlayacak dosyaları yazar.</span><span class="sxs-lookup"><span data-stu-id="e34e3-109">Developers author `.proto` files to describe services, along with their inputs and outputs.</span></span> <span data-ttu-id="e34e3-110">Bu `.proto` dosyalar daha sonra istemciler ve sunucular için dile veya platforma özel saplamalar oluşturmak için kullanılabilir ve birden çok farklı platformun iletişim kurmasına olanak tanır.</span><span class="sxs-lookup"><span data-stu-id="e34e3-110">These `.proto` files can then be used to generate language- or platform-specific stubs for clients and servers, allowing multiple different platforms to communicate.</span></span> <span data-ttu-id="e34e3-111">`.proto`Ekipler, dosyaları paylaşarak, bir kod bağımlılığı almak zorunda kalmadan diğerlerinin her birinin hizmetlerini kullanmak için kod oluşturabilir.</span><span class="sxs-lookup"><span data-stu-id="e34e3-111">By sharing `.proto` files, teams can generate code to use each others' services, without needing to take a code dependency.</span></span>

<span data-ttu-id="e34e3-112">Prototipsel IDL 'nin avantajlarından biri özel bir dil olarak, gRPC 'nin tamamen dil ve platform belirsiz olmasını sağlar, başka bir teknolojiyi de favoring etmez.</span><span class="sxs-lookup"><span data-stu-id="e34e3-112">One of the advantages of the Protobuf IDL is that as a custom language, it enables gRPC to be completely language and platform agnostic, not favoring any technology over another.</span></span>

<span data-ttu-id="e34e3-113">Prototiplik IDL Ayrıca insanlar için hem okuma hem de yazma için tasarlanmıştır, WSDL ise makine tarafından okunabilen/yazılabilir biçim olarak tasarlanmıştır.</span><span class="sxs-lookup"><span data-stu-id="e34e3-113">The Protobuf IDL is also designed for humans to both read and write, whereas WSDL is intended as a machine-readable/writable format.</span></span> <span data-ttu-id="e34e3-114">Bir WCF hizmetinin WSDL 'sini değiştirmek, genellikle hizmetin değiştirilmesini, hizmetin çalıştırılmasını ve WSDL dosyasını sunucudan yeniden üretmeyi gerektirir.</span><span class="sxs-lookup"><span data-stu-id="e34e3-114">Changing the WSDL of a WCF service typically requires changing the service, running the service, and regenerating the WSDL file from the server.</span></span> <span data-ttu-id="e34e3-115">Bunun aksine, bir `.proto` dosyayla birlikte, değişiklikler bir metin Düzenleyicisi ile uygulanır ve oluşturulan kodla otomatik olarak akar.</span><span class="sxs-lookup"><span data-stu-id="e34e3-115">By contrast, with a `.proto` file, changes are simple to apply with a text editor, and automatically flow through the generated code.</span></span> <span data-ttu-id="e34e3-116">Visual Studio 2019 `.proto` , kaydedildiklerinde arka planda dosyaları oluşturur.</span><span class="sxs-lookup"><span data-stu-id="e34e3-116">Visual Studio 2019 builds `.proto` files in the background when they are saved.</span></span> <span data-ttu-id="e34e3-117">VS Code gibi diğer düzenleyicilerle, proje oluşturulduğunda değişiklikler uygulanır.</span><span class="sxs-lookup"><span data-stu-id="e34e3-117">With other editors, such as VS Code, the changes are applied when the project is built.</span></span>

<span data-ttu-id="e34e3-118">XML ile karşılaştırıldığında ve özellikle SOAP kullanılarak kodlanan iletiler birçok avantaj sağlar.</span><span class="sxs-lookup"><span data-stu-id="e34e3-118">When compared with XML, and particularly SOAP, messages encoded by using Protobuf have many advantages.</span></span> <span data-ttu-id="e34e3-119">Prototip iletiler SOAP XML ile serileştirilmiş aynı verilerden daha küçük olmaya eğilimlidir ve kodlama, kod çözme ve ağ üzerinden iletilmesi daha hızlı olabilir.</span><span class="sxs-lookup"><span data-stu-id="e34e3-119">Protobuf messages tend to be smaller than the same data serialized as SOAP XML, and encoding, decoding, and transmitting them over a network can be faster.</span></span>

<span data-ttu-id="e34e3-120">Prototiplinin SOAP ile karşılaştırıldığında potansiyel dezavantajı, iletiler insanların okunmadığı için ileti içeriğinin hata ayıklaması için ek araçlar gerektirir.</span><span class="sxs-lookup"><span data-stu-id="e34e3-120">The potential disadvantage of Protobuf compared to SOAP is that, because the messages aren't readable by humans, additional tooling is required to debug message content.</span></span>

> [!TIP]
> <span data-ttu-id="e34e3-121">gRPC *,* önceden derlenmiş saplamalar olmadan hizmetlere dinamik olarak erişmek için sunucu yansımasını destekler, ancak uygulamaya özgü istemcilerden daha fazla genel amaçlı araçlara yöneliktir.</span><span class="sxs-lookup"><span data-stu-id="e34e3-121">gRPC *does* support server reflection for dynamically accessing services without pre-compiled stubs, although it's intended more for general-purpose tools than application-specific clients.</span></span> <span data-ttu-id="e34e3-122">Daha fazla bilgi için GitHub 'da [GRPC sunucu yansıma Protokolü](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md) ' ne bakın.</span><span class="sxs-lookup"><span data-stu-id="e34e3-122">For more information, see [GRPC Server Reflection Protocol](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md) on GitHub.</span></span>
