# <a name="replay-attacks"></a><span data-ttu-id="5b5bd-102">Yeniden Yürütme Saldırıları</span><span class="sxs-lookup"><span data-stu-id="5b5bd-102">Replay Attacks</span></span>
<span data-ttu-id="5b5bd-103">Bir saldırgan iki taraf arasında bir ileti akışını kopyaladığında veya bir veya daha fazla tarafın akışını yeniden oynadığında bir yeniden *yürütme saldırısı* meydana gelir.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-103">A *replay attack* occurs when an attacker copies a stream of messages between two parties and replays the stream to one or more of the parties.</span></span> <span data-ttu-id="5b5bd-104">Hafiflemediği sürece, saldırıya tabi olan bilgisayarlar akışı meşru iletiler olarak işler ve bu da bir öğenin gereksiz sıraları gibi hatalı sonuçlar oluşmasına neden olur.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-104">Unless mitigated, the computers subject to the attack process the stream as legitimate messages, resulting in a range of bad consequences, such as redundant orders of an item.</span></span>  
  
## <a name="bindings-may-be-subject-to-reflection-attacks"></a><span data-ttu-id="5b5bd-105">Bağlamalar, yansıma saldırılarına maruz kalabilir</span><span class="sxs-lookup"><span data-stu-id="5b5bd-105">Bindings May Be Subject to Reflection Attacks</span></span>  
 <span data-ttu-id="5b5bd-106">*Yansıma saldırıları* , yanıt olarak alıcıdan gelmiş gibi iletiler bir gönderene geri yeniden oynatılır.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-106">*Reflection attacks* are replays of messages back to a sender as if they came from the receiver as the reply.</span></span> <span data-ttu-id="5b5bd-107">Windows Communication Foundation (WCF) mekanizmasında standart yeniden *yürütme algılaması* bunu otomatik olarak işlemez.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-107">The standard *replay detection* in the Windows Communication Foundation (WCF) mechanism does not automatically handle this.</span></span>  
  
 <span data-ttu-id="5b5bd-108">WCF hizmet modeli istek iletileri için imzalı bir ileti KIMLIĞI eklediğinden ve yanıt iletilerinde imzalı bir üst bilgi beklediği için yansıma saldırıları varsayılan olarak azaltıldığında `relates-to` .</span><span class="sxs-lookup"><span data-stu-id="5b5bd-108">Reflection attacks are mitigated by default because the WCF service model adds a signed message ID to request messages and expects a signed `relates-to` header on response messages.</span></span> <span data-ttu-id="5b5bd-109">Sonuç olarak, istek iletisi yanıt olarak yeniden yürütülemez.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-109">Consequently, the request message cannot be replayed as a response.</span></span> <span data-ttu-id="5b5bd-110">Güvenli güvenilir ileti (RM) senaryolarında, şu nedenle yansıma saldırıları azaltıldığında:</span><span class="sxs-lookup"><span data-stu-id="5b5bd-110">In secure reliable message (RM) scenarios, reflection attacks are mitigated because:</span></span>  
  
- <span data-ttu-id="5b5bd-111">Oluşturma sırası ve sıra oluşturma yanıtı ileti şemaları farklıdır.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-111">The create sequence and create sequence response message schemas are different.</span></span>  
  
- <span data-ttu-id="5b5bd-112">Simpleks dizileri için, istemci gönderdiği sıralı iletiler, istemci bu iletileri anlamadığından bu iletiyi yeniden oynamayabilir.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-112">For simplex sequences, sequence messages the client sends cannot be replayed back to it because the client cannot understand such messages.</span></span>  
  
- <span data-ttu-id="5b5bd-113">Çift yönlü diziler için, iki sıra kimliği benzersiz olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-113">For duplex sequences, the two sequence IDs must be unique.</span></span> <span data-ttu-id="5b5bd-114">Bu nedenle, bir giden sırası iletisi gelen sıralı ileti olarak yeniden oynatılamaz (tüm sıralı üstbilgiler ve gövdeler, çok fazla).</span><span class="sxs-lookup"><span data-stu-id="5b5bd-114">Thus, an outgoing sequence message cannot be replayed back as an incoming sequence message (all sequence headers and bodies are signed, too).</span></span>  
  
 <span data-ttu-id="5b5bd-115">Yansıma saldırılarına açık olan tek bağlamalar, WS-Addressing olmayan özel bağlamalardır ve simetrik anahtar tabanlı güvenliği kullanır.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-115">The only bindings that are susceptible to reflection attacks are those without WS-Addressing: custom bindings that have WS-Addressing disabled and use the symmetric key-based security.</span></span> <span data-ttu-id="5b5bd-116"><xref:System.ServiceModel.BasicHttpBinding>Varsayılan olarak ws-Addressing kullanmaz, ancak simetrik anahtar tabanlı güvenliği bu saldırıya karşı savunmasız çalışmasına izin verecek şekilde kullanmaz.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-116">The <xref:System.ServiceModel.BasicHttpBinding> does not use WS-Addressing by default, but it does not use symmetric key-based security in a way that allows it to be vulnerable to this attack.</span></span>  
  
 <span data-ttu-id="5b5bd-117">Özel bağlamalar için azaltma, güvenlik bağlamı kurmamalıdır veya WS-Addressing üst bilgileri gerektirmez.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-117">The mitigation for custom bindings is to not establish security context or to require WS-Addressing headers.</span></span>

 <span data-ttu-id="5b5bd-124">Azaltmaları şunları içerir:</span><span class="sxs-lookup"><span data-stu-id="5b5bd-124">Mitigations include:</span></span>

[nasıl yapılır: güvenli bir oturum Için güvenlik bağlamı belirteci oluşturma](how-to-create-a-security-context-token-for-a-secure-session.md).</span><span class="sxs-lookup"><span data-stu-id="5b5bd-126">For more information, see [How to: Create a Security Context Token for a Secure Session](how-to-create-a-security-context-token-for-a-secure-session.md).</span></span>  
  
- <span data-ttu-id="5b5bd-127">Hizmeti, aktarım düzeyi güvenliği kullanacak şekilde yapılandırın.</span><span class="sxs-lookup"><span data-stu-id="5b5bd-127">Configure the service to use transport-level security.</span></span>
