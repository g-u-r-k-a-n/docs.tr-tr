## <a name="choosing-an-authentication-scheme"></a><span data-ttu-id="39598-145">Kimlik doğrulama düzeni seçme</span><span class="sxs-lookup"><span data-stu-id="39598-145">Choosing an Authentication Scheme</span></span>  
 <span data-ttu-id="39598-146">Bir HTTP sunucusu için olası kimlik doğrulama düzenlerini seçerken, dikkate alınması gereken birkaç öğe şunlardır:</span><span class="sxs-lookup"><span data-stu-id="39598-146">When selecting the potential authentication schemes for an HTTP server, a few items to consider include the following:</span></span>  
  
- <span data-ttu-id="39598-147">Kaynağın korunması gerekip gerekmediğini göz önünde bulundurun.</span><span class="sxs-lookup"><span data-stu-id="39598-147">Consider whether the resource needs to be protected.</span></span> <span data-ttu-id="39598-148">HTTP kimlik doğrulamasının kullanılması, daha fazla veri iletilmesi gerektirir ve istemcilerle birlikte çalışabilirliği sınırlayabilir.</span><span class="sxs-lookup"><span data-stu-id="39598-148">Using HTTP authentication requires transmitting more data and can limit interoperability with clients.</span></span> <span data-ttu-id="39598-149">Korunması gerekmeyen kaynaklara anonim erişime izin verin.</span><span class="sxs-lookup"><span data-stu-id="39598-149">Allow anonymous access to resources that do not need to be protected.</span></span>  
  
- <span data-ttu-id="39598-150">Kaynağın korunması gerekiyorsa, hangi kimlik doğrulama düzenlerinin gerekli güvenlik düzeyini sağlamasını düşünün.</span><span class="sxs-lookup"><span data-stu-id="39598-150">If the resource needs to be protected, consider which authentication schemes provide the required level of security.</span></span> <span data-ttu-id="39598-151">Burada ele alınan en zayıf standart kimlik doğrulama şeması, temel kimlik doğrulamadır.</span><span class="sxs-lookup"><span data-stu-id="39598-151">The weakest standard authentication scheme discussed here is Basic authentication.</span></span> <span data-ttu-id="39598-152">Temel kimlik doğrulaması, kullanıcının kimlik bilgilerini korumaz.</span><span class="sxs-lookup"><span data-stu-id="39598-152">Basic authentication does not protect the user's credentials.</span></span> <span data-ttu-id="39598-153">En güçlü standart kimlik doğrulama şeması, Negotiate kimlik doğrulamadır ve Kerberos protokolüne yol açar.</span><span class="sxs-lookup"><span data-stu-id="39598-153">The strongest standard authentication scheme is Negotiate authentication, resulting in the Kerberos protocol.</span></span>
