# <a name="compiler-error-cs1001"></a><span data-ttu-id="b4f85-102">Derleyici Hatası CS1001</span><span class="sxs-lookup"><span data-stu-id="b4f85-102">Compiler Error CS1001</span></span>
<span data-ttu-id="b4f85-103">Tanımlayıcı bekleniyor</span><span class="sxs-lookup"><span data-stu-id="b4f85-103">Identifier expected</span></span>  
  
 <span data-ttu-id="b4f85-104">Bir tanımlayıcı sağlamadın.</span><span class="sxs-lookup"><span data-stu-id="b4f85-104">You did not supply an identifier.</span></span> <span data-ttu-id="b4f85-105">Tanımlayıcı, sağladığınız sınıf, yapı, ad alanı, yöntem, değişken vb. adıdır.</span><span class="sxs-lookup"><span data-stu-id="b4f85-105">An identifier is the name of a class, struct, namespace, method, variable and so on that you provide.</span></span>  
  
 <span data-ttu-id="b4f85-106">Aşağıdaki örnek basit bir sınıf bildirir, ancak sınıfa bir ad vermez:</span><span class="sxs-lookup"><span data-stu-id="b4f85-106">The following example declares a simple class but does not give the class a name:</span></span>  
  
```csharp  
//cs1001.cs  
public class              //CS1001  
    {  
        public int Num {get; set;}  
        void MethodA(){}  
    }  
```  
  
 <span data-ttu-id="b4f85-107">Aşağıdaki örnek CS1001 oluşturur, çünkü bir enum bildirirken, üyeleri belirtmeniz gerekir:</span><span class="sxs-lookup"><span data-stu-id="b4f85-107">The following sample generates CS1001 because, when declaring an enum, you must specify members:</span></span>  
  
```csharp  
// CS1001.cs  
public class clx  
{  
   enum Colors : int  
   {  
      'a', 'b' // CS1001, 'a' is not a valid int identifier  
       // The following line shows examples of valid identifiers:  
       // Blue, Red, Orange  
   };  
  
   public static void Main()  
   {  
   }  
}  
```  
  
 <span data-ttu-id="b4f85-108">Parametre adları, derleyici bunları bir arabirim tanımında, örneğin kullanmasa bile gereklidir.</span><span class="sxs-lookup"><span data-stu-id="b4f85-108">Parameter names are required even if the compiler doesn't use them, for example, in an interface definition.</span></span> <span data-ttu-id="b4f85-109">Bu parametreler, arabirimi tüketen programcıların parametrelerin ne anlama geldiğini bilmeleri için gereklidir.</span><span class="sxs-lookup"><span data-stu-id="b4f85-109">These parameters are required so that programmers who are consuming the interface know something about the what the parameters mean.</span></span>  
  
```csharp  
// CS1001-2.cs  
// compile with: /target:library  
interface IMyTest  
{  
   void TestFunc1(int, int);  // CS1001  
   // Use the following line instead:  
   // void TestFunc1(int a, int b);  
}  
  
class CMyTest : IMyTest  
{  
   void IMyTest.TestFunc1(int a, int b)  
   {  
   }  
}  
```
