# <a name="compare-testing-options-between-aspnet-mvc-and-aspnet-core"></a><span data-ttu-id="dbdc3-103">ASP.NET MVC ve ASP.NET Core arasındaki test seçeneklerini karşılaştırma</span><span class="sxs-lookup"><span data-stu-id="dbdc3-103">Compare testing options between ASP.NET MVC and ASP.NET Core</span></span>

<span data-ttu-id="dbdc3-104">ASP.NET MVC Apps, denetleyicilerin birim testlerini destekler, ancak bu yaklaşım genellikle Yönlendirme, yetkilendirme, model bağlama, model doğrulama, filtreler ve daha fazlası gibi birçok MVC özelliğini atlar.</span><span class="sxs-lookup"><span data-stu-id="dbdc3-104">ASP.NET MVC apps support unit testing of controllers, but this approach often omits many MVC features like routing, authorization, model binding, model validation, filters, and more.</span></span> <span data-ttu-id="dbdc3-105">Bu MVC özelliklerini denetleyici eyleminin içindeki mantığa ek olarak test etmek için, genellikle uygulamanın dağıtılması ve ardından Selenium gibi bir araçla test olması gerekir.</span><span class="sxs-lookup"><span data-stu-id="dbdc3-105">To test these MVC features in addition to the logic within the controller action itself, frequently the app would need to be deployed and then tested with a tool like Selenium.</span></span> <span data-ttu-id="dbdc3-106">Bu testler önemli ölçüde daha pahalı, daha Brittle ve tüm uygulamayı barındırmak ve çalıştırmak zorunda kalmadan çalıştırılabilen tipik birim testlerinden daha yavaştır.</span><span class="sxs-lookup"><span data-stu-id="dbdc3-106">These tests are substantially more expensive, more brittle, and slower than typical unit tests that can be run without the need for hosting and running the entire app.</span></span>

<span data-ttu-id="dbdc3-107">[ASP.NET Core denetleyicileri](/aspnet/core/mvc/controllers/testing) , TıPKı ASP.NET MVC denetleyicileri gibi, ancak aynı sınırlamalara sahip birim test edilebilir.</span><span class="sxs-lookup"><span data-stu-id="dbdc3-107">[ASP.NET Core controllers can be unit tested](/aspnet/core/mvc/controllers/testing) just like ASP.NET MVC controllers, but with the same limitations.</span></span> <span data-ttu-id="dbdc3-108">Ancak, [ASP.NET Core hızlı, kolay yazarı tümleştirme testlerini de destekler](/aspnet/core/test/integration-tests) .</span><span class="sxs-lookup"><span data-stu-id="dbdc3-108">However, [ASP.NET Core supports fast, easy-to-author integration tests](/aspnet/core/test/integration-tests) as well.</span></span> <span data-ttu-id="dbdc3-109">Tümleştirme testleri bir sınıf tarafından barındırılır `TestHost` ve tipik olarak, `WebApplicationFactory` uygulama bağımlılıklarını geçersiz kılabileceğiniz veya değiştiren özel olarak yapılandırılır.</span><span class="sxs-lookup"><span data-stu-id="dbdc3-109">Integration tests are hosted by a `TestHost` class and are typically configured in a custom `WebApplicationFactory` that can override or replace app dependencies.</span></span> <span data-ttu-id="dbdc3-110">Örneğin, genellikle tümleştirme testi sırasında uygulama farklı bir veri kaynağını hedefleyecek ve sahte veya sahte uygulamalarla e-posta gönderen Hizmetleri değiştirebilir.</span><span class="sxs-lookup"><span data-stu-id="dbdc3-110">For instance, frequently during integration tests the app will target a different data source and may replace services that send emails with fake or mock implementations.</span></span>

<span data-ttu-id="dbdc3-111">ASP.NET MVC ve Web API ASP.NET Core ' de bulunan tümleştirme testi senaryolarına benzer bir şeyi desteklemez.</span><span class="sxs-lookup"><span data-stu-id="dbdc3-111">ASP.NET MVC and Web API did not support anything like the integration testing scenarios available in ASP.NET Core.</span></span> <span data-ttu-id="dbdc3-112">Herhangi bir geçiş çabasında, beklendiği gibi çalıştığından emin olmak için yeni geçirilen sisteminiz için bazı tümleştirme testlerini yazmak üzere zaman ayırmanız gerekir ve devam edin.</span><span class="sxs-lookup"><span data-stu-id="dbdc3-112">In any migration effort, you should allocate time to write some integration tests for your newly migrated system to ensure it's working as expected and continues to do so.</span></span> <span data-ttu-id="dbdc3-113">Taşıma işleminden önce Web uygulaması mantığınızın testlerini yazmasanız bile, ASP.NET Core ' a taşırken kesinlikle bunu yapmanız gerekir.</span><span class="sxs-lookup"><span data-stu-id="dbdc3-113">Even if you weren't writing tests of your web app logic before the migration, you should strongly consider doing so as you move to ASP.NET Core.</span></span>
