# <a name="nested-types"></a><span data-ttu-id="a5148-102">İç içe Geçmiş Türler</span><span class="sxs-lookup"><span data-stu-id="a5148-102">Nested Types</span></span>
<span data-ttu-id="a5148-103">İç içe bir tür, kapsayan tür olarak adlandırılan başka bir türün kapsamı içinde tanımlanan bir türdür.</span><span class="sxs-lookup"><span data-stu-id="a5148-103">A nested type is a type defined within the scope of another type, which is called the enclosing type.</span></span> <span data-ttu-id="a5148-104">İç içe bir tür, kapsayan türünün tüm üyelerine erişebilir.</span><span class="sxs-lookup"><span data-stu-id="a5148-104">A nested type has access to all members of its enclosing type.</span></span> <span data-ttu-id="a5148-105">Örneğin, kapsayan tür ' de tanımlanan özel alanlara ve kapsayan türün tüm yokler ' de tanımlanan korumalı alanlara erişimi vardır.</span><span class="sxs-lookup"><span data-stu-id="a5148-105">For example, it has access to private fields defined in the enclosing type and to protected fields defined in all ascendants of the enclosing type.</span></span>  
  
 <span data-ttu-id="a5148-106">Genel olarak, iç içe türler gelişigüzel kullanılmalıdır.</span><span class="sxs-lookup"><span data-stu-id="a5148-106">In general, nested types should be used sparingly.</span></span> <span data-ttu-id="a5148-107">Bunun birçok nedeni vardır.</span><span class="sxs-lookup"><span data-stu-id="a5148-107">There are several reasons for this.</span></span> <span data-ttu-id="a5148-108">Bazı geliştiriciler kavram ile tam olarak tanıdık değildir.</span><span class="sxs-lookup"><span data-stu-id="a5148-108">Some developers are not fully familiar with the concept.</span></span> <span data-ttu-id="a5148-109">Bu geliştiriciler Örneğin, iç içe geçmiş türlerin değişkenlerini bildirme söz dizimi ile ilgili sorunlarla karşılaşabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="a5148-109">These developers might, for example, have problems with the syntax of declaring variables of nested types.</span></span> <span data-ttu-id="a5148-110">İç içe türler aynı zamanda kapsayan türleriyle çok sıkı bir şekilde bağlanmış ve bu nedenle genel amaçlı türler gibi uygun değildir.</span><span class="sxs-lookup"><span data-stu-id="a5148-110">Nested types are also very tightly coupled with their enclosing types, and as such are not suited to be general-purpose types.</span></span>  
  
 <span data-ttu-id="a5148-111">İç içe türler, kapsayan türlerin modelleme uygulama ayrıntıları için idealdir.</span><span class="sxs-lookup"><span data-stu-id="a5148-111">Nested types are best suited for modeling implementation details of their enclosing types.</span></span> <span data-ttu-id="a5148-112">Son Kullanıcı nadiren iç içe bir türün değişkenlerini bildirmeli ve neredeyse asla iç içe geçmiş türleri örneklemek gerekmez.</span><span class="sxs-lookup"><span data-stu-id="a5148-112">The end user should rarely have to declare variables of a nested type and almost never should have to explicitly instantiate nested types.</span></span> <span data-ttu-id="a5148-113">Örneğin, bir koleksiyonun numaralandırıcısı, o koleksiyonun iç içe bir türü olabilir.</span><span class="sxs-lookup"><span data-stu-id="a5148-113">For example, the enumerator of a collection can be a nested type of that collection.</span></span> <span data-ttu-id="a5148-114">Numaralandırıcılar genellikle kapsayan türlerine göre örneklenmiştir ve birçok dil foreach ifadesini destekledikleri için, Numaralandırıcı değişkenlerinin Son Kullanıcı tarafından bildirilmelidir.</span><span class="sxs-lookup"><span data-stu-id="a5148-114">Enumerators are usually instantiated by their enclosing type, and because many languages support the foreach statement, enumerator variables rarely have to be declared by the end user.</span></span>  
  
 <span data-ttu-id="a5148-115">**✓ DO** üye erişilebilirlik semantiği istenen şekilde iç içe geçmiş tür ve dış türü arasındaki ilişkiyi olduğunda iç içe geçmiş türler kullanın.</span><span class="sxs-lookup"><span data-stu-id="a5148-115">**✓ DO** use nested types when the relationship between the nested type and its outer type is such that member-accessibility semantics are desirable.</span></span>  
  
 <span data-ttu-id="a5148-116">**X DO NOT** kullanım ortak iç içe geçmiş türler, mantıksal bir gruplandırma olarak oluşturun; ad alanları için bunu kullanın.</span><span class="sxs-lookup"><span data-stu-id="a5148-116">**X DO NOT** use public nested types as a logical grouping construct; use namespaces for this.</span></span>  
  
 <span data-ttu-id="a5148-117">**X AVOID** herkese açık şekilde iç içe geçmiş türler açık.</span><span class="sxs-lookup"><span data-stu-id="a5148-117">**X AVOID** publicly exposed nested types.</span></span> <span data-ttu-id="a5148-118">Bunun tek istisnası, iç içe türün değişkenlerinin yalnızca altsınıflama veya diğer gelişmiş özelleştirme senaryoları gibi nadir senaryolarda bildirilmesine ihtiyaç duymalıdır.</span><span class="sxs-lookup"><span data-stu-id="a5148-118">The only exception to this is if variables of the nested type need to be declared only in rare scenarios such as subclassing or other advanced customization scenarios.</span></span>  
  
 <span data-ttu-id="a5148-119">**X DO NOT** türü içeren türde dışında başvurulacak olasılığı olan iç içe geçmiş türler kullanın.</span><span class="sxs-lookup"><span data-stu-id="a5148-119">**X DO NOT** use nested types if the type is likely to be referenced outside of the containing type.</span></span>  
  
 <span data-ttu-id="a5148-120">Örneğin, bir sınıfta tanımlanan bir metoda geçirilen Enum, sınıfta iç içe geçmiş bir tür olarak tanımlanmamalıdır.</span><span class="sxs-lookup"><span data-stu-id="a5148-120">For example, an enum passed to a method defined on a class should not be defined as a nested type in the class.</span></span>  
  
 <span data-ttu-id="a5148-121">**X DO NOT** istemci kodu tarafından örneğinin oluşturulması gerekirse iç içe geçmiş türler kullanın.</span><span class="sxs-lookup"><span data-stu-id="a5148-121">**X DO NOT** use nested types if they need to be instantiated by client code.</span></span>  <span data-ttu-id="a5148-122">Bir türün ortak Oluşturucusu varsa, muhtemelen iç içe olmamalıdır.</span><span class="sxs-lookup"><span data-stu-id="a5148-122">If a type has a public constructor, it should probably not be nested.</span></span>  
  
 <span data-ttu-id="a5148-123">Bir tür örneği oluşturulabilir, bu, türün kendi çerçevesinde bir yerde yer aldığı anlamına gelir (bunu oluşturabilir, onunla çalışabilir ve dış türü kullanmadan yok edebilir) ve bu nedenle iç içe geçmemelidir.</span><span class="sxs-lookup"><span data-stu-id="a5148-123">If a type can be instantiated, that seems to indicate the type has a place in the framework on its own (you can create it, work with it, and destroy it without ever using the outer type), and thus should not be nested.</span></span> <span data-ttu-id="a5148-124">İç türler, dış tür herhangi bir ilişki olmadan dış türün dışında yaygın olarak yeniden kullanılmamalıdır.</span><span class="sxs-lookup"><span data-stu-id="a5148-124">Inner types should not be widely reused outside of the outer type without any relationship whatsoever to the outer type.</span></span>  
  
 <span data-ttu-id="a5148-125">**X DO NOT** iç içe geçmiş tür bir arabirim üye olarak tanımlayın.</span><span class="sxs-lookup"><span data-stu-id="a5148-125">**X DO NOT** define a nested type as a member of an interface.</span></span> <span data-ttu-id="a5148-126">Birçok dil böyle bir yapıyı desteklemez.</span><span class="sxs-lookup"><span data-stu-id="a5148-126">Many languages do not support such a construct.</span></span>
