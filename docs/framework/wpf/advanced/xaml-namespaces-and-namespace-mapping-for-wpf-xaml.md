## <a name="wpf-and-assembly-loading"></a><span data-ttu-id="2ff2b-162">WPF ve Montaj Yükleme</span><span class="sxs-lookup"><span data-stu-id="2ff2b-162">WPF and Assembly Loading</span></span>  
 <span data-ttu-id="2ff2b-163">WPF için XAML şeması, WPF uygulama modeliyle bütünleşir ve bu da CLR <xref:System.AppDomain>tarafından tanımlanan .</span><span class="sxs-lookup"><span data-stu-id="2ff2b-163">The XAML schema context for WPF integrates with the WPF application model, which in turn uses the CLR-defined concept of <xref:System.AppDomain>.</span></span> <span data-ttu-id="2ff2b-164">Aşağıdaki sıra, XAML şema bağlamının WPF kullanımı <xref:System.AppDomain> ve diğer etkenlere bağlı olarak derlemeleri nasıl yükleyeceklerini veya çalışma zamanında veya tasarım zamanında türleri nasıl bulabileceğinizi açıklar.</span><span class="sxs-lookup"><span data-stu-id="2ff2b-164">The following sequence describes how XAML schema context interprets how to either load assemblies or find types at run time or design time, based on the WPF use of <xref:System.AppDomain> and other factors.</span></span>  
  
1. <span data-ttu-id="2ff2b-165"><xref:System.AppDomain>En son yüklenen derlemeden başlayarak, adın tüm yönleriyle eşleşen zaten yüklü bir derleme yi arayarak,</span><span class="sxs-lookup"><span data-stu-id="2ff2b-165">Iterate through the <xref:System.AppDomain>, looking for an already-loaded assembly that matches all aspects of the name, starting from the most recently loaded assembly.</span></span>  
  
2. <span data-ttu-id="2ff2b-166">Ad nitelikliyse, nitelikli <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType> adı arayın.</span><span class="sxs-lookup"><span data-stu-id="2ff2b-166">If the name is qualified, call <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType> on the qualified name.</span></span>  
  
3. <span data-ttu-id="2ff2b-167">Nitelikli bir ismin kısa adı + ortak anahtar belirteci biçimlendirmenin yüklendiği derlemeyle eşleşiyorsa, bu derlemeyi döndürün.</span><span class="sxs-lookup"><span data-stu-id="2ff2b-167">If the short name + public key token of a qualified name matches the assembly that the markup was loaded from, return that assembly.</span></span>  
  
4. <span data-ttu-id="2ff2b-168">Aramak <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType>için kısa ad + ortak anahtar belirteci kullanın.</span><span class="sxs-lookup"><span data-stu-id="2ff2b-168">Use the short name + public key token to call <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType>.</span></span>  
  
5. <span data-ttu-id="2ff2b-169">Ad niteliksizse, <xref:System.Reflection.Assembly.LoadWithPartialName%2A?displayProperty=nameWithType>çağırın.</span><span class="sxs-lookup"><span data-stu-id="2ff2b-169">If the name is unqualified, call <xref:System.Reflection.Assembly.LoadWithPartialName%2A?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="2ff2b-170">Gevşek XAML Adım 3 kullanmaz; montajdan yüklenmiş bir yük yoktur.</span><span class="sxs-lookup"><span data-stu-id="2ff2b-170">Loose XAML does not use Step 3; there is no loaded-from assembly.</span></span>  
  
 <span data-ttu-id="2ff2b-171">WPF için derlenmiş XAML (XamlBuildTask üzerinden oluşturulan) (Adım <xref:System.AppDomain> 1) zaten yüklenmiş derlemeleri kullanmaz.</span><span class="sxs-lookup"><span data-stu-id="2ff2b-171">Compiled XAML for WPF (generated via XamlBuildTask) does not use the already-loaded assemblies from <xref:System.AppDomain> (Step 1).</span></span> <span data-ttu-id="2ff2b-172">Ayrıca, ad XamlBuildTask çıktısından asla niteliksiz olmamalıdır, bu nedenle Adım 5 geçerli değildir.</span><span class="sxs-lookup"><span data-stu-id="2ff2b-172">Also, the name should never be unqualified from XamlBuildTask output, so Step 5 does not apply.</span></span>  
  
 <span data-ttu-id="2ff2b-173">Derlenmiş BAML (PresentationBuildTask aracılığıyla oluşturulan) tüm adımları kullanır, ancak BAML de niteliksiz montaj adları içermemelidir.</span><span class="sxs-lookup"><span data-stu-id="2ff2b-173">Compiled BAML (generated via PresentationBuildTask) uses all steps, although BAML also should not contain unqualified assembly names.</span></span>
