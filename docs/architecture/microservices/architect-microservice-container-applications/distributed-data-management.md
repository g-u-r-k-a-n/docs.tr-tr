Örneğin, Katalog mikro hizmeti ve sepet mikro hizmeti 'nde ürünün adına veya açıklamasına sahip olmanız gerektiğinde.</span><span class="sxs-lookup"><span data-stu-id="3a829-175">This challenge is also related to the question of how to propagate changes across multiple microservices when certain data needs to be redundant—for example, when you need to have the product's name or description in the Catalog microservice and the Basket microservice.</span></span>

## <a name="challenge-4-how-to-design-communication-across-microservice-boundaries"></a><span data-ttu-id="3a829-178">Zorluk \# 4: mikro hizmet sınırları genelinde iletişim tasarımı</span><span class="sxs-lookup"><span data-stu-id="3a829-178">Challenge \#4: How to design communication across microservice boundaries</span></span>

<span data-ttu-id="3a829-179">Mikro hizmet sınırları genelinde iletişim kurmak gerçek bir zorluk dır.</span><span class="sxs-lookup"><span data-stu-id="3a829-179">Communicating across microservice boundaries is a real challenge.</span></span> <span data-ttu-id="3a829-180">Bu bağlamda iletişim, kullanmanız gereken Protokolü (HTTP ve REST, AMQP, mesajlaşma vb.) ifade etmez.</span><span class="sxs-lookup"><span data-stu-id="3a829-180">In this context, communication doesn't refer to what protocol you should use (HTTP and REST, AMQP, messaging, and so on).</span></span> <span data-ttu-id="3a829-181">Bunun yerine, hangi iletişim stilini kullanacağınızı ve özellikle mikro hizmetlerinizin ne şekilde bağlanmış olduğunu ele alınmaktadır.</span><span class="sxs-lookup"><span data-stu-id="3a829-181">Instead, it addresses what communication style you should use, and especially how coupled your microservices should be.</span></span> <span data-ttu-id="3a829-182">Bir başarısızlık düzeyine bağlı olarak, hata oluştuğunda sisteminizde bu hatanın etkisi önemli ölçüde farklılık gösterecektir.</span><span class="sxs-lookup"><span data-stu-id="3a829-182">Depending on the level of coupling, when failure occurs, the impact of that failure on your system will vary significantly.</span></span>
