# <a name="compiler-error-cs1705"></a>Derleyici Hatası CS1705
' AssemblyName1 ' derlemesi, başvurulan ' AssemblyName2 ' derlemesinden daha yüksek sürüme sahip ' TypeName ' kullanır  
  
 Başvurulan bir derlemede sürüm numarasından daha yüksek sürüm numarasına sahip bir türe erişiyorsunuz. Genellikle, bu hata, aynı derlemenin iki sürümünün yanlışlıkla kullanılması nedeniyle oluşur.  
  
 Örneğin, Asmb1 ve Asmb2 olmak üzere iki derlemeye sahip olduğunuzu varsayalım. Assembly Asmb1, Asmb2 derlemesinin 1,0 sürümüne başvurur. Derleme Asmb1, 2,0 sürümünde derleme Asmb2 'e eklenmiş bir sınıf `MyClass` de kullanır. Derleyicinin bağlama başvuruları için birleşme kuralları vardır ve sürüm 2,0 ' deki `MyClass` bir başvuru, sürüm 1,0 tarafından karşılanamıyor.  
  
## <a name="example"></a>Örnek  
 Aşağıdaki daha ayrıntılı örnek dört kod modülünden oluşur:  
  
- Bir sürüm özniteliği hariç özdeş olan iki dll.  
  
- İlk iki başvuruda bulunan üçüncü bir DLL.  
  
- Yalnızca özdeş dll 'lerin 1,0 sürümüne başvuran, ancak 2,0 sürümünden bir sınıfa erişen istemci.  
  
 Aşağıdaki kod, aynı dll 'Lerden ilki oluşturur. Anahtar dosyası oluşturma hakkında daha fazla bilgi için bkz. [-keyfile (C# derleyici seçenekleri)](../compiler-options/keyfile-compiler-option.md).  
  
```csharp  
// CS1705a.cs  
  
// Compile by using the following command:   
//      csc /target:library /out:C:\\CS1705.dll /keyfile:mykey.snk CS1705a.cs  
  
// The DLL is created in the C:\ directory.  
  
// The AssemblyVersion attribute specifies version 1.0 for this DLL.  
  
[assembly:System.Reflection.AssemblyVersion("1.0")]  
public class Class1   
{  
   public void Method1() {}  
}  
```  
  
## <a name="example"></a>Örnek  
 Aşağıdaki kod, <xref:System.Reflection.AssemblyVersionAttribute> özniteliği tarafından belirtilen şekilde derlemenin 2,0 sürümünü tanımlar.  
  
```csharp  
// CS1705b.cs  
  
// Compile by using the following command:   
//     csc /target:library /out:CS1705.dll /keyfile:mykey.snk CS1705b.cs  
  
// The DLL is created in the current directory.  
  
// The AssemblyVersion attribute specifies version 2.0 for this DLL.  
  
[assembly:System.Reflection.AssemblyVersion("2.0")]  
public class Class1  
{  
    public void Method1() { }  
}  
```  
  
## <a name="example"></a>Örnek  
 Aşağıdaki kod, önceki kodda tanımlanan iki DLL sürümüne başvurur. `AssemblyA`, CS1705a.cs tarafından oluşturulan DLL 'ye (sürüm 1,0) başvurur. `AssemblyB`, CS1705b.cs tarafından oluşturulan DLL 'ye (sürüm 2,0) başvurur. @No__t_0, iki yöntem tanımlanmıştır. Birincisi `Return1A`, DLL 'nin 1,0 sürümünden `Class1` için bir diğer ad olan `Class1A` türünde bir nesne döndürür. İkinci `Return1B`, DLL 'nin 2,0 sürümünden `Class1` için bir diğer ad olan `Class1B` türünde bir nesne döndürür. @No__t_0 tanımı sürüm 1,0 ' de bir bağımlılık oluşturur; `Return1B` tanımı sürüm 2,0 ' de bir bağımlılık oluşturur.  
  
```csharp  
// CS1705c.cs  
  
// Compile by using the following command. AssemblyA refers to the DLL created by  
// CS1705a.cs (version 1.0). AssemblyB refers to the DLL created by CS1705b.cs  
// (version 2.0).  
//    csc /target:library /r:AssemblyA=C:\\CS1705.dll /r:AssemblyB=CS1705.dll CS1705c.cs  
  
extern alias AssemblyA;  
extern alias AssemblyB;  
  
// Class1A is an alias for type Class1 from VS1705a.cs, which is in version 1.0   
// of the assembly. Class1B is an alias for type Class1 from CS1705b.cs, which  
// is in version 2.0 of the assembly.  
  
using Class1A = AssemblyA::Class1;  
using Class1B = AssemblyB::Class1;  
  
// Method Return1A in ClassC returns an object of type Class1A, which is  
// Class1 from version 1.0 of the DLL. Method Return1B returns an object  
// of type Class1B, which is Class1 from version 2.0 of the DLL.  
  
public class ClassC  
{  
    // The following line creates a dependency on version 1.0 of CS1705.dll.  
    // This is not the source of the problem when ClassC is accessed from  
    // CS1705d.cs because CS1705d.cs references version 1.0 of the DLL.   
    // Therefore, type Class1A and the assembly have the same version.  
    public static Class1A Return1A() { return new Class1A(); }  
  
    // The following line creates a dependency on version 2.0 of CS1705.dll.  
    // This causes compiler error CS1705 when ClassC is accessed from     
    // CS1705d.cs, because CS1705d.cs does not reference version 2.0 of the   
    // DLL. Class1B is the alias for Class1 in version 2.0, and CS1705d.cs   
    // references version 1.0.  
    public static Class1B Return1B() { return new Class1B(); }  
}  
```  
  
## <a name="example"></a>Örnek  
 Aşağıdaki kod derleyici hatası CS1705 oluşturur. CS1705a.cs tarafından oluşturulan DLL 'ye (sürüm 1,0) başvurur. Ancak, `Main` yönteminde, kod CS1705c.cs 'ten `ClassC` erişir. `ClassC`, CS1705b.cs (sürüm 2,0) içinde tanımlanan bir tür kullanır. Bu, tür CS1705. dll ' nin başvurulan sürümünden daha yüksek bir sürüm numarasına sahip olduğundan, derleyici hatasına neden olur.  
  
```csharp  
// CS1705d.cs  
  
// Compile by using the following command:  
//     csc /reference:C:\\CS1705.dll /reference:CS1705c.dll CS1705d.cs  
  
// C:\\CS1705.dll is version 1.0 of the assembly.  
  
class Tester   
{  
    static void Main()  
    {  
        // Return1A returns a type defined in version 1.0.  
        ClassC.Return1A().Method1();  
        // Return1B returns a type defined in version 2.0.  
        ClassC.Return1B().Method1();  
    }  
}  
```  
  
 Hatayı aşağıdaki yollarla çözebilirsiniz:  
  
- Tüm dosyaların aynı DLL sürümünü kullanabilmesi için kodu güncelleştirin.  
  
- Derlemek için aşağıdaki komutu kullanarak DLL 'nin 2,0 sürümüne bir başvuru ekleyin:  
  
     `csc /reference:C:\\CS1705.dll /reference:CS1705.dll /reference:CS1705c.dll CS1705d.cs`  
  
     Bu komutu kullandığınızda program derlense de, hala çalıştırılmaz. Programın çalışmasını sağlamak için, [\<assemblyIdentity >](../../../framework/configure-apps/file-schema/runtime/assemblyidentity-element-for-runtime.md) ve [\<codeBase >](../../../framework/configure-apps/file-schema/runtime/codebase-element.md) alt öğelerini kullanan bir [\<dependentAssembly > öğesi](../../../framework/configure-apps/file-schema/runtime/dependentassembly-element.md) içeren bir uygulama yapılandırma dosyası sağlayabilirsiniz. DLL sürümünün 1,0 konumu. Yapılandırma dosyaları hakkında daha fazla bilgi için bkz. [uygulamaları yapılandırma](../../../framework/configure-apps/index.md).
