## <a name="choice-of-passing-mechanism"></a><span data-ttu-id="e35d1-113">Geçirme mekanizması seçimi</span><span class="sxs-lookup"><span data-stu-id="e35d1-113">Choice of Passing Mechanism</span></span>

- <span data-ttu-id="e35d1-119">**Performans**.</span><span class="sxs-lookup"><span data-stu-id="e35d1-119">**Performance**.</span></span> <span data-ttu-id="e35d1-120">Geçirme mekanizması kodunuzun performansını etkileyebilse de, fark genellikle önemsizdir.</span><span class="sxs-lookup"><span data-stu-id="e35d1-120">Although the passing mechanism can affect the performance of your code, the difference is usually insignificant.</span></span> <span data-ttu-id="e35d1-121">Bu bir özel durum geçildi değer türüdür `ByVal` .</span><span class="sxs-lookup"><span data-stu-id="e35d1-121">One exception to this is a value type passed `ByVal`.</span></span> <span data-ttu-id="e35d1-122">Bu durumda Visual Basic bağımsız değişkenin tüm veri içeriğini kopyalar.</span><span class="sxs-lookup"><span data-stu-id="e35d1-122">In this case, Visual Basic copies the entire data contents of the argument.</span></span> <span data-ttu-id="e35d1-123">Bu nedenle, yapı gibi büyük bir değer türü için, daha verimli olabilir `ByRef` .</span><span class="sxs-lookup"><span data-stu-id="e35d1-123">Therefore, for a large value type such as a structure, it can be more efficient to pass it `ByRef`.</span></span>  
  
     <span data-ttu-id="e35d1-124">Başvuru türleri için, yalnızca verilerin işaretçisi kopyalanır (32-bit platformlarda dört bayt, 64 bit platformlarda sekiz bayt).</span><span class="sxs-lookup"><span data-stu-id="e35d1-124">For reference types, only the pointer to the data is copied (four bytes on 32-bit platforms, eight bytes on 64-bit platforms).</span></span> <span data-ttu-id="e35d1-125">Bu nedenle, bu tür bağımsız değişkenleri, `String` performansı çok fazla `Object` performans olmadan veya değere göre geçirebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="e35d1-125">Therefore, you can pass arguments of type `String` or `Object` by value without harming performance.</span></span>

## <a name="when-to-pass-an-argument-by-value"></a><span data-ttu-id="e35d1-132">Bir bağımsız değişkenin değere göre ne zaman geçirileceğini</span><span class="sxs-lookup"><span data-stu-id="e35d1-132">When to Pass an Argument by Value</span></span>

- <span data-ttu-id="e35d1-139">Kodun doğru yürütülmesi, çağıran koddaki temeldeki öğeyi değiştirme yordamına bağımlıysa, parametresini bildirin `ByRef` .</span><span class="sxs-lookup"><span data-stu-id="e35d1-139">If the correct execution of the code depends on the procedure changing the underlying element in the calling code, declare the parameter `ByRef`.</span></span> <span data-ttu-id="e35d1-140">Bunu değere göre geçirirseniz veya çağırma kodu `ByRef` bağımsız değişkeni parantez içine alarak geçen mekanizmayı geçersiz kılıyorsa, yordam çağrısı beklenmedik sonuçlar üretebilir.</span><span class="sxs-lookup"><span data-stu-id="e35d1-140">If you pass it by value, or if the calling code overrides the `ByRef` passing mechanism by enclosing the argument in parentheses, the procedure call might produce unexpected results.</span></span>
