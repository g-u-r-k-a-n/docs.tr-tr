# <a name="service-oriented-architecture"></a><span data-ttu-id="784ef-103">Hizmet odaklı mimari</span><span class="sxs-lookup"><span data-stu-id="784ef-103">Service-oriented architecture</span></span>

<span data-ttu-id="784ef-104">Hizmet odaklı mimari (SOA) aşırı kullanılan bir terim di ve farklı insanlar için farklı şeyler anlamına geliyordu.</span><span class="sxs-lookup"><span data-stu-id="784ef-104">Service-oriented architecture (SOA) was an overused term and has meant different things to different people.</span></span> <span data-ttu-id="784ef-105">Ancak ortak bir payda olarak SOA, uygulamanızı alt sistemler veya katmanlar gibi farklı türler olarak sınıflandırılabilen birden çok hizmete (en yaygın olarak HTTP hizmetleri olarak) ayrıştırarak yapılandırdığınız anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="784ef-105">But as a common denominator, SOA means that you structure your application by decomposing it into multiple services (most commonly as HTTP services) that can be classified as different types like subsystems or tiers.</span></span>

<span data-ttu-id="784ef-106">Tüm bağımlılıklar kapsayıcı görüntüsüne dahil olduğundan, bu hizmetler artık dağıtım sorunlarını çözen Docker kapsayıcıları olarak dağıtılabilir.</span><span class="sxs-lookup"><span data-stu-id="784ef-106">Those services can now be deployed as Docker containers, which solves deployment issues, because all the dependencies are included in the container image.</span></span> <span data-ttu-id="784ef-107">Ancak, SOA uygulamalarını ölçeklendirmeniz gerektiğinde, tek Docker ana bilgisayarlarına göre dağıtım yapıyorsanız ölçeklenebilirlik ve kullanılabilirlik zorluklarına sahip olabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="784ef-107">However, when you need to scale up SOA applications, you might have scalability and availability challenges if you're deploying based on single Docker hosts.</span></span> <span data-ttu-id="784ef-108">Burası Docker kümeleme yazılımının veya bir orkestratörün, mikro hizmetler için dağıtım yaklaşımlarının açıklandığı sonraki bölümlerde açıklandığı gibi size yardımcı olabileceği yerdir.</span><span class="sxs-lookup"><span data-stu-id="784ef-108">This is where Docker clustering software or an orchestrator can help you, as explained in later sections where deployment approaches for microservices are described.</span></span>

<span data-ttu-id="784ef-109">Docker kapsayıcıları hem geleneksel hizmet odaklı mimariler hem de daha gelişmiş microservices mimarileri için kullanışlıdır (ancak gerekli değildir).</span><span class="sxs-lookup"><span data-stu-id="784ef-109">Docker containers are useful (but not required) for both traditional service-oriented architectures and the more advanced microservices architectures.</span></span>

<span data-ttu-id="784ef-110">Microservices SOA türetilmiştir, ancak SOA microservices mimarisi farklıdır.</span><span class="sxs-lookup"><span data-stu-id="784ef-110">Microservices derive from SOA, but SOA is different from microservices architecture.</span></span> <span data-ttu-id="784ef-111">Büyük merkezi brokerlar, organizasyon düzeyinde merkezi orkestrasyonlar ve [Enterprise Service Bus (ESB)](https://en.wikipedia.org/wiki/Enterprise_service_bus) gibi özellikler SOA'da tipiktir.</span><span class="sxs-lookup"><span data-stu-id="784ef-111">Features like large central brokers, central orchestrators at the organization level, and the [Enterprise Service Bus (ESB)](https://en.wikipedia.org/wiki/Enterprise_service_bus) are typical in SOA.</span></span> <span data-ttu-id="784ef-112">Ama çoğu durumda, bu mikro hizmet toplumda anti-desenler vardır.</span><span class="sxs-lookup"><span data-stu-id="784ef-112">But in most cases, these are anti-patterns in the microservice community.</span></span> <span data-ttu-id="784ef-113">Aslında, bazı insanlar "microservice mimarisi SOA doğru yapılır" savunuyorlar.</span><span class="sxs-lookup"><span data-stu-id="784ef-113">In fact, some people argue that "The microservice architecture is SOA done right."</span></span>

<span data-ttu-id="784ef-114">Soa yaklaşımı, mikro hizmet mimarisinde kullanılan gereksinimve tekniklerden daha az açıklayıcı olduğundan, bu kılavuz mikro hizmetlere odaklanır.</span><span class="sxs-lookup"><span data-stu-id="784ef-114">This guide focuses on microservices, because a SOA approach is less prescriptive than the requirements and techniques used in a microservice architecture.</span></span> <span data-ttu-id="784ef-115">Mikro hizmet tabanlı bir uygulama oluşturmayı biliyorsanız, daha basit bir hizmet odaklı uygulamayı nasıl oluşturabileceğinizi de bilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="784ef-115">If you know how to build a microservice-based application, you also know how to build a simpler service-oriented application.</span></span>
