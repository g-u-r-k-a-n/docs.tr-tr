## <a name="rule-description"></a><span data-ttu-id="29757-115">Kural açıklaması</span><span class="sxs-lookup"><span data-stu-id="29757-115">Rule description</span></span>

<span data-ttu-id="29757-116">Yöntemler, IF deyimleri, ikili ifadeler ( `==` , `!=` , `<` , `>` ), null denetimleri vb. gibi koşullu koda sahip olabilir. Örneğin, aşağıdaki kodu göz önünde bulundurun:</span><span class="sxs-lookup"><span data-stu-id="29757-116">Methods can have conditional code, such as if statements, binary expressions (`==`, `!=`, `<`, `>`), null checks, etc. For example, consider the following code:</span></span>

```csharp
public void M(int i, int j)
{
    if (i != 0)
    {
        return;
    }

    if (j != 0)
    {
        return;
    }

    // Below condition will always evaluate to 'false' as 'i' and 'j' are both '0' here.
    if (i != j)
    {
        // Code in this 'if' branch is dead code.
        // It can either be removed or refactored.
        ...
    }
}
```

<span data-ttu-id="29757-117">C# ve VB derleyicileri, her zaman veya olarak değerlendirme yapan derleme zamanı _sabit değerlerini_ içeren koşullu denetimler analizini gerçekleştirir `true` `false` .</span><span class="sxs-lookup"><span data-stu-id="29757-117">C# and VB compilers perform analysis of conditional checks involving compile-time _constant values_ that always evaluate to `true` or `false`.</span></span> <span data-ttu-id="29757-118">Bu çözümleyici sabit olmayan değişkenlerin veri akışı analizini gerçekleştirerek _sabit olmayan değerler_ içeren gereksiz koşullu denetimleri tespit eder.</span><span class="sxs-lookup"><span data-stu-id="29757-118">This analyzer performs data flow analysis of non-constant variables to determine redundant conditional checks involving _non-constant values_.</span></span> <span data-ttu-id="29757-119">Yukarıdaki kodda, çözümleyici, `i` `j` her ikisi de `0` Check 'e ulaşan tüm kod yolları için olduğunu belirler `i != j` .</span><span class="sxs-lookup"><span data-stu-id="29757-119">In the preceding code, the analyzer determines that `i` and `j` are both `0` for all code paths that reach `i != j` check.</span></span> <span data-ttu-id="29757-120">Bu nedenle, bu denetim her zaman `false` çalışma zamanında olarak değerlendirilir.</span><span class="sxs-lookup"><span data-stu-id="29757-120">Hence, this check will always evaluate to `false` at runtime.</span></span> <span data-ttu-id="29757-121">If ifadesinin içindeki kod, ölü koddur ve kaldırılabilir veya yeniden düzenlenmiş.</span><span class="sxs-lookup"><span data-stu-id="29757-121">The code inside the if statement is dead code and can be removed or refactored.</span></span> <span data-ttu-id="29757-122">Benzer şekilde, çözümleyici değişkenlerin boş düzeyini ve rapor gereksiz null denetimleri izler.</span><span class="sxs-lookup"><span data-stu-id="29757-122">Similarly, the analyzer tracks nullness of variables and reports redundant null checks.</span></span>
