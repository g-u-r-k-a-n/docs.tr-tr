## <a name="how-to-fix-violations"></a><span data-ttu-id="ebdad-116">İhlalleri çözme</span><span class="sxs-lookup"><span data-stu-id="ebdad-116">How to fix violations</span></span>

<span data-ttu-id="ebdad-117">Çoğu durumda, gerekli olan düzeltmeler gereksiz başlatmayı silmektir.</span><span class="sxs-lookup"><span data-stu-id="ebdad-117">In most cases, the proper fix is to delete the unnecessary initialization.</span></span>

```csharp

{
    // Violation
    int _value1 = 0;

    // Fixed
    int _value1;
}
```

<span data-ttu-id="ebdad-118">Bazı durumlarda, başlatmanın silinmesi, alan varsayılan değerini süresiz olarak korurken sonraki [CS0649](../../../csharp/misc/cs0649.md) uyarılarının verilmesinin oluşmasına neden olabilir.</span><span class="sxs-lookup"><span data-stu-id="ebdad-118">In some cases, deleting the initialization may result in subsequent [CS0649](../../../csharp/misc/cs0649.md) warnings being issued due to the field retaining its default value forever.</span></span>  <span data-ttu-id="ebdad-119">Böyle durumlarda, alanı tamamen silmek veya bir özelliği ile değiştirmek daha iyi bir çözüm olabilir:</span><span class="sxs-lookup"><span data-stu-id="ebdad-119">In such cases, a better fix may be to delete the field entirely or replace it with a property:</span></span>

```csharp
class C
{
    // Violation
    private static readonly int s_value = 0;

    // Fixed
    private static int Value => 0;
}
```
