## <a name="rule-description"></a><span data-ttu-id="bec53-113">Kural açıklaması</span><span class="sxs-lookup"><span data-stu-id="bec53-113">Rule description</span></span>

<span data-ttu-id="bec53-114">C# derleyicisi bir uyarı [CS1717: aynı değişkene yapılan atama oluşturur; başka bir şey atamanız mı istediniz?](../../../csharp/misc/cs1717.md) bir alan, yerel veya parametre sembolü kendisine atandığında.</span><span class="sxs-lookup"><span data-stu-id="bec53-114">C# compiler generates a warning [CS1717: Assignment made to same variable; did you mean to assign something else?](../../../csharp/misc/cs1717.md) when a field, local or parameter symbol is assigned to itself.</span></span> <span data-ttu-id="bec53-115">Bu tür bir hata, bir yerel, parametre veya alan sembolü, kapsamdaki başka bir simgeye benzer bir ada sahip olduğunda yaygındır.</span><span class="sxs-lookup"><span data-stu-id="bec53-115">Such a mistake is common when a local, parameter, or field symbol has a name similar to another symbol in scope.</span></span> <span data-ttu-id="bec53-116">Atamanın sol tarafındaki ve sağ tarafında farklı semboller kullanmak yerine, her iki tarafta da aynı sembol kullanılmıştır.</span><span class="sxs-lookup"><span data-stu-id="bec53-116">Instead of using different symbols on the left-hand and right-hand side of the assignment, the same symbol was used on both sides.</span></span> <span data-ttu-id="bec53-117">Bu, değerin bir yedekli atamaya ve genellikle işlevsel bir hata olduğunu gösterir.</span><span class="sxs-lookup"><span data-stu-id="bec53-117">This leads to a redundant assignment of the value to itself and generally indicates a functional bug.</span></span>

<span data-ttu-id="bec53-122">`CA2245`Bu boşluğun doldurulmasıyla, kural amaçlar.</span><span class="sxs-lookup"><span data-stu-id="bec53-122">Rule `CA2245` aims at filling this gap.</span></span> <span data-ttu-id="bec53-123">Bu işlevsel hataları gidermeye yardımcı olmak için kendi kendine atamaya yönelik ihlalin olduğunu bildirir.</span><span class="sxs-lookup"><span data-stu-id="bec53-123">It reports the violation for property self-assignment to help fix these functional bugs.</span></span> <span data-ttu-id="bec53-124">Özelliğin kendi kendine atamasının tercih edildiği küçük köşe örnekleri kümesi için, `CA2245` uygun bir gerekçe açıklamasıyla kaynak içinde ihlaller gizlenebilir.</span><span class="sxs-lookup"><span data-stu-id="bec53-124">For the small set of corner cases where property self-assignment is desirable, `CA2245` violations can be suppressed in source with an appropriate justification comment.</span></span>

## <a name="how-to-fix-violations"></a><span data-ttu-id="bec53-125">İhlalleri çözme</span><span class="sxs-lookup"><span data-stu-id="bec53-125">How to fix violations</span></span>

<span data-ttu-id="bec53-126">İhlalleri onarmak için, sol taraftaki ve atamanın sağ tarafında farklı semboller kullanın.</span><span class="sxs-lookup"><span data-stu-id="bec53-126">To fix violations, use different symbols on the left-hand and the right-hand side of the assignment.</span></span> <span data-ttu-id="bec53-127">Örneğin, aşağıdaki kod parçacığı kural ihlalini ve nasıl düzeltileceğini gösterir:</span><span class="sxs-lookup"><span data-stu-id="bec53-127">For example, the following code snippet shows a violation of the rule and how to fix it:</span></span>

```csharp
public class C
{
    private int p = 0;
    public int P { get; private set; }

    public void M(int p)
    {
        // CS1717: Accidentally assigned the parameter 'p' to itself.
        p = p;

        // CA2245: Accidentally assigned the property 'P' to itself.
        P = P;
    }
}
```

```csharp
public class C
{
    private int p = 0;
    public int P { get; private set; }

    public void M(int p)
    {
        // No violation, now the parameter is assigned to the field.
        this.p = p;

        // No violation, now the parameter is assigned to the property.
        P = p;
    }
}
```
