#### <a name="setting-a-clientcredentials-value-in-configuration"></a><span data-ttu-id="ec46b-182">Yapılandırmada \<müşteri kimlik bilgilerini> Değer Ayarlama</span><span class="sxs-lookup"><span data-stu-id="ec46b-182">Setting a \<clientCredentials> Value in Configuration</span></span>  
 <span data-ttu-id="ec46b-183">Kimlik bilgileri değerleri, [ \<istemci Kimlik Bilgileri>](../configure-apps/file-schema/wcf/clientcredentials.md) öğesinin alt öğeleri olarak bir uç nokta davranışı kullanılarak belirtilir.</span><span class="sxs-lookup"><span data-stu-id="ec46b-183">Credential values are specified by using an endpoint behavior as child elements of the [\<clientCredentials>](../configure-apps/file-schema/wcf/clientcredentials.md) element.</span></span> <span data-ttu-id="ec46b-184">Kullanılan öğe istemci kimlik bilgisi türüne bağlıdır.</span><span class="sxs-lookup"><span data-stu-id="ec46b-184">The element used depends on the client credential type.</span></span> <span data-ttu-id="ec46b-185">Örneğin, aşağıdaki örnek, <[ \<istemci Sertifikası>](../configure-apps/file-schema/wcf/clientcertificate-of-clientcredentials-element.md)kullanarak bir X.509 sertifikası ayarlamak için yapılandırmayı gösterir.</span><span class="sxs-lookup"><span data-stu-id="ec46b-185">For example, the following example shows the configuration to set an X.509 certificate using the <[\<clientCertificate>](../configure-apps/file-schema/wcf/clientcertificate-of-clientcredentials-element.md).</span></span>

```xml  
<configuration>  
  <system.serviceModel>  
    <behaviors>  
      <endpointBehaviors>
        <behavior name="myEndpointBehavior">

            <clientCertificate findvalue="myMachineName"

        </behavior>
      </endpointBehaviors>
    </behaviors>
  </system.serviceModel>  
</configuration>  
```  
  
 <span data-ttu-id="ec46b-186">İstemci kimlik bilgisini yapılandırmada ayarlamak için, yapılandırma dosyasına [ \<bir uç nokta davranışı>](../configure-apps/file-schema/wcf/endpointbehaviors.md) öğesi ekleyin.</span><span class="sxs-lookup"><span data-stu-id="ec46b-186">To set the client credential in configuration, add an [\<endpointBehaviors>](../configure-apps/file-schema/wcf/endpointbehaviors.md) element to the configuration file.</span></span> <span data-ttu-id="ec46b-187">Ayrıca, eklenen davranış öğesi, aşağıdaki örnekte gösterildiği gibi `behaviorConfiguration` [ \< \<istemci>](../configure-apps/file-schema/wcf/endpoint-of-client.md) öğesinin bitiş noktası> özniteliğini kullanarak hizmetin bitiş noktasına bağlanmalıdır.</span><span class="sxs-lookup"><span data-stu-id="ec46b-187">Additionally, the added behavior element must be linked to the service's endpoint using the `behaviorConfiguration` attribute of the [\<endpoint> of \<client>](../configure-apps/file-schema/wcf/endpoint-of-client.md) element as shown in the following example.</span></span> <span data-ttu-id="ec46b-188">Özniteliğin `behaviorConfiguration` değeri davranış `name` özniteliğinin değeriyle eşleşmelidir.</span><span class="sxs-lookup"><span data-stu-id="ec46b-188">The value of the `behaviorConfiguration` attribute must match the value of the behavior `name` attribute.</span></span>

```xml
  </system.serviceModel>
```
