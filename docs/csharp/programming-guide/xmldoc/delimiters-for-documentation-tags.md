<span data-ttu-id="fd9e4-105">XML belgesi açıklamalarının kullanılması, bir belge açıklamasının başladığı ve bittiği derleyicisine işaret eden sınırlayıcıları gerektirir.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-105">The use of XML doc comments requires delimiters, which indicate to the compiler where a documentation comment begins and ends.</span></span> <span data-ttu-id="fd9e4-106">XML belge etiketleriyle aşağıdaki tür sınırlayıcıları kullanabilirsiniz:</span><span class="sxs-lookup"><span data-stu-id="fd9e4-106">You can use the following kinds of delimiters with the XML documentation tags:</span></span>

- `///`

  <span data-ttu-id="fd9e4-107">Tek satırlık sınırlayıcı.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-107">Single-line delimiter.</span></span> <span data-ttu-id="fd9e4-108">Bu, belge örneklerinde gösterilen ve C# proje şablonları tarafından kullanılan formdur.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-108">This is the form that is shown in documentation examples and used by the C# project templates.</span></span> <span data-ttu-id="fd9e4-109">Sınırlayıcıyı izleyen bir boşluk karakteri varsa, bu karakter XML çıktısına dahil edilmez.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-109">If there is a white space character following the delimiter, that character is not included in the XML output.</span></span>

  <span data-ttu-id="fd9e4-112">Çok satırlı sınırlayıcılar.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-112">Multiline delimiters.</span></span>

  <span data-ttu-id="fd9e4-113">Sınırlayıcıları kullandığınızda izlenecek bazı biçimlendirme kuralları vardır `/** */` :</span><span class="sxs-lookup"><span data-stu-id="fd9e4-113">There are some formatting rules to follow when you use the `/** */` delimiters:</span></span>
  
  - <span data-ttu-id="fd9e4-114">Sınırlayıcısı içeren satırda `/**` , satırın geri kalanı boşluk ise, satır Yorumlar için işlenmez.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-114">On the line that contains the `/**` delimiter, if the remainder of the line is white space, the line is not processed for comments.</span></span> <span data-ttu-id="fd9e4-115">Sınırlayıcıdan sonraki ilk karakter boşluk `/**` ise, bu boşluk karakteri yok sayılır ve satırın geri kalanı işlenir.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-115">If the first character after the `/**` delimiter is white space, that white space character is ignored and the rest of the line is processed.</span></span> <span data-ttu-id="fd9e4-116">Aksi halde, ayırıcıdan sonraki satırın metnin tamamı `/**` açıklamanın bir parçası olarak işlenir.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-116">Otherwise, the entire text of the line after the `/**` delimiter is processed as part of the comment.</span></span>

  - <span data-ttu-id="fd9e4-117">Sınırlandırıcının bulunduğu satırda, `*/` sınırlandırıcının yalnızca beyaz alanı varsa `*/` , bu satır yok sayılır.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-117">On the line that contains the `*/` delimiter, if there is only white space up to the `*/` delimiter, that line is ignored.</span></span> <span data-ttu-id="fd9e4-118">Aksi takdirde, ayırıcıya kadar olan satırdaki metin `*/` yorumun bir parçası olarak işlenir.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-118">Otherwise, the text on the line up to the `*/` delimiter is processed as part of the comment.</span></span>
  
  - <span data-ttu-id="fd9e4-119">Sınırlayıcı ile başlayan satırlar için `/**` , derleyici her bir satırın başlangıcında ortak bir model arar.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-119">For the lines after the one that begins with the `/**` delimiter, the compiler looks for a common pattern at the beginning of each line.</span></span> <span data-ttu-id="fd9e4-120">Bu kalıp, isteğe bağlı boşluk ve bir yıldız işareti ( `*` ) ve ardından daha isteğe bağlı boşluk içerebilir.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-120">The pattern can consist of optional white space and an asterisk (`*`), followed by more optional white space.</span></span> <span data-ttu-id="fd9e4-121">Derleyici sınırlayıcı veya sınırlayıcıyla başlamayan her bir satırın başlangıcında ortak bir model bulursa `/**` `*/` , bu kalıbı her satır için yoksayar.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-121">If the compiler finds a common pattern at the beginning of each line that does not begin with the `/**` delimiter or the `*/` delimiter, it ignores that pattern for each line.</span></span>

  <span data-ttu-id="fd9e4-122">Aşağıdaki örneklerde bu kurallar gösterilmektedir.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-122">The following examples illustrate these rules.</span></span>

    ```csharp
    <summary>text</summary>
    */

    /**
     * <summary>text</summary>
    */
    ```

  - <span data-ttu-id="fd9e4-125">Derleyici, \* ikinci ve üçüncü çizgilerin başlangıcında "" ortak bir modelini tanımlar.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-125">The compiler identifies a common pattern of " \* " at the beginning of the second and third lines.</span></span> <span data-ttu-id="fd9e4-126">Bu kalıp çıkışa dahil değildir.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-126">The pattern is not included in the output.</span></span>

    ```csharp

     * <summary>
     * text </summary>*/
    ```

  - <span data-ttu-id="fd9e4-127">Derleyici, aşağıdaki açıklamada ortak bir model bulmadığından, üçüncü satırdaki ikinci karakter bir yıldız işareti değildir.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-127">The compiler finds no common pattern in the following comment because the second character on the third line is not an asterisk.</span></span> <span data-ttu-id="fd9e4-128">Bu nedenle, ikinci ve üçüncü satırlardaki tüm metinler açıklamanın bir parçası olarak işlenir.</span><span class="sxs-lookup"><span data-stu-id="fd9e4-128">Therefore, all text on the second and third lines is processed as part of the comment.</span></span>
