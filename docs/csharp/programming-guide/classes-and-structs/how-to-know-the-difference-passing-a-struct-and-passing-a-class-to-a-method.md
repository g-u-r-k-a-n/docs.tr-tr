 <span data-ttu-id="f8cdb-106">Bir struct bir [değer türü](../../language-reference/builtin-types/value-types.md)olduğundan, bir yapıya [değere göre](./passing-value-type-parameters.md) bir yönteme geçirdiğinizde, yöntemi struct bağımsız değişkeninin bir kopyası üzerinde alır ve çalışır.</span><span class="sxs-lookup"><span data-stu-id="f8cdb-106">Because a struct is a [value type](../../language-reference/builtin-types/value-types.md), when you [pass a struct by value](./passing-value-type-parameters.md) to a method, the method receives and operates on a copy of the struct argument.</span></span> <span data-ttu-id="f8cdb-107">Yöntemi çağıran yöntemde orijinal yapıya erişemez ve bu nedenle herhangi bir şekilde değiştirilemez.</span><span class="sxs-lookup"><span data-stu-id="f8cdb-107">The method has no access to the original struct in the calling method and therefore can't change it in any way.</span></span> <span data-ttu-id="f8cdb-108">Yöntemi yalnızca kopyayı değiştirebilir.</span><span class="sxs-lookup"><span data-stu-id="f8cdb-108">The method can change only the copy.</span></span>  
  
 <span data-ttu-id="f8cdb-109">Bir sınıf örneği, bir değer türü değil [başvuru türüdür](../../language-reference/keywords/reference-types.md).</span><span class="sxs-lookup"><span data-ttu-id="f8cdb-111">Diğer bir deyişle, çağrılan yöntem örneğin adresinin bir kopyasını alır ve çağırma yöntemi örneğin özgün adresini korur.</span><span class="sxs-lookup"><span data-ttu-id="f8cdb-112">Çağıran yöntemdeki sınıf örneği bir adrese sahiptir, çağrılan yöntemdeki parametresi adresin bir kopyasına sahiptir ve her iki adres de aynı nesneye başvurur.</span><span class="sxs-lookup"><span data-stu-id="f8cdb-112">The class instance in the calling method has an address, the parameter in the called method has a copy of the address, and both addresses refer to the class="sxs-lookup"><span data-stu-id="f8cdb-113">Because the parameter contains only a copy of the address, the called method cannot change the address of the class instance in the calling kullanabilir.</span><span class="sxs-lookup"><span data-stu-id="f8cdb-114">However, the called method can use the copy of the address to access the class members that both the original address and the copy of the address reference.</span></span> <span data-ttu-id="f8cdb-115">Çağrılan yöntem bir sınıf üyesini değiştirirse, çağıran yöntemdeki özgün sınıf örneği de değişir.</span><span class="sxs-lookup"><span data-stu-id="f8cdb-115">If the called method changes a class member, the original class instance in the calling method also changes.</span></span>  
  
 <span data-ttu-id="f8cdb-116">Aşağıdaki örnek çıktıda fark gösterilmektedir.</span><span class="sxs-lookup"><span data-stu-id="f8cdb-116">The output of the following example illustrates the difference.</span></span> <span data-ttu-id="f8cdb-117">Sınıf örneğinin `willIChange` alanının değeri, yöntem, sınıf örneğinin belirtilen alanını bulmak için parametresindeki adresi kullandığından, yöntem `ClassTaker` çağrısıyla değiştirilir.</span><span class="sxs-lookup"><span data-stu-id="f8cdb-117">The value of the `willIChange` field of the class instance is changed by the call to method `ClassTaker` because the method uses the address in the parameter to find the specified field of the class instance.</span></span> <span data-ttu-id="f8cdb-118">Bağımsız değişkenin değeri, adresinin bir kopyası değil, yapısının kendisi bir kopyası olduğundan, çağırma yöntemindeki yapının `willIChange` alanı, `StructTaker` yöntemi çağrısıyla değiştirilmez.</span><span class="sxs-lookup"><span data-stu-id="f8cdb-118">The `willIChange` field of the struct in the calling method is not changed by the call to method `StructTaker` because the value of the argument is a copy of the struct itself, not a copy of its address.</span></span> <span data-ttu-id="f8cdb-119">`StructTaker` kopyayı değiştirir ve `StructTaker` çağrısı tamamlandığında kopya kaybedilir.</span><span class="sxs-lookup"><span data-stu-id="f8cdb-119">`StructTaker` changes the copy, and the copy is lost when the call to `StructTaker` is completed.</span></span>
