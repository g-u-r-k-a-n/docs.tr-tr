## <a name="container-design-principles"></a><span data-ttu-id="46735-113">Kapsayıcı tasarım ilkeleri</span><span class="sxs-lookup"><span data-stu-id="46735-113">Container design principles</span></span>

<span data-ttu-id="46735-116">Bir kapsayıcı görüntüsü tasarladığınızda Dockerfile dosyasında bir [giriş noktası](https://docs.docker.com/engine/reference/builder/#entrypoint) tanımı görürsünüz.</span><span class="sxs-lookup"><span data-stu-id="46735-116">When you design a container image, you'll see an [ENTRYPOINT](https://docs.docker.com/engine/reference/builder/#entrypoint) definition in the Dockerfile.</span></span> <span data-ttu-id="46735-117">Bu tanım, ömrü kapsayıcının ömrünü denetleyen işlemi tanımlar.</span><span class="sxs-lookup"><span data-stu-id="46735-117">This definition defines the process whose lifetime controls the lifetime of the container.</span></span> <span data-ttu-id="46735-118">İşlem tamamlandığında kapsayıcı yaşam döngüsü sona erer.</span><span class="sxs-lookup"><span data-stu-id="46735-118">When the process completes, the container lifecycle ends.</span></span> <span data-ttu-id="46735-119">Kapsayıcılar Web sunucuları gibi uzun süreli işlemleri temsil edebilir, ancak daha önce Azure [WebJobs](https://github.com/Azure/azure-webjobs-sdk/wiki)olarak uygulanan toplu işler gibi kısa süreli işlemleri de temsil edebilir.</span><span class="sxs-lookup"><span data-stu-id="46735-119">Containers might represent long-running processes like web servers, but can also represent short-lived processes like batch jobs, which formerly might have been implemented as Azure [WebJobs](https://github.com/Azure/azure-webjobs-sdk/wiki).</span></span>

<span data-ttu-id="46735-120">İşlem başarısız olursa, kapsayıcı sonlanır ve Orchestrator üzerinde sürer.</span><span class="sxs-lookup"><span data-stu-id="46735-120">If the process fails, the container ends, and the orchestrator takes over.</span></span> <span data-ttu-id="46735-121">Orchestrator beş örneği çalışır durumda tutmak üzere yapılandırıldıysa ve bir hata başarısız olursa, Orchestrator başarısız işlemi değiştirmek için başka bir kapsayıcı örneği oluşturacaktır.</span><span class="sxs-lookup"><span data-stu-id="46735-121">If the orchestrator was configured to keep five instances running and one fails, the orchestrator will create another container instance to replace the failed process.</span></span> <span data-ttu-id="46735-122">Batch işinde, işlem parametrelerle başlatılır.</span><span class="sxs-lookup"><span data-stu-id="46735-122">In a batch job, the process is started with parameters.</span></span> <span data-ttu-id="46735-123">İşlem tamamlandığında, çalışma tamamlanmıştır.</span><span class="sxs-lookup"><span data-stu-id="46735-123">When the process completes, the work is complete.</span></span> <span data-ttu-id="46735-124">Bu kılavuz, daha sonra, bu kılavuzda daha sonra detaya gider.</span><span class="sxs-lookup"><span data-stu-id="46735-124">This guidance drills-down on orchestrators, later on.</span></span>
